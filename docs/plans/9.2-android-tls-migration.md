# 9.2 Android TLS migration (upstream-aligned)

**Status:** тЬЕ Done (2026-06-17) тАФ device verified.

**Canonical doc:** [`docs/android-tls.md`](../android-tls.md) тАФ how Android TLS works today.

**Upstream references:**

- [gtk-android-builder#20](https://github.com/sp1ritCS/gtk-android-builder/issues/20) тАФ app loads TLS via static `g_io_openssl_load()`, not GDK / GIO modules in assets
- [GLib #3449](https://gitlab.gnome.org/GNOME/glib/-/work_items/3449) тАФ long-term Android init belongs in GLib, not GTK hacks
- Prior art: GStreamer on Android (static TLS, per #3449); Tuba (per #20 / GeopJr)

**Layout:** `docs/guide-to-writing-plans.md` тАФ checklist, emoji bullets, concrete file targets.

---

## Purpose

- **ЁЯФ╖** Move from **rejected** approach (GDK TLS scan, `libgioopenssl.so` in shared assets, `g_setenv` for TLS) to **maintainer-aligned** approach (app-only init, static `gioopenssl`, no env mutation).
- **ЁЯФ╖** Keep HTTPS working on **arm64 chat POC** throughout тАФ no regression while migrating.
- **ЁЯФ╖** Split GTK upstream work: **IME/paste/popup** patches stay; **TLS in GDK** drops.
- **тЬЕ** Phase 1 тАФ app-side cleanup; no `g_setenv` for TLS/CA; icon theme in `AndroidApplication`; GDK TLS removed from `android-bugs.patch`.
- **тЬЕ** Phase 2 тАФ static `g_io_openssl_load()`; no `libgioopenssl.so` in APK assets.
- **ЁЯЪл** Phase 3 deferred (Java `GTlsBackend`, GLib !4203 Context, split APK, system CA store).

---

## Why the current fix was rejected

| Maintainer objection | Our old code | Source |
|---------------------|--------------|--------|
| TLS is **not GTK's job** | `gdk_android_scan_gio_modules`, `g_setenv("GIO_MODULE_DIR")` in `android-bugs.patch` | #20 |
| **Never `g_setenv`** after threading | `SSL_CERT_FILE`, `GIO_MODULE_DIR` in `android-gio-tls.c` | #20, #3449 |
| **Multi-arch / split APK** breaks shared `assets/share/gio/modules/libgioopenssl.so` | Single-arch `aarch64` POC only today | #20 |
| Android platform glue belongs in **GLib**, not GDK exports | `g_set_user_dirs` hack, TLS in `gdkandroidruntime.c` | #3449 |

## Phase 1 тАФ App cleanup + drop GDK TLS

**Goal:** HTTPS on arm64 without GTK carrying TLS init.

### GTK / upstream

| Action | Status |
|--------|--------|
| **Drop GDK TLS** from `android-bugs.patch` (`gdk_android_prepare_tls_modules`, `gdk_android_scan_gio_modules`, `g_setenv("GIO_MODULE_DIR")`) | тЬЕ |
| **Keep TLS init app-only** in `android-gio-tls.c`, called from `main()` | тЬЕ |
| **Keep** `assets/share/ssl/certs/ca-certificates.crt` | тЬЕ |

### `android-gio-tls.c` cleanup

- [x] **Remove `g_setenv("SSL_CERT_FILE")`** тАФ `find_bundled_ca_cert_path()` + `GTlsFileDatabase` on sessions
- [x] **Stop setting `GIO_MODULE_DIR`** (superseded by Phase 2 static load)
- [x] **Icon theme** тАФ `Gtk.Settings` in `AndroidApplication`
- [x] **Remove `g_setenv("G_MESSAGES_DEBUG")`**

## Phase 2 тАФ Static `g_io_openssl_load()`

**Goal:** Satisfy [#20](https://github.com/sp1ritCS/gtk-android-builder/issues/20).

### Init pattern (GStreamer / Tuba)

```c
extern void g_io_openssl_load (gpointer data);

gboolean
ollmapp_configure_android_gio_tls_modules (void)
{
  GTlsBackend *backend;

  g_io_openssl_load (NULL);
  backend = g_tls_backend_get_default ();
  /* log + return backend_is_real(backend) */
}
```

### Checklist

- [x] `android/pixiewood-extra.cross` тАФ `default_library = 'static'` for glib-networking
- [x] `ollmapp/meson.build` тАФ `dependency('gioopenssl')` on android POC
- [x] `android-gio-tls.c` тАФ `g_io_openssl_load(NULL)` only; ~180 lines (was ~400)
- [x] Drop `assets/share/gio/modules/libgioopenssl.so` from build + `verify-apk.sh`
- [x] Keep `libssl.so` / `libcrypto.so` in `lib/arm64-v8a/` (jniLibs)

### Risk

Link-time duplicate symbols if multiple deps statically pull `gioopenssl`. Link into app binary only. Validate with full APK build + device HTTPS.

## Phase 3 тАФ Defer (not needed for chat POC)

| Item | Defer because |
|------|----------------|
| Java `GTlsBackend` (sp1ritCS long-term on #3449) | Large; static OpenSSL is established path |
| GLib !4203 Android `Context` integration | Upstream |
| Universal / split APK support | Ship `aarch64` only today |
| Remove CA bundle from assets | System CAs need OpenSSL patches (WPE-Android on #3449) |

---

## Keep vs drop

| Keep | Drop |
|------|------|
| `android-gio-tls.c` init in `main()` | GDK TLS in `android-bugs.patch` |
| `AndroidConnectionTls` / `GTlsFileDatabase` | `g_setenv` for TLS/CA |
| CA PEM in `assets/share/ssl/certs/` | `assets/share/gio/modules/libgioopenssl.so` |
| `libssl` / `libcrypto` in jniLibs | Dynamic GIO module scan in app |
| IME/paste/popup GTK patches | GLib ensure-before-scan **for Android** (optional follow-up) |
