### Progress notification (✅)

```json
{"method":"event.vector.scan_update","object-type":"Vector","message":"12 /path/to/file.vala"}
```

- **`message`** = `queue_size` + space + `current_file`; **`"0 "`** when queue empty
- **`Codebase.start`** / **`stop`** reply immediately — **🚫** do not block RPC for full index run

### `Response.result` — always a list

**`Response.result`** is **`Gee.ArrayList<GLib.Object>`** (never null) — **`default = new Gee.ArrayList<GLib.Object>()`**. Length 0, 1, or N after handlers populate it. **🚫** no bare single-object **`result`**. One row → one-element list. **🚫** no **`result_type`** / **`is_array`** — see **Phase 1.5**.

**Clients:** **`response.result`** is always a list. One **`(Gee.ArrayList<T>) response.result`** cast per site — **🚫** no **`(Gee.ArrayList<GLib.Object>)`** intermediate (**`raw`**, etc.). **🚫** no **`response.result == null`** / **`!= null`** — guard **`response.error`** only, then cast. List-returning methods: cast and return (empty list casts fine). Single-row methods: cast typed list, then **`list.get(0)`** when **`list.size > 0`**. **🚫** no **`response.result as File`** / **`(File) response.result`**. **🚫** no copy-**`foreach`** loops.

**Encode:** **`bin_write_prop`** omits the **`result`** property when **`result.size == 0`** (default empty, msg-only, and intentional empty replies). **Decode:** property absent → default empty list remains.

**ℹ️** **`libocrpc/Bin/Stream.vala`** **`parse_object_array`** — **unchanged**.

### 1.5.22 `docs/bin-rpc-protocol.md` — § root result arrays

**Why:** document encode/decode without handler metadata; **no wire reorder**.

**Where:** **`### Root result arrays (`Response.result`)`** section.

**Depends on:** §1.5.2–§1.5.3.

#### Remove

```markdown
**Handler metadata (not on the wire):** set **`is_array = true`** and **`result_type`** (element alias string) before reply. **`Response.bin_write_prop`** omits **`result-type`** and **`is-array`** from the encoded property stream — they exist only to drive encode. On decode, **`is_array`** is set when the **`result`** type byte is an object array (`0xD0`); **`result_type`** is not round-tripped.

Wire layout matches the object-array form above (`0xD0` + element `reg_id` + count + element streams).

**Empty arrays:** when **`list.size == 0`**, the encoder uses **`result_type`** to pick the element class (no sample object to infer `GType` from). **`result_type`** must be a registered alias.
```

#### Replace with

```markdown
**Property type:** **`Gee.ArrayList<GLib.Object>`** — **`default = new Gee.ArrayList<GLib.Object>()`**; never null. Handlers populate **`result`** (length 0, 1, or N). Single-row RPCs use a one-element list — **🚫** no bare object on **`result`**.

**Encode:** omit **`result`** when **`result.size == 0`**. When **`size > 0`**, **`Response.bin_write_prop`** writes reg-id-first object arrays (`0xD0` + element `reg_id` + count + bodies); element **`GType`** from **`result.get(0)`**. **🚫** no **`is_array`** / **`result_type`** on handlers.

**Decode:** property present → **`Stream.parse_object_array()`**; property absent → default empty list. Clients: guard **`response.error`** only — **🚫** no **`response.result == null`**; one **`(Gee.ArrayList<T>) response.result`** cast — **🚫** no **`(Gee.ArrayList<GLib.Object>)`** hop. Single-row methods use **`list.get(0)`** when **`list.size > 0`**.
```
