## Purpose

**2.10.4.23** cannot delete shipping **`libocvector`** (FAISS, **`Indexer`**, in-process search) until **`oc-vector-index`** / **`oc-vector-search`** CLIs migrate to RPC.

**🔷** CodebaseSearch agent tool: **V2 already written** — **not** a V1 rewrite.

| | V1 (legacy, `v2testing=false` only) | V2 (replacement, in tree) |
|--|--------------------------------------|---------------------------|
| Tool | `OLLMvector.Tool.CodebaseSearchTool` | `OLLMtools.CodebaseSearch.CodebaseSearchTool` |
| Request | `OLLMvector.Tool.RequestCodebaseSearch` — local FAISS | `OLLMtools.CodebaseSearch.RequestCodebaseSearch` — **`Codebase.search`** RPC |
| Registered by | `libocvector/Registry.vala` (full lib) | **✅** `liboctools/Registry.vala` (V2 `octools_src`) |
| Default app (`v2testing=true`) | **not linked** | **✅ registered** via `tools_registry.fill_tools` |

**🔷** This plan **wires the existing V2 tool**, adds missing daemon **`vector.*`** RPC for CLIs, and deletes V1 **`libocvector`** when CLIs no longer need it.

**🚫** Do **not** port CodebaseSearch from V1 — compile and register **`liboctools/CodebaseSearch/*`** instead.

## Client migration

### A — Wire existing V2 **`CodebaseSearch`** tool (app) **✅**

**V2 implementation is done** (`liboctools/CodebaseSearch/` → **`Codebase.search`**). **✅ applied 2026-07-05:**

1. **✅** **`CodebaseSearch/Tool.vala`**, **`Request.vala`**, **`Config.vala`** in V2 **`octools_src`**
2. **✅** **`liboctools/Registry.vala`** — `init_config`, `setup_config_defaults`, `fill_tools` register **`OLLMtools.CodebaseSearch.CodebaseSearchTool`**
3. **✅** no **`init_databases`** / local FAISS — daemon owns vector DB
4. **⏳** Delete **`OLLMvector.Tool.CodebaseSearch*`** when V1 **`libocvector`** tree removed (**2.10.4.23**)

### B — Rewrite **`oc-vector-search`** example

**🔷** RPC-only thin client (like **`ollmchat-cli`**):

1. Connect to **`ollmfilesd`** (spawn or **`OLLM_OLLMFILESD`** / stdio — match app pattern)
2. **`Codebase.search`** for normal query (+ filters via **`VectorParams`**)
3. **`vector.list`** for **`--show-info`**
4. **`vector.get`** for **`--dump-vector`**
5. **🚫** no local **`SQ.Database`**, **🚫** no **`OLLMvector.Database`**

**Deps:** **`libocrpc`**, **`libocfiles` V2 PM** (RPC only), **`libollmchat`** (config/embed client if still needed for model checks — **⏳** audit: embed may be daemon-only after index migration)

### C — Rewrite **`oc-vector-index`** example

1. Same daemon connection
2. **`ProjectManager.create_project`** / **`Folder.*`** RPC for **`--create-project`** (existing wire)
3. **`vector.reset`** for **`--reset-database`**
4. **`Folder.project_description`** for **`--project-summary`**
5. **`vector.index`** for main index run
6. **🚫** no in-process **`read_dir`** / **`Indexer`** on client
