### Two scans (do not conflate)

| Step | What | Who | Code |
|------|------|-----|------|
| **Filesystem scan** | Walk disk → update **`filebase`** rows | **CLI / app** (not **`Codebase.start`**) | **`Folder.read_dir`** via **`ProjectManager.activate_project`** |
| **Queue build** | DB → enqueue where **`last_vector_scan < mtime_on_disk()`** | **`Codebase.start`** | **`BackgroundScan`** **`queueProject`** path — **🚫** no disk walk |
| **Index run** | FAISS embed + metadata | daemon (async) | **`BackgroundScan.startQueue`** |

**`stop`** does **not** clear the queue — a later **`start`** appends newly stale files and resumes pending work.

### Progress notification (✅)

```json
{"method":"event.vector.scan_update","object-type":"Vector","message":"12 /path/to/file.vala"}
```

- **`message`** = `queue_size` + space + `current_file`; **`"0 "`** when queue empty
- **`Codebase.start`** / **`stop`** reply immediately — **🚫** do not block RPC for full index run

### `Response.result` — always a list

**`Response.result`** is **`Gee.ArrayList<GLib.Object>`** (never null) — **`default = new Gee.ArrayList<GLib.Object>()`**. Length 0, 1, or N after handlers populate it. **🚫** no bare single-object **`result`**. One row → one-element list. **🚫** no **`result_type`** / **`is_array`** — see **Phase 1.5**.

**Clients:** **`response.result`** is always a list. One **`(Gee.ArrayList<T>) response.result`** cast per site — **🚫** no **`(Gee.ArrayList<GLib.Object>)`** intermediate (**`raw`**, etc.). **🚫** no **`response.result == null`** / **`!= null`** — guard **`response.error`** only, then cast. List-returning methods: cast and return (empty list casts fine). Single-row methods: cast typed list, then **`list.get(0)`** when **`list.size > 0`**. **🚫** no **`response.result as File`** / **`(File) response.result`**. **🚫** no copy-**`foreach`** loops.

**Encode:** **`bin_write_prop`** omits the **`result`** property when **`result.size == 0`** (default empty, msg-only, and intentional empty replies). **Decode:** property absent → default empty list remains.

**Why:** document encode/decode without handler metadata; **no wire reorder**.

**Where:** **`### Root result arrays (`Response.result`)`** section.



### Phase 6 — `Codebase.start` / `Codebase.stop` ✔️

**Task:** RPC control of the background **vector index queue** only — not filesystem scan, not project creation.

**Handlers:** **`call_start`** and **`call_stop`** on **`Codebase.vala`** (`construct` — inline, no separate handler method).

**`call_stop`:**

1. **`background_scan.stop_requested = true`**
2. Reply **`msg`**: `"ok"`

**`call_start`:**

1. **`stop_requested = false`** — resume after **`stop`**
2. **`queueProject(path, only_file, auto_start=true)`** — stale-from-DB enqueue (same rules as before Phase 6); **`only_file`** early-return path in **`BackgroundScan`**
3. Reply **`msg`**: `"ok"` or queued count (when enqueue finishes — **🚫** not when index run completes)

**🚫** **`read_dir`** / filesystem scan — CLI responsibility (Phase 9: **`ProjectManager.activate_project`** before **`Codebase.start`**).

**`VectorParams` for `start`:**

| Field | CLI flag | Notes |
|-------|----------|-------|
| `path` | project path | must already exist in DB (post-scan) |
| `only_file` | `--only-file` | enqueue single file when set |

**🚫** **`recurse`**, **`create_project`** on **`start`** — filesystem / project setup stays on CLI wire (**`activate_project`**, **`create_project`**).

**CLI mapping:** **`oc-vector-index`** calls **`Codebase.start`** after scan; progress on **`OLLMrpc.Client.notification`**.

**Verify:** RPC **`start`** → **`event.vector.scan_update`** until **`"0 "`**; **`stop`** mid-run pauses with queue intact; **`start`** without prior scan enqueues only what DB already knows.
