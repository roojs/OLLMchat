# Table support Phase 1: Renderer interface (1.10.1)

Part of the phased implementation for markdown table support; see **1.8.2-table-support-current-design.md** for the overall design and phase order.

---

## Goal

Add the table callback methods to **RenderBase** so all renderers declare the interface. Stubs or default empty implementations so everything still compiles.

## Files

- `libocmarkdown/RenderBase.vala` – add the four virtual table methods (see below).
- **All extended classes** – add placeholder overrides for the four table callbacks in every RenderBase subclass so they explicitly have the methods. Subclasses: `MarkdownGtk.Render` (libocmarkdowngtk), `HtmlRender`, `DummyRenderer`, `PangoRender` (libocmarkdown). Placeholder = empty body or `base()` call.

## Code to add (RenderBase)

After the existing block-level callbacks (e.g. after `on_hr()`), add four new virtual methods with default empty implementations:

```vala
// Table callbacks (block-level; default empty so subclasses can override)
public virtual void on_table(bool is_start) {}
public virtual void on_table_row(bool is_start) {}
public virtual void on_table_hcell(bool is_start) {}
public virtual void on_table_cell(bool is_start) {}
```

## Placeholder methods in extended classes

Add placeholder overrides for the four table callbacks in every RenderBase subclass so that it builds. Each subclass (MarkdownGtk.Render, HtmlRender, DummyRenderer, PangoRender) gets explicit overrides with empty body or `base.on_*(...)` call.

## Notes

- Use `virtual` and `{}` in RenderBase so subclasses can override when they add table support.
- Build and run tests to confirm nothing breaks.
