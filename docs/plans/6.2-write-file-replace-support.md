# Plan 6.2: Write tool — `search_text` replace (`liboctools/WriteFile` only)

## How to read this plan

Every edit is split into:

- **REMOVE** — exact text **deleted** from the tree (line ranges refer to
  [`Request.vala`](../../liboctools/WriteFile/Request.vala) /
  [`Tool.vala`](../../liboctools/WriteFile/Tool.vala) at plan time).
- **ADD** — exact text **inserted** after the removal, or **INSERT** when
  nothing is deleted (anchor lines shown).

If there is no **REMOVE**, the section states **insert only** and quotes the
surrounding lines so the gap is visible.

**ADD is not a style rewrite:** **`ADD`** must keep the same formatting as the
**`REMOVE`** it replaces (line breaks, ternary **`modes`** sum, single-line
**`return`** strings where the tree already does that). Only introduce new lines
for **`search_text`** behavior. Do not “pretty-print” or reflow approved code.

## Coding standards checklist

Verified against [.cursor/rules/CODING_STANDARDS.md](../../.cursor/rules/CODING_STANDARDS.md):

- **`this.`**, **`GLib.`**, no `@"` in Vala fences (multi-line **`"""`** only in
  `Tool` help text).
- Braces: class/method — own line; **`if`/`switch`** — inline `{`.
- Docblocks: multiline **`/** … */`**; **`{{{…}}}`** for literals; wrap near
  72 characters.
- **`try`** only around **`read_async`**.

## Purpose

Add **`search_text`** on **`Request`**: find an excerpt in an existing file,
replace with **`content`**, validate with **`FileBuffer.locate`**, then convert
the match to **`start_line` / `end_line`** and use the same **`FileChange`**
line-range path as explicit line mode (**`apply_change (false)`** → **`apply_edit`**).

**Out of scope:** **`liboccoder`**, **`libocfiles`**.

### Buffer and edits — `search_text` → normal `FileChange` line range

**No new `FileBuffer` API in 6.2.** [`FileBuffer.locate`](../../libocfiles/FileBuffer.vala) returns **`Gee.HashMap<int, string>`**: **0-based start line** of the match → **full line text** covering the match (through the end of the last line touched), same as in **`validate()`**.

**Convert that single match into the same shape as user-supplied line range** ([`Request.execute_request`](../../liboctools/WriteFile/Request.vala) **`else if (this.start_line >= 1 …)`** branch → **`FileChange`** with **`start` / `end`** → **`apply_change (false)`** → **`apply_edit`**):

| Derived field | Rule |
|---------------|------|
| **`start` (1-based, inclusive)** | **`locate` key + 1** (map key is 0-based first line). |
| **`end` (1-based, exclusive)** | **`start` + `n_lines`**. Derive **`n_lines`** from the map **value** (same span **`locate`** uses): split with **`"\n"`**, then if the last segment is empty (Vala **`split`** after a trailing newline), subtract **1** — e.g. **`"foo\nbar\n"`** → **2** lines, not **3**. |
| **`replacement`** | **`this.content`** (unchanged — the tool argument body). |

Then clear **`search_text`** so the rest of **`execute_request`** treats the request as **line-range mode** (same **`FileChange`** construction as **`start_line` / `end_line`** from JSON).

**Implementation note:** Extract the **single** map entry ( **`validate()`** already guarantees **`matches.size == 1`** ). Use **`locate`** again in **`execute_request`** with the same **`(needle, match_trimmed, case_sensitive)`** arguments as **`validate()`** — do **not** use **`index_of`** on **`get_text()`** here; line range comes only from **`locate`**.

**Caveat:** **`apply_edit`** replaces **whole lines**. The **`locate`** value is defined to cover full lines for the match; replacing those lines with **`content`** is the intended chunk edit. If **`locate`** and **`apply_edit`** ever disagree on line boundaries (should not happen when both use the same buffer), fix in one place.

| Mechanism | Role |
|-----------|------|
| **`FileBuffer.apply_edit (FileChange)`** | Line-based splice ([`DummyFileBuffer.apply_edit`](../../libocfiles/DummyFileBuffer.vala)). |
| **`FileChange.apply_change (false)`** | **`yield buffer.apply_edit (this)`** for line / AST modes ([`FileChange.vala`](../../libocfiles/FileChange.vala)). |
