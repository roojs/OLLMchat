# 6.12 Consecutive fenced frame spacing (placeholder `TextView` + CSS)

**Status:** proposed — implement when approved (see **`.cursor/rules/plan-implementation-workflow.mdc`**).

**Checklist:** **`.cursor/rules/CODING_STANDARDS.md`** — *Checklist for all plans*.

**Layout:** **`docs/guide-to-writing-plans.md`**.

---

## Purpose

- **Streaming markdown GTK:** When a **new** fenced block **starts**, we **`append`** a frame and **`create_textview()`**. Before that, drop whitespace-only placeholder **`Gtk.TextView`**s:
  - **`+1` previous block:** **`this.current_textview.get_prev_sibling()`** — if it is a **`Gtk.TextView`** and its buffer **`strip()`**s to empty, **`remove`** it (one sibling only; e.g. orphan empty view before the current placeholder).
  - **Current placeholder:** **`this.current_textview`** — same **`strip()`** test; **`remove`** if empty.
- **Do not** remove the **new** text view created **after** the frame. **Do not** scan the whole box (**no** **`get_last_child`** loops); only **`current_textview`** and **one** **`prev_sibling`**.
- **CSS:** **`.oc-frame`** **`margin-bottom`** (e.g. **8px**) so frames are not doubly spaced.

---

## Scope

| In scope | Out of scope |
| -------- | ------------- |
| **`Render.vala`** **`on_code_block(true)`**: **`prev_sibling`** + **`current_textview`**, each whitespace-only → **`remove`**, then frame + **`create_textview()`** | Walking all box children, **`get_last_child`** chains |
| **`resources/frame.css`** — **`.oc-frame`** margin | **`Parser.vala`** newline suppression |

---

## Approved behaviour

1. **`Render.vala`:** With **`this.current_textview`** and **`this.box`** set: (a) if **`get_prev_sibling()`** is a **`Gtk.TextView`** and buffer strips empty, **`remove`** it; (b) if **`this.current_textview`** buffer strips empty, **`remove`** it. Then null **`current_textview`** / **`current_buffer`** / **`top_state`** / **`current_state`**; **`RenderSourceView`**, **`source_view_handlers.add`**, **`create_textview()`**.
2. **`frame.css`:** **`.oc-frame`** **`margin-bottom: 8px`** (or reviewed value).

### 2. `libocmarkdowngtk/Render.vala` — **`on_code_block`**

#### Replace with ( **`is_start == true`** through **`create_textview();`** )

```vala
			// Code block started - clear old textview/buffer/states
			// Streaming: +1 previous sibling (whitespace-only TextView) then current
			// placeholder; do not remove views after the frame.
			// During the code block, all text goes to sourceview, not textview
			if (this.current_textview != null && this.box != null) {
				Gtk.Widget? prev_w = this.current_textview.get_prev_sibling();
				Gtk.TextView? prev_tv = prev_w as Gtk.TextView;
				if (prev_tv != null) {
					Gtk.TextBuffer pbuf = prev_tv.buffer;
					Gtk.TextIter p_start;
					Gtk.TextIter p_end;
					pbuf.get_start_iter(out p_start);
					pbuf.get_end_iter(out p_end);
					if (pbuf.get_text(p_start, p_end, false).strip() == "") {
						this.box.remove(prev_tv);
					}
				}
				Gtk.TextBuffer buf = this.current_textview.buffer;
				Gtk.TextIter start_i;
				Gtk.TextIter end_i;
				buf.get_start_iter(out start_i);
				buf.get_end_iter(out end_i);
				if (buf.get_text(start_i, end_i, false).strip() == "") {
					this.box.remove(this.current_textview);
				}
			}
			this.current_textview = null;
			this.current_buffer = null;
			this.top_state = null;
			this.current_state = null;

			this.childview = new RenderSourceView(this, lang);
			this.source_view_handlers.add(this.childview);
			this.create_textview();
		}
```

## Acceptance criteria

1. Consecutive fenced blocks: less gap ( **`prev_sibling`** + **`current`** removal + margin) without full box scans.
2. **Non-whitespace** in either view: that view **not** removed.
3. **No** **`get_last_child`** / multi-step box walks — only **`get_prev_sibling()`** once plus **`current_textview`**.
4. Build passes.
