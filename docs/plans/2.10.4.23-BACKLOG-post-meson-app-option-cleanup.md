## Purpose

- **🔷** Remove **app CLI options** that only applied to **in-process** indexing / SQLite startup and have **no caller** after v2.
- **🔷** First target: **`--disable-indexer`** / **`OllmchatApplication.opt_disable_indexer`**.
- **🚫** not in **2.10.4.22** scope — leave **`Application.vala`** alone until post-Meson cleanup.

---

## `--disable-indexer` today

| Piece | Role |
|-------|------|
| **`ollmapp/Application.vala`** | **`opt_disable_indexer`** static + **`OptionEntry`** **`disable-indexer`** |
| **Shipping `ollmapp/Window.vala`** | Skips creating **`OLLMvector.BackgroundScan`** when flag set |
| **V2 `ollmapp/V2/Window.vala`** | **🚫** no **`BackgroundScan`** — flag is unused when **`window_src`** is V2 |

**ℹ️** Semantic indexing runs on **`ollmfilesd`** (**`Vector/BackgroundScan`**) after cutover. Disabling it from the **GUI app** via this flag no longer maps to behaviour.

**💩** If product still wants “disable indexer”, add a **daemon** CLI/config hook later — out of scope here unless requested.
