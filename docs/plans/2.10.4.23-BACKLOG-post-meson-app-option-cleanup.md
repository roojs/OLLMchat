### Display vs domain: methods on backend, properties on client

**🔷** Split rule after cleanup:

- **Backend (`ollmfilesd/`)** — domain checks and derived values as **methods** (or inline `this is FileAlias`), **not** GObject computed **properties**. Stops `bin_write` / `list_properties()` from seeing display or derived surface; no omit switches needed for things that are not properties.
- **Client (`libocfiles/V2/`)** — keep **computed properties** where Gtk needs them (`bind_property`, list models, templates). Display derives locally from stored wire fields (`path`, `is_need_approval`, …).

| Surface | Client (`libocfiles/V2`) | Daemon (`ollmfilesd`) | Wire (bin) |
| -------- | ------------------------ | --------------------- | ----------- |
| `path_basename` | **🔷** computed **property** (binding) | **⏳** method or inline `Path.get_basename(path)` | **🚫** not sent |
| `icon_name` | **🔷** computed **property** (binding) | **🚫** not on type | **🚫** not sent |
| `display_with_indicators` | **🔷** computed **property** (binding) | **🚫** not on type | **🚫** not sent |
| `is_alias` | computed **property** (`this is FileAlias`) | **⏳** replace property with **method** / type check; used by Indexer, FileHistory | **🚫** not sent |

**⏳** Backend: convert **`is_alias`** (and any other derived **property** left on **`ollmfilesd/FileBase`**) to a **method**; update call sites (`FileHistory`, **`Vector/Indexer`**, …).

**⏳** Remove interim **bin** omit cases on daemon for properties that **do not exist** on daemon types (`path-basename`, `icon-name`, `display-with-indicators`) once audit confirms they never appear in `list_properties()`.

**⏳** After backend conversion, drop **`is-alias`** from daemon **bin** / **JSON** omit switches — it will no longer be a **`ParamSpec`**.

**ℹ️** **JSON** (`serialize_property` / stdio NDJSON tests) still omits graph edges only (`manager`, `parent`, …). Display props were **never** on daemon JSON types; do **not** extend JSON omits for client-only display surface.

### Move types to the right layer

**⏳** Audit shared / duplicated **`FileBase`** surface after v2 default:

- **Daemon (`ollmfilesd/`)** — DB, scan, disk I/O, vector index, RPC handlers only; **derived/domain as methods**, not computed properties
- **Client (`libocfiles/V2/`)** — RPC hydration, **computed properties for Gtk binding**, buffers, **`ProjectManager`** list UX
- **⏳** Remove dead JSON/bin omit switches that reference client-only or non-existent **`ParamSpec`** names
- **⏳** Confirm **`ollmfilesd/*`** vs **`libocfiles/V2/*`** naming and registration (`Bin.Stream.register`) match one class per wire alias

**ℹ️** Shipping **`libocfiles/Folder.vala`** (non-V2) deletion is tied to **`v2testing`** default — same milestone as shipping **`Window.vala`** removal.
