## Purpose

- **🔷** Serialize outgoing NDJSON writes on the single **`DataOutputStream`** in **`libocrpc/Client.vala`**
- **🔷** Keep one socket, many in-flight calls — only the **write lane** is queued
- **🔷** Fix **`write: Stream has outstanding operation`** on session restore (**`Folder.fetch_files`**, **`Folder.fetch_pending_approvals`**, **`ProjectManager.activate_project`**)
- **⏳** Smoke — app startup / session restore: no write warnings; all three RPCs reach daemon
- **ℹ️** Root cause — concurrent **`yield this.write()`** → concurrent **`flush_async()`** on one stream

---

## Current behaviour

- **ℹ️** **`call()`** and **`connect()`** both call private **`write()`** directly
- **ℹ️** **`write()`** does **`put_string`** + **`yield flush_async`** with no mutual exclusion
- **ℹ️** **`activate_project()`** emits **`active_project_changed`**, then starts **`review_files.refresh.begin()`** and **`rpc.call.begin()`** — three writes can hit the stream in the same main-loop tick
- **ℹ️** GLib rejects overlapping async ops → client-side failure before wire

---

## Proposed behaviour

- **🔷** **`write()`** enqueues a job and yields until that job's flush completes
- **🔷** One **`drain_writes()`** loop drains the queue FIFO — one **`flush_async`** at a time
- **🔷** **`call()`** / **`connect()`** unchanged at call sites — fix is entirely inside **`Client`**
- **🔷** **`disconnect()`** fails any queued jobs with **`Client: disconnected`**

## Concrete code proposals

Hunks are **Remove** / **Replace with** / **Add** from the tree; verify context before applying.

### 1. `libocrpc/Client.vala` — nested job type + queue fields

**Why:** Pair each payload with a completion promise so concurrent **`write()`** callers can wait without touching the stream in parallel.

**Where:** **`PendingWrite`** immediately before **`public class Client`**; queue fields inside **`Client`** after the **`pending`** field block.

**Depends on:** none

#### Add — `PendingWrite` holder before `public class Client`

```vala
	private class PendingWrite : GLib.Object
	{
		public GLib.Object payload { get; construct; }
		public Gee.Promise<bool> done { get; construct; }

		public PendingWrite(GLib.Object payload)
		{
			Object(
				payload: payload,
				done: new Gee.Promise<bool>()
			);
		}
	}
```

#### Add — queue fields inside `Client`, after `pending` block

```vala
		private Gee.ArrayList<PendingWrite> write_queue {
			get; private set;
			default = new Gee.ArrayList<PendingWrite>();
		}
		private bool write_draining { get; set; default = false; }
```

### 2. `libocrpc/Client.vala` — `write()` queue entry + `drain_writes()` drain loop

**Why:** Move actual I/O into a single drain loop so **`flush_async`** never overlaps.

**Where:** Replace existing **`write()`** method; add **`drain_writes()`** immediately after it.

**Depends on:** §1

#### Remove

```vala
		private async void write(GLib.Object gobject) throws GLib.Error
		{
			var generator = new Json.Generator();
			generator.set_pretty(false);
			generator.set_root(Json.gobject_serialize(gobject));
			this.output.put_string(generator.to_data(null) + "\n");
			yield this.output.flush_async(GLib.Priority.DEFAULT, null);
		}
```

#### Replace with — enqueue, kick drain once, yield per job

```vala
		private async void write(GLib.Object gobject) throws GLib.Error
		{
			var job = new PendingWrite(gobject);
			this.write_queue.add(job);
			if (!this.write_draining) {
				this.write_draining = true;
				this.drain_writes.begin();
			}
			yield job.done.future.wait_async();
		}

		private async void drain_writes()
		{
			while (this.write_queue.size > 0) {
				var job = this.write_queue.get(0);
				this.write_queue.remove_at(0);
				if (!this.connected || this.output == null) {
					job.done.set_exception(
						new GLib.IOError.FAILED("Client: disconnected")
					);
					continue;
				}
				try {
					var generator = new Json.Generator();
					generator.set_pretty(false);
					generator.set_root(
						Json.gobject_serialize(job.payload)
					);
					this.output.put_string(
						generator.to_data(null) + "\n"
					);
					yield this.output.flush_async(
						GLib.Priority.DEFAULT,
						null
					);
					job.done.set_value(true);
				} catch (GLib.Error e) {
					job.done.set_exception(e);
				}
			}
			this.write_draining = false;
		}
```

**ℹ️** **`connect()`** hello and every **`call()`** still use **`yield this.write(...)`** — no caller changes.

### 3. `libocrpc/Client.vala` — `disconnect()`: fail queued writes

**Why:** Callers blocked in **`yield write()`** must not hang after disconnect.

**Where:** Start of **`disconnect()`**, after **`if (!this.connected) { return; }`**, before **`this.connected = false`**.

**Depends on:** §1

#### Add

```vala
			foreach (var job in this.write_queue) {
				job.done.set_exception(
					new GLib.IOError.FAILED("Client: disconnected")
				);
			}
			this.write_queue.clear();
			this.write_draining = false;
```
