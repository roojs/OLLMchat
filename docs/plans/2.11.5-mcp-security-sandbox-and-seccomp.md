## Purpose

- Harden **stdio** MCP servers: bubblewrap filesystem policy, seccomp user-notify with **actionable** user messages, spawn rules for host vs parent sandbox (Flatpak).
- Reuse **`run_command`** sandbox code (**R9**). Do **not** duplicate bwrap/seccomp in `libocmcp`.
- HTTP MCP: trust `url` only (**R11**). No local child sandbox.

## Scope

| In scope | Out of scope |
|----------|----------------|
| `allow_write`, `trust_sandbox` on `OLLMmcp.Config` (**R10**) | Runtime permission prompts (**R4**) |
| Project overlay + `$HOME/playground` when project open (**R1**, same as RunCommand) | Per-call `allow_write` / `network` on MCP tools |
| Seccomp user-notify on stdio spawn (**R5**) | Validating MCP `args` paths (**R7**) |
| Spawn: host+bwrap; refuse host without bwrap; Flatpak + `trust_sandbox` (**R6–R8**, **R7**) | HTTP URL allowlists (**R11**) |
| `liboctools`: refactor `Bubble.build_bubble_args` (`exec_argv` after `--`) | Duplicate bwrap mount builder for MCP |
| Seccomp appendix on tool **results** when a call hits policy (**R5**) | `sandbox_note` / extra text in `parameter_description`; pretooler; MCP supply-chain audit |

## Policy (authoritative)

### 🔷 User requirements (stdio)

- **🔷 Project overlay** when a project is open — wire `project_manager` through Loader (**R1**). **`$HOME/playground`** when project is open — same as RunCommand (no separate MCP opt-out).
- **🔷 `ProjectManager` non-nullable** on `fill_tools`, `Loader.run`, `create_client`, and client constructors (same signature as tools/vector registries). **Http** takes it and does not use it (no local sandbox). **No project open** is `project_manager.active_project == null` only — **not** a nullable manager parameter.
- **🔷 `allow_write` in mcp.json** — string (`:`-separated absolute roots) **or** JSON array (**R10**). Additive with project overlay when project is open.
- **🔷 No project:** read-only host (`--ro-bind / /`); writable **`/tmp` only** unless `allow_write` lists extra roots. No config-load validation of paths or project state.
- **🔷 No permission prompts** — policy fixed in `mcp.json` at spawn (**R4**).
- **🔷 Seccomp** must tell the user what was blocked and how to fix (`network`, `allow_write`, open project) (**R5**).
- **🔷 No raw unsandboxed spawn on host** (**R6**). **🔷 Inside parent sandbox** (e.g. Flatpak): stdio MCP only if `trust_sandbox: true`; then direct spawn, no nested bwrap (**R7**). **🔷 Host without bwrap:** refuse stdio MCP (**R8**).
- **🔷 Reuse `Bubble` / `RunSeccomp`** from `liboctools` (**R9**).
- **🔷 Do not validate `command` / `args`** against binds (**R7**).

### 🚫 Rejected / do not implement

- **🚫** Spawn-time or tool-call approval UI for MCP.
- **🚫** HTTP URL filtering / SSRF blocklists.
- **🚫** Injecting sandbox policy into MCP `tools/call` JSON.
- **🚫** `sandbox_note`, `build_sandbox_note`, or other sandbox prose baked into MCP `Tool.parameter_description` (agent guidance belongs in seccomp **result** text per **R5**, not static tool schema).

### ℹ️ HTTP

- Trust `config.url`. No bwrap on HTTP transport.

## Current behaviour

- **⏳** `Client.Stdio` — minimal bwrap (`--ro-bind /`, `/tmp`, optional `--unshare-net`); Flatpak skips bwrap and uses raw spawn; no overlay, seccomp, or `allow_write` (`libocmcp/Client/Stdio.vala`).
- **⏳** `Registry.fill_tools` — takes `project_manager` but `Loader.run` does not (`libocmcp/Registry.vala`, `libocmcp/Loader.vala`).
- **⏳** `Config` — `network`, `env` only; no `allow_write` or `trust_sandbox` (`libocmcp/Config.vala`).
- **✅** `config.env` applied at spawn via `SubprocessLauncher.setenv`.

## mcp.json fields (stdio)

| Field | Meaning |
|-------|---------|
| `network` | `true` → omit `--unshare-net` (default `false`) |
| `allow_write` | Omitted: no extra host binds (no project: `/tmp` only; project: overlay only). String or array of absolute roots — **additive** with project. |
| `trust_sandbox` | Required `true` to run stdio MCP when app is inside parent sandbox (e.g. Flatpak) |
| `command` / `args` / `env` | Unchanged; **not** validated against binds |
