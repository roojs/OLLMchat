## Concrete code proposals

Land these **before** [`2.10.4.9.1`](2.10.4.9.1-design-project-files-and-client-api.md) client hunks.

### 1. `ollmfilesd/CallParam.vala` тАФ params for scoped Folder/File RPC

**Why:** `File.fetch` and `Folder.contains_folder` need project + path.

**Where:** After `FileParams` class.

**Depends on:** none.

#### Add тАФ `project_path` on `FileParams`

Placement: inside `FileParams`, immediately after the `path` property.

```vala
		public string project_path { get; set; default = ""; }
```

#### Add тАФ `FolderParams` class

Placement: after closing `}` of `FileParams`, before `VectorParams`.

```vala
	/** {@code Folder.*} request params (project-scoped). */
	public class FolderParams : OLLMrpc.CallParam
	{
		public string project_path { get; set; default = ""; }
		public string path { get; set; default = ""; }
	}
```

**Also:** register `FolderParams` in RPC type registry (same pattern as `ProjectParams` / `FileParams` in `RpcClient` / daemon startup).

### 2. `ollmfilesd/Listen.vala` тАФ dispatch `Folder.*`

**Why:** Wire object matches client API ([`2.10.4.9.1`](2.10.4.9.1-design-project-files-and-client-api.md)).

**Where:** `Listen` properties and RPC dispatch routing (mirror `ProjectManager` / `File` pattern from [`2.10.4.4`](done/2.10.4.4-DONE-projectmanager.md)).

**Depends on:** ┬з3 daemon `Folder` relays.

*(Full Remove/Replace hunks when implementing тАФ add `Folder` property on `Listen` + route `Folder.*` methods to `app.folder` relay. Same pattern as `File.*` in [`2.10.4.6`](done/2.10.4.6-DONE-file.md).)*

### 3. `ollmfilesd/Folder.vala` тАФ `Folder.fetch` and project-scoped relays

**Why:** Daemon reads `project_files` here; not via PM indirection.

**Where:** `ollmfilesd/Folder.vala` тАФ new `rpc_*` handlers in `construct` (mirror `ollmfilesd/File.vala` relay style).

**Depends on:** ┬з1 params.

**Handler behaviour (implement verbatim logic in relays):**

- **`Folder.fetch`** тАФ `ProjectParams.path` тЖТ lookup folder in any project `folder_map` or DB тЖТ return `Folder` row
- **`File.fetch`** тАФ `FileParams.path` + `project_path` тЖТ `project_files.child_map` on daemon project folder
- **`Folder.contains_folder`** тАФ `FolderParams` тЖТ `folder_map.has_key`
- **`Folder.fetch_file_list`** тАФ `ProjectParams.path` тЖТ project root тЖТ return `ArrayList<File>` from `project_files` items
- **`Folder.fetch_pending_approvals`** тАФ same root тЖТ files from `review_files`

*(Full construct-block fences when opening `ollmfilesd/Folder.vala` for edit тАФ file is large; one **Add** relay block per method.)*

### 4. Remove `ProjectManager.get_folder_at_path` daemon relay

**Why:** Replaced by **`Folder.fetch`** ([`2.10.4.9.1`](2.10.4.9.1-design-project-files-and-client-api.md)).

**Where:** `ollmfilesd/ProjectManager.vala` and any `Listen` route for `ProjectManager.get_folder_at_path`.

**Depends on:** ┬з3 `Folder.fetch` green.

#### Remove

Remove `rpc_get_folder_at_path` handler and `get_folder_at_path` RPC entry point if present; keep internal `get_folder_at_path` only if still used locally тАФ rename to private helper or inline into `Folder.fetch` relay.
