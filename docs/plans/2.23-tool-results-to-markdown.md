# Tool results as markdown in skills agent only

**Status:** Plan  
**Scope:** BaseTool; skills agent (Task.Tool, Details.executor_precursor); tools used by skills agent.

**Goal:** Standard tools keep current return; skills agent path only calls `result_to_markdown(result)` and uses **header_raw** for tool output. Tool markdown uses **####** headings (level 4 under `## Tool call … result`); no extra indent inside fenced blocks.

---

## 1. BaseTool — add result_to_markdown

**File:** `libollmchat/Tool/BaseTool.vala`

```vala
public virtual string result_to_markdown(string raw_result)
{
	return raw_result;
}
```

---

## 2. Skills agent — store markdown, use header_raw

**File:** `liboccoder/Task/Tool.vala` (in execute(), after yield tool_impl.execute):

```vala
var result = yield tool_impl.execute(this.task.chat(), this.tool_call);
this.task.tool_outputs.set(this.tool_call.id, tool_impl.result_to_markdown(result));
```

**File:** `liboccoder/Task/Details.vala` (executor_precursor), tool output line:

```vala
// was:
parts += this.header_fenced("### Tool call " + e.key + " Output", e.value, "text");
// change to:
parts += this.header_raw("Tool call " + e.key + " result", e.value);
```

## 3. Per-tool code changes

### codebase_search — heading level only

**File:** `libocvector/Search/SearchResult.vala` in `to_markdown()`:

```vala
// change:
return "## Result (distance: " + ...
// to:
return "#### Result (distance: " + ...
```

No override on CodebaseSearchTool (default identity).

### read_file — override result_to_markdown

**File:** `liboctools/ReadFile/Tool.vala`

```vala
public override string result_to_markdown(string raw_result)
{
	return "#### Read file\n\n**Result**\n\n" + "```\n" + raw_result + "\n```\n";
}
```

(If raw is error/short, could use no fence; same pattern, one expression.)
