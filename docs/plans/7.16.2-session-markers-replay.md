## Purpose

- **🔷** Bump **session `schema_version`** on new skill sessions.
- **🔷** Add **richer transcript markers** so **`Runner.on_replay`** can hydrate **`Action`** runs without **`replay_tool_pos`**-style guessing.
- **🔷** Replay **phase 1** flows only — path **B** still **`RunTools`** (legacy multi-tool), not **`Iterate`** yet.
- **🔷** Old transcripts: **`can_replay = false`** or chat-only — no progress lie.

---

## Scope

- **`SessionJson`** / save-load — **`schema_version`** field.
- **`Skill.Runner`** — emit markers from **`Action.*`** / **`Details`** during live runs.
- **`Runner.on_replay`** — rewrite for **`Action`** parity paths.
- **🚫** No **`REPLAY_PROTOCOL`** const on **`Runner`** — version lives on session (**🔷** parent).
- **ℹ️** Related done work: [`docs/plans/done/7.13-DONE-agent-save-restore.md`](done/7.13-DONE-agent-save-restore.md), [`docs/plans/done/7.14.5-DONE-restore-replay.md`](done/7.14.5-DONE-restore-replay.md).

## Today

- **ℹ️** **`can_replay`** set when any **`agent-stage`** message exists.
- **ℹ️** Replay maps **`agent-stage`** body via **`PhaseEnum.from_string`** (`refinement`, `exec`, `post_exec`, …).
- **ℹ️** Multi-tool tasks advance **`replay_tool_pos`** across N **`Tool`** rows then **`POST_EXEC`** (`Runner.vala` ~978–1090).
- **ℹ️** Fragile when message order and cursor assumptions diverge.

## Target markers (**💩** draft — expand in implementation)

### Session schema

- **🔷** New skill sessions: **`schema_version = 2`** (or next free int) on create and in JSON on disk.
- **🔷** Replay requires **`schema_version >= 2`** and recognized marker set.
- **ℹ️** Skills agent: **`SessionJson.agent_name`**.

### `agent-stage` bodies (parity / phase 1)

- **💩** `⏳` Stable ids tied to **`Action`** runners, e.g.:
  - **`action_exam`** — exam sequence active
  - **`action_tool`** — registry tool + executor pass
  - **`action_post_exam`** — post-exam merge
  - **`action_ref_only`**, **`action_write_exec`** — paths **C / D**
- **ℹ️** Keep existing where still needed: **`refinement`**, **`exec`**, **`exec_validate`** during transition **💩** `⏳` or map 1:1.

### Optional meta rows (**💩**)

- **💩** `⏳` **`agent-action-meta`** or fenced **`ui`** JSON, e.g. **`tool_id`**, **`action_class`**, **`exam_index`**.
- **🔷** Goal: replay reads meta instead of inferring from message order alone.
- **ℹ️** Phase 3 extends with **`cycle`** — see **7.16.3**.

## Replay changes

- **🔷** Hydrate **`Action`** dispatch paths using new markers.
- **🔷** Path **B**: still multi-tool + merge transcript shape until **7.16.3**.
- **🔷** Drop reliance on **`replay_tool_pos`** stepping where markers suffice.
- **🚫** Do not require replay of pre-schema sessions.
- **ℹ️** **`docs/plans/7.14.11-ON-HOLD-tool-call-summary-replay-tooltip.md`** — revisit after marker shape settles.
