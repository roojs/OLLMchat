# 2.11.4. MCP Meson and App Wiring

## Overview

Phase 4 of the MCP loader: add libocmcp to the build and wire `OLLMmcp.Registry` into the application so MCP tools are loaded at runtime. This is the first phase where libocmcp is compiled and included in the build.

Parent: [2.11-mcp-loader-tool.md](2.11-mcp-loader-tool.md)

## Scope

- **Meson:** Add libocmcp as a library (or subproject): list all sources under `libocmcp/` (Config, Client, Factory/, Tool/, Loader, Registry, Stdio/Client, Http/Client). Link libocmcp where needed (e.g. main app, ollmchat-cli). Ensure dependencies (e.g. libollmchat, etc.) are declared. Add to top-level meson.build or equivalent so the project builds with libocmcp.
- **Application:** Add `mcp_registry` property (e.g. `OLLMmcp.Registry`). In startup path (after tools_registry and vector_registry): instantiate `mcp_registry`, call `mcp_registry.init_config()`, and if config is loaded call `mcp_registry.setup_config_defaults(config)`.
- **Window:** After `tools_registry.fill_tools()` and `vector_registry.fill_tools()`, call `app.mcp_registry.fill_tools(this.history_manager, this.project_manager)` so MCP tools are registered for the session.
- **ollmchat-cli:** If it uses registries the same way, add `mcp_registry`, `init_config()`, `setup_config_defaults()`, and `fill_tools()` in the same order as in the GUI app.

## Files to touch (outside libocmcp)

- Root or relevant `meson.build` – add libocmcp library and its sources; add dependency for executables that use it.
- `ollmapp/Application.vala` (or equivalent) – add `mcp_registry`, create it, call `init_config()` and `setup_config_defaults(config)`.
- `ollmapp/Window.vala` – call `mcp_registry.fill_tools(history_manager, project_manager)` after the other registries.
- `ollmapp/ollmchat-cli.vala` – same wiring if it has its own registry flow.

## Completion

- Full build succeeds with libocmcp.
- On startup, if `~/.config/ollmchat/mcp.json` exists and has enabled servers, MCP tools appear in the tool list and are callable by the agent.
- **First integration test:** Chrome MCP (stdio, with `network: true` in config) running under bwrap; confirm tools list and at least one tool call works.
