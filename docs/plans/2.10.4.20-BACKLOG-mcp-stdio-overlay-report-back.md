## Purpose

- **🔷** Sandboxed **MCP stdio** (`libocmcp/Client/Stdio.vala`) spawns inside a bwrap overlay with project write roots.
- **🔷** Today it uses **`OLLMbwrap.NoOpFileVerification`** — overlay creates/edits/deletes are **not** merged to disk or the project index.
- **🔷** On **`disconnect`**, overlay upper-layer changes are **lost** unless we run the same scan + apply pipeline as **`run_command`**.

**Reference (working):** `liboctools/RunCommandV2/Request.vala` — **`OLLMtools.FileVerification`** + **`yield bubble.exec(...)`** (scan + apply inside **`Bubble.exec`** finally).

---

## Gap today

| Path | Overlay scan on exit | Index / disk apply |
|------|----------------------|-------------------|
| **`run_command`** (V2) | **✔️** in **`Bubble.exec`** | **✔️** **`FileVerification`** RPC |
| **MCP stdio** | **🚫** | **🚫** **`NoOpFileVerification`** |

**Code markers:**

- `libocmcp/Client/Stdio.vala` — class docblock + **`disconnect()`** TODO + **`build_spawn_argv()`** TODO
- `libocbwrap/NoOpFileVerification.vala` — intentional no-op (keep for callers that never scan)

**Implementation note:** **`build_spawn_argv()`** constructs **`Bubble`** on the stack; **`disconnect()`** has no **`Bubble`** / overlay handle today. Plan must **retain **`Bubble`**** (or upper path + write roots) on the **`Stdio`** instance for teardown scan.

---

## Agreed direction (🔷 sketch — not approved)

1. **Keep **`NoOpFileVerification`** during the MCP session** (no per-hook RPC while server runs).
2. **On `disconnect`** (before **`force_exit`** if upper layer must be read):
   - run **`bubble.overlay.scan.run()`** (same as **`RunCommandV2`**)
   - use a real **`FileVerification`** impl for apply hooks — likely **`OLLMtools.FileVerification`** once V2 / **`ollmfilesd`** is linked; shipping in-process apply until then
3. **Wire **`RunSeccomp`** teardown** so scan runs while overlay paths still exist.
4. **🚫** no **`libocmcp/V2/`** — in-place **`Stdio.vala`** change only (same as bwrap repoint in cutover plan).

**Open product question:** report-back **only on disconnect**, or also after each **`tools/call`** that used the overlay? Class docblock mentions both; **`run_command`** applies once per exec.
