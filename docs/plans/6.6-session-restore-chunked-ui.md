# 6.6 Session restore: chunked UI replay and `GLib.Cancellable`

**Status:** Not started

**Note on numbering:** The previous **6.6** item (write executor `next_section` heading validation) is completed and archived as `docs/plans/done/6.6-DONE-write-executor-next-section-heading-validation.md`.

---

## Purpose

Restoring a large saved session blocks the GTK main loop: `Session.restore_messages()` runs synchronously inside `Session.activate()` and calls `Manager.message_added` for every stored message, each of which drives heavy chat UI work (`append_complete_assistant_message`, markdown, etc.). The UI appears to hang until the full history is rendered.

This plan defines an implementation that **yields between messages** (e.g. ~100 ms via `GLib.Timeout`) so the main loop stays responsive, and uses **`GLib.Cancellable`** to stop in-flight replay when the user switches to another session (or another operation cancels the same token). **No extra visible “loading” chrome** is required: an empty chat filling in progressively is enough feedback.

---

## Current behaviour (reference)

- `Manager.switch_to_session()` calls `loaded_session.activate()`.
- `Session.activate()` calls `restore_messages()`, which loops over `session.messages` and emits `manager.message_added(...)` for each non-transient message in one shot.
- `ChatWidget` connects to `message_added` and renders each message; `ChatWidget.load_messages()` exists but is not the primary path for session switch (restore goes through signals).

---

## Proposed behaviour

1. **Do not** replay the full history synchronously inside `activate()`.
2. After `activate()`, run **`async` replay** that emits one stored message, **`yield`**s (~100 ms), repeats. Optionally tune delay or batching later.
3. **Cancellation:** `Manager` owns a **`GLib.Cancellable`** used only for the current session-restore replay. At each **`switch_to_session` start**, **`cancel()`** the previous cancellable (if any), then **`new GLib.Cancellable()`** for this switch. Pass that instance into **`restore_messages_async`**. Between chunks, if **`cancellable.is_cancelled()`**, return immediately (optionally also check **`manager.session != this`** as a defensive guard).
4. **Signals:** Emit **`session_history_restored(session)`** after replay completes normally, after cancel (early return), or immediately when there is nothing to replay—so `ChatWidget` can clear `restoring_session`, re-enable scroll, and scroll to bottom. Adjust **`session_activated`** so it does not clear `restoring_session` before history is on screen.

---

## Why `GLib.Cancellable` instead of a generation counter

- Standard GLib pattern for cooperative cancellation; works with other APIs that already take a `Cancellable?`.
- **`cancel()`** is explicit: starting a new switch invalidates the previous restore without maintaining a separate monotonic **`uint`**.
- Optional: still assert **`manager.session == this`** in the loop if you want belt-and-suspenders against stray main-loop continuations.

---

## Scope

| In scope | Out of scope |
| -------- | -------------- |
| `libollmchat/History/Session.vala` — move restore out of `activate()`; `async` replay + `Cancellable` | Chunking **content** inside a single huge assistant message |
| `libollmchat/History/Manager.vala` — cancellable lifecycle, `yield` replay after activate, `session_history_restored` | Changing persistence format or DB load |
| `libollmchatgtk/ChatWidget.vala` — `session_history_restored`, `session_activated` / `restoring_session`, error paths (**no** waiting indicator) | Redesign of history sidebar |

---

## Implementation notes (for whoever implements)

- Reuse the **streaming tail** logic at the end of the old `restore_messages()` (last `content-stream` / `think-stream` when `is_running`) unchanged in ordering; run it **after** all prior messages have been replayed.
- **`ChatWidget.load_messages()`** — optional follow-up: delete dead code.
- Verify **EmptySession** and **SessionPlaceholder → Session** paths: replay only for real **`Session`** with **`messages.size > 0`**.
- Test: very large session, switch away mid-restore, ensure cancel stops old replay and new session loads cleanly.

---

## Acceptance criteria

- Restoring a session with many messages keeps the window responsive (UI events processed between chunks).
- User can select another session while the first is still loading; in-flight replay is cancelled via **`Cancellable`** and the new session restores correctly.
- No duplicate messages; streaming mid-restore edge cases remain correct.

---

## Concrete code proposals

Below is **fully usable Vala** aligned with the current tree. **No** `show_waiting_indicator` / **`clear_waiting_indicator`** for restore—only `restoring_session` + scroll handling.

Tune **`RESTORE_MESSAGE_DELAY_MS`** (e.g. `50`–`150`) as needed.

### 1. `libollmchat/History/Manager.vala`

**Add** after `public SessionBase session` (or with other private fields):

```vala
		/** Cancelled when a new `switch_to_session` starts so in-flight `restore_messages_async` stops. */
		private GLib.Cancellable? session_restore_cancellable;
```

**Add** after `session_activated` signal:

```vala
		/** After async history replay finishes, was cancelled, or there was nothing to replay. */
		public signal void session_history_restored(SessionBase session);
```

**Replace** the entire body of `switch_to_session` with:

```vala
		public async void switch_to_session(SessionBase session) throws Error
		{
			if (this.session_restore_cancellable != null) {
				this.session_restore_cancellable.cancel();
			}
			this.session_restore_cancellable = new GLib.Cancellable();
			var cancellable = this.session_restore_cancellable;

			this.session.deactivate();

			SessionBase? loaded_session = yield session.load();

			if (loaded_session == null) {
				throw new GLib.IOError.FAILED("Session load returned null");
			}

			this.session = loaded_session;
			loaded_session.activate();

			this.session_activated(loaded_session);
			this.agent_status_change();

			if (loaded_session is Session) {
				var hist = loaded_session as Session;
				if (hist.messages.size > 0) {
					yield hist.restore_messages_async(cancellable);
				}
			}

			this.session_history_restored(loaded_session);
		}
```

### 2. `libollmchat/History/Session.vala`

**Replace** `activate()` with:

```vala
		public override void activate()
		{
			base.activate();
			// History replay: `Manager.switch_to_session` calls `restore_messages_async` after `activate()`.
		}
```

**Remove** the private method **`restore_messages()`** entirely.

**Add** in its place:

```vala
		private const uint RESTORE_MESSAGE_DELAY_MS = 100;

		private async void yield_ms(uint ms)
		{
			GLib.Timeout.add(ms, () => {
				this.yield_ms.callback();
				return false;
			});
			yield;
		}

		/**
		 * Replays persisted messages to the UI with delays so the main loop stays responsive.
		 * Stops when `cancellable` is cancelled (e.g. user switched session) or this session is no longer active.
		 */
		internal async void restore_messages_async(GLib.Cancellable cancellable)
		{
			if (this.messages.size == 0) {
				return;
			}

			for (int i = 0; i < this.messages.size - 1; i++) {
				if (cancellable.is_cancelled() || this.manager.session != this) {
					return;
				}
				if (this.messages[i].role == "ui-waiting") {
					continue;
				}
				this.manager.message_added(this.messages[i], this);
				yield this.yield_ms(RESTORE_MESSAGE_DELAY_MS);
			}

			if (cancellable.is_cancelled() || this.manager.session != this) {
				return;
			}

			var last_msg = this.messages[this.messages.size - 1];
			if (last_msg.role == "ui-waiting") {
				return;
			}

			if (!this.is_running ||
				(last_msg.role != "content-stream" && last_msg.role != "think-stream")) {
				this.manager.message_added(last_msg, this);
				return;
			}

			this.current_stream_message = last_msg;
			this.current_stream_is_thinking = (last_msg.role == "think-stream");

			var dummy_message = new OLLMchat.Message("assistant", "", "");
			var dummy_response = new Response.Chat(
					this.agent.chat().connection, this.agent.chat()) {
				message = dummy_message,
				done = false,
				is_thinking = this.current_stream_is_thinking
			};

			base.handle_stream_chunk(last_msg.content, this.current_stream_is_thinking, dummy_response);
		}
```

### 3. `libollmchatgtk/ChatWidget.vala`

**Replace** the `session_activated` handler and **add** `session_history_restored` (no waiting-indicator calls):

```vala
			this.manager.session_activated.connect((session) => {
				GLib.Idle.add(() => {
					if (!this.restoring_session && session.is_running) {
						this.chat_view.scroll_enabled = true;
						this.chat_view.scroll_to_bottom();
					}
					return false;
				});
			});

			this.manager.session_history_restored.connect((_session) => {
				this.restoring_session = false;
				GLib.Idle.add(() => {
					this.chat_view.scroll_enabled = true;
					this.chat_view.scroll_to_bottom();
					return false;
				});
			});
```

**Replace** `switch_to_session` with (error path does **not** touch waiting indicator):

```vala
		public async void switch_to_session(OLLMchat.History.SessionBase session)
		{
			this.chat_view.finalize_assistant_message();
			this.streaming_state(true);
			this.clear_chat();
			this.chat_view.scroll_enabled = false;
			this.restoring_session = true;

			try {
				yield this.manager.switch_to_session(session);
			} catch (Error e) {
				GLib.warning("Error loading session: %s", e.message);
				this.streaming_state(false);
				this.chat_view.scroll_enabled = true;
				this.restoring_session = false;
				return;
			}
		}
```

**Do not** add any **`clear_waiting_indicator`** / **`show_waiting_indicator`** in **`on_message_created`** for restore.

### 5. Behaviour summary (post-patch)

| Step | What happens |
| ---- | ------------ |
| User switches session | `ChatWidget` clears chat, sets `restoring_session`; chat area repopulates as messages replay. |
| `Manager.switch_to_session` | Cancels previous **`session_restore_cancellable`**, allocates a new one, loads, **`activate()`**, **`session_activated`** + **`agent_status_change`**, **`yield restore_messages_async(cancellable)`**, then **`session_history_restored`**. |
| Mid-restore switch | New switch **`cancel()`**s the old cancellable; old **`restore_messages_async`** returns at next check; new switch clears UI and runs a new replay. |
| End | **`session_history_restored`** clears **`restoring_session`**, re-enables scroll, scrolls to bottom. |
