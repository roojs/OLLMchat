# 2.10.4.2 — `ollmfilesd` RPC coverage map (implementers)

**Status:** PROPOSAL — appendix to [`2.10.4.1-ollmfilesd-rpc-api.md`](2.10.4.1-ollmfilesd-rpc-api.md).

Use this when wiring `libocfiles` client stubs: check every today’s caller has a matching `kind.verb` or is listed client-only.

---

## Client-only (no RPC)

- `File.buffer`, `buffer_provider`, `create_buffer` after `file.read`
- Gtk: re-emit `active_project_changed`, `active_file_changed`, `file_metadata_changed` locally
- Chat, session, history
- `File.new_fake` until `file.register`

---

## Daemon-internal (no RPC)

- `GitProvider` during scan
- `FileHistory.cleanup_old_backups`
- `delete_manager.on_cleanup` / `VectorMetadata.cleanup_all_deleted` (triggered by `file.delete`)
- Full `Folder.children` graph in daemon memory
- `VectorBuilder`, `Analysis`, `DocumentationAnalysis`, `FolderAnalysis` (inside index queue)
- `Database.save_index`, `add_vectors_batch` (inside indexer)

---

## Map: today → RPC

**ProjectManager / projects**

- `load_projects_from_db` → `projects.list`
- `create_project` → `projects.create`
- `remove_project` → `projects.remove`
- `activate_project` → `project.activate` (`skip_scan`)
- `restore_active_state` → client: `projects.list` + `project.activate` + `file.activate`
- `get_folder_at_path` → `file.get` (or `path` row in snapshot)
- `notify_project_changed` → `event.project.changed`

**Files / editor**

- `get_file_from_active_project` → `file.get`
- `read` / reload buffer → `file.read`
- `write_buffer_to_disk`, `on_file_contents_change` → `file.write`
- `check_active_file_changed` → `file.changed.check`
- `on_file_metadata_change` → `file.metadata.set`
- `activate_file` → `file.activate`
- `convert_fake_file_to_real` → `file.register`

**Scan / index / search (`libocfiles` + `libocvector`)**

- `read_dir` → `project.scan`
- per-path scan → `path.scan`
- `BackgroundScan` → `project.index`, `file.index`, `event.project.index.progress`
- `Indexer.index_filebase` (force, recurse) → `project.index` params; queue in daemon
- `Indexer.element_scanned` → `event.project.index.element` (optional UI)
- `ProjectAnalysis` / `--project-summary` → `vector.project.describe` or `project.index` + `project_summary_only`
- `CodebaseSearchTool.init_databases` → daemon startup + `vector.status`
- `RequestCodebaseSearch` / `Search.execute` → `vector.search` (`format`: tool | json)
- `oc-vector-search --show-info` → `vector.list`
- `oc-vector-search --dump-vector` → `vector.get`
- `Indexer.reset_database` → `vector.reset`
- `Folder.get_project_description` → `vector.project.describe`

**Deletes / migrate**

- `DeleteManager.remove` → `file.delete`
- `ProjectMigrate.migrate_all` → `daemon.migrate`

**UI lists**

- `project_files` / dropdown → `project.snapshot`
- `review_files` → filter `filebase` from snapshot or `project.get` (field TBD)
- `new_file_added` → `event.file.added`

**Deferred**

- `Sandbox.Scan` → `path.scan.overlay`
- `tree_factory` → local tree-sitter or `file.parse.ast`
- `project_files.get_recent_list` → `project.files.recent`
- Git blame UI → `file.git.blame`
