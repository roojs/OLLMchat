- Gtk: re-emit `active_project_changed`, `active_file_changed`, `file_metadata_changed` locally

## Daemon-internal (no RPC)

- `GitProvider` during scan
- `FileHistory.cleanup_old_backups`
- `delete_manager.on_cleanup` / `VectorMetadata.cleanup_all_deleted` (triggered by `file.delete`)

- `Database.save_index`, `add_vectors_batch` (inside indexer)

## Map: today → RPC

**`ProjectManager` object** (`ProjectManager.*` methods — no `projects`, no `project_manager`, no separate `project` object)

- `load_projects_from_db` (startup) + enumerate `projects` → `ProjectManager.list`
- `create_project` → `ProjectManager.create`
- `remove_project` → `ProjectManager.remove`
- `activate_project` → `ProjectManager.activate` (`skip_scan`)
- `restore_active_state` → client: `ProjectManager.list` + `ProjectManager.activate` + `File.activate`
- `get_folder_at_path` → `File.get` (or `path` row in snapshot)
- `notify_project_changed` → `event.project.changed`

**Files / editor**

- `get_file_from_active_project` → `File.get`
- `read` / reload buffer → `File.read`
- `write_buffer_to_disk`, `on_file_contents_change` → `File.write`
- `check_active_file_changed` → `File.changed.check`
- `on_file_metadata_change` → `File.metadata.set`
- `activate_file` → `file.activate`
- `convert_fake_file_to_real` → `file.register`

**Scan / index / search (`libocfiles` + `libocvector`)**

- `read_dir` → `Folder.read_dir` or `ProjectManager.scan` (pick one in 2.10.4.1)
- `BackgroundScan` → `ProjectManager.index`, `File.index`, `event.project.index.progress`
- `Indexer.index_filebase` (force, recurse) → `ProjectManager.index` params; queue in daemon
- `Indexer.element_scanned` → `event.project.index.element` (optional UI)
- `ProjectAnalysis` / `--project-summary` → `vector.project.describe` or `ProjectManager.index` + `project_summary_only`
- `CodebaseSearchTool.init_databases` → daemon startup + `vector.status`
- `RequestCodebaseSearch` / `Search.execute` → `vector.search` (`format`: tool | json)
- `oc-vector-search --show-info` → `vector.list`
- `oc-vector-search --dump-vector` → `vector.get`
- `Indexer.reset_database` → `vector.reset`
- `Folder.get_project_description` → `vector.project.describe`

**Deletes / migrate**

- `DeleteManager.remove` → `File.delete`
- `ProjectMigrate.migrate_all` → **daemon startup only** (not RPC); then `ProjectManager.list`

**UI lists**

- `project_files` / dropdown → `ProjectManager.snapshot`
- `review_files` → filter `filebase` from snapshot or `ProjectManager.get` (field TBD)
- `new_file_added` → `event.file.added`

**Deferred**

- `Sandbox.Scan` → `folder.*` overlay (TBD)
- `tree_factory` → local tree-sitter or `file.parse.ast`
- `project_files.get_recent_list` → `project.files.recent`
- Git blame UI → `file.git.blame`
