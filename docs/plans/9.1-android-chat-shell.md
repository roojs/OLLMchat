# 9.1 Android chat shell (thin window skins + reused config widgets)

**Status:** Phase 0 + Phase 1 **done** — Phase 2 wired in code — Phase 3 (device verify) **blocked on bootup bug**

**Port status (read first):** [`docs/android-port-status.md`](../android-port-status.md)  
**Active device debug loop:** [`docs/bugs/2026-06-17-android-chat-poc-device-issues.md`](../bugs/2026-06-17-android-chat-poc-device-issues.md)

**Layout:** `docs/guide-to-writing-plans.md` — **Checklist for plans**, **Concrete code proposals**; **🚫** no ollmapp desktop refactors except the approved **`Config1` removal** in `libollmchat`.

> **Note:** `docs/plans/1.0-summary.md` is **not** updated until this plan is done and archived.

---

## Purpose

- **🔷** Replace `ollmapp/AndroidPoc.vala` with **new Android-only shell files** under `ollmapp/android/` that **host existing widgets** — not a fork of chat rendering or desktop window code.
- **🔷** **`ConnectionAdd`** is the bootstrap and add-connection UI; **`CheckingConnectionDialog`** for connection verification.
- **🔷** Main screen is **chat-first full width** — **`OverlaySplitView`** overlay for history, **no** `WindowPane` tool split.
- **🔷** **Do not refactor the existing desktop codebase** to support Android — no new shared interfaces, no method renames, no extra `ApplicationInterface` hooks.
- **🔷** **Drop `Config1` / `config.json` migration everywhere** — Android never had v1 config; main app no longer needs backward compatibility either.
- **⏳** Phase 0 — remove `Config1` from `libollmchat`; simplify `base_load_config()` to **Config2 only**.
- **⏳** Phase 1 — `AndroidApplication` + app-private **Config2-only** load + bootstrap + Android-local startup + **`AndroidSettingsDialog`**.
- **⏳** Phase 2 — expand Android cross-build for **`libollmchatgtk`**; wire **`ChatWidget`** + **`HistoryBrowser`** into **`AndroidMainWindow`**.
- **⏳** Phase 3 — device verification (debug APK on arm64).
- **ℹ️** Context: `docs/android-build.md`, `android/pixiewood-chat-poc.xml`, desktop reference behaviour in `ollmapp/Window.vala` and `ollmapp/Initialize.vala` (**read only — do not modify for Phase 1**).

---

## Design principle (review feedback)

- **🚫 Golden rule (2026-06-17):** **Never edit shared / non-Android code** without explicit user permission. If a fix appears to require `ConnectionAdd`, `ChatWidget`, `Initialize`, `Window`, or other desktop/shared files — **stop**, propose the change and why Android-only is insufficient, wait for approval. Reuse widgets **as-is**; copy behaviour into `ollmapp/android/` instead.
- **🔷** **New Android files:** yes.
- **🔷** **Existing desktop files:** leave untouched **except** deleting `Config1` (user-approved — no Android-specific reason required).
- **🔷** Where desktop logic is needed on Android, **copy the pattern into an Android file** (bootstrap, init loop, error alerts) rather than extracting interfaces or changing signatures on `OllmchatWindow`, `Initialize`, or `MainDialog`.
- **🚫** **`MainShellHost`**, **`config_dir()` on `ApplicationInterface`**, **`as_window`**, renaming **`show_connection_error_dialog`**, changing **`MainDialog` parent type**, wiring Android through **`Initialize`** — all rejected for this plan.
- **🚫** Do not instantiate desktop **`OllmchatWindow`** or **`MainDialog`** on Android.

## Phase 1 — Android shell + config + settings

### Config on Android (Android file only)

- **🔷** Config and data live under app-private storage, not `~/.config/ollmchat`.
- **💩** Paths:
  - `data_dir` = `{GLib.Environment.get_user_data_dir()}/ollmchat`
  - config file = `{data_dir}/config/config.2.json` only
- **🔷** **Config2 only** — no `Config1`, no `config.json`, no migration.
- **ℹ️** Why not call `base_load_config()` on Android?
  - It hardcodes `~/.config/ollmchat` for `config_path`.
  - **`AndroidApplication.load_config()`** sets `Config2.config_path` under `{data_dir}/config/`, then same two-branch load as simplified `base_load_config()` (exists → load, else → new).

### `AndroidApplication.vala`

- **🔷** `Adw.Application` + `ApplicationInterface`.
- **🔷** `load_config()` — Config2-only, app-private path (see above).
- **🔷** `activate` → `AndroidMainWindow`, `add_window`, `present`.
- **🔷** No `tools_registry`, `vector_registry`, or `mcp_registry`.
- **🔷** Bootstrap saves minimal `Config2`: one connection + empty `default_model` / `title_model` usage entries — **no** `tools_registry.setup_config_defaults()`.

### `AndroidMainWindow.vala`

- **🔷** `Adw.ApplicationWindow` + `Adw.ToolbarView` + compact `HeaderBar` + `Adw.OverlaySplitView` (sidebar hidden at start).
- **🔷** On `realize` → `load_config_and_initialize.begin()` (Android-local, not `Initialize`).
- **🔷** Bootstrap when `!app.config.loaded`:
  - Copy **signal wiring pattern** from `OllmchatWindow.show_bootstrap_dialog()` into this file.
  - Uses **`ConnectionAdd.show_bootstrap()`** + **`present(this)`**.
  - Verified close → save config → `initialize_after_bootstrap.begin()`.
  - No verified connection on close → `quit()`.
- **🔷** Startup after config exists — **`AndroidStartup.run()`** (see below), not `Initialize`.

### `AndroidStartup.vala` (new — Android copy of init loop)

- **🔷** New class in `ollmapp/android/`; **does not modify** `ollmapp/Initialize.vala`.
- **ℹ️** Reference implementation: `Initialize.run()` in `ollmapp/Initialize.vala`.
- **🔷** Same outcomes Android needs:
  - `CheckingConnectionDialog` + `config.check_connections()`
  - pick working connection or show **`Adw.AlertDialog`** with Configure / Close (inline — **do not** call `show_connection_error_dialog`)
  - `initialize_model()` equivalent — first non-embedding chat model into `default_model`
  - **skip** `ensure_required_models()` tool loop — empty `config.tools` on Android bootstrap
  - create `OLLMchat.History.Manager(app)` and assign to `AndroidMainWindow.history_manager`
  - `ensure_model_usage()` with same retry → models alert path as desktop
- **💩** On Configure, open **`AndroidSettingsDialog`** on the relevant tab instead of `MainDialog`.

### `AndroidSettingsDialog.vala` (new — Android settings skin)

- **🔷** `Adw.Dialog` + `ViewStack` / `ViewSwitcher` with **Connections** and **Models** tabs only.
- **🔷** **Connections tab** (Android-local layout):
  - `Adw.PreferencesGroup` + `ConnectionRow` per entry in `app.config.connections`
  - Add button → **`ConnectionAdd.show_add()`** + `present(parent_window)`
  - Save on dialog close (`config.save()`, `config.changed()`)
  - Connection check loop mirrors `MainDialog.check_all_connections()` — copy into this file
- **🔷** **Models tab** (Phase 1 minimal):
  - **💩** Simple default-model picker (connection + model name) sufficient for v1 — full `ModelsPage` port is optional Phase 1 polish
  - **⏳** If reusing desktop model list UI is needed later, add `AndroidModelsTab.vala` as a **new** file rather than changing `ModelsPage.vala`
- **🔷** Header settings button on `AndroidMainWindow` opens this dialog.

### Meson (Android target sources only)

- **🔷** Replace `AndroidPoc.vala` with android shell files + **unchanged** shared widgets the shell links:
  - `SettingsDialog/ConnectionAdd.vala`
  - `SettingsDialog/CheckingConnectionDialog.vala`
  - `SettingsDialog/ConnectionRow.vala`
- **🔷** Link `Initialize.vala`, `Window.vala`, `MainDialog.vala`, etc. **only if** the build requires them — prefer **not** linking desktop-only sources; Phase 1 should compile without `Initialize.vala` or `Window.vala`.

```vala
	/**
	 * Android remote-chat application entry.
	 *
	 * Loads config from app-private storage. Does not use desktop tool registries.

	 * @since 1.0

		public AndroidApplication()
		{
			Object(
				application_id: "org.roojs.ollmchat.androidpoc",
				flags: GLib.ApplicationFlags.DEFAULT_FLAGS
			);

			this.data_dir = GLib.Path.build_filename(
				GLib.Environment.get_user_data_dir(), "ollmchat"
			);

			this.config = this.load_config();

			this.activate.connect(() => {
				this.ensure_data_dir();
				var window = new AndroidMainWindow(this);
				this.add_window(window);
				window.present();
			});
		}

		/**
		 * Loads config from app-private paths (Config2 only).
		 *
		 * Sets Config2.config_path under {data_dir}/config/ then loads or
		 * returns empty Config2. Does not use base_load_config() because
		 * that hardcodes ~/.config/ollmchat.
		 */
		public OLLMchat.Settings.Config2 load_config()
		{
			var config_dir = GLib.Path.build_filename(this.data_dir, "config");
			var config_dir_file = GLib.File.new_for_path(config_dir);
			if (!config_dir_file.query_exists()) {
				try {
					config_dir_file.make_directory_with_parents(null);
				} catch (GLib.Error e) {
					GLib.warning("AndroidApplication: config dir: %s", e.message);
				}
			}

			var dummy2 = new OLLMchat.Settings.Config2();

			OLLMchat.Settings.Config2.config_path = GLib.Path.build_filename(
				config_dir, "config.2.json"
			);

			if (GLib.FileUtils.test(
				OLLMchat.Settings.Config2.config_path, GLib.FileTest.EXISTS
			)) {
				return OLLMchat.Settings.Config2.load();
			}

			return new OLLMchat.Settings.Config2();
		}
	}
}
```

**Why:** Connection/model verification without touching `Initialize.vala`.

**Where:** New file.

**Depends on:** §1, §3.

#### Add — skeleton; implementer fills model-pick body by copying from `Initialize.initialize_model()` verbatim into this class.

```vala
/*
 * Copyright (C) 2026 Alan Knowles <alan@roojs.com>
 *
 * … license header …
 */

namespace OLLMapp
{
	/**
	 * Android startup: connection check, default model, history manager.
	 *
	 * Android-only counterpart to ollmapp/Initialize.vala — do not modify Initialize.
	 *
	 * @since 1.0
	 */
	public class AndroidStartup : Object
	{
		private AndroidMainWindow window;

		public signal void reinitialize();

		public AndroidStartup(AndroidMainWindow window)
		{
			this.window = window;
		}

		public async bool run(OLLMchat.Settings.Config2 config)
		{
			while (true) {
				var checking_dialog = new SettingsDialog.CheckingConnectionDialog(
					this.window
				);
				checking_dialog.show_dialog();
				yield config.check_connections();
				checking_dialog.hide_dialog();

				var working_conn = config.working_connection();
				if (working_conn == null) {
					if (!(yield this.prompt_settings(
						"No working connection found.",
						"connections"
					))) {
						return false;
					}
					continue;
				}

				if (!(yield this.initialize_model(config, working_conn))) {
					if (!(yield this.prompt_settings(
						"No chat model found.",
						"models"
					))) {
						return false;
					}
					continue;
				}

				this.window.history_manager = new OLLMchat.History.Manager(
					this.window.app
				);

				try {
					yield this.window.history_manager.ensure_model_usage();
				} catch (GLib.Error e) {
					GLib.warning(
						"AndroidStartup: model verify failed: %s", e.message
					);
					if (!(yield this.prompt_settings(
						"No chat model found.",
						"models"
					))) {
						return false;
					}
					continue;
				}

				break;
			}
			return true;
		}

		private async bool prompt_settings(string message, string page)
		{
			var alert = new Adw.AlertDialog(
				page == "connections" ? "Connection Failed" : "No Chat Model",
				message + "\n\nOpen settings to fix this?"
			);
			alert.add_response("cancel", "Close");
			alert.add_response("settings", "Configure");
			alert.set_response_appearance(
				"settings", Adw.ResponseAppearance.SUGGESTED
			);
			var response = yield alert.choose(this.window, null);
			if (response != "settings") {
				(this.window.app as Gtk.Application).quit();
				return false;
			}
			ulong signal_id = 0;
			signal_id = this.window.settings_dialog.closed.connect(() => {
				this.window.settings_dialog.disconnect(signal_id);
				this.reinitialize();
			});
			this.window.settings_dialog.show(page);
			return true;
		}

		private async bool initialize_model(
			OLLMchat.Settings.Config2 config,
			OLLMchat.Settings.Connection working_conn
		) {
			// Copy body from Initialize.initialize_model() — do not refactor Initialize
			return false;
		}
	}
}
```
