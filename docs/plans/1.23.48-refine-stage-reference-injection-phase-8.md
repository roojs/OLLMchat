**Coding standards:** [`.cursor/rules/CODING_STANDARDS.md`](../../.cursor/rules/CODING_STANDARDS.md) · [`docs/code-documentation.md`](../../docs/code-documentation.md). Do not paste the full checklist here.

**When this plan is approved for implementation**, landed Vala must follow that document. In particular for these fences:

- **Temporary variables:** No one-use locals; no trivial aliases such as `var doc = this.document` — use **`this.document`** at each site (or a local only if it has **multiple** non-trivial uses). **`var last`** in **`run_exec`** is fine (two assignments). Inline **`defn`** / **`run_post`** unless a **4+ step** chain warrants a single alias per **§ Temporary variables — EXCEPTION**. **Exception:** **`WriteChange.exec`** may use **`args`**, **`tc`**, and **`impl`** (see fence): **`args`** / **`tc`** split serialization from the **`ToolCall`**; **`impl`** splits **`tools.get("write_file")`** from **`yield impl.execute`** for readability.
- **Null checks:** Per **§ Null checks** — do **not** add defensive **`!= null`** guards unless an **external** contract truly allows null. Here, a **`Tool`** that **`run()`** finishes without throwing has already had **`exec_extract`** set **`summary`** and **`document`** on success; redundant checks hide bugs and are **not** in the fence below.
- **Docblocks / `this.` / braces / string building:** Match **CODING_STANDARDS.md** and **`docs/code-documentation.md`** (multiline **`/** … */`**, **`this.`** on **`ResultParser`** instance members, control-structure brace rules, no **`+=`** string build in loops). **`exec_extract`:** after **`ex.summary`** / **`ex.document`**, re-parse **`ex.summary.to_markdown_with_content()`** with **`Markdown.Document.Render`**, **`foreach`** **`sum_render.document.links`**, **`ex.parent.validate_link(..., MarkdownPhase.POST_EXEC, sum_render.document)`**; merge **`ex.parent.issues`** into **`this.issues`**, clear **`ex.parent.issues`**, **`return false`** on failure (same POST_EXEC rules as **`exec_post_extract`**, but **`#`** anchors must resolve inside the summary slice only). Write path: **`foreach`** + **`continue`** for **`result-summary`** slug (§ Loops); inline **`this.document.headings.get(slug)`** into **`from_header`**; keep **`wc`** for **`single_output`**, **`issues`**, **`add`**. When **`wc.issues`** is set, **`+=`** to **`this.issues`** and **`continue`**; after the loop, **`return false`** if **`ex.writes.size == 0`** or **`write_parse_issues`**.

**`use_write_executor_template` / `synthesized_write_arguments`:** **Removed** from the design. Gate write parsing with **`this.skill_manager.fetch(ex.parent).tools.contains("write_file")`** (same predicate as [Phase 5](1.23.45-refine-stage-reference-injection-phase-5.md) **`executor_prompt`**). Push **`WriteChange`** instances onto **`ex.writes`** — **zero or more**; [Phase 5](1.23.45-refine-stage-reference-injection-phase-5.md) **`run()`** **`foreach`** **`writes`** after **`exec_ok`**.

**Phase 5 vs Phase 8:** [Phase 5](1.23.45-refine-stage-reference-injection-phase-5.md) adds **`WriteChange`** as **`Json.Serializable`** **shell** (no payload fields, empty **`exec`**). **This phase** **Replace**s **`WriteChange.vala`** with GObject properties, a real **`exec`**, and **`exec_extract`** logic that constructs **`WriteChange`** from markdown — **no** **`Json.Object arguments`** field on **`WriteChange`**.

**Multiple writes:** The **`exec_extract`** fence uses **`break`** after the first **`single_output`** (v1). When **`task_execution_write.md`** defines several writes, drop **`break`** and **`add`** each, or scan for multiple qualifying headings — **no** extra branch in **`Tool.run()`**.

**Replace** — **`liboccoder/Task/WriteChange.vala`** — **full** class (**Replace** the Phase 5 skeleton). Approved **`Json.Serializable`** + **`write_file`**-aligned fields + **`exec`**. **Arguments for **`CallFunction`:**** derive the **`Json.Object`** from **`Json.gobject_serialize(this).get_object()`**. That keeps one source of truth (GObject properties ↔ tool JSON) and matches how other code builds nested tool JSON (e.g. **`Json.gobject_serialize`** elsewhere in **`libollmchat`**). **Why not guard `get_node_type() == OBJECT`?** For a normal **`GObject`** with default **`Json.Serializable`** behaviour, json-glib’s **`gobject_serialize`** produces a **JSON object** root — not an array or scalar. You would only see a non-**`OBJECT`** root if **`WriteChange`** (or a base class) overrode serialization incorrectly; that is a **programmer error**, not an executor/runtime outcome worth branching for in **`exec`**. Fix the serializer; do not add a user-facing **`throw`** for “wrong node type” here. **`Json.gobject_to_data`** is for **string** payloads (logging, wire), not for filling **`CallFunction.with_values`** — use **`gobject_serialize` → `Json.Object`** here. **`OLLMchat.Response.CallFunction`** only needs **`with_values(string name, Json.Object arguments)`** — no separate constructor required unless you introduce a dedicated args type later. **Tool lookup:** **`run.parent.runner.session.manager.tools`** is **`Gee.HashMap<string, OLLMchat.Tool.BaseTool>`** (see **`libollmchat/History/Manager.vala`**). **`get("write_file")`** is already typed **`BaseTool?`**; an **`as OLLMchat.Tool.BaseTool`** cast is redundant. The fence uses a local **`impl`** before **`yield impl.execute`** for readability only — **no** **`if (impl == null)`** branch (invalid here: **`write_file`** is a precondition of this path).

**`ToolCall` id:** First argument is **`Response.ToolCall.id`**. The fence uses **`"fake-write-id"`** (same spirit as **`"http-fake-id"`** in **`Tool.load_http`**) so logs and the IDE show a stable label for this **synthetic** call — **`BaseTool.execute`** still only reads **`function.arguments`**.

**`WriteChange.from_header` (named constructor):** **`public WriteChange.from_header (Markdown.Document.Block header)`** — **only** the heading **`Block`** (e.g. from **`headings.get(slug)`**). Do **not** pass **`Document`**; use **`header.contents()`** (and **`header.document()`** only if something truly needs the root) to read the section under that heading: first block list → **`to_key_map`**, then **`output_mode`** and body (**`fenced`** vs **`next_section`**). If this heading is **not** the write-output section (e.g. wrong slug/title), leave **`single_output`** false and **`issues`** empty. If it **is** the write section and parsing fails, set **`issues`** and keep **`single_output`** false. On success set tool fields, **`output_mode`**, and **`single_output`** true.

**`single_output`:** **`public bool single_output { get; private set; default = false; }`** — true when **`from_header`** successfully parsed **one** write from **this** heading’s section. **`exec_extract`** uses it to **`add`** and **`break`** (v1: first successful section wins).

**`issues`:** **`public string issues { get; private set; default = ""; }`** — parse failure message(s) for a heading that was treated as the write section; empty when not applicable or success. **`exec_extract`** appends each non-empty **`wc.issues`** and keeps scanning (**`continue`**) so the user sees **all** section failures, then returns false if any occurred (along with the no-write fallback when **`ex.writes`** stayed empty).

**`output_mode`:** **`public string output_mode { get; set; default = ""; }`** — normalized **`fenced`** / **`next_section`** from the list. Parse metadata; **not** sent to **`write_file`**.

**Hide parse-only fields from tool JSON:** **`Json.gobject_serialize(this)`** in **`exec`** must emit **only** **`write_file`** arguments. Override **`serialize_property`** ( **`OLLMchat.Call.Options`** pattern — **`libollmchat/Call/Options.vala`**) to return **`null`** for **`output_mode`** / **`output-mode`**, **`single_output`** / **`single-output`**, and **`issues`** (and hyphen variants if needed).

**Optional later:** **`async`** validation before **`exec`** by delegating to **`OLLMtools.WriteFile.Tool.validate`** (see **`liboctools/WriteFile/Tool.vala`**) using **`this`** fields — not required in the core fence v1.

**KV → properties:** Implement in **`from_header`** without pasting a long **`if (kv.has_key (...))`** ladder into this plan — one place in code (e.g. **`apply_kv`** with **`set_property`** like **`OLLMchat.Call.Options.fill_from_model`**, or a tight loop over known keys).

**No** persisted **`Json.Object`** field on **`WriteChange`** and **no** **`this.arguments`**. If serializing default sentinels (e.g. **`start_line == -1`**) is a problem for **`write_file`**, extend **`serialize_property`** on **`WriteChange`** (omit or default-strip) — do not hand-build a parallel **`Json.Object`** unless **`gobject_serialize`** cannot be made correct.

**Replace** — **`ResultParser.exec_extract(Tool ex)`** — use the shape below (result-summary links, then write path: **`header_list`** + **`new WriteChange.from_header(header)`**; **`ex.writes`** — **not** **`wc.arguments`**):

```vala
// ResultParser.exec_extract(Tool ex) — summary + document; result-summary links; write path → ex.writes
public bool exec_extract(Tool ex)
{
	if (!this.document.headings.has_key("result-summary")) {
		this.issues += "\nThis task's executor output must include a \"Result summary\" section.";
		return false;
	}
	ex.summary = this.document.headings.get("result-summary");
	ex.document = this.document;
	var sum_render = new Markdown.Document.Render();
	sum_render.parse(ex.summary.to_markdown_with_content());
	foreach (var link in sum_render.document.links) {
		ex.parent.validate_link(link, MarkdownPhase.POST_EXEC, sum_render.document);
	}
	if (ex.parent.issues != "") {
		this.issues += ex.parent.issues;
		ex.parent.issues = "";
		return false;
	}
	if (!ex.parent.skill_manager.fetch(ex.parent).tools.contains("write_file")) {
		return true;
	}
	ex.writes.clear();
	var write_parse_issues = false;
	foreach (var slug in this.document.header_list) {
		if (slug == "result-summary") {
			continue;
		}
		var wc = new WriteChange.from_header(this.document.headings.get(slug));
		if (wc.single_output) {
			ex.writes.add(wc);
			break;
		}
		if (wc.issues != "") {
			this.issues += "\n" + wc.issues;
			write_parse_issues = true;
			continue;
		}
	}
	if (ex.writes.size == 0) {
		this.issues += "\nWrite executor output must include a recognizable Change details section.";
		return false;
	}
	if (write_parse_issues) {
		return false;
	}
	return true;
}
```
