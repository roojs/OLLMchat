# 7.14.4 — `ProgressView` sizing + selection + `ChatView` integration (phase 4)

**Status:** proposed

**Parent:** `docs/plans/7.14-task-progress-tree-ui.md`

**Checklist:** `.cursor/rules/CODING_STANDARDS.md` — **Checklist for all plans**.

**Layout:** `docs/guide-to-writing-plans.md`

**Depends on:** **`docs/plans/done/7.14.2-DONE-progressview-columnview.md`** (see **`docs/plans/7.14.1-2-task-progress.md`**) — **`ProgressView`** ships with **minimal** **`ScrolledWindow`** height, **`single_click_activate = false`**, and **no** row tooltips. This plan adds **sizing**, **tooltips**, **`single_click_activate`**, and **activate** → **`ChatView`**.

---

## Purpose

- **`ScrolledWindow` height** — Apply min/max content height and natural-height behaviour for the progress strip, aligned with **Height & placement** in **`docs/plans/7.14-task-progress-tree-ui.md`** (show **all tools** for active detail + that detail + **two prior `Details`** with tools **collapsed** — not row filtering).
- **Row tooltips** — **Title** column: plain **`ProgressItem.title`**. **Stage** column: **plain** text derived from **`status_str`** (Pango markup — **strip** markup for the tooltip string).
- **`Gtk.ColumnView`**: set **`single_click_activate = true`** when row **activate** should scroll the chat (see § **`ColumnView` `single_click_activate`**); wire **activate** / selection → **`ChatView`** scroll.
- **`Gtk.SingleSelection`** remains from **`7.14.2`**; this phase adds **behaviour**: row **activate** (and selection as needed) → scroll/focus **chat transcript** to the **`Message`** / marker region for that **`ProgressItem`**.
- Depends on a **stable link** between progress rows and **`Session.messages`** (indices, ids, or marker sequence) — spike if missing.

---

## Scope

| In scope | Out of scope |
|----------|----------------|
| **`ProgressView`** / **`ScrolledWindow`** sizing (min/max, **`propagate_natural_height`**) | **Continue** / replay (**`7.14.6`**) |

| Stubs if row ↔ message linkage incomplete | Re-implementing base **`ColumnView`** / **`7.14.2`** wiring |

---

## Acceptance criteria

1. **`ScrolledWindow`** uses the **height policy** hunk in § **ScrolledWindow height** below (min/max content height **96–280px**, **`propagate_natural_height`**) unless a **documented** deviation is required for the parent **Height & placement** rules.
2. **Tooltips:** title column shows **plain** **`title`**; stage column tooltip shows **plain** text derived from **`status_str`** (§ **Row tooltips — title column** and normative text for **stage**).
3. **`ColumnView`** uses **`single_click_activate = true`** (§ **`ColumnView` `single_click_activate`**) when **activate** → chat scroll is implemented.
4. Activating a progress row triggers **scroll** to the linked transcript region (or **documented gap** with TODO).
5. Does **not** block shipping **`7.14.1`–`7.14.3`** or **`7.14.2`** (this phase **extends** **`7.14.2`**).

### `ScrolledWindow` height

**Target:** `liboccoder/Task/ProgressView.vala` — constructor, after **`7.14.2`** has landed the placeholder **`set_min_content_height(48)`** block.

**Normative:** Replace the placeholder with the following (restores the **96–280px** strip policy planned before the **`7.14.2`** split).

#### Remove — `ProgressView` ctor: placeholder **`ScrolledWindow`** block (**`7.14.2`**)

```vala
			this.scrolled = new Gtk.ScrolledWindow() {
				vexpand = false,
				hexpand = true,
				has_frame = true
			};
			this.scrolled.set_policy(Gtk.PolicyType.AUTOMATIC, Gtk.PolicyType.AUTOMATIC);
			this.scrolled.set_min_content_height(48);
			this.scrolled.set_child(this.column_view);
```

#### Replace with — `ScrolledWindow` height policy (**`7.14.4`**)

```vala
			this.scrolled = new Gtk.ScrolledWindow() {
				vexpand = false,
				hexpand = true,
				has_frame = true,
				propagate_natural_height = true
			};
			this.scrolled.set_policy(Gtk.PolicyType.AUTOMATIC, Gtk.PolicyType.AUTOMATIC);
			this.scrolled.set_min_content_height(96);
			this.scrolled.set_max_content_height(280);
			this.scrolled.set_child(this.column_view);
```

*Further tuning* to match the parent doc’s **Height** story (e.g. showing all tools for active detail + two prior **`Details`**) may adjust these numbers or policy in a follow-up if UX requires it — document any change in the implementing PR.

### Row tooltips — title column

**Target:** `liboccoder/Task/ProgressView.vala` — **`title_factory`** **`bind`** / **`unbind`** (after **`7.14.2`**).

**Normative:** Tooltip text tracks **`ProgressItem.title`** (plain). Second **`bind_property`** from **`title`** to **`Gtk.Widget`** **`tooltip-text`** keeps the tooltip in sync; **`unbind`** must release **both** bindings.

#### Remove — `title_factory.bind` (**`7.14.2`**)

```vala
			title_factory.bind.connect((obj) => {
				var li = (Gtk.ListItem) obj;
				((Gtk.Label) li.child).set_data<GLib.Binding>(
					"ollm-column-bind",
					((ProgressItem) li.item).bind_property(
						"title",
						(Gtk.Label) li.child,
						"label",
						GLib.BindingFlags.SYNC_CREATE));
			});
```

#### Replace with — title label + **tooltip-text** (**`7.14.4`**)

```vala
			title_factory.bind.connect((obj) => {
				var li = (Gtk.ListItem) obj;
				var label = (Gtk.Label) li.child;
				var item = (ProgressItem) li.item;
				label.set_data<GLib.Binding>(
					"ollm-column-bind",
					item.bind_property(
						"title",
						label,
						"label",
						GLib.BindingFlags.SYNC_CREATE));
				label.set_data<GLib.Binding>(
					"ollm-tooltip-bind",
					item.bind_property(
						"title",
						label,
						"tooltip-text",
						GLib.BindingFlags.SYNC_CREATE));
			});
```

#### Remove — `title_factory.unbind` (**`7.14.2`**)

```vala
			title_factory.unbind.connect((obj) => {
				var b = ((Gtk.Label) ((Gtk.ListItem) obj).child).get_data<GLib.Binding>(
					"ollm-column-bind");
				if (b == null) {
					return;
				}
				b.unbind();
				((Gtk.Label) ((Gtk.ListItem) obj).child).set_data<GLib.Binding>(
					"ollm-column-bind",
					null);
			});
```

#### Replace with — unbind **both** bindings (**`7.14.4`**)

```vala
			title_factory.unbind.connect((obj) => {
				var label = (Gtk.Label) ((Gtk.ListItem) obj).child;
				var b = label.get_data<GLib.Binding>("ollm-column-bind");
				if (b != null) {
					b.unbind();
					label.set_data<GLib.Binding>("ollm-column-bind", null);
				}
				var tb = label.get_data<GLib.Binding>("ollm-tooltip-bind");
				if (tb != null) {
					tb.unbind();
					label.set_data<GLib.Binding>("ollm-tooltip-bind", null);
				}
			});
```
