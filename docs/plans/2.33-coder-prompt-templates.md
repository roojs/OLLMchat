# 2.33. Coding Assistant — standard prompt templates

Status: proposed

> **Note:** `docs/plans/1.0-summary.md` is not updated until this plan is done
> and archived.

ℹ️ Plan checklist: `docs/guide-to-writing-plans.md` — Checklist for plans.
Proposed Vala code must follow `docs/coding-standards.md`. Plan layout and
implementation workflow: `docs/guide-to-writing-plans.md`.

ℹ️ Related: **`docs/plans/2.32-coder-summarized-history.md`** (summarized
history — `coder_followup.md` already added there). This plan migrates the
**main** Coder prompt off the legacy section-loader path.

## Purpose

- 🔷 Move the **Coding Assistant** system and user prompts onto
  **`OLLMchat.Prompt.Template`** and **`resources/chat-prompts/`** — same
  pattern as Chatter (`chatter_initial.md`, `chatter_followup.md`) and skills
  (`skill-prompts/`).
- 🔷 Replace **`AgentFactory.system_message()`** string stitch +
  **`Factory.load_section()`** over
  **`resources/ollmchat-agents/code-assistant/*.md`** with template load +
  **`system_fill()`**.
- 🔷 Replace **`generate_user_prompt()`** XML wrapper in Vala with a **user
  template** and **`fill()`** — keep **`generate_context_section()`** in Vala
  (dynamic editor/project state).
- ⏳ Add **`AgentFactory.load_prompt()`** (mirror **`Chatter.Factory`**).
- ⏳ Add **`coder_initial.md`** (main system rules + `{user_info}`).
- ⏳ Add **`coder_user.md`** (user half: `{additional_data}` + `{user_query}`).
- ⏳ Wire **`coder_followup.md`** through **`load_prompt()`** (drop inline
  **`Prompt.Template`** in **`OLLMcoder.Agent`**).
- ⏳ Remove legacy **`/ollmchat-agents/code-assistant/`** gresource group and
  section files after migration.
- ⏳ Manual verification (see **Verification**).

## Architecture

- **`resources/chat-prompts/coder_initial.md`** — system half: migrated content
  from today's eight section files; `{model_name}`, `{user_info}` placeholders.
- **`resources/chat-prompts/coder_user.md`** — user half: `{additional_data}`,
  `{user_query}`.
- **`resources/chat-prompts/coder_followup.md`** — unchanged role (summary tail
  on follow-up turns); loaded via factory after this plan.
- **`OLLMcoder.AgentFactory.load_prompt()`** — loads from `chat-prompts`
  gresource prefix (same four-line pattern as **`Chatter.Factory`**).
- **`OLLMcoder.AgentFactory.system_message()`** — `load_prompt("coder_initial.md")`
  + `system_fill(...)`.
- **`OLLMcoder.AgentFactory.generate_user_prompt()`** — `load_prompt("coder_user.md")`
  + `fill(...)`.
- **`OLLMcoder.Agent.send_async()`** — follow-up append via
  `((AgentFactory) this.factory).load_prompt("coder_followup.md")`.
- **`Factory.load_section()`** — unused after migration; remove in this plan
  if no other callers remain.

## Current behaviour

- ℹ️ **`AgentFactory.system_message()`** concatenates eight
  **`load_section()`** files under **`/ollmchat-agents/code-assistant/`**, wraps
  some in XML tags in Vala, and builds **`<user_info>`** inline.
- ℹ️ **`generate_user_prompt()`** builds **`<additional_data>`** in Vala and
  wraps the query in **`<user_query>`**.
- ℹ️ Placeholder syntax is **`$(model_name)`** (manual **`replace()`**), not
  **`{model_name}`** template keys.
- ℹ️ **`coder_followup.md`** (plan 2.32) is loaded with an inline
  **`new Prompt.Template(...)`** in **`OLLMcoder.Agent`**.

## Proposed behaviour

- 🔷 First turn and every turn: **`coder_initial.md`** via
  **`system_message()`** (same effective rules as today).
- 🔷 Every user message: **`coder_user.md`** via **`generate_user_prompt()`**.
- 🔷 Follow-up with summary: **`system_message()`** + filled
  **`coder_followup.md`** (unchanged contract from 2.32).
- 🔷 All prompt prose lives in **`chat-prompts/`**; Vala only fills dynamic
  blocks (**`user_info`**, **`additional_data`**, **`user_query`**,
  **`conversation_summary`**).

## Phase tasks

- ⏳ **Phase 1** — Add **`coder_initial.md`**, **`coder_user.md`**; register in
  **`gresources.xml`**; migrate section file content.
- ⏳ **Phase 2** — **`AgentFactory.load_prompt()`**; rewrite

- ⏳ **Phase 3** — **`OLLMcoder.Agent`**: factory **`load_prompt()`** for

- ⏳ **Phase 4** — Delete legacy section files; remove

  **`load_section()`** if unused.
- ⏳ **Phase 5** — Manual verification.

### 4. `liboccoder/AgentFactory.vala` — `load_prompt()`

**Why:** Shared loader for all Coder chat templates.

**Where:** **`AgentFactory` class**, after constructor, before
**`get_working_directory()`**.

**Depends on:** §3.

#### Add

```vala
		/**
		 * Loads a prompt template from the chat-prompts gresource prefix.
		 *
		 * @param filename template file name (e.g. coder_initial.md)
		 * @return loaded template with system and user halves parsed
		 */
		public OLLMchat.Prompt.Template load_prompt(string filename) throws GLib.Error
		{
			var tpl = new OLLMchat.Prompt.Template(filename) {
				source = "resource:///",
				base_dir = "chat-prompts"
			};
			tpl.load();
			return tpl;
		}

```
