# 7.14.2 — `ProgressView` (`Gtk.ColumnView`) + layout (phase 2)

**Status:** proposed

**Parent:** `docs/plans/7.14-task-progress-tree-ui.md`

**Checklist:** `.cursor/rules/CODING_STANDARDS.md` — **Checklist for all plans**.

---

## Purpose

- Implement **`ProgressView`**: **`Gtk.ScrolledWindow`** + **`Gtk.ColumnView`** ( **no `Gtk.TreeView`** ), factories/columns for title + stage (and tooltips).
- **Integrate into the app in this phase** — pack **between main chat view and text entry** (above input) so **height**, margins, and **read-only** behaviour can be judged early.
- Bind **`Runner.progress`** — **`ProgressList`** **`implements GLib.ListModel`** (same pattern as **`libocfiles/ProjectList.vala`** + **`Gtk.SingleSelection`** as in **`libollmchatgtk/HistoryBrowser.vala`**). Flat **`Details`** rows first; **tool** expansion may use **`Gtk.TreeListModel`** / **`TreeExpander`** later if the UI is hierarchical — **TBD** in this phase (shell can start flat).

---

## Scope

| In scope | Out of scope |
|----------|----------------|
| Widget + layout + **read-only** interaction | Full stage text / **`to_human()`** (**`7.14.3`**) |
| Tooltips wired (can show placeholders) | Chat scroll (**`7.14.4`**) |

---

## Acceptance criteria

1. **`ProgressView`** visible **below chat**, **above** chat input; **`Gtk.ColumnView`** used.
2. Title + stage columns present; height policy per parent doc (**Height** section).
3. User can **see** progress structure early (**read-only** — no requirement for double-click yet).
