## Purpose

- Implement **`ProgressView`** in **liboccoder**: **`Gtk.ScrolledWindow`** + **`Gtk.ColumnView`** ( **no `Gtk.TreeView`** ), factories/columns for title + stage (and tooltips).
- **Integrate into the app in this phase** — strip visible **above the chat input** so **height**, margins, and **read-only** behaviour can be judged early.
- Bind **`Runner.progress`** only — **`OLLMcoder.Task.ProgressList`** (same row type as task code). Flat **`Details`** rows first; **tool** expansion may use **`Gtk.TreeListModel`** / **`TreeExpander`** later — **TBD** in this phase.

**Layering:** **`libollmchatgtk` must not depend on liboccoder**. **`activate`/`deactivate`** receive **`GLib.Object window`**; **`liboccoder`** casts to **`OLLMchatGtk.ChatUserInterface`** (§3b) — **never** to **`OLLMapp.OllmchatWindow`**. **`OllmchatWindow`** implements **`ChatUserInterface`** in **`ollmapp/`** (§9). **`liboccoder`** gains dependencies on **`libollmchatgtk`** (interface) and **`libadwaita`** ( **`Adw.ViewStack`** for **`tab_view()`** inlining) — **`libollmchatgtk`** must not link **`liboccoder`**. **`ChatWidget.above_input`** is the strip parent (§3). **`libollmchat`** **`Agent.Factory`**: **`activate`/`deactivate`** (§4). **`Skill.Runner`** comes from **`host.session_agent()`** ( **`session.agent`** ). **`ProgressList`’s** back-ref to **`Runner`** stays **`weak`** where the model already uses it. **Do not** unparent **`above_input`**.

**Guards (`.cursor/rules/CODING_STANDARDS.md` — *Defensive code and null checks*):** No generic null-guards in **`activate`** / **`deactivate`**. **`(OLLMchatGtk.ChatUserInterface) window`** — wrong type is a bug. DB readiness for **`SourceView`** — call graph, not **`if (db == null)`** inside **`activate`**.

## Agent integration (existing design)

**Where the agent instance comes from:** **`OLLMchat.History.SessionBase`** already has **`public OLLMchat.Agent.Base? agent { get; set; }`**. It is set when a session activates an agent (**`Session.activate_agent`** → **`factory.create_agent`** → assign **`this.agent`**). That is the “agent initialized” handle the rest of the stack already uses.

**Two methods — no boolean flag, no `agent` on `activate`:**

| Method | Purpose |
|--------|---------|
| **`activate(window)`** | **`var host = (OLLMchatGtk.ChatUserInterface) window`**. **`Skill.Runner`** from **`host.session_agent()`**. First run: **`ProgressView()`** + **`SourceView`**; **always** **`progress_view.set_runner(runner)`**; strip: **`((Gtk.Box) host.above_input_widget()).append(progress_view)`**. Side pane: cast **`host.tab_view()`** to **`Adw.ViewStack`** and **inline** the former **`WindowPane.add_or_show_agent_widget`** behaviour (add/show child by **`widget_id`**) in the factory; then **`host.schedule_pane_update(true)`** (§3b, §5, §9). |
| **`deactivate(window)`** | Hide strip; **`host.schedule_pane_update(false)`** — **no** unparent. |

**Window:** **`OllmchatWindow`** implements **`OLLMchatGtk.ChatUserInterface`** (§9). Factories use **only** **`ChatUserInterface`** — not **`window_pane`** / **`chat_widget`** fields from **`liboccoder`**.

**Layout ownership:** **`ChatWidget.above_input`** (§3). **`ChatUserInterface.above_input_widget()`** returns that **`Gtk.Box`** as **`GLib.Object`**. **Right pane:** **`ChatUserInterface.tab_view()`** exposes **`WindowPane.tab_view`** (**`Adw.ViewStack`**) as **`GLib.Object`** — factories **inline** tab add/show there; **`ChatUserInterface`** does **not** expose **`add_or_show_agent_widget`**.

**Model ownership:** **`Skill.Runner`** has **`public Task.ProgressList progress { get; }`**. **`Task.ProgressList`** has **`public weak Skill.Runner runner { get; }`**. Rows are **`ProgressItem`**.

**Who fills the slot:** **`create_agent`** returns **`Runner`**; **`session.agent`** is set before **`activate`**. **`session_agent()`** on the host reads the same. **No** factory **`runner`** field, **no** **`agent`** parameter on **`activate`**. **Never** unparent **`above_input`**.

**`get_widget` is removed** — **`activate`** / **`deactivate`** replace **`Window.handle_agent_widget`**.

**Deactivation:** **`agent_deactivated`** (§7) → **`factory.deactivate.begin(this, …)`**. **`on_agent_activated`** → **`factory.activate.begin(this, …)`** (window only).

## Acceptance criteria

1. **`ProgressView`** visible **below chat**, **above** chat input; **`Gtk.ColumnView`** used.
2. Title + stage columns present; **`ScrolledWindow`** min/max content height (e.g. 96–280px) for early layout; full **Height** policy remains parent doc / later phases.
3. User can **see** progress structure early (**read-only** — no double-click yet).

### 1. `liboccoder/Task/ProgressView.vala` — new file

Widget lives in **liboccoder** next to **`Task.ProgressList`** / **`Task.ProgressItem`**; **libollmchatgtk** does not reference this type.

#### Lifecycle — placeholder first, **`set_runner`** always

- **Model:** The live data is **`Runner.progress`** (**`Task.ProgressList`**). **`ProgressList.runner`** is the weak back-ref from list to **`Skill.Runner`**.
- **Construction:** **`new Task.ProgressView()`** builds **`Gtk.ColumnView`** + factories on a **placeholder** **`Gio.ListStore`** of **`ProgressItem`** with **zero rows** (no real **`Runner`** yet — layout and columns exist; strip is empty).
- **Binding:** **`set_runner(Runner)`** is called **every** time **`activate`** runs (first time and later). It replaces the selection with **`new Gtk.SingleSelection(runner.progress)`**. One code path; no “constructor bind vs else **`set_runner`**” split.
- **When the skills agent is not active:** **`Skill.Factory.deactivate(window)`** hides **`ProgressView`**. The widget **stays** in **`above_input`** — **no** unparent.
- **Selection model:** **`Gtk.NoSelection`** = list is not selectable (display-only). **`Gtk.SingleSelection`** wraps **`Task.ProgressList`** so **at most one** **`ProgressItem`** row is selected — use this here so a later phase can **`connect`** to **`Gtk.SingleSelection.selection-changed`** (or **`notify::selected-item`**) and scroll **`ChatView`** to the related message (**`7.14.x`**). Keep **`private Gtk.SingleSelection progress_selection`** on **`ProgressView`** so that hook does not hunt through **`ColumnView`** internals.
- **GTK detail:** **`set_runner`** assigns **`this.progress_selection = new Gtk.SingleSelection(runner.progress)`** and **`this.column_view.model = this.progress_selection`**. Prefer **`single_click_activate = true`** on **`ColumnView`** if one click should select a row (tune with UX).
- **Cells:** Use **`GLib.Object.bind_property`** from each row’s **`OLLMcoder.Task.ProgressItem`** to **`Gtk.Label`**: **`title`** → **`label`** (plain), **`status_str`** → **`label`** on the stage column with **`use_markup = true`** (Pango markup). Store the returned **`GLib.Binding`** on the label with **`set_data`** (inline the **`bind_property`** call — no one-use **`var`** for the binding). In **`unbind`**, read **`GLib.Binding?`** from **`get_data`**; if null, **`return`** (GObject slot may be empty); otherwise **`unbind()`** and clear **`set_data`**. No custom **`CellBinding`** class — optional tooltip text can be a tiny **`notify`** hook or left to **`7.14.3`**. Bindings update when the **`ProgressItem`** emits **`notify`** for those properties ( **`status_str`** already tracks **`status`**); if **`title`** can change without **`notify`**, fix the row type or **`ProgressList`** signalling separately.
- **Style (`.cursor/rules/CODING_STANDARDS.md`):** Multiline docblocks with **`@param`** on **`set_runner`**; **`this.`** on instance members; fully qualified **`GLib.*`**; break long calls across lines; **`GLib.debug()`** without class/name prefixes in the message text.

```vala
namespace OLLMcoder.Task
{
	/**
	 * Read-only task progress strip: {@link Gtk.ScrolledWindow} plus
	 * {@link Gtk.ColumnView} (title and stage columns). Built with an empty
	 * placeholder model; call {@link #set_runner} to attach {@link Skill.Runner.progress}.
	 */
	public class ProgressView : Gtk.Box
	{
		private Gtk.SingleSelection progress_selection;
		private Gtk.ColumnView column_view;
		private Gtk.ScrolledWindow scrolled;

		/**
		 * Creates the column view with an empty placeholder list. Call
		 * {@link #set_runner} before expecting rows.
		 */
		public ProgressView()
		{
			Object(orientation: Gtk.Orientation.VERTICAL, spacing: 0);
			var placeholder = new Gio.ListStore(typeof(ProgressItem));
			this.progress_selection = new Gtk.SingleSelection(placeholder);
			this.column_view = new Gtk.ColumnView(this.progress_selection) {
				vexpand = true,
				hexpand = true,
				show_row_separators = true,
				single_click_activate = true,
				enable_rubberband = false
			};

			var title_factory = new Gtk.SignalListItemFactory();
			title_factory.setup.connect((obj) => {
				((Gtk.ListItem) obj).set_child(new Gtk.Label("") {
					halign = Gtk.Align.START,
					hexpand = true,
					xalign = 0,
					ellipsize = Pango.EllipsizeMode.END,
					single_line_mode = true
				});
			});
			title_factory.bind.connect((obj) => {
				var li = (Gtk.ListItem) obj;
				((Gtk.Label) li.child).set_data<GLib.Binding>(
					"ollm-column-bind",
					((ProgressItem) li.item).bind_property(
						"title",
						(Gtk.Label) li.child,
						"label",
						GLib.BindingFlags.SYNC_CREATE));
			});
			title_factory.unbind.connect((obj) => {
				var b = ((Gtk.Label) ((Gtk.ListItem) obj).child).get_data<GLib.Binding>(
					"ollm-column-bind");
				if (b == null) {
					return;
				}
				b.unbind();
				((Gtk.Label) ((Gtk.ListItem) obj).child).set_data<GLib.Binding>(
					"ollm-column-bind",
					null);
			});

			var stage_factory = new Gtk.SignalListItemFactory();
			stage_factory.setup.connect((obj) => {
				((Gtk.ListItem) obj).set_child(new Gtk.Label("") {
					halign = Gtk.Align.START,
					hexpand = true,
					xalign = 0,
					ellipsize = Pango.EllipsizeMode.END,
					single_line_mode = true,
					use_markup = true
				});
			});
			stage_factory.bind.connect((obj) => {
				var li = (Gtk.ListItem) obj;
				((Gtk.Label) li.child).set_data<GLib.Binding>(
					"ollm-column-bind",
					((ProgressItem) li.item).bind_property(
						"status_str",
						(Gtk.Label) li.child,
						"label",
						GLib.BindingFlags.SYNC_CREATE));
			});
			stage_factory.unbind.connect((obj) => {
				var b = ((Gtk.Label) ((Gtk.ListItem) obj).child).get_data<GLib.Binding>(
					"ollm-column-bind");
				if (b == null) {
					return;
				}
				b.unbind();
				((Gtk.Label) ((Gtk.ListItem) obj).child).set_data<GLib.Binding>(
					"ollm-column-bind",
					null);
			});

			this.column_view.append_column(new Gtk.ColumnViewColumn("Title", title_factory));
			this.column_view.append_column(new Gtk.ColumnViewColumn("Stage", stage_factory));

			this.scrolled = new Gtk.ScrolledWindow() {
				vexpand = false,
				hexpand = true,
				has_frame = true,
				propagate_natural_height = true
			};
			this.scrolled.set_policy(Gtk.PolicyType.AUTOMATIC, Gtk.PolicyType.AUTOMATIC);
			this.scrolled.set_min_content_height(96);
			this.scrolled.set_max_content_height(280);
			this.scrolled.set_child(this.column_view);
			this.append(this.scrolled);
		}

		/**
		 * Binds the column view to {@link Skill.Runner.progress} (first activation
		 * and every re-activation, e.g. new session).
		 *
		 * @param runner the active skill runner
		 */
		public void set_runner(OLLMcoder.Skill.Runner runner)
		{
			this.progress_selection = new Gtk.SingleSelection(runner.progress);
			this.column_view.model = this.progress_selection;
		}
	}
}
```

### 3b. `libollmchatgtk/ChatUserInterface.vala` — host API for **`activate` / `deactivate`**

**Rationale:** **`liboccoder`** must **not** reference **`OLLMapp`** or cast to **`OllmchatWindow`**. Factories cast **`window`** to **`OLLMchatGtk.ChatUserInterface`** (§5–§6). **`OllmchatWindow`** implements the interface in **`ollmapp/`** (§9) and forwards to **`history_manager`**, **`chat_widget`**, and **`window_pane`** where needed.

**Tab pane — inlined into the agent factory:** **`ChatUserInterface.add_or_show_agent_widget`** is **not** part of the host API. **`WindowPane.add_or_show_agent_widget`** (today’s tree) is the **reference implementation**; each factory’s **`activate`** **inlines** that logic against **`(Adw.ViewStack) host.tab_view()`** ( **`get_child_by_name`** / **`add_named`** / **`visible`** / **`set_visible_child_name`** ), then **`schedule_pane_update(true)`**. **`OllmchatWindow`** does **not** store **`current_agent_widget`** — that field only served **`get_widget`** + **`handle_agent_widget`**. Which agent widget is visible is determined by **`tab_view`**’s visible child and by **`activate`/`deactivate`** (strip + pane), not by a **`Gtk.Widget?`** on the window.

**`liboccoder` + libadwaita:** Inlining uses **`Adw.ViewStack`**. Add **`dependency('libadwaita-1')`** and **`--pkg libadwaita-1`** to **`liboccoder`** (meson + **`occoder-vapi`** command) alongside existing GTK deps.

**Pane visibility:** **`ChatUserInterface`** exposes **`schedule_pane_update(bool visible)`** only — **no** separate **`intended_pane_visible`** on the interface. **`OllmchatWindow`** sets **`window_pane.intended_pane_visible = visible`** then calls the existing parameterless **`window_pane.schedule_pane_update()`** ( **`WindowPane`** keeps the bool internally).

### 5. `liboccoder/Skill/Factory.vala` — **`Task.ProgressView`** + **`activate`** + **`deactivate`**
