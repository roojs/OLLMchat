## 3. HTTP URLs — stage-by-stage behaviour (critical)

- **Task iteration and task creation:** Allow http(s) URLs as references; currently blocked — unblock at these stages.
- **Refinement stage:** Do **not** use reference contents for http(s) URLs (do not fetch or inject URL body into precursor).
- **Exec stage:** Ignore http(s) refs in the executor’s Reference section (return ""). Refinement should have created tool calls (e.g. web_fetch) to fetch URLs; we do not surface or fetch them in code.

## 4. Iteration stage — completed vs new tasks (critical)

- **Issue:** Completed stages may still be added to the pending task list when the model outputs the next iteration; or the behaviour is ambiguous.
- **Fix (choose one or both):**
  - **Option A:** Ensure completed stages are **not** re-appended to the pending task list when building the list for the next iteration (code/Runner fix).
  - **Option B:** Instruct the iteration (post-completion) prompt explicitly: **DO NOT OUTPUT existing completed stages — only output new tasks.** So the model returns only new tasks; Runner merges with completed list.
