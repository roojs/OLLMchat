## Purpose

- **🔷** **`RunCommandV2`** must **not** call **`fetch_files`** before bubble exec — whole-project prefetch is wrong and heavy.
- **🔷** Overlay **`Scan`** must know, **per changed path**, whether that path was **already in the project index** (file / folder / symlink) so it can call **`created`** vs **`modified`** vs **`removed`** correctly.
- **🔷** Fix with **small, targeted** index lookups — **overlay paths only** — **🚫** not bulk **`fetch_files`**, **🚫** not “scan the project first”.

---

## Mental model (read this first)

Two separate things get conflated in the old write-up:

| Layer | What it is | When it’s populated |
|-------|------------|---------------------|
| **Daemon index** | DB + in-process **`project_files`** on **`ollmfilesd`** | Background scan + FS watch while daemon runs; usually current for on-disk project files |
| **Client `file_cache`** | Sparse in-memory map on the app | Only when something explicitly RPC-fetches rows (**`fetch_file`**, paged **`fetch_files`**, etc.) |

**Assumption (reasonable):** with **`ollmfilesd`** running, the daemon **already knows** about normal project files on disk. You do **not** need to “ensure the whole project is indexed” before every bubble run.

**Actual gap:** **`FileVerification.indexed_file_type`** is **sync** and reads **client `file_cache` only**. Shipping **`RunCommand`** never had this problem — the client held **`project_files`** in RAM. V2 does not.

So the bug is **not** “indexing probably isn’t done on the daemon”. It is **“the overlay scanner asks the wrong layer, with an empty client cache”**. Without a lookup, every path looks **`UNKNOWN`** → **`Scan`** treats edits as **`created`**, whiteouts on indexed files are skipped, type changes mis-classified.

**What we need:** for each **overlay upper** path the scan visits (typically **O(changed paths)**, not **O(project)**), ask the daemon **“was this path already in the index, and as what type?”** — then classify.

That is **not** the same as prefetching the whole project via **`fetch_files()`**, and usually **not** a one-shot “ensure indexed” RPC for the entire tree.

---

## Problem

**Today in `RunCommandV2/Request.vala`:** unfiltered **`yield project.fetch_files()`** (return value discarded).

- Return value **discarded**.
- Side effect: fills **`file_cache`** from every row returned.
- Shipping never did this — client already had **`project_files`** in RAM.

**Why `fetch_files()` was added (interim hack):** populate client **`file_cache`** so **`indexed_file_type`** stops returning **`UNKNOWN`** for everything.

**🚫** Wrong fix — loads every project row (or first page semantics) when the scan only cares about **paths that changed in the overlay**. Conflicts with paged dropdown (**[B-10](2.10.4.9-ACTIVE-v2-caller-cutover.md)**).

**When daemon index might still miss a path (edge cases, not the common case):**

- Path existed on disk but background scan queue has not reached it yet (race right after project open or external copy-in).
- Path never under the project tree.
- Brand-new path created **only inside the bubble** (correctly **`UNKNOWN`** → **`created`**).

For those, **`File.fetch`** returning “not in index” is the right answer; an “ensure indexed” RPC that **scans disk and inserts rows** is only needed if we decide **`created`** must not run when a file exists on disk but is not yet in DB — **⏳** out of scope unless we hit that in practice.

---

## Agreed direction (🔷)

1. **Remove** prefetch from **`liboctools/RunCommandV2/Request.vala`** (**`yield project.fetch_files()`** with no filter — loads a page of dropdown rows, not overlay paths).
2. **Extend existing **`Folder.fetch_files`** (reuse RPC + client API; **🚫** no new method name unless wire forces it):
   - **`FolderParams.paths`** — optional **`string[]`**. When non-empty: return rows **only for those paths** (overlay upper set). **🚫** ignore dropdown **`query`** / pagination for this mode (or set **`limit`** = paths.length).
   - **`FolderParams.metadata_only`** (name TBC) — when **true**: index row only (**path**, **id**, **base_type** / **is_alias**); **🚫** no buffer, **🚫** no content. When **false**: today’s full **`File`** payload (dropdown / other callers).
3. **Daemon lookup** for path-filter mode: **`project_files.all_files`** (files + folders + aliases), **not** **`cached_search`** alone — dropdown search only indexes **`items`** (regular files); overlay can touch dirs and symlinks.
4. **Before overlay classify:** collect overlay **`actual_real_path`** list → **`yield project.fetch_files(..., paths, metadata_only: true)`** → fill client **`file_cache`** → run **`Scan`**.

### Do we need full file information?

| Step | Need full `File` (content / buffer)? | Need from index |
|------|--------------------------------------|-----------------|
| **`Scan` classify** (`indexed_file_type`) | **No** | **Yes** — path + type (regular / directory / symlink) |
| **`created` / `modified` apply** | **No** — reads bytes from **overlay upper** on disk; writes via **`File.write`** RPC | **No** (uses overlay + fake file) |
| **`removed` apply** | **No** content | **Yes** — real **id** + path for **`delete_manager.remove`** |

So for bubble overlay: **`metadata_only: true`** is enough for the batch prefetch before **`Scan`**. That row also satisfies **`removed()`** on cache hit (same as today’s **`fetch_file`** without loading buffer). Full **`File`** payloads stay for dropdown and any caller that needs them (**`metadata_only: false`**, default).

**🚫** not project-wide unfiltered **`fetch_files`**. **🚫** not disk ingest — lookup against daemon index only.

### ⏳ 2. `ollmfilesd/Folder.vala` — `rpc_fetch_files`: path-filter via `all_files`

**Why:** Overlay classify needs folder / alias / file rows from daemon index, not **`cached_search`** (regular files only).

**Where:** `rpc_fetch_files` signal handler, after **`project == null`** early return, before **`p.limit = …`**.

**Depends on:** §1.

##### Part 1 — Path-filter branch

**Keep**

```vala
			this.rpc_fetch_files.connect((request) => {
				var p = (FolderParams) request.param;
				var project = this.manager.project_root(p.path);
				if (project == null) {
					request.reply(new OLLMrpc.Response() {
						id = request.id,
						result = new Gee.ArrayList<GLib.Object>(),
						result_type = "File",
						is_array = true,
						msg = "project not found"
					});
					return;
				}
```

**Add** — When **`paths`** non-empty, look up **`project_files.all_files`** and return metadata **`File[]`**.

```vala
				if (p.paths.length > 0) {
					var list = new Gee.ArrayList<GLib.Object>();
					foreach (var lookup_path in p.paths) {
						if (
							!project.project_files.all_files.has_key(
								lookup_path
							)
						) {
							continue;
						}
						var source = project.project_files.all_files.get(
							lookup_path
						);
						var row = new File(this.manager);
						row.copy_from(
							source,
							{
								"manager",
								"buffer",
								"parent",
								"last_modified"
							}
						);
						list.add(row);
					}
					request.reply(new OLLMrpc.Response() {
						id = request.id,
						result = list,
						result_type = "File",
						is_array = true,
						msg = list.size.to_string()
					});
					return;
				}
```

**Keep**

```vala
				p.limit = p.limit < 1 ? 50 : p.limit;
				p.offset = p.offset < 0 ? 0 : p.offset;
				var matched = project.project_files.cached_search(
					p.query.strip().down()
				);
```

##### Part 2 — `collect_overlay_paths` private method

**Add** — After **`run()`** method, before **`scan_dir()`** method.

```vala
		private void collect_overlay_paths(
			string overlay_path,
			Gee.ArrayList<string> paths)
		{
			var overlay_dir = GLib.File.new_for_path(overlay_path);
			if (!overlay_dir.query_exists()) {
				return;
			}

			GLib.FileEnumerator enumerator;
			try {
				enumerator = overlay_dir.enumerate_children(
					GLib.FileAttribute.STANDARD_NAME + ","
						+ GLib.FileAttribute.STANDARD_TYPE,
					GLib.FileQueryInfoFlags.NONE,
					null
				);
			} catch (GLib.Error e) {
				GLib.warning(
					"Cannot enumerate overlay directory %s: %s",
					overlay_path,
					e.message
				);
				return;
			}

			var folders_list = new Gee.ArrayList<string>();
			GLib.FileInfo? file_info;
			while ((file_info = enumerator.next_file(null)) != null) {
				var item_overlay_path = GLib.Path.build_filename(
					overlay_path,
					file_info.get_name()
				);
				paths.add(this.to_real_path(item_overlay_path));
				if (
					file_info.get_file_type()
					== GLib.FileType.DIRECTORY
				) {
					folders_list.add(item_overlay_path);
				}
			}

			foreach (var folder_overlay_path in folders_list) {
				this.collect_overlay_paths(
					folder_overlay_path,
					paths
				);
			}
		}

```
