## Purpose

- **🔷** Extract bubblewrap + overlay + seccomp from **`libocfiles/Sandbox/`** into a **small standalone library** **`libocbwrap`** (namespace **`OLLMbwrap`**).
- **🔷** Define a **`FileVerification`** interface in **`libocbwrap`** — post-exec overlay changes are **applied by the host** (`libocfiles` today, **`ollmfilesd`** later), not hard-wired to **`ProjectFiles`** inside the sandbox library.
- **🔷** **`liboctools/RunCommand`** and **`libocmcp/Client/Stdio`** link **`libocbwrap`** directly — **MCP and run_command keep working** while **`Exec.*` RPC** and V2 cutover proceed in parallel.
- **🔷** **Optional / stub build** on **Windows** and **Android** — same vapi, **`Bubble.can_wrap()` → false**, **no `libseccomp`** link.
- **ℹ️** This **delays** copying sandbox into **`ollmfilesd`** — daemon **`Exec.run`** will **`--pkg=ocbwrap`** + supply a daemon **`FileVerification`** impl when RPC lands ([`2.10.4.15`](2.10.4.15-ACTIVE-execution-rpc-sandbox.md)).

---

## Why this approach (🔷)

- **🔷** **No duplicate sandbox tree** — **🚫** do not copy **`Sandbox/*`** into **`ollmfilesd/`** and renamespace (see superseded section in [`2.10.4.15`](2.10.4.15-ACTIVE-execution-rpc-sandbox.md)).
- **🔷** **`libocbwrap` has no `libocfiles` dependency** — avoids coupling bwrap to PM / **`Folder`** / SQLite; **`ollmfilesd`** namespace ban is unaffected.
- **🔷** **File index sync stays in the domain layer** — overlay *walking* is generic; *verification* (DB rows, approvals, history) is injected via **`FileVerification`**.
- **🔷** **Platform stubs in one place** — Android + Windows build the same library name with stub **`Bubble`**; callers keep one code path (`can_wrap()` branch).
- **🔷** **Unblocks V2 `libocfiles`** — shipping **`Sandbox/*`** removed from **`libocfiles/meson.build`**; **`liboctools`** / **`libocmcp`** depend on **`libocbwrap`** instead.

## Library layout

### New tree: `libocbwrap/`

- **`namespace.vala`** — **`OLLMbwrap`**
- **`SandboxProfile.vala`** — bind-mount inputs (replaces **`Folder`** ctor arg on **`Bubble`**)
- **`Bubble.vala`** (Linux) — **`can_wrap()`**, **`exec()`**, **`build_bubble_args()`**
- **`BubbleStub.vala`** (Windows / Android) — same public API; **`can_wrap()` → false**
- **`Overlay.vala`** — overlay dir create/mount/cleanup; uses **`SandboxProfile`**, not **`OLLMfiles.Folder`**
- **`RunSeccomp.vala`** (Linux full build only) — NOTIFY + evidence strings
- **`FileVerification.vala`** — **interface** (see below)
- **`OverlaySync.vala`** — generic overlay upper-dir walk; calls **`FileVerification`** per change
- **`meson.build`** — platform split (see **Meson**)

### Move out of `libocfiles/Sandbox/` (delete after migration)

- **`Bubble.vala`** → **`libocbwrap/Bubble.vala`**
- **`BubbleWindows.vala`** → **`libocbwrap/BubbleStub.vala`** (rename for clarity)
- **`Overlay.vala`** → **`libocbwrap/Overlay.vala`**
- **`RunSeccomp.vala`** → **`libocbwrap/RunSeccomp.vala`**
- **`Scan.vala`** → **split**:
  - overlay directory walk → **`libocbwrap/OverlaySync.vala`**
  - project-files / DB / history apply → **`libocfiles/OverlayVerification.vala`** (implements **`FileVerification`**)

### Stay in `libocfiles` (adapter — not in `libocbwrap`)

- **`libocfiles/OverlayVerification.vala`** — **`OLLMfiles.OverlayVerification : OLLMbwrap.FileVerification`**
  - today’s **`Scan.handle_file`**, **`handle_folder`**, **`handle_remove`**, … logic
  - holds **`OLLMfiles.Folder?`** + **`ProjectManager`** refs

### Later: `ollmfilesd` adapter (with [`2.10.4.15`](2.10.4.15-ACTIVE-execution-rpc-sandbox.md))

- **`ollmfilesd/OverlayVerification.vala`** — **`OLLMfilesd.OverlayVerification : OLLMbwrap.FileVerification`**
  - same contract, daemon **`OLLMfilesd.Folder`** / **`ProjectFiles`**

## `SandboxProfile` (🔷)

Replaces passing **`OLLMfiles.Folder?`** into **`Bubble`**. Callers build from active project (or null).

- **`project_path`** (string) — project root; empty = no-project mode
- **`write_roots`** (`Gee.HashMap<string,string>`) — absolute path → overlay subdir label (from today’s **`Folder.build_roots()`** + **`allow_write`** parsing)
- **`allow_network`** (bool)
- **`write_tokens`** (string[]) — raw **`allow_write`** array for policy checks inside **`Bubble`**

**ℹ️** **`RunCommand`** and **`Stdio`** construct **`SandboxProfile`** from **`active_project`** before **`new Bubble(profile)`**.

---

## `FileVerification` interface (🔷)

**🔷** Host implements this; **`OverlaySync.run()`** invokes it for each detected overlay change.

Suggested methods (names TBD at implementation — one **`###` per method in code proposals):

- **`async void file_created(string real_path, string overlay_path)`**
- **`async void file_modified(string real_path, string overlay_path, …)`**
- **`async void file_removed(string real_path)`**
- **`async void folder_created(…)`** / **`folder_removed(…)`** / **`alias_*`** — mirror today’s **`Scan.handle_*`** split
- **`async void finish()`** — today’s **`Scan.run()`** tail: **`review_files.refresh()`**, cleanup maps

**🚫** Interface lives in **`libocbwrap`** — **🚫** no **`ProjectFiles`**, **`FileHistory`**, or SQLite types in the interface signature (pass paths + optional opaque ids only).

**💩** Alternative: single **`async void apply(OverlayChange change)`** with a small **`OverlayChange`** enum + path fields — fewer interface methods; confirm at implementation.
