## `Bubble` / `Overlay` — Vala construct properties (🔷)

**🚫 No **`SandboxProfile`** wrapper class**

- **💩** dropped

**🔷** **`Bubble`** and **`Overlay`** extend **`GLib.Object`**

- declare optional inputs as **`{ get; construct; }`** properties with **`default =`** where sensible
- **`verification`** is the **only** ctor parameter — non-null **`FileVerification`**, forwarded with **`Object (verification: verification)`**
- **🚫** no multi-parameter **`public Bubble (…)`** ctor (no positional laundry list)
- **🚫** not plain fields assigned in ctor body (shipping **`Bubble`** style today)
- other properties: object initializer on the same **`new`** expression

**🔷 Class shape** (illustrative — see **`## Concrete code proposals`**)

```vala
public class Bubble : Object {
    public string project_path { get; construct; default = ""; }
    public bool allow_network { get; construct; default = false; }
    public string[] write_tokens { get; construct; default = {}; }
    public Gee.HashMap<string, string> write_roots {
        get; construct; default = new Gee.HashMap<string, string> ();
    }
    public FileVerification verification { get; construct; }

    public Bubble (FileVerification verification) {
        Object (verification: verification);
    }
}
```

**🔷 **`Bubble`** construct properties** (semantics)

- **`project_path`** (`string`, default `""`)
  - project root
  - empty = no-project mode (today’s **`project == null`**)
- **`allow_network`** (`bool`, default `false`)
- **`write_tokens`** (`string[]`, default `{}`)
  - parsed **`allow_write`** tokens
  - **`project`**, absolute paths
  - same as today’s **`write_array`**
- **`write_roots`** (`Gee.HashMap<string,string>`, default empty map)
  - caller fills from **`Folder.build_roots()`** + policy (in-app)
  - daemon RPC relay resolves from project row + **`allow_write`**
- **`verification`** (**`FileVerification`**, required — sole ctor arg)
  - ctor parameter is non-null; **`this.verification`** always set after construction
  - wired into **`Overlay`** / **`Scan`**
  - scan paths: real impl (**`RunCommand/FileVerification`**, **`ollmfilesd/FileVerification`**)
  - no-scan paths (MCP **`build_bubble_args`**): caller passes a no-op **`FileVerification`**

**🔷 **`Overlay`**

- same pattern: **`verification`** sole ctor arg; other inputs via object initializer
- **`Bubble.exec`** builds **`new Overlay (this.verification) { … }`** from remaining **`this.*`**

**ℹ️ Caller** (ctor + object initializer — like **`Daemon`**, plus **`ModelUsage.clone`**-style **`{ … }`**)

```vala
var bubble = new OLLMbwrap.Bubble (verification) {
    project_path = project.path,
    allow_network = this.network,
    write_tokens = this.write_array,
    write_roots = roots
};
```

**ℹ️ In-app tools** (after **`libocbwrap`** lands)

  - still uses shipping **`libocfiles`** for **`ProjectManager`**, **`Folder`**, etc.
- **`libocmcp/Client/Stdio`**
  - **`OLLMbwrap.Bubble (no_op_verification) { … }`**, **`RunSeccomp`**, **`build_bubble_args`**

## `FileVerification` (🔷)

**Naming**

- **`OLLMbwrap.FileVerification`** — interface (`libocbwrap/FileVerification.vala`)
- **`OLLMfilesd.FileVerification`** — daemon (`ollmfilesd/FileVerification.vala`)
- **`OLLMtools.RunCommand.FileVerification`** — in-app (`liboctools/RunCommand/FileVerification.vala`)
- **🚫** not in **`libocfiles/`**

**🔷 Caller**

- **`Scan`** (same class as shipping)
- calls from **`handle_*`**
- three operations + **`GLib.FileType`**
- **🚫** no custom enum

**Interface methods**

- **`async void created(GLib.FileType file_type, string real_path, string overlay_path)`**
- **`async void modified(GLib.FileType file_type, string real_path, string overlay_path)`**
- **`async void removed(GLib.FileType file_type, string real_path, string overlay_path)`**
  - whiteout / delete
  - **`overlay_path`** when needed (today’s **`handle_remove`**)
- **`async void finish()`**
  - today’s **`Scan.run()`** tail
  - **`review_files.refresh()`**, cleanup maps

**ℹ️ **`file_type`**

- reuse **`GLib.FileType`** from overlay **`FileInfo`**
- same as **`Scan.scan_dir`**:
  - **`DIRECTORY`** → folder
  - **`SYMBOLIC_LINK`** / symlink branch → alias
  - otherwise → regular file

**ℹ️ **`Scan`** vs **`FileVerification`**

- **`Scan`** decides created vs modified vs removed
  - whiteout
  - existing index lookup
  - stays in **`Scan`** walk
- **`FileVerification`** only applies the change

**🚫 Interface constraints**

- lives in **`libocbwrap`**
- **🚫** no **`ProjectFiles`**, **`FileHistory`**, or SQLite types in signature
- paths only

- **`property-initialization`** — construct **`default =`** on properties; no ctor-body defaults

- **`docblocks`** — public **`FileVerification`**, **`Bubble`**, **`Overlay`** API
- **`agent-compliance-gate`** — run before marking implementation done
