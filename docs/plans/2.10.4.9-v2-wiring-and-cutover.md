### T8 ‚Äî `notify_config_updated()` (detail)

**Problem:** Config persistence is scattered ‚Äî settings UI knows to call `config.changed()`, but there is no standard method on `ApplicationInterface` for ‚Äúconfig was saved / reloaded; react.‚Äù

**Proposed (option A ‚Äî minimal):**

```vala
// ApplicationInterface ‚Äî default implementation
public void notify_config_updated() {
    this.config.changed();
}
```

**Call sites after cutover / tidy-up:**

| Caller | When |
| ------ | ---- |
| `ollmapp/SettingsDialog/MainDialog` | Replace direct `this.app.config.changed()` with `this.app.notify_config_updated()` |
| `ollmfilesd/Application` | After any future config reload path (e.g. SIGHUP or RPC) ‚Äî same method as GUI app |
| Tests / CLI | Same entry point if they mutate and save config |

**Listeners:** unchanged for option A ‚Äî still `config.changed.connect(...)`. Option B adds `app.config_updated` for components that should not depend on `Config2` directly.

**Out of scope for T8:** daemon RPC to push config to `ollmfilesd`; file-indexer config keys ‚Äî those stay on `Config2` / tool registries until a separate plan says otherwise.
