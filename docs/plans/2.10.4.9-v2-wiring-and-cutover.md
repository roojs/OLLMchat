## Goal

[`2.10.4.5`](2.10.4.5-DONE-folder.md) and sibling plans **remove** daemon-only code from client types. That leaves:

1. **Caller reshape** — UI and tools still read `project_files`, `child_map`, `folder_map` on shipping paths.
2. **Folder fallout** — client `Folder` no longer has `project_files`; V2 code still references it in places.
3. **Meson cutover** — swap `libocfiles/*.vala` → `libocfiles/V2/*.vala` in the app build.

This plan owns **wiring those together** — decide first, then implement. **No ad-hoc stubs** in V2 PM (e.g. reusing `file_cache` as a project index).

Exception: **file dropdown** — RPC to fill its **own** Gtk model on open/refresh (wire TBD — [`2.10.4.1`](done/2.10.4.1-DONE-ollmfilesd-rpc-api.md)).
