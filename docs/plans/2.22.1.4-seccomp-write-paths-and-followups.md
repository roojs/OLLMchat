# 2.22.1.4 Seccomp file evidence: write paths, allowlists, follow-ups

**Parent:** [2.22.1-run-command-seccomp-network-and-path-evidence.md](./2.22.1-run-command-seccomp-network-and-path-evidence.md) · **Prior:** [2.22.1.3](./2.22.1.3-seccomp-reporting-after-bwrap.md)

## Unchanged behaviour

- **Flatpak / no `bwrap`:** **`FLATPAK_ID`** **or** missing **`bwrap`** on **`PATH`** → **`can_wrap()`** false → **`execute_with_subprocess()`** → **no bwrap, no seccomp-based sandbox reporting** for that run. Stays as today; do not change this branch to load seccomp.

## 1. Approved (this change only)

- **Scope:** **Reporting only** for this work, plus **supporting `allow_write`**. **Not in this change:** general logging, tracking, or overlay-based auditing for arbitrary host usage — separate project later.

- **`allow_write`:** **`no`** (default), **`project`**, or **a set of paths** in one string; **user approval** when the tool uses a **non-empty** parsed path set. No second parameter name (reuse this field only).

- **File warnings vs what is allowed:** give **`RunSeccomp`** a way to ask the same policy as bubblewrap (e.g. **inject a reference from the bwrap side into the seccomp setup / ctor** so it can answer **“is this file allowed?”**).
  - When **not** allowed: record in **`file_writes`** as **`string → bool`** (presence map for quick lookups).
  - Build the appendix from **unique keys** of that map (e.g. collect keys → **`string.joinv`**), capped at **50** entries.

- **`unlink` / `rename`:** if the model triggers them, **pick them up** — **`unlink`:** affected path / name; **`rename`:** source **and** target. Fold into the same reporting path as disallowed writes where it fits.
  - **LLM-facing copy (intent):** you tried to write / change these paths but the sandbox prevented it because you did not request permission using **`allow_write`** (exact wording in implementation).

### 0. Parse order, validation, and a single `write_roots`-style list

**Why “validate after permission” is wrong:** `Request.execute()` runs **`build_perm_question()`** → **`get_permission_provider().request()`** → **`execute_tool_async()`**. Any rule enforced only in **`execute_tool_async()`** (for example **“each path segment must be absolute”**) runs **after** the user may have already approved. That ordering is incorrect for input the UI should never have offered.

**Fix:** Parse **`allow_write` once** (shared **`private` helper** on **`Request`**, or **`out`** parameters filled from **`execute()`** before **`build_perm_question()`**) so that **invalid combinations return `ERROR: …` from `execute()` before any permission prompt**. **`build_perm_question()`** then only reads the already-validated parse (cached on **`this`**, or the helper is idempotent and cheap). Same parse drives the permission string and **`Bubble`** construction.

**Treat keywords as the first list segment (`write_roots` mental model):** After **`strip`** and **`split (GLib.SEARCHPATH_SEPARATOR_S)`**, drop empty segments into **`string[] tokens`**. If **`this.allow_write.strip() == ""`**, normalize **before permission** to a default token: **`project`** when an active project exists, otherwise **`no`**. Then if **`tokens.length == 0`**, treat as the same normalized default so there is always at least one logical value. Interpret **`tokens[0].down()`**:

- **`no`** or **`project`** — key values only; no extra path looping for these values.
- **Otherwise** — every token must be an absolute path; those are the only additional write roots.

**Single write policy shape:** After parse, **only `write_array`** carries write intent — tokens are **`no`**, **`project`**, and/or **absolute paths** (§0). **`Request`** caches **`write_array` only** (no parallel **`bool`** / **`string[]`** for the same policy). **`Bubble`** takes **`(project, network, write_array)`** only; **inside `Bubble`**, keep root bind decision inline and derive extra **`--bind`** roots from **`write_array`** directly.

**No repeated split/strip on `allow_write`:** Tokenize the tool string **exactly once** per run, **inline in `Request.execute()` before `build_perm_question()`** (no helper method). Store only **`this.write_array`**. **`build_perm_question()`**, **`execute_tool_async()`**, **`build_bubble_args()`**, and **`can_write()`** consume this validated array and must not normalize again.

**`one_time_only` (not the same as “session allow” in the UI):** When **`RequestBase.one_time_only`** is **true**, **`libollmchatgtk/ChatPermission.request`** hides **“Allow Always”** and **“Deny Always”**; the user still sees **“Allow”** and **“Deny”** (mapped to **`ALLOW_ONCE`** / **`DENY_ONCE`**). The **`ChatPermission.PermissionResponse`** enum also includes **`ALLOW_SESSION`** / **`DENY_SESSION`**, and **`Provider`** can store session-scoped decisions — but the **current GTK permission row does not add separate “for this session” buttons**; session vs always is mainly relevant when the provider maps responses into its store. Use **`one_time_only = true`** for prompts whose **`permission_target_path`** is **intentionally unique each time** (**`network#…`**, **`allow_write_paths#…`**) so **“Always” could not match the next run anyway**; it avoids implying a durable **Always** that would never hit the cache. For stable keys (e.g. resolved binary **`realpath`**), leave **`one_time_only`** **false** so the user can persist or session-cache if the provider supports it.

#### 2.4 `can_write` (new public API)

**Status:** **`Bubble.vala` in the tree today does not define this method** — it exists only as a plan item. Shipping requires a real body (not a stub).

#### Add

(after **`build_bubble_args`** or next to other **`public`** helpers — place is implementer choice; paste **below** the closing **`}`** of **`build_bubble_args`**)

```vala
		/**
		 * Whether writes to @path are permitted for this sandbox profile.
		 * Must match {@link build_bubble_args} (bind / overlay / tmpfs / dev-bind).
		 *
		 * @param path absolute path from NOTIFY (caller must not pass raw tool strings to re-tokenize)
		 */
		public bool can_write (string path)
		{
			if (path == "/tmp" || GLib.str_has_prefix (path, "/tmp/")) {
				return true;
			}
			if (path == "/dev/null") {
				return true;
			}
			var home = GLib.Environment.get_home_dir ();
			var bind_play = GLib.Path.build_filename (home, "playground");
			if (path == bind_play || GLib.str_has_prefix (path, bind_play + "/")) {
				return true;
			}
			if (this.overlay.overlay_map.size > 0) {
				foreach (var e in this.overlay.overlay_map.entries) {
					var lower = e.value;
					if (path == lower || GLib.str_has_prefix (path, lower + "/")) {
						return true;
					}
				}
			}
			for (var i = 0; i < this.write_array.length; i++) {
				var root = this.write_array[i];
				if (i == 0 && (root.down () == "no" || root.down () == "project")) {
					return false;
				}
				if (path == root || GLib.str_has_prefix (path, root + "/")) {
					return true;
				}
			}
			return false;
		}
```

Tighten **`/tmp`** / **`write_array`** path-prefix rules in the PR if needed (e.g. **`/tmpfoo`** must not match). Keep **`build_bubble_args`** and **`can_write`** loop semantics identical. **Do not normalize/validate in these methods**; they consume pre-validated **`write_array`** only.
