# 6.5 Write-plan output: allow `#section` links and resolve them to absolute file paths

**Status:** Open

**Purpose:** When a **write_file** / plan-style executor writes a plan under **## Change details**, the model often puts `**[Label](#slug)`** in **## Result summary** pointing at headings inside the written body. `**ValidateLink`** must accept those anchors when they exist in the **write** `**Document`** or the **fallback** `**Document`**, and `**ResultParser.exec_extract**` may rewrite matching fragment links to **absolute** file paths. **Validation** only checks that the anchor exists — it **must not** replace or mutate write body content.

**Phases (`liboccoder/Task/Details.vala` — `MarkdownPhase`):** `**EXECUTION`** = executor LLM output (`**ResultParser.exec_extract**`, `**Tool.run**`). `**POST_EXEC**` = post-exec synthesis (`**exec_post_extract**`, `**Details.run_post_exec**`). This plan uses `**MarkdownPhase.EXECUTION**` for `**ValidateLink**` constructed in `**exec_extract**`, and `**MarkdownPhase.POST_EXEC**` only for `**exec_post_extract**`. The fragment / `**check_writes**` branch in `**validate()**` must apply to `**EXECUTION**` (executor) and `**POST_EXEC**` (synthesis) — not `**POST_EXEC**` alone for both call sites.

**Checklist (see `.cursor/rules/CODING_STANDARDS.md`):** **No nullable fields** for this feature — use **default empty** `**Gee.ArrayList`**, `**new Markdown.Document.Document ()**`, or a **dummy** `**Tool`** (empty `**writes**`) where a reference type is required; `**this.**` / `**GLib.**`; multiline docblocks. **One** new helper on `**ValidateLink`** only: `**check_writes**` (see §4.2). No other new helpers unless explicitly expanded later.

---

## 1. Problem

- `**Document.headings**` on the **root** executor `**Document`** only registers **top-level** headings, so `**#slug`** targets inside **Change details** bodies are missing.
- Fragment validation needs a **clear order**: **write** bodies first (parsed as their own `**Document`**), then the **main** parsed document for the response (e.g. summary / top-level `**header_list`**).

---

## 2. Goals

1. `**WriteChange**` holds a `**Markdown.Document.Document document**` — **empty** by default; filled when the write body is parsed (same `**Render.parse`** pipeline as elsewhere) so `**document.headings**` / `**header_list**` match that body.
2. `**ValidateLink**` holds everything it needs **without nullable types**: e.g. `**Gee.ArrayList<WriteChange> writes`** (default **empty**) — in `**exec_extract**`, set from the tool’s `**ex.writes**` (may be **empty** — see goal 3); and `**Markdown.Document.Document document`** on `**ValidateLink**` (default `**new Document ()**`) holding the fallback parse for `**#**` in **Result summary** (e.g. assign `**sum_render.document**`). **Alternatively** a non-null `**Tool`** reference using a **dummy `**Tool`** with empty `**writes**` when not in an exec-with-writes context — **pick one** pattern, **no** `**Tool?`**. Fragment `**#**` validation: `**check_writes**` checks **each** `**WriteChange.document.headings.has_key (link.hash)`**, then `**this.document.headings.has_key (link.hash)**`.
3. **Non–write skills / no parsed writes:** the list assigned to `**ValidateLink.writes`** (from `**ex.writes`** on the tool in `**exec_extract**`) may be an **empty** `**Gee.ArrayList`** — `**foreach**` over it does nothing; `**check_writes**` then uses `**ValidateLink.document**` only. **No** placeholder `**WriteChange`** instances. **Strict** “must have **Change details** or **no changes needed**” rules apply **only** where the skill is a **write** skill (`**details.skill.tools`** contains `**write_file**`), not at every call site — see §4.3.
4. **Rewrite** (absolute path) stays in `**ResultParser.exec_extract`** after validation — **out of scope** for `**ValidateLink`** (validation only).
5. **Post-exec** synthesis: link rewriting with `**task://`** prefixes and related rules — **defer** (see §6).

---

## 3. Non-goals

- Building a **union** of every slug from every `**header_list`** into `**ValidateLink**` for bulk membership — **no**; resolution is **per link** via `**check_writes`** + fallback `**Document**`.
- `**ValidateLink**` mutating `**WriteChange.content**` or `**WriteChange.document**` during validation.
- **Post-exec** `**task://`** fragment rewriting in this iteration.

---

## 4. Proposed approach

### 4.1 `WriteChange.document`

- `**public Markdown.Document.Document document { get; set; }**` — default `**new Markdown.Document.Document ()**` (or equivalent empty document per project style).
- **Populate** in `**WriteChange.from_header`** when setting `**content**`: `**Render.parse (content)**` (or `**code_text**` for fenced `**markdown**`) into `**this.document**` so `**headings**` / `**header_list**` reflect the body. **Idempotent** or guarded so repeated parse does not duplicate state if `**from_header`** runs once.

### 4.2 `ValidateLink`: **`writes`** + **`document`** + **`check_writes`**

- **No nullable properties.** Recommended shape:
  - `**public Gee.ArrayList<WriteChange> writes { get; set; default = new Gee.ArrayList<WriteChange> (); }`**
  - `**public Markdown.Document.Document document { get; set; default = new Markdown.Document.Document (); }**`
  - Non-exec / no writes: `**ValidateLink.writes**` stays **empty**; `**ValidateLink.document**` is replaced or left empty depending on phase — `**check_writes**` only consults `**this.document.headings**` when the write loop finds nothing (empty `**this.writes**` skips the loop).
- **If** the design insists on `**Tool`**: store `**Tool exec_tool**` with a **static** or **factory-built dummy `**Tool`** (empty `**writes**`) as default — **still no** `**Tool?`**.
- `**public bool check_writes (Markdown.Document.Format link)**`:  
  - `**if (link.path != "" || link.hash == "")**` return **false**.  
  - `**foreach (var wc in this.writes)`** `**if (wc.document.headings.has_key (link.hash))**` return **true**.  
  - `**if (this.document.headings.has_key (link.hash))`** return **true**.  
  - return **false**.  
  - **Open:** that guard assumes fragment links are **bare `#anchor`** (`**link.path == ""**`). If the model emits **`somefile.md#anchor`** (path and hash both set), `**check_writes**` will not run — may need a follow-up to resolve the path (target file vs summary) and still validate the hash.
- In `**validate()`** for `**link.path == ""**`, `**case MarkdownPhase.EXECUTION:**` and `**case MarkdownPhase.POST_EXEC:**` (same body): after `**user_request**` / `**out_doc**` checks, `**if (this.check_writes (link))**` **return**; **else** existing error. **Do not** use `**POST_EXEC`** for `**exec_extract**` — that path is `**EXECUTION**`.
- `**http()**`, `**file()**`, `**task()**` also `**switch (this.stage)**` on `**POST_EXEC**`. If `**exec_extract**` passes `**EXECUTION**`, those methods must treat `**EXECUTION**` like `**POST_EXEC**` wherever `**POST_EXEC**` is currently grouped (otherwise `**http()**` would reject URLs, `**task()**` would use the `**LIST**` branch, etc.). Add `**case MarkdownPhase.EXECUTION:**` next to each relevant `**POST_EXEC**` case (see current `**ValidateLink.vala**`).

### 4.3 `ResultParser` / `**exec_extract**` and `**Details.skill**`

- `**Details.skill**` — `**public OLLMcoder.Skill.Definition skill { get; set; }**` (or `**{ get; private set; }**` if you only assign from `**Details**` internals). Holds the resolved `**Skill.Definition**` for `**task_data["skill"]**`. Set **once** when the skill name is known and valid (same moment you would `**skill_manager.fetch (this)`** today — e.g. end of `**fill_task_data**`, or immediately after `**skill_manager.validate (this)**` in code paths that guarantee a registered skill). **Usage:** `**details.skill.tools.contains ("write_file")`** for **write-contract** rules; `**details.skill.execute`**, `**details.skill.header**`, etc. anywhere `**fetch(this)**` was repeated. **Do not** add a second cached field (e.g. `**skill_definition`**) — `**skill**` is the single stored reference.
- **Invariant — `task.skill` must not be null after the task is live in the pipeline:** Once a `**Details**` instance is past **initial** setup (task list parsed, skill name resolved, and validation has accepted the task for refinement/execution), **`skill`** is **always** set to a valid `**Skill.Definition**`. **Executor, post-exec, `**ResultParser.exec_extract**`, link validation, prompts, etc.** must **assume** `**ex.parent.skill**` / `**task.skill**` is non-null — **do not** add routine `**if (task.skill != null)**` guards; reaching those stages with **`skill == null`** is a **bug** (failed validation or missing assignment), not a case to handle quietly. Invalid or unknown skills should **fail earlier** (`**skill_manager.validate**`, issues on the task) and **not** reach tool runs. Only the **earliest** construction / `**fill_task_data**` path may run before assignment; after that, **`assert (this.skill != null)**` in debug builds is OK, **not** production fallbacks to `**fetch (this)**` “in case skill was unset.”
- **Stop branching link validation on `write_file`.** After `**sum_render`**, `**ex.writes.clear()**`, and filling `**ex.writes**` from `**## Change details**` (zero or more `**WriteChange**` rows — **empty array** when there are no change-detail sections or the skill is not a write skill), run **one** `**ValidateLink`** pass: set `**ValidateLink.writes**` from `**ex.writes**`, `**ValidateLink.document = sum_render.document**`. An **empty** `**ValidateLink.writes`** is fine: the `**foreach**` in `**check_writes**` runs **zero** times, then `**this.document`** on `**ValidateLink`** is checked. No separate “validate summary then `**return true**` if !`**write_file**`” block for links.
- **Where `write_file` still matters:** `**Skill.Definition.tools`** declares whether the **write** executor contract applies (**## Change details**, **“no changes needed”** escape hatch, unexpected top-level sections). Use `**ex.parent.skill.tools.contains ("write_file")`** **only** in that **strict** tail (empty writes / section errors), **not** duplicated before every `**ValidateLink`** or stored as a separate boolean on `**Details**`.
- **Populate** `**WriteChange.document`** before `**validate_all**` for **real** rows so `**check_writes`** sees parsed bodies.
- **Rewrite** fragment links under `**ex.summary`** after validation (unchanged intent): **first** `**WriteChange`** in `**ex.writes**` whose `**document.headings.has_key (link.hash)**` → absolute `**file_path**` (if `**ex.writes.size == 0**`, nothing to rewrite from writes — summary-only).

### 4.4 `exec_post_extract`

- `**ValidateLink.writes**` default **empty**; `**ValidateLink.document**` = the parsed **Result summary** / synthesis slice (same `**Markdown.Document.Document**` as `**ResultParser.document**` after parse where applicable). `**check_writes**`: `**foreach**` over `**this.writes**` is a no-op when empty, then `**this.document**` is checked.
- **Link rewriting** with `**task://…#`** — **not** in this plan iteration (§6).

### 5.3 `ValidateLink.vala` — new fields + **`check_writes`**

**Current `ValidateLink`** (`liboccoder/Task/ValidateLink.vala`) has no `**writes**` or `**document**` properties — only `**runner**`, `**details**`, `**stage**`, `**issues**`. The fields below are **add**itions from this plan.

**add** (after `**MarkdownPhase stage;`**, ~line 28)

```vala

	 * Change-detail sections used to resolve fragment links (hash targets).
	 * In exec_extract, assign the same list as the Tool's writes; may be empty.
	 */
	public Gee.ArrayList<WriteChange> writes {
		get; set; default = new Gee.ArrayList<WriteChange> ();
	}

	/**
	 * Parsed Result summary (or synthesis slice) for fragment fallback when writes do not match.
	 * Default empty until set by ResultParser before validate_all.
	 */
	public Markdown.Document.Document document {
		get; set; default = new Markdown.Document.Document ();
	}
```

**add** (before `**validate`**, or after `**validate_all**`)

```vala
	/**
	 * Whether link.hash matches a heading in any WriteChange body or in this.document.
	 *
	 * Only considers bare fragment links (empty path and non-empty hash). If the model
	 * emits path-plus-hash links, extend this helper once that shape appears in executor output.
	 *
	 * @param link parsed link (fragment-style: path empty, hash set)
	 * @return true if the anchor resolves in writes or document
	 */
	public bool check_writes (Markdown.Document.Format link)
	{
		if (link.path != "" || link.hash == "") {
			return false;
		}
		foreach (var wc in this.writes) {
			if (wc.document.headings.has_key (link.hash)) {
				return true;
			}
		}
		return this.document.headings.has_key (link.hash);
	}
```

### 5.5 `ResultParser.vala` — `**exec_extract**`: fill `**ex.writes**` then **one** `**ValidateLink`**

**remove** (current block: summary links validated **before** `**ex.writes`** are filled — ~lines 500–510). Note: today’s code wrongly passes `**MarkdownPhase.POST_EXEC**` here; use `**EXECUTION**`.

```vala
		var sum_render = new Markdown.Document.Render();
		sum_render.parse(ex.summary.to_markdown_with_content());
		var vl_sum = new ValidateLink(this.runner, ex.parent, MarkdownPhase.POST_EXEC);
		vl_sum.validate_all(sum_render.document.links);
		if (vl_sum.issues != "") {
			this.issues += vl_sum.issues;
			return false;
		}
		if (!ex.parent.skill_manager.fetch(ex.parent).tools.contains("write_file")) {

		}
```

**replace with** (**single** link-validation path): `**sum_render`** → `**ex.writes.clear()**` → **existing** `**foreach (slug … header_list)`** / `**WriteChange.from_header**` (adds **zero** or more rows; **empty `ex.writes`** is normal for non-write output) → **if** `**this.issues != ""**` **return false** (change-details loop failures only — see note below) → **one** `**ValidateLink`** with `**writes**` / `**document**` set via **object initializer** (see §5 legend), `**MarkdownPhase.EXECUTION**`. `**ex.parent.skill**` must be set — expected non-null here per §4.3 invariant. Use `**ex.parent.skill.tools.contains ("write_file")**` **only** for the **strict** tail (“no changes needed”, missing change details when writes empty) — **not** to skip `**ValidateLink`**.

```vala
		var sum_render = new Markdown.Document.Render ();
		sum_render.parse (ex.summary.to_markdown_with_content ());
		ex.writes.clear ();
		foreach (var slug in this.document.header_list) {
			/* … same body as current exec_extract (change-details, wc.issues, ex.writes.add) … */
		}
		if (this.issues != "") {
			return false;
		}
		var vl_sum = new ValidateLink (this.runner, ex.parent, MarkdownPhase.EXECUTION) {
			writes = ex.writes,
			document = sum_render.document
		};
		vl_sum.validate_all (sum_render.document.links);
		if (vl_sum.issues != "") {
			this.issues += vl_sum.issues;
			return false;
		}
		if (ex.parent.skill.tools.contains ("write_file") && ex.writes.size == 0) {
			var md = this.document.to_markdown ().strip ();
			if (md.down ().contains ("**no changes needed**")) {
				return true;
			}
			this.issues += "\nWrite executor output must include a recognizable Change details section, or Path 2 (full markdown must contain **no changes needed**).";
			return false;
		}
		return true;
```

**Why `write_file` *and* `ex.writes.size == 0`?** It is not redundant — they are different axes. Implement as **one** `if` with `&&`, not nested `if`s:

- **`skill.tools` contains `write_file`** — From YAML: this task uses the **write executor** (plan/change-details style). The model is instructed to produce **## Change details** **or** the Path 2 escape phrase (**no changes needed**). Skills that **do not** list `write_file` use a **different** executor template; **empty `ex.writes`** is normal and must **not** produce “missing Change details” errors.
- **`ex.writes.size == 0`** — After the header loop, **no** section became a valid `WriteChange`. For a **write** skill only, that means: either the model chose Path 2 (check full markdown for **no changes needed**) **or** the output is incomplete. When `ex.writes.size > 0`, structured writes exist and this tail does not apply.

So: **write contract** and **no parsed change-detail rows** together mean: require Path 2 or fail. Omitting the `write_file` check would wrongly punish non-write executors for empty `ex.writes`.

**Note — `this.issues` vs `write_path_issues_start` (today’s code):** `**ResultParser.exec_extract**` currently sets `**var write_path_issues_start = this.issues.length**` before the change-details `**foreach**` because **summary link validation runs first** and, on success, `**this.issues**` is still empty — so “issues added during the loop” equals `**this.issues.length > write_path_issues_start**`. That is a **delta**, not “non-empty issues” in general. If anything ever appends to `**this.issues**` **before** that loop, the delta is still correct but `**this.issues != ""**` alone would not be. **Unified order** (loop **before** `**ValidateLink`**): as long as **nothing** appends to `**this.issues**` before the loop, **`if (this.issues != "") return false`** after the loop is the usual style and matches “we check for issues, not for a length change.” If a future refactor prepends parser warnings, use a **local** `**string write_issues**` built only inside the loop, then **`if (write_issues != "") { this.issues += write_issues; return false; }**`.

**Note:** merge/adjust with the **rest** of `**exec_extract`** as implemented — the **replace with** fence above shows **strict** write rules **after** link validation, keyed off `**ex.parent.skill.tools`**, not a second `**ValidateLink**`.

### 5.7 Rewrite (after successful `**exec_extract`** validation)

**Depends on §5.1–§5.2** — `**WriteChange.document**` must be filled when `**content**` is set so heading lookup does **not** re-parse `**content**` (same parsed tree as `**ValidateLink.check_writes**` / validation).

```vala
	/**
	 * Bare fragment in Result summary → project-relative file link. Uses [[document]] from §5.2.
	 */
	public bool update_link(Markdown.Document.Format link)
	{
		if (link.path != "" || link.hash == "") {
			return false;
		}
		if (!this.document.headings.has_key (link.hash)) {
			return false;
		}
		link.up_relpath (this.file_path.strip ());
		return true;
	}
```

## 6. Testing

- **Write** path: `**#slug**` in **Result summary** matches heading in `**WriteChange.document`** → valid.
- **Fallback:** `**#slug`** only in **summary** `**Document`** → valid.
- **No writes:** fragment validates against `**ValidateLink.document`** only; **no** content replacement.
- Regression: **user_request**, **out_doc**, **http(s)** unchanged for **post-exec**; **exec** summary uses `**EXECUTION`** with `**http`/`file`/`task**` rules aligned to `**POST_EXEC**` (§5.4b).

## 8. Open decisions

- **`ValidateLink.writes`** + **`document`** only vs **dummy `Tool` + `document`** — both avoid nulls; `**ArrayList**` + `**Document**` is usually simpler.
- `**document**` for **exec**: `**sum_render.document`** vs `**this.document**` when nested `**###**` in **Result summary** only appear in `**sum_render.document`**.
- **When to set `**Details.skill`** — typically `**fill_task_data**` when `**skill_name**` is in `**by_name**`; invalid tasks fail validation and never reach exec. **Invariant:** `**task.skill == null**` after initial setup is disallowed (§4.3) — not an open design choice.
