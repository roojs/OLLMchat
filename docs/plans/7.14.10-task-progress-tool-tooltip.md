This plan aims to satisfy the **plan checklist** (verify before marking **done**):

- Avoid generic null-check / defensive noise unless the contract requires it (**null checks**, **defensive code**).
- **`@"` interpolation**, one-use temps, **`get_*`**, long method names — avoid per standards.
- Brace style, **`this.`**, **`GLib.`** qualification, line length — match project rules.
- No **`StringBuilder`** unless loop-heavy; **`string.joinv`** / **`+`** for simple joins; **`string[]`** not **`Gee.ArrayList<string>`** for string accumulation.
- **New methods**: only where this plan names them (default: edit **existing** methods). **`⚠️`** Refactoring **`execute()`** to assign **`last_request`** is **in-place** inside **`execute()`**, not a new public API unless explicitly added later.
- **Single canonical proposals**: mechanical **Remove** / **Replace with** / **Add** fences below — no duplicate full-file copies that could drift (**`docs/guide-to-writing-plans.md`**).
- Every **`#### Add`** documents **Where** + **What** (placement + purpose), not only a fence (**`docs/guide-to-writing-plans.md`** — **Don’t** “only a code fence”).

## Purpose

- Task progress rows show **`ProgressItem.title`** (**[`7.14.9`](done/7.14.9-DONE-task-tool-display-name.md)**). Tooltip text is **`ProgressItem.tooltip_text`** (**`string`**, **`""`** = none). **`BaseTool.execute()`** sets **`BaseTool.last_request`** to the **`Request`** it built — then **`Task.Tool.run()`**, **after** **`yield tool_impl.execute(...)`**, **assigns** **`this.tool_request = tool_impl.last_request`** (**same object reference**, not a deep copy) so **this progress row** holds the **`Request`** for **`to_summary()`** (**§ Tooltip implementation**). **`ProgressItem`** exposes **`tool_request`** (**`null`** on **`Details`** / **`ProgressRunner`**). **`tooltip_text`** on **`Task.Tool`** is **`tool_request.to_summary()`** when set. **`ProgressView`** binds **`tooltip_text`** → **`tooltip-text`**.
- **⚠️** **`RequestBase.to_summary()`** only (**§ API**). **`last_request`** is the live slot on **`BaseTool`** during / after **`execute`**; **`tool_request`** on the **`Task.Tool`** row is what the UI reads for tooltip text.

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| **`RequestBase.to_summary()`**, **`BaseTool.last_request`**, **`ProgressItem.tool_request`**, **`tooltip_text`**, **`ProgressView`** binds | **`to_summary_title()`**; changing **result** **`add_message`** bodies; **`tool_call_details()`** |
| **`BaseTool`** private generic fallback when **`deserialize`** fails (**optional**) | Public **`BaseTool`** JSON summarizer API (**`summarize_*`**) for **`ProgressItem`** |

## Acceptance criteria

- **`to_summary()`** text for **`codebase_search`** / **`read_file`** / **`web_fetch`** request rows **matches** previous **`request_message`** / **`url`** strings **byte-for-byte** where the same inputs and **`tool`** are available.
- Chat fences and progress tooltip both use **`Request.to_summary()`**. Tooltip reads **`Task.Tool.tool_request`** — assign **`this.tool_request = tool_impl.last_request`** after **`yield tool_impl.execute(...)`** (same **`Request`** reference **`execute`** built). **`to_summary`** has **no** parameters.
- **`ProgressView`**: **`bind_property ("tooltip_text", …)`** alongside title — **no** manual **`Binding.unbind()`** by default; **`Gtk.ListItem`** / row teardown typically drops bindings when the row is recycled (**ℹ️** GTK/GObject lifecycle). **⚠️** Only add explicit **`unbind`** if profiling or leaks prove stacked bindings — not the project norm.

## API — **`to_summary()`** on **`RequestBase`** only

**Rule:** **`to_summary()` takes no parameters.** Only **`RequestBase`** (after **`deserialize`** from the tool call’s **`arguments`**) implements **`to_summary()`**.

**Progress tree:** **`BaseTool.last_request`** in **`execute()`**; **`Task.Tool.tool_request`** copied from **`tool_impl.last_request`** after **`execute`** returns (**§ Tooltip implementation**) — **not** a second **`deserialize`**. **Do not** add **`BaseTool.summarize_from_arguments(Json.Object)`** for tooltip (**wrong layer**).

| Location | Signature | Role |
| -------- | --------- | ---- |
| **`RequestBase`** | **`public virtual string to_summary ()`** | Chat fences + **`Task.Tool.tool_request.to_summary()`** for tooltip. **`""`** = no typed lines (tooltip empty unless **⚠️** generic fallback elsewhere). |
| **`BaseTool`** | **`public RequestBase? last_request { get; set; }`** *(or **`private set`**)* | Set in **`execute()`** to the **`request`** built from **`tool_call.function.arguments`**. |
| **`Task.Tool`** / **`ProgressItem`** | **`public RequestBase? tool_request { get; set; }`** *( **`Task.Tool`** only; **`null`** elsewhere)* | **Reference copy** from **`tool_impl.last_request`** after **`yield execute()`** in **`run()`**. |

### Semantics ( **`RequestBase.to_summary()`** )

- **`Message.fenced` fence titles** stay **at the call site** unless you add **`to_summary_title()`** later — **out of scope**.
- **`RequestBase.to_summary()` must not** call **`this.agent.add_message`**, **`yield`**, touch buffers, or depend on **`execute_request`** having run (except fields derivable from **`tool`** + deserialized JSON).

### **`RequestBase`** — default virtual

**ℹ️** Verbatim **`#### Add`** (includes docblock) is **§ Core — `RequestBase.vala`** — **do not** paste a second variant here.

### **`BaseTool` / `execute()`** — **⚠️** implementation alignment

**`execute()`** **`deserialize`s** into a **`Request`**. Add **`this.last_request = request`** (after **`request.tool`** / **`request.agent`** / monitoring registration, **before** **`yield request.execute()`**). Any **generic JSON fallback** when **`Request`** cannot be built stays **private** on **`BaseTool`** — **not** a public JSON summarizer for **`ProgressItem`**.

## 6. `WriteFile.Request` — `to_summary()` (tooltip-first; optional chat later)

### Add — compact intent from JSON fields (**no** `validate()` / disk)

```vala
		public override string to_summary ()
		{
			if (this.file_path.strip () == "") {
				return "";
			}
			var norm = this.tool != null ? this.normalize_file_path (this.file_path) : this.file_path;
			string[] lines = {};
			lines += "File: " + norm;
			if (this.ast_path.strip () != "") {
				lines += "AST path: " + this.ast_path;
			} else if (this.start_line >= 1 && this.end_line >= this.start_line) {
				lines += "Lines: %d-%d".printf (this.start_line, this.end_line);
			} else if (this.complete_file) {
				lines += "Mode: complete file";
			} else if (this.search_text.strip () != "") {
				lines += "Mode: search/replace";
			}
			if (this.overwrite) {
				lines += "Overwrite: yes";
			}
			return string.joinv ("\n", lines);
		}
```
