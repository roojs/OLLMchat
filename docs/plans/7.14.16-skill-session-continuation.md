# 7.14.16 — Skill session continuation (follow-up prompt + revised task list)

**Status:** proposed

**Parent:** `docs/plans/done/7.14-DONE-task-progress-tree-ui.md` · `docs/plans/done/7.13-DONE-agent-save-restore.md`

**Pointer:** `.cursor/rules/CODING_STANDARDS.md` — **Checklist for all plans** · `docs/guide-to-writing-plans.md`

**Related:** `docs/plans/7.14.6-continue-replay-as-new.md` — fork / replay-as-new (**🚫** not this plan)

---

## Purpose

- 🔷 After **session restore**, the user sends a message in the **prompt box** to **continue** the same Skills session (not a fresh **`task_creation_initial`** run).
- 🔷 **Continuation** revises **outstanding** tasks from **completed history** + the new user message (same family as **`run_task_list_iteration`**, user-triggered).
- 🔷 **Lead sections:**
  - **## Original prompt** — locked after first creation
  - **## Follow-up prompts** — **`Gee.ArrayList<string>`** on **`Runner`**; append one tidied statement per continuation; earlier entries unchanged
  - **## Goals / summary** — existing section; **updated** on continuation (expanded plan intent)
- 🔷 **Original prompt:** prompt says **“Do not put the original prompt in your output.”**; parser **ignores** **`original-prompt`** in the model response.

---

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| **`send_async`** branch → **`run_task_list_continuation`** | **7.14.6** continue-from-here / **`replay_as_new`** |
| **`task_list_continuation.md`**, **`parse_task_list_continuation()`**, **`PhaseEnum`** + replay | **General information for all tasks** (removed in **1.23.18**) |
| **`Runner.original_prompt_md`**, **`Runner.follow_up_prompts`**, unified **`task_list.md`** write | Injecting follow-up into **executor** prompts (**💩** fast follow-up) |
| **`agent-stage`** `task_list_continuation` + **`on_replay`** case | Full **iteration** prompt refresh in same PR (**ℹ️** optional **`{follow_up_prompts}`**) |

---

## Current behaviour

- ℹ️ **`send_async`** always runs **`task_creation_initial.md`** — even on restored / completed sessions.
- ℹ️ **`parse_task_list()`** reads **`goals-summary`** into **`pending.goals_summary_md`** but does **not** persist **`original-prompt`** on **`Runner`**.
- ℹ️ **`run_task_list_iteration()`** — model outputs **`## Tasks` only**; **`goals_summary_md`** preserved (**`Runner.vala`** ~518).
- ℹ️ **`parse_task_list_iteration()`** — tasks only; no goals / follow-up (**`ResultParser.vala`** ~157).

---

## Proposed behaviour

- ⏳ **`send_async`:** if **`original_prompt_md != ""`** and **`session.can_replay`** and **`!in_replay`** → **`run_task_list_continuation`**; else creation path.
- ⏳ **First creation success:** set **`original_prompt_md`** from parsed **`original-prompt`**; **`follow_up_prompts`** empty.
- ⏳ **Continuation model output:** **`## Follow-up prompts`** (this turn only), **`## Goals / summary`** (revised), **`## Tasks`** — **not** **`## Original prompt`**.
- ⏳ **On success:** **`follow_up_prompts.add(tidied)`**; **`pending.goals_summary_md`** ← parsed goals; replace **`pending`** tasks; **`write_task_list_file()`** from Runner lead fields + lists.
- ⏳ **Replay:** wire id **`task_list_continuation`**; **`on_replay`** case mirrors **`TASK_LIST_ITERATION`** (no step migrate — user continuation when idle).

---

## Acceptance criteria

- ⏳ Restored session with prior work: new user message → **no** second task-creation pass; **`handle_task_list`** runs on revised **`pending`**.
- ⏳ After two continuations: **`follow_up_prompts.size == 2`** and index **0** text unchanged.
- ⏳ **`task_list.md`:** stable **## Original prompt**; numbered **### Follow-up N**; **## Goals / summary** updated.
- ⏳ Continuation template contains **“Do not put the original prompt in your output.”**
- ⏳ Model echo of **`## Original prompt`** does not change **`original_prompt_md`** or on-disk original.
- ⏳ **Completed** tasks not re-run; **`agent-stage`** **`task_list_continuation`** replays like live.

```markdown
You are the **continuation planner**. The user is continuing an existing Skills session. You do **not** execute anything.

## Do not put the original prompt in your output

**Do not put the original prompt in your output.** You receive the original prompt for context only. It is locked and must not appear in your response.

## What you receive

- **Original prompt** (context only — do not output)
- **Follow-up prompts so far** (prior continuations — do not re-output them)
- **Goals / summary** (current plan intent — you will output a **revised** version)
- **Completed tasks** and **outstanding tasks**
- **This follow-up message** (raw user text — tidy for your new follow-up entry)
- **Skill catalog**, environment, project description

{skill_catalog}

## Your job

1. Output **only this turn's** tidied text under **`## Follow-up prompts`** (typos/grammar only; do not change intent). Do **not** repeat prior follow-ups.
2. Output a **revised** **`## Goals / summary`** — one paragraph for the **whole** plan (original + all follow-ups + completed context).
3. Output **`## Tasks`** — revised **outstanding** work only; do **not** list completed tasks (same rules as task list iteration).
4. **Do not** output **`## Original prompt`**.

## Output format

Use exactly: `## Follow-up prompts`, `## Goals / summary`, `## Tasks`. Under **Tasks** use `### Task section 1`, … and task bullets per task creation / iteration.

{original_prompt}
{follow_up_prompts}
{goals_summary}
{completed_task_list}
{outstanding_task_list}
{user_follow_up}
{environment}
{project_description}
{previous_proposal_issues}
```

### 3. `liboccoder/Skill/Runner.vala` — `assemble_task_list_file` and `follow_up_prompts_markdown`

#### Add — after `fill_tools()` (~196)

**Where:** **`Runner`**, after **`fill_tools`**. **What:** Render follow-ups + write unified **`task_list.md`**.

#### Add

```vala
		/**
		 * Markdown for ## Follow-up prompts from {@link follow_up_prompts}.
		 */
		public string follow_up_prompts_markdown()
		{
			if (this.follow_up_prompts.size == 0) {
				return "";
			}
			var parts = new string[this.follow_up_prompts.size];
			for (var i = 0; i < this.follow_up_prompts.size; i++) {
				parts[i] = "### Follow-up " + (i + 1).to_string() + "\n\n"
					+ this.follow_up_prompts[i].strip() + "\n";
			}
			return "## Follow-up prompts\n\n" + string.joinv("\n", parts);
		}

		/**
		 * Write task_list.md from locked lead sections and current lists.
		 */
		public void write_task_list_file()
		{
			var body = "## Original prompt\n\n" + this.original_prompt_md.strip() + "\n\n";
			var follow_md = this.follow_up_prompts_markdown();
			if (follow_md != "") {
				body += follow_md + "\n\n";
			}
			body += "## Goals / summary\n\n" + this.pending.goals_summary_md.strip() + "\n\n";
			body += this.pending.to_markdown(OLLMcoder.Task.PhaseEnum.LIST);
			this.pending.write("task_list.md", body);
		}
```

### 5. `liboccoder/Skill/Runner.vala` — `continuation_prompt` and `run_task_list_continuation`

#### Add — after `iteration_prompt` (~448)

**Where:** After **`iteration_prompt`**. **What:** Fill **`task_list_continuation.md`**.

#### Add

```vala
		/**
		 * Build session continuation prompt (task_list_continuation.md).
		 */
		public PromptTemplate continuation_prompt(
			string user_follow_up,
			string previous_proposal_issues,
			string previous_proposed_md) throws GLib.Error
		{
			this.sr_factory.skill_manager.scan();
			var tpl = PromptTemplate.template("task_list_continuation.md");
			tpl.system_fill(1, "skill_catalog", this.sr_factory.skill_manager.to_markdown());
			var follow_md = this.follow_up_prompts_markdown();
			tpl.fill(9,
				"original_prompt", tpl.header_raw("Original prompt", this.original_prompt_md),
				"follow_up_prompts", follow_md == "" ? "" :
					tpl.header_raw("Follow-up prompts so far", follow_md),
				"goals_summary", tpl.header_raw("Goals / summary", this.pending.goals_summary_md),
				"completed_task_list",
					this.completed.to_markdown(OLLMcoder.Task.PhaseEnum.REFINE_COMPLETED),
				"outstanding_task_list",
					this.pending.to_markdown(OLLMcoder.Task.PhaseEnum.LIST),
				"user_follow_up", tpl.header_fenced("This follow-up message", user_follow_up, "text"),
				"environment", tpl.header_raw("Environment", this.env()),
				"project_description",
					this.sr_factory.project_manager.active_project == null ? "" :
					this.sr_factory.project_manager.active_project.project_description(),
				"previous_proposal_issues", previous_proposal_issues == "" ? "" :
					tpl.header_raw("Issues with the tasks", previous_proposal_issues),
				"previous_proposed_task_list", previous_proposed_md == "" ? "" :
					tpl.header_raw("Proposed (your last response — had issues)", previous_proposed_md));
			return tpl;
		}

		/**
		 * User continuation: revise goals, append follow-up, replace outstanding tasks.
		 */
		public async void run_task_list_continuation(
			OLLMchat.Message in_message,
			GLib.Cancellable? cancellable = null) throws GLib.Error
		{
			var existing_proposed = this.pending;
			var response = "";
			var parser = new OLLMcoder.Task.ResultParser(this, "");

			var cr = new OLLMcoder.Task.ProgressRunner(this) {
				in_creation = false,
				try_max = 5,
				try_no = 0,
				status = OLLMcoder.Task.PhaseEnum.TASK_LIST_CONTINUATION,
			};
			this.progress.add(cr);
			this.progress.active_item_changed(cr);
			for (; cr.try_no < cr.try_max; cr.try_no++) {
				this.progress.active_item_changed(cr);
				this.progress.clear_pending(false);
				var tpl = this.continuation_prompt(
					in_message.content, parser.issues, response);
				this.fill_tools();
				if (cr.try_no > 0) {
					this.add_message(new OLLMchat.Message("ui",
						"Trying again (attempt %d/5).".printf((int) (cr.try_no + 1))));
				}
				var model_label = this.session.model_usage.model != "" ?
					this.session.model_usage.display_name_with_size() : "Unknown model";
				this.add_message(new OLLMchat.Message("ui", OLLMchat.Message.fenced(
					"markdown.oc-frame-info.collapsed Continuing plan with " + model_label,
					tpl.filled_user)));
				this.session.add_message(new OLLMchat.Message("system", tpl.filled_system));
				this.session.add_message(new OLLMchat.Message("user", tpl.filled_user));
				this.add_message(new OLLMchat.Message("ui-waiting",
					"waiting for " + model_label + " to reply"));
				var messages = new Gee.ArrayList<OLLMchat.Message>();
				messages.add(new OLLMchat.Message("system", tpl.filled_system));
				messages.add(new OLLMchat.Message("user", tpl.filled_user));
				this.add_message(new OLLMchat.Message("agent-stage", "task_list_continuation"));
				var response_obj = yield this.chat_call.send(messages, cancellable);
				response = response_obj != null ? response_obj.message.content : "";

				this.pending = new OLLMcoder.Task.List(this);
				parser = new OLLMcoder.Task.ResultParser(this, response);
				parser.parse_task_list_continuation();
				this.add_message(new OLLMchat.Message("agent-issues", parser.issues));

				if (parser.issues != "") {
					this.pending = existing_proposed;
					cr.status = OLLMcoder.Task.PhaseEnum.TASK_LIST_CONTINUATION_RETRY;
					continue;
				}
				this.follow_up_prompts.add(parser.continuation_follow_up_md);
				this.pending.goals_summary_md = parser.continuation_goals_md;
				this.write_task_list_file();
				this.pending.write("task_list_latest.md", response);
				this.pending.write("task_list_completed.md",
					this.completed.to_markdown(OLLMcoder.Task.PhaseEnum.LIST));
				cr.status = OLLMcoder.Task.PhaseEnum.COMPLETED;
				this.progress.clear_pending(true);
				this.progress.add_pending(true);
				yield this.handle_task_list(cancellable);
				return;
			}
			cr.status = OLLMcoder.Task.PhaseEnum.ERROR;
			if (cancellable != null) {
				cancellable.cancel();
			}
			this.add_message(new OLLMchat.Message("ui", OLLMchat.Message.fenced(
				"text.oc-frame-danger.collapsed Continuation: could not get valid task list after 5 tries",
				parser.issues != "" ? parser.issues.strip() : "")));
		}
```

```vala
		/**
		 * Parses continuation output: follow-up (this turn), goals, tasks.
		 * Ignores ## Original prompt if present. On failure leaves runner.pending
		 * as built so far for retry capture.
		 */
		public void parse_task_list_continuation()
		{
			this.issues = "";
			this.continuation_follow_up_md = "";
			this.continuation_goals_md = "";

			foreach (var key in new string[] { "follow-up-prompts", "goals-summary", "tasks" }) {
				if (!this.document.headings.has_key(key)) {
					this.issues += "\n" + "Continuation response must contain ## Follow-up prompts,"
						+ " ## Goals / summary, and ## Tasks. Missing: \"" + key + "\".";
				}
			}
			if (this.issues != "") {
				return;
			}

			this.continuation_follow_up_md =
				this.document.headings.get("follow-up-prompts").to_markdown_with_content().strip();
			if (this.continuation_follow_up_md == "") {
				this.issues += "\n" + "Follow-up prompts: this turn's tidied text must not be empty.";
				return;
			}
			this.continuation_goals_md =
				this.document.headings.get("goals-summary").to_markdown_with_content().strip();
			if (this.continuation_goals_md == "") {
				this.issues += "\n" + "Goals / summary: revised paragraph must not be empty.";
				return;
			}

			foreach (var key in this.document.header_list) {
				if (!key.has_prefix("task-section-")) {
					continue;
				}
				var step = this.parse_step(this.document.headings.get(key));
				if (step == null) {
					continue;
				}
				this.runner.pending.steps.add(step);
			}
			this.runner.pending.fill_names();
			var steps = this.runner.pending.steps;
			for (var i = 0; i < steps.size; i++) {
				var step = steps.get(i);
				step.register_slugs(i);
				foreach (var t in step.children) {
					this.validate_task(t, PhaseEnum.LIST);
					var vl_list = new ValidateLink(t.runner, t, PhaseEnum.LIST);
					vl_list.validate_all(t.references);
					t.issues += vl_list.issues;
					if (t.issues != "") {
						this.issues += "\n" + t.issue_label() + " (References): " + t.issues;
					}
				}
			}
		}
```
