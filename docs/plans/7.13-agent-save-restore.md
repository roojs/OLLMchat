# 7.13 Agent save and restore — session replay (option B)

**Status:** Phases **1–4** implemented. **Next:** GTK hydration — **`## Concrete code proposals — Runner.on_replay`** (double-`switch` sketch). **`Runner.replay` / `ReplayChat`** = CLI; no **`Process.exit`** in app paths.

---

## Purpose

- On **open saved session**, restore whatever the **agent** needs to **Continue** — **without** calling the model during load. (Concrete task-list / `Details` graphs are a **follow-up hydration** step if/when you need them — not the same ordering as a live **Runner.send_async** run.)
- **Transcript:** `session.messages` is source of truth; markers **`agent-stage`** / **`agent-issues`** (Phase 1) gate capability (`can_replay`, Phase 2).
- **Optional `tool-output`** (Phase 1.5) only if we ever need persisted tool strings — not required for Continue.

---

## Stage messages (contract)

| Role | Body |
|------|------|
| `agent-stage` | `stage_id` (e.g. `task_list_parse`, `refinement`, `exec`, …) |
| `agent-issues` | Issues string after that stage; **empty = success** |

Emit **stage** before work, **issues** after parse/validation. Masked from LLM requests; persisted in JSON.

## Live pipeline walkthrough (formal)

One section per **logical step**. Classes and **properties** are what the code sets today; **restore** reuses the **same parsers** with a **stored response string** instead of **`chat_call.send`** for that step (see end).

### 1. User prompt → build request → LLM

| Piece | Class / API |
|-------|-------------|
| Entry | **`OLLMcoder.Skill.Runner.send_async`** (`liboccoder/Skill/Runner.vala`) |
| Build prompt | **`Runner.task_creation_prompt`** → **`OLLMcoder.Skill.PromptTemplate`** (`task_creation_initial.md`) — fills system/user from user text, env, project, skill catalog, previous proposal + issues on retry |
| Store user doc | **`Runner.user_request`** ← **`tpl.user_to_document()`** (markdown) |
| Send | **`OLLMchat.Call.Chat.send`** on **`runner.chat_call`** — **`Gee.ArrayList<Message>`** with `system` + `user` only for the API call |

There is **no** separate `replay_phase` flag in code: the **task-list creation** retry uses **`try_count`** (0…4) inside **`send_async`**.

### 2. LLM response → parse task list

| Piece | Class / API |
|-------|-------------|
| Parser | **`OLLMcoder.Task.ResultParser`** (`liboccoder/Task/ResultParser.vala`) — **`new ResultParser(runner, response_text)`** builds an internal **`Markdown.Document`** from the raw string |
| Method | **`parse_task_list()`** |
| Writes into | **`runner.pending`** (**`OLLMcoder.Task.List`**) — must be a **fresh** `List` before call (runner does `pending = new List(this)` right before) |

**`ResultParser`** (after successful parse):

| Property | Meaning |
|----------|---------|
| **`issues`** | `""` if OK; else validation errors (structure, tasks, links, …) |
| **`proposal`** | Exposed for retry prompt; overlapping with input string |

**`runner.pending`** (**`Task.List`**):

| Property | Set by parser |
|----------|----------------|
| **`steps`** | **`Step`** list from `##` task sections |
| **`goals_summary_md`** | From `goals-summary` section |
| **`slugs`** | Via **`Step.register_slugs`** → **`Details`** keyed by slug |

**Per task `OLLMcoder.Task.Details`:**

| Property | Set by parser |
|----------|----------------|
| **`task_data`** | Map of list-item blocks (what is needed, skill, references, …) |
| **`skill`** | Later via **`skill_manager.validate`** in **`validate_task`** path during parse |

### 3. Failure → retry (same “phase”: task list parse)

- **`parser.issues != ""`** → do **not** call **`handle_task_list`**.
- **`Runner`** sets **`previous_proposal = parser.proposal`**, **`previous_proposal_issues`** (with try label), rebuilds **`task_creation_prompt`** with those, **`chat_call.send`** again — loop **`try_count`** until 5 failures or success.
- **`in_replay`** (CLI **`Runner.replay`**) uses the **same** loop; **`ReplayChat`** returns the **next** stored assistant body instead of HTTP.

### 4. Success → run the task list (forward)

**`Runner.handle_task_list`** (private):

- **`pending.refine`** — per **`Details`**, builds refinement prompt, **`chat_call.send`**, **`new ResultParser(runner, response).extract_refinement(details)`** — fills **`Details.task_data`**, **`result_parser`**, refinement issues.
- **`pending.run_step_until_approval`** / **`run_step`** — moves work through steps; may call **`run_task_list_iteration`**.
- **Iteration** new list from LLM: **`parse_task_list_iteration()`** (tasks-only sections) into a **new** **`pending`** shape — **`goals_summary_md`** preserved by caller contract (see **`ResultParser`** doc).

**Objects:** still **`Task.List`**, **`Step`**, **`Details`**; state moves **`pending`** ↔ **`completed`** inside **`List`** helpers (see `liboccoder/Task/List.vala`).

### 5. Execution: **`OLLMcoder.Task.Tool.run`**

| Piece | Role |
|-------|------|
| Build executor prompt | **`Tool.executor_prompt`** (template + tool output + refs) |
| Send | **`chat_call.send`** (system + user) |
| Parse | **`new ResultParser(runner, response_text).exec_extract(this)`** — fills executor **`summary`**, **`document`**, validates structure |
| Parent **`Details`** | **`exec_runs`** ( **`Tool`** instances ), **`exec_done`** after successful **`run_exec`** path |

**`Tool`**: tool call args, **`writes`**, summaries — populated through **`exec_extract`** and validate passes.

### 6. What “restore without LLM” is, formally

Almost everything above that **mutates** graph state goes through **`ResultParser`** with a **string**. Live flow adds **before** that:

1. Template fill (**`PromptTemplate`** / **`executor_prompt`** / refinement template), and  
2. **`chat_call.send`**.

**Hydration / restore:** for each logical step you need to reconstruct, take the **saved assistant markdown** from **`session.messages`** — the visible persisted body is **`content-stream`** (see **`Session.handle_stream_chunk`**: streaming appends to a **`content-stream`** message; **`assistant`** is internal / not the chat transcript row). Pair with **`agent-stage` / `agent-issues`** order, then:

- **`new ResultParser(runner, that_string)`**  
- call the **same** method as live: **`parse_task_list`**, **`extract_refinement`**, **`exec_extract`**, etc.

Skip **(1)** and **(2)** for that step; **do not** skip the **parser** if you want the **same** **`Task.List` / `Details` / `Tool`** shapes.

**Phases 3–4** in the tree: **`in_replay`** avoids **duplicate `session.messages.add`** and skips **`tool_impl.execute`** — orthogonal to **parser** inputs.

### `liboccoder/Task/PhaseEnum.vala` — `PhaseEnum`

#### Add — new members + **`;`**, then methods on the enum (Vala)

```vala
public enum PhaseEnum
{
	COARSE,
	REFINEMENT,
	LIST,
	REFINE_COMPLETED,
	EXECUTION,
	POST_EXEC,
	NONE,
	TASK_LIST_ITERATION,
	EXEC_VALIDATE;

	public static PhaseEnum from_string(string id)
	{
		switch (id) {
		case "task_list_parse":
			return PhaseEnum.LIST;
		case "task_list_iteration":
			return PhaseEnum.TASK_LIST_ITERATION;
		case "refinement":
			return PhaseEnum.REFINEMENT;
		case "exec":
			return PhaseEnum.EXECUTION;
		case "post_exec":
			return PhaseEnum.POST_EXEC;
		case "exec_validate":
			return PhaseEnum.EXEC_VALIDATE;
		default:
			return PhaseEnum.NONE;
		}
	}

	public string to_string()
	{
		switch (this) {
		case NONE:
			return "";
		case LIST:
			return "task_list_parse";
		case TASK_LIST_ITERATION:
			return "task_list_iteration";
		case REFINEMENT:
			return "refinement";
		case EXECUTION:
			return "exec";
		case POST_EXEC:
			return "post_exec";
		case EXEC_VALIDATE:
			return "exec_validate";
		default:
			return "";
		}
	}
}
```

```vala
		/**
		 * Apply one stored transcript message during GTK session restore. Dispatches on
		 * replay_phase and message role; uses ResultParser on content-stream where live
		 * code would have used the LLM response string.
		 */
		public override void on_replay(OLLMchat.Message m)
		{
			if (!this.session.can_replay) {
				return;
			}

			switch (this.replay_phase) {
			case NONE:
				switch (m.role) {
				case "agent-stage":
					// Bootstrap: first wire stages only. Iteration step move is under EXEC_VALIDATE.
					this.replay_phase = PhaseEnum.from_string(m.content);
					break;
				default:
					break;
				}
				break;

			case LIST:
				switch (m.role) {
				case "content-stream":
					this.pending = new OLLMcoder.Task.List(this);
					var p0 = new OLLMcoder.Task.ResultParser(this, m.content);
					p0.parse_task_list();
					break;
				case "agent-stage":
					this.replay_phase = PhaseEnum.from_string(m.content);
					break;
				case "agent-issues":
					// this.pending.write("task_list.md", TBC);
					break;
				default:
					break;
				}
				break;

			case TASK_LIST_ITERATION:
				switch (m.role) {
				case "content-stream":
					this.pending = new OLLMcoder.Task.List(this);
					var p1 = new OLLMcoder.Task.ResultParser(this, m.content);
					p1.parse_task_list_iteration();
					this.replay_step_pos = 0;
					this.replay_details_pos = 0;
					this.replay_tool_pos = 0;
					break;
				case "agent-stage":
					this.replay_phase = PhaseEnum.from_string(m.content);
					break;
				case "agent-issues":
					// this.pending.write("task_list_latest.md", TBC);
					// this.pending.write("task_list_completed.md", TBC);
					break;
				default:
					break;
				}
				break;

			case REFINEMENT:
				switch (m.role) {
				case "content-stream":
					var pr = new OLLMcoder.Task.ResultParser(this, m.content);
					var st_r = this.pending.steps.get(this.replay_step_pos);
					pr.extract_refinement(st_r.children.get(this.replay_details_pos));
					break;
				case "agent-stage":
					this.replay_phase = PhaseEnum.from_string(m.content);
					break;
				case "agent-issues":
					if (m.content != "") {
						break;
					}
					var st_ri = this.pending.steps.get(this.replay_step_pos);
					if (this.replay_details_pos >= st_ri.children.size - 1) {
						break;
					}
					this.replay_details_pos++;
					break;
				default:
					break;
				}
				break;

			case POST_EXEC:
				switch (m.role) {
				case "content-stream":
					var pp = new OLLMcoder.Task.ResultParser(this, m.content);
					var st_p = this.pending.steps.get(this.replay_step_pos);
					pp.exec_post_extract(st_p.children.get(this.replay_details_pos));
					break;
				case "agent-stage":
					this.replay_phase = PhaseEnum.from_string(m.content);
					break;
				case "agent-issues":
					// Align with Details.run_post_exec when issues are empty (live).
					break;
				default:
					break;
				}
				break;

			case EXECUTION:
				switch (m.role) {
				case "content-stream":
					var st_e = this.pending.steps.get(this.replay_step_pos);
					var d_exec = st_e.children.get(this.replay_details_pos);
					var px = new OLLMcoder.Task.ResultParser(this, m.content);
					px.exec_extract(d_exec.exec_runs.get(this.replay_tool_pos));
					break;
				case "agent-stage":
					this.replay_phase = PhaseEnum.from_string(m.content);
					// from_string: exec → EXECUTION, exec_validate → EXEC_VALIDATE.
					break;
				case "agent-issues":
					// Outcome only; next replay_phase comes from the next agent-stage.
					break;
				default:
					break;
				}
				break;

			case EXEC_VALIDATE:
				switch (m.role) {
				case "agent-stage":
					var new_phase = PhaseEnum.from_string(m.content);
					if (new_phase != PhaseEnum.TASK_LIST_ITERATION || this.pending.steps.size == 0) {
						this.replay_phase = new_phase;
						break;
					}
					// Same pending → completed move as List.run_step* (List.vala ~218–223).
					var step = this.pending.steps.get(this.replay_step_pos);
					this.completed.steps.add(step);
					step.list = this.completed;
					foreach (var t in step.children) {
						this.completed.slugs.set(t.slug(), t);
					}
					this.pending.steps.remove_at(this.replay_step_pos);
					this.replay_step_pos = 0;
					this.replay_details_pos = 0;
					this.replay_tool_pos = 0;
					this.replay_phase = new_phase;
					break;
				case "agent-issues":
					if (m.content != "") {
						break;
					}
					var st_v = this.pending.steps.get(this.replay_step_pos);
					var det_v = st_v.children.get(this.replay_details_pos);
					if (this.replay_tool_pos < det_v.exec_runs.size - 1) {
						this.replay_tool_pos++;
						break;
					}
					this.replay_tool_pos = 0;
					if (this.replay_details_pos < st_v.children.size - 1) {
						this.replay_details_pos++;
					}
					// Else: finished last run of last task in step — transcript’s next agent-stage matches live.
					// Stay on EXEC_VALIDATE until the next agent-stage; do not set NONE here.
					break;
				default:
					break;
				}
				break;
			}
		}
```

### `libollmchat/History/Session.vala` — `restore_messages` (caller)

**`on_replay`** must run in **transcript order** for every persisted row that **`message_added`** replays to the UI (same message instance **`this.messages[i]`** / **`last_msg`**). Call **`ensure_agent_handler()`** once before the loop: on success it either leaves an existing **`this.agent`** or sets **`this.agent = factory.create_agent(this)`** — so **`this.agent`** is non-null for the rest of **`restore_messages`** unless **`ensure_agent_handler()`** throws (early **`return`** in **`catch`**). No separate **`if (this.agent != null)`** is needed after a successful call.

Sketch (merge into existing **`restore_messages`** in **`libollmchat/History/Session.vala`**; ellipsis = unchanged body):

```vala
internal async void restore_messages(GLib.Cancellable cancellable)
{
	if (this.messages.size == 0) {
		return;
	}

	try {
		this.ensure_agent_handler();
	} catch (GLib.Error e) {
		GLib.warning("restore_messages: %s", e.message);
		return;
	}

	for (int i = 0; i < this.messages.size - 1; i++) {
		if (cancellable.is_cancelled() || this.manager.session != this) {
			return;
		}
		if (this.messages[i].role == "ui-waiting") {
			continue;
		}
		this.manager.message_added(this.messages[i], this);
		this.agent.on_replay(this.messages[i]);
		GLib.Timeout.add(100, () => {
			this.restore_messages.callback();
			return false;
		});
		yield;
	}

	if (cancellable.is_cancelled() || this.manager.session != this) {
		return;
	}

	var last_msg = this.messages[this.messages.size - 1];
	if (last_msg.role == "ui-waiting") {
		return;
	}

	if (!this.is_running ||
			(last_msg.role != "content-stream" && last_msg.role != "think-stream")) {
		this.manager.message_added(last_msg, this);
		this.agent.on_replay(last_msg);
		return;
	}

	// Existing interrupted-stream tail: dummy_response + handle_stream_chunk — unchanged.
	// If that path must hydrate **`on_replay`** for the partial stream row, add explicitly
	// when the stream is finalized or in one place consistent with **`can_replay`** (TBC).
	// ...
}
```

## Restore flow (order matters — live Runner pipeline does not)

This is **only** the **GTK load / open session** path. It is **not** the same sequence as **`send_async` → `handle_task_list` → refine → exec**.

1. **`SessionPlaceholder.load()`** (or equivalent) → **`Session`** with **`messages`**, **`can_replay`**, **`project_path`**, etc., from JSON + DB.
2. **`Manager.switch_to_session`** → **`restore_messages`** on **`Session`**.
3. Start of **`restore_messages`**: **`ensure_agent_handler()`** (see caller sketch).
4. **Same loop** as today: for each stored row (except **`ui-waiting`** handling as now), **`message_added`** to UI, then **`this.agent.on_replay`** with the **same** **`Message`** (see caller sketch).
5. **Done** for the wire-up. No **`Runner.replay`**, no **`ReplayChat`**.

**Markers (`agent-stage` / `agent-issues`):** carried in **`messages`**; **`on_replay`** reads them **in transcript order** — that **is** the restore-time sequence that matters.

**Task graph for Continue:** **`on_replay`** above — **`ResultParser`** on **`content-stream`** (**`m.content`**); **`agent-issues`** markers only where noted. With **`replay_as_new == false`**, rely on **existing** **`task_dir()`** files from when the session was saved; do not rewrite **`task_list*.md`** during GTK load.

---

## Continue + `replay_as_new` (later — not GTK open by default)

**Default:** **`Runner.replay_as_new = false`**. Opening a saved session only **hydrates** **`pending` / `Details`** (etc.) from the transcript; **does not** regenerate **`task_list.md`**, **`task_list_latest.md`**, **`task_list_completed.md`**.

**Continue** (when you want a **fresh** task-dir mirror from replay): create a **new session**, attach the agent / **`Runner`**, set **`replay_as_new = true`**, then **replay the same message sequence** into that session (same **`on_replay`** loop as restore). That pass may **`write`** the task-list files like live. Exact wiring (who allocates the session, copies **`messages`**, triggers the replay) is **TBD** with the Continue UI flow.

## Acceptance criteria (draft)

1. Markers persisted in live runs; **`can_replay`** when appropriate.
2. Without markers: transcript-only; no task restore.
3. **GTK:** **`on_replay`** hydrates **or** no-ops; **no** extra full-message loop; **no** `Process.exit` from restore.
4. **CLI:** `Runner.replay` / `ReplayChat` behaviour acceptable until drift/`exit` cleanup.
5. Continue / inject prompt works once state is restored.
