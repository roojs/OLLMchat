# 7.14.1.3 — `Details.vala` — `proposed_tools`, `children`, `build_run_queue`, **`ProgressItem`**

**Status:** proposed

**Parent:** `docs/plans/7.14.1-progresslist-runner.md`

## Purpose

- **`proposed_tools`** (**`ToolList`**) — refined proposals; **`children`** (**`ToolList`**) — execution queue (**`build_run_queue`** → **`run_exec`**).
- Rename **`build_exec_runs`** → **`build_run_queue`**; use **`this.children`** inside (was **`exec_runs`**).
- Port **`ResultParser`**, **`to_markdown`**, **`run_exec`**, **`foreach`** sites to **`ToolList`** API.
- **`Details` implements `ProgressItem`** — **`status`**, **`title`**, **`status_str`**, **`children`** property returns **execution** **`ToolList`** (same field name as **`ProgressItem.children`**).

## Status updates (normative — coordinate with **`7.14.1.2`** / **`7.14.1.1`**)

| When | Set **`Details.status`** |
|------|---------------------------|
| Not refining / not executing yet | **`NONE`** |
| **`refine()`** in progress | **`REFINEMENT`** |
| **`run_exec`** active | **`TOOLS_RUNNING`** |
| Task row done | **`COMPLETED`** |

Wire in **`refine()`**, **`run_exec()`**, and completion paths — grep **`status_value`** / **`this.status`**.

## Edits

### 4a — class declaration

Find:

```vala
public class Details : OLLMchat.Agent.Base
```

Replace with:

```vala
public class Details : OLLMchat.Agent.Base, ProgressItem
```

### 4b — fields — **`tools`** / **`exec_runs`** → **`proposed_tools`** / **`children`**

**Find** (full replace):

```vala
	/**
	 * Tool instances from `## Tool Calls` section (ResultParser);
	 * build_exec_runs() uses them for scenario 1.
	 */
	public Gee.ArrayList<Tool> tools { get; set; default = new Gee.ArrayList<Tool>(); }

	/**
	 * Ordered executor passes for run_exec() (tool + LLM per pass); not always the same as ## Tool Calls.
	 * After run, summaries here drive REFINE_COMPLETED / iteration.
	 */
	public Gee.ArrayList<Tool> exec_runs { get; set; default = new Gee.ArrayList<Tool>(); }
```

**Replace with:**

```vala
	/**
	 * Refined tool-call **proposals** ({@link ResultParser} / {@link to_markdown}). **Not** nested progress rows — see {@link children}.
	 */
	public ToolList proposed_tools { get; set; default = new ToolList(); }

	/**
	 * **Execution queue** — {@link build_run_queue} / {@link run_exec} (**activity**). May alias {@link proposed_tools} or use exam-only / synthetic ''exec'' rows. Same field as {@link ProgressItem.children}.
	 * @see ToolList
	 */
	public ToolList children { get; set; default = new ToolList(); }
```

Then rename **`build_exec_runs`** → **`build_run_queue`** everywhere; **`this.tools`** → **`this.proposed_tools`**; **`this.exec_runs`** → **`this.children`**; **`Runner`** replay **`d_exec.exec_runs`** → **`d_exec.children`**.

### 4c — `ProgressItem` accessors

```vala
	private PhaseEnum status_value = PhaseEnum.NONE;

	public PhaseEnum status {
		get { return this.status_value; }
		set {
			if (this.status_value == value) {
				return;
			}
			this.status_value = value;
			this.notify_property("status_str");
		}
	}

	public string title {
		owned get { return this.slug(); }
	}

	public string status_str {
		owned get { return this.status.to_human(); }
	}
```

**Note:** **`ProgressItem.children`** on **`Details`** is the **`ToolList`** field **`children`** — **`ToolList`** implements **`GLib.ListModel`**; **`public ToolList children`** satisfies the interface (no alias getter).
