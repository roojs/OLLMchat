# 7.14.3 — Data + `PhaseEnum` + `to_human()` + bindable `ProgressItem` (phase 3)

**Status:** proposed

**Parent:** `docs/plans/7.14-task-progress-tree-ui.md`

**Checklist:** `.cursor/rules/CODING_STANDARDS.md` — **Checklist for all plans**.

---

## Purpose

- **`ProgressItem`** surface: **`kind`** (**`PhaseEnum`** — shared with markdown / wire / validation; **7.14.1** stubs **`REFINEMENT`** on **`Details`**, **`EXECUTION`** on **`Tool`**), **`title`**, **`stage`**, **`is_active`** — getters/setters, **`notify`**, bindable from **`Gtk.ColumnView`** factories.
- **`PhaseEnum`**: store **category** where possible; **extend** enum if a **kind** value is missing (**no** separate **`TaskKind`** — **7.14.1**). Add **`PhaseEnum.to_human()`** (or **`kind.to_human()`** on **`ProgressItem`**) so **`ProgressItem.stage`** returns that string — **stage column tracks **`kind`**** and **`notify["stage"]`** / bindings refresh **ColumnView** when **`kind`** updates (**7.14.1**).
- **Status** (fine-grained, optional): wire on **`Runner`** / rows where **7.14** tables need more than **`kind`** (e.g. retrying vs idle); use for **tooltips** or combined display — **not** a second competing source for the base **`stage`** string unless **`kind`** alone is insufficient. Align with parent doc **Stage column — UI contract** (Creating, Retrying, Refine, Executing, Review output, Post review, empty, grey tick for tool done).
- **Bold vs not bold**, **blank**, **tick**: driven by **`is_active`** + **`kind`** / **`stage`** + optional **status** where the renderer branches.

## Scope

| In scope | Out of scope |
|----------|----------------|
| Enum + human strings + **`ProgressItem`** completion | Chat navigation (**`7.14.4`**) |
| Optional “Post review” merge | Restore (**`7.14.5`**) |

---

## Acceptance criteria

1. Stage column matches **7.14** tables: task create (**Creating** / **Retrying** / empty); **`Details`** (**Refine**, **Retrying**, **Post review** optional, …); **`Tool`** (**Executing**, **Review output**, **Retrying**, grey **✓**, blank).
2. **Active** states use **bold**, **≤ 2 words**; completed/idle follow parent rules.
3. **`PhaseEnum.to_human()`** / **`kind.to_human()`** is the single source for **`ProgressItem.stage`** (column text); optional **status** helpers only where **7.14** requires finer strings than **`kind`** alone.
