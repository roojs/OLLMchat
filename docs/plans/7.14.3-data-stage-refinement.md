# 7.14.3 — Data + `PhaseEnum` + `to_human()` + bindable `ProgressItem` (phase 3)

**Status:** proposed

**Parent:** `docs/plans/7.14-task-progress-tree-ui.md`

**Checklist:** `.cursor/rules/CODING_STANDARDS.md` — **Checklist for all plans**.

---

## Purpose

- **`ProgressItem`** properties match **renderer needs**: getters/setters, **`notify`**, bindable from **`Gtk.ColumnView`** factories.
- **`PhaseEnum`**: store logical phase where possible; **extend** enum if creation/iteration/tool states are missing.
- Add **`to_human()`** (or extend existing helper) for **short** labels — used for stage column + tooltips; align with parent doc **Stage column — UI contract** (Creating, Retrying, Refine, Executing, Review output, Post review, empty, grey tick for tool done).
- **Bold vs not bold**, **blank**, **tick**: driven by **active flag** + enum + row kind in the factory/renderer.

---

## Scope

| In scope | Out of scope |
|----------|----------------|
| Enum + human strings + **`ProgressItem`** completion | Chat navigation (**`7.14.4`**) |
| Optional “Post review” merge | Restore (**`7.14.5`**) |

---

## Acceptance criteria

1. Stage column matches **7.14** tables: task create (**Creating** / **Retrying** / empty); **`Details`** (**Refine**, **Retrying**, **Post review** optional, …); **`Tool`** (**Executing**, **Review output**, **Retrying**, grey **✓**, blank).
2. **Active** states use **bold**, **≤ 2 words**; completed/idle follow parent rules.
3. **`to_human()`** (or equivalent) is the single source for displayed phase text where applicable.
