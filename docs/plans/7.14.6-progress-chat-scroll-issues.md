## Purpose

- рџ”· Stop storing duplicate `msg_idx` on progress rows.
- рџ”· Make `message.idx` the only source of row Idx truth.
- рџ”· Let each row react to `message.notify["idx"]` only when needed.
- рџ”· Keep Idx column updates automatic and centralized.

## Scope

| In scope | Out of scope |
| --- | --- |
| рџ”· `ProgressItem` owns `message` + `idx_notify_id` contract | рџљ« Any parse/overlay/walk to infer row ids |
| рџ”· Replace row-local `msg_idx` with `message`; keep **`msg_idx_txt`** as the computed label (`a ? b : c`) | рџљ« Scroll viewport behavior changes |
| рџ”· Use object signal API (`this.message.disconnect(id)`) for watcher lifecycle | рџљ« Progress tree hit-test / expander refactors |
| рџ”· Replace manual row id assignment with `assign_message(m)` calls | рџљ« Extra manager/helper classes |

## Acceptance criteria

- рџ”· `ProgressItem` has abstract `message`, `idx_notify_id`, `msg_idx_txt`, and a **non-abstract** default **`assign_message(...)`** on the interface.
- рџ”· `ProgressItem` no longer has stored `msg_idx` (only `msg_idx_txt`, computed from `message.idx`).
- рџ”· `assign_message(...)` always notifies `msg_idx_txt` immediately.
- рџ”· If `message.idx < 0`, row subscribes to `message.notify["idx"]`.
- рџ”· If `message.idx >= 0`, row does not subscribe.
- рџ”· Re-assignment disconnects previous watcher with `this.message.disconnect(this.idx_notify_id)`.
- рџ”· All direct `*.msg_idx = ...` sites are removed from this flow.

## Proposed behavior

- рџ”· Row carries the message reference, not a copied idx number.
- рџ”· Idx column text stays **`msg_idx_txt`**, same ternary style as today: `(message != null && message.idx >= 0) ? message.idx.to_string() : "вЂ”"`.
- рџ”· Watcher is short-lived and conditional (unset idx only).
- рџљ« Do not add parsing, scanning, or lookup overlays.

### 1. `liboccoder/Task/ProgressItem.vala` вЂ” interface contract вЏі

#### Remove

```vala
	/** Last **`Message.idx`** for this row (**continue-from** / **`scroll_to_idx`**); **-1** if unset. */
	public abstract int msg_idx { get; set; }
```

#### Add вЂ” message-first contract with watcher id (**keep `msg_idx_txt` name**).

```vala
	public abstract OLLMchat.Message? message { get; set; }

	public abstract ulong idx_notify_id { get; set; }
```

#### Add вЂ” **`assign_message` on the interface as a default implementation** (not `abstract`), so implementors only declare **`message`**, **`idx_notify_id`**, and **`msg_idx_txt`**.

```vala
	public void assign_message(OLLMchat.Message m)
	{
		if (this.message != null && this.idx_notify_id != 0) {
			this.message.disconnect(this.idx_notify_id);
			this.idx_notify_id = 0;
		}
		this.message = m;
		this.notify_property("msg_idx_txt");
		if (m.idx >= 0) {
			return;
		}
		this.idx_notify_id = m.notify["idx"].connect(() => {
			this.notify_property("msg_idx_txt");
			if (m.idx >= 0 && this.idx_notify_id != 0) {
				m.disconnect(this.idx_notify_id);
				this.idx_notify_id = 0;
			}
		});
	}
```

#### Keep (unchanged signature вЂ” implementation derives idx text from `message.idx`)

```vala
	/** Label text for the Idx column (**digits** or **вЂ”**). */
	public abstract string msg_idx_txt { owned get; }
```
