## Purpose

- **🔷** Extend the **activity / progress notification** UI so a long-running job can expose **one primary action button** (Cancel or Pause) on the banner.
- **🔷** Add a **routing mechanism**: button click → invoke the right cancel/pause for that job (browser download, Hub download, vector scan, …).
- **🔷** Consumers (same banner):
  - **Browser downloads** ([`5.0.5`](5.0.5-webkit-download.md)) — **Cancel** → WebKit `Download.cancel()`
  - **Hugging Face downloads** — **Cancel** when/if Hub download cancel is wired (theoretically; same chrome)
  - **Vector / filesystem scanning** — **Pause** (not cancel): call existing `Codebase.stop` / `stop_requested` path; resume stays `Codebase.start` elsewhere if needed
- **⏳** Ship banner action + routing **before** or **with** browser download progress wiring — download should not invent a one-off cancel UI.

---

## Current behaviour

- **ℹ️** `ActivityBanner` shows label + `Gtk.ProgressBar` from `notification` / `OLLMrpc.Notification.method`.
- **ℹ️** No button; user cannot cancel a download or pause a scan from the banner.
- **ℹ️** Vector scan can already be paused via RPC `Codebase.stop`; the UI never surfaces that.

---

## Proposed behaviour

- **🔷** Banner may show an optional action control (e.g. **Cancel** or **Pause**) when the active job advertises one.
- **🔷** Click routes to a registered handler for that job id / method family (not a giant `switch` that hardcodes every future tool forever — prefer a small registry or callback carried with the start event).
- **🔷** While no action is advertised, banner looks as today (label + progress only).
- **🔷** On cancel/pause success, progress updates continue via existing end/update events (`failed` / `scan_end` / etc.).

---

## Action kinds

- **🔷** **Cancel** — abort the job (browser `Download.cancel()`, Hub download cancel when available).
- **🔷** **Pause** — stop after current unit; keep queue (vector `stop_requested` / `Codebase.stop`). Label the button **Pause**, not Cancel.
- **💩** Resume button on the banner — optional later; scan resume may stay elsewhere for v1.

---

## Routing (design lock needed at implement)

- **🔷** Start/progress notifications must carry enough identity to route the click (e.g. method family + opaque job id, or a session-local handle).
- **💩** Shape options (pick one when coding):
  - Extend `OLLMrpc.Notification` with optional `action` / `action_label` / `job_id` fields
  - Or parallel `client.activity.action` metadata only on the UI side when emitting from the app
- **🔷** App owns the registry: “job X → callback / RPC call”. Daemon-originated scans route to `Codebase.stop`; in-process WebKit downloads route to the live `Download` object.
- **🚫** Do not put WebKit types inside `ActivityBanner` — banner only emits “action clicked for job id”; owners handle it.

---

## Consumers (wire-up backlog)

1. **⏳** **🔷** **ActivityBanner** — optional action button + emit/route click.
2. **⏳** **🔷** **Vector scan** — show **Pause** while `event.vector.scan_*` active → `Codebase.stop`.
3. **⏳** **🔷** **Filesystem scan** — **Pause** if a stop exists; else omit button until API exists.
4. **⏳** **🔷** **Browser download (`5.0.5`)** — show **Cancel** while WebKit download in progress → `download.cancel()`.
5. **⏳** **💩** **Hub / HF download** — **Cancel** when download cancellable is available; same banner contract.

---

## Relation to **5.0.5**

- **🔷** Browser download progress still uses Hub-style events into `ActivityBanner`.
- **🔷** Cancel for those downloads is **this** plan’s button + routing — not a separate download-only widget.
- **ℹ️** Basic **5.0.5** can land permission + save path first; cancel button should land via **5.0.6** as soon as progress is shown.

---

## Suggested order

1. **⏳** **🔷** Banner UI: optional action button + click signal / job id.
2. **⏳** **🔷** Routing registry in the app (or thin owner callbacks).
3. **⏳** **🔷** Vector scan **Pause** (existing `Codebase.stop`) — proves the mechanism.
4. **⏳** **🔷** Browser download **Cancel** when **5.0.5** progress is wired.
5. **⏳** **💩** HF download **Cancel** when ready.

**🚫** No code fences yet — urgent design lock; add Remove / Replace / Add at implement.
