# 2.11.2. MCP Loader

## Overview

Phase 2 of the MCP loader: abstract Client, ToolFactory, ToolWrapper, and Loader. Registry is the client factory (creates Client from Config; no ClientFactory/Stdio.Factory/Http.Factory). Loader takes configs and gets Client per config from Registry, fetches tool factories from client, registers ToolWrappers. No concrete transport implementations yet — those come in 2.11.3.

## Scope

- **Abstract base:** `OLLMmcp.Client` – abstract class or interface with: `connect()` / `disconnect()`, `list_tools()` (return list of **tool factories** — see below), `call_tool(name, arguments)` (return result string or throw). Optionally `initialize()` for MCP handshake if needed before list_tools.
- **Registry is the client factory:** No ClientFactory. Registry creates Client from Config (switch on transport).
- **Tool factory = our “tool description”:** In our design the MCP tool description (name, description, inputSchema) is represented as a **factory** that creates the BaseTool. So `list_tools()` returns something like a list of `OLLMmcp.ToolFactory` (or “tool wrapper factory”): each one holds the MCP tool metadata and has e.g. `create_wrapper(Client client, string server_id) → Tool.BaseTool`. So the descriptor is the factory; we don’t have a separate data-only descriptor and then manual wrapper construction.
- **ToolWrapper:** `OLLMmcp.ToolWrapper` extends `OLLMchat.Tool.BaseTool`. Created by that factory; holds client, server_id, MCP tool name, description, params. `execute(Json.Object parameters)` calls `client.call_tool(tool_name, parameters)`. Tool name exposed to the model: `mcp:{server_id}:{tool_name}`.
- **Loader:** Takes array of Config. For each enabled Config: get Client from Registry (Registry creates it), connect, list_tools, for each tool factory create_wrapper and register_tool. Store clients for later disconnect.
- **Registry:** Loads mcp.json → array of Config. Creates Client (switch on config.transport). fill_tools() uses Loader; Loader gets Client from Registry per config.

## Files

Under **libocmcp/** (still not in meson):

- `libocmcp/Client.vala` – abstract `OLLMmcp.Client`: connect, disconnect, list_tools, call_tool (and optional initialize).
- `libocmcp/ToolFactory.vala` – `OLLMmcp.ToolFactory`: holds MCP tool name, description, input_schema; `create_wrapper(client, server_id)` → ToolWrapper (our “tool description” is the factory).
- `libocmcp/ToolWrapper.vala` – `OLLMmcp.ToolWrapper` extends BaseTool; takes client + server_id + ToolFactory; execute() → client.call_tool().
- `libocmcp/Loader.vala` – takes array of Config; for each enabled Config gets Client from Registry, list_tools, register ToolWrappers.
- `libocmcp/Registry.vala` – loads mcp.json; creates Client (switch on transport); fill_tools() uses Loader. No client factories.

## Concrete code mock-up (in plan only, not in codebase)

### Client (abstract)

```vala
namespace OLLMmcp {
  public abstract class Client : Object {
    public abstract async void connect() throws Error;
    public abstract void disconnect();
    // list_tools returns tool factories (our "tool description" = factory that creates the wrapper)
    public abstract async Gee.ArrayList<ToolFactory> list_tools() throws Error;
    public abstract async string call_tool(string name, Json.Object arguments) throws Error;
  }
  // In our design the "tool description" from MCP is a factory: it creates the ToolWrapper.
  public class ToolFactory : Object {
    public string name { get; set; }
    public string description { get; set; }
    public Json.Node? input_schema { get; set; }
    public OLLMchat.Tool.BaseTool create_wrapper(Client client, string server_id) {
      return new ToolWrapper(client, server_id, this);
    }
  }
}
```

### Registry creates Client (no ClientFactory)

```vala
// In Registry.fill_tools() or a helper:
Client create_client(Config config) {
  switch (config.transport) {
    case "stdio": return new Stdio.Client(config);
    case "http":  return new Http.Client(config);
    default: return null;
  }
}
```

### ToolWrapper

```vala
namespace OLLMmcp {
  public class ToolWrapper : OLLMchat.Tool.BaseTool {
    private Client client;
    private string server_id;
    private ToolFactory tool_factory;  // descriptor = factory; holds name, description, input_schema
    public ToolWrapper(Client client, string server_id, ToolFactory tool_factory) { ... }
    public override string name { get { return "mcp:%s:%s".printf(server_id, tool_factory.name); } }
    public override async string? execute(Json.Object parameters) {
      return yield client.call_tool(tool_factory.name, parameters);
    }
  }
}
```

### Loader

```vala
namespace OLLMmcp {
  public class Loader : Object {
    private Gee.ArrayList<Client> clients;  // keep for disconnect later
    public async void run(OLLMchat.History.Manager manager, Gee.ArrayList<Config> configs, ClientCreator create_client) {
      foreach (var config in configs) {
        if (!config.enabled) continue;
        var client = create_client(config);  // Registry creates Client (switch on transport)
        if (client == null) continue;
        yield client.connect();
        clients.add(client);
        var tool_factories = yield client.list_tools();
        foreach (var tf in tool_factories) {
          var wrapper = tf.create_wrapper(client, config.id);
          manager.register_tool(wrapper);
        }
      }
    }
  }
}
```

### Registry (fill_tools change)

```vala
// In Registry.fill_tools():
var configs = Config.load();
var loader = new Loader();
loader.run.begin(manager, configs, (config) => this.create_client(config), (obj, res) => { loader.run.end(res); });
// create_client(config): switch on config.transport → new Stdio.Client(config) or new Http.Client(config)
```
