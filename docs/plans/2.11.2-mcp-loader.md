# 2.11.2. MCP Loader

## Overview

Phase 2 of the MCP loader: abstract Client, **Factory** (libocmcp/Factory/), **Tool** (libocmcp/Tool/), and Loader. Registry is the client factory (creates Client from Config; no ClientFactory/Stdio.Factory/Http.Factory). Loader takes configs and gets Client per config from Registry, fetches factories from client, registers Tools. Standard design: **Mcp/Tool** and **Mcp/Factory**. No concrete transport implementations yet — those come in 2.11.3.

Parent: [2.11-mcp-loader-tool.md](2.11-mcp-loader-tool.md)

## Scope

- **Client directory/namespace:** `libocmcp/Client/` — abstract base and transport implementations live under the same namespace. **Client/Base** = abstract class; **Client/Stdio** and **Client/Http** (2.11.3) extend it. So: `OLLMmcp.Client.Base`, `OLLMmcp.Client.Stdio`, `OLLMmcp.Client.Http`.
- **Abstract base:** `OLLMmcp.Client.Base` – abstract class with: `connect()` / `disconnect()`, `list_tools()` (return list of **Factory**), `call_tool(name, arguments)` (return result string or throw). Optionally `initialize()` for MCP handshake if needed before list_tools.
- **Registry is the client factory:** No ClientFactory. Registry creates Client from Config (switch on transport).
- **Tool factory = our “tool description”:** In our design the MCP tool description (name, description, inputSchema) is represented as a **factory** that creates the Tool. So `list_tools()` returns something like a list of `OLLMmcp.Factory` (or “tool wrapper factory”): each one holds the MCP tool metadata and has e.g. `create_tool(Client client, string server_id) → Tool (BaseTool)`. So the descriptor is the factory; we don’t have a separate data-only descriptor and then manual tool construction.
- **Tool** (libocmcp/Tool/): `OLLMmcp.Tool` extends `OLLMchat.Tool.BaseTool`. Created by that factory; holds client, server_id, MCP tool name, description, params. `execute(Json.Object parameters)` calls `client.call_tool(tool_name, parameters)`. Tool name exposed to the model: `mcp:{server_id}:{tool_name}`. Standard **Mcp/Tool** layout.
- **Loader:** Takes array of Config. For each enabled Config: get Client from Registry (Registry creates it), connect, list_tools, for each Factory create_tool and register_tool. Store clients for later disconnect.
- **Registry:** Loads mcp.json → array of Config. Creates Client (switch on config.transport → `Client.Stdio` or `Client.Http`). fill_tools() uses Loader; Loader gets Client from Registry per config.

## Files

Under **libocmcp/** (still not in meson):

- `libocmcp/Client/Base.vala` – abstract `OLLMmcp.Client.Base`: connect, disconnect, list_tools, call_tool (and optional initialize). Concrete transports in 2.11.3: `Client/Stdio.vala`, `Client/Http.vala`.
- `libocmcp/Factory/` – `OLLMmcp.Factory`: holds MCP tool name, description, input_schema; `create_tool(client, server_id)` → Tool. Standard **Mcp/Factory**.
- `libocmcp/Tool/` – `OLLMmcp.Tool` extends BaseTool; takes client + server_id + Factory; execute() → client.call_tool().
- `libocmcp/Loader.vala` – takes array of Config; for each enabled Config gets Client from Registry, list_tools, register Tools.
- `libocmcp/Registry.vala` – loads mcp.json; creates Client (switch on transport → `new Client.Stdio(config)` or `new Client.Http(config)`); fill_tools() uses Loader. No client factories.

## Concrete code mock-up (in plan only, not in codebase)

### Client.Base (abstract) — libocmcp/Client/Base.vala

```vala
namespace OLLMmcp.Client {
  public abstract class Base : Object {
    public abstract async void connect() throws Error;
    public abstract void disconnect();
    // list_tools returns Factory list (Mcp/Factory standard)
    public abstract async Gee.ArrayList<OLLMmcp.Factory> list_tools() throws Error;
    public abstract async string call_tool(string name, Json.Object arguments) throws Error;
  }
}
// Factory (libocmcp/Factory/): creates the Tool from MCP tool metadata.
namespace OLLMmcp {
  public class Factory : Object {
    public string name { get; set; }
    public string description { get; set; }
    public Json.Node? input_schema { get; set; }
    public OLLMchat.Tool.BaseTool create_tool(OLLMmcp.Client.Base client, string server_id) {
      return new Tool(client, server_id, this);
    }
  }
}
```

### Registry creates Client (no ClientFactory)

```vala
// In Registry.fill_tools() or a helper. Client namespace: Base (abstract), Stdio, Http (2.11.3).
OLLMmcp.Client.Base create_client(Config config) {
  switch (config.transport) {
    case "stdio": return new OLLMmcp.Client.Stdio(config);
    case "http":  return new OLLMmcp.Client.Http(config);
    default: return null;
  }
}
```

### Tool

```vala
namespace OLLMmcp {
  public class Tool : OLLMchat.Tool.BaseTool {
    private OLLMmcp.Client.Base client;
    private string server_id;
    private Factory factory;  // MCP tool metadata (name, description, input_schema)
    public Tool(OLLMmcp.Client.Base client, string server_id, Factory factory) { ... }
    public override string name { get { return "mcp:%s:%s".printf(server_id, factory.name); } }
    public override async string? execute(Json.Object parameters) {
      return yield client.call_tool(factory.name, parameters);
    }
  }
}
```

### Loader

```vala
namespace OLLMmcp {
  public class Loader : Object {
    private Gee.ArrayList<OLLMmcp.Client.Base> clients;  // keep for disconnect later
    public async void run(OLLMchat.History.Manager manager, Gee.ArrayList<Config> configs, ClientCreator create_client) {
      foreach (var config in configs) {
        if (!config.enabled) continue;
        var client = create_client(config);  // Registry creates Client (switch on transport)
        if (client == null) continue;
        yield client.connect();
        clients.add(client);
        var factories = yield client.list_tools();
        foreach (var f in factories) {
          var tool = f.create_tool(client, config.id);
          manager.register_tool(tool);
        }
      }
    }
  }
}
```

### Registry (fill_tools change)

```vala
// In Registry.fill_tools():
var configs = Config.load();
var loader = new Loader();
loader.run.begin(manager, configs, (config) => this.create_client(config), (obj, res) => { loader.run.end(res); });
// create_client(config): switch on config.transport → new Client.Stdio(config) or new Client.Http(config)
```
