## Coding standards checklist

Copied from **`.cursor/rules/CODING_STANDARDS.md`** — “Checklist for all plans”. If a bullet conflicts with **§ Conventions** in the [parent plan](1.23.40-refine-stage-reference-injection.md), **the standards file wins**.

- **Nullable types**: Design new APIs/properties to avoid nullable types where possible (default objects / flags).
- **Null checks**: Avoid generic null checks; only where the design explicitly requires null (see standards for the narrow exceptions).
- **String interpolation**: No `@"..."` except multi-line usage/help/documentation; use `+` concatenation otherwise.
- **Temporary variables**: No one-use temporaries; no trivial aliases (allowed exception: long property chains **4+** steps).
- **Brace placement**: Line breaks before `{` for namespace/class/method; **inline** `{` for `if`/`for`/`while`/`switch`; never a one-line `if` with body on the same line.
- **`this.` prefix**: Use for instance member access in new/modified Vala.
- **GLib prefix & `using`**: Fully qualify `GLib.*`; do not add `using` for new code.
- **Property initialization**: Prefer `get; set; default = …` (or field defaults), not constructor assignment for those defaults.
- **Line length & breaking**: Break long method calls, concatenations, docblocks, and comments (docblock/comment line length per full standards).
- **StringBuilder**: Avoid `GLib.StringBuilder` unless building strings in loops with **hundreds** of iterations; use `+` and `string.joinv()`.
- **String building in loops**: Never accumulate with `+=` inside a loop; use `string.nfill()` / `replace()`, `string.joinv()`, etc.
- **ArrayList for strings**: Do not use `Gee.ArrayList<string>` only to join; use `string[]` with **`string[] name = {}`** only for empty init (not `new string[0]`).
- **Loops and nesting**: In loops, use **`continue`** to skip cases and keep the body flat; avoid **`else`** and nested **`if`** inside loops.
- **Character looping**: Do not iterate characters unless there is truly no alternative; prefer string methods and `GLib.Regex`.
- **File info try/catch**: Wrap `query_info()`-style calls only when file existence is unknown — not right after a successful read of the same path.
- **Try/catch scope**: Wrap only the minimal expression(s) that throw; keep setup outside `try`.
- **Underscore prefix**: No leading `_` on variables, fields, or properties.
- **`get_*` methods**: Prefer properties or verb-less/action names over `get_foo()` (see standards for exceptions).
- **Docblocks**: Multiline `/** … */` with `@param` / `@return` where relevant; not one-liners for real behavior.
- **Debug / warning messages**: No timestamps or monotonic time in `GLib.debug()` / `GLib.warning()`; no class/method prefixes in the message (file/line come from the log).
- **Gee.HashMap**: Use `.set()` / `.get()` / `.has_key()` / `.unset()` — not `[]`.
- **Gee.ArrayList**: Use `.get()` / `.set()` / `.add()` / `.remove_at()` — not `[]`.
- **Reducing nesting**: Prefer early `return` / `continue`; avoid `else` where flow stays clear; prefer `switch` over long `if` / `else if` chains when editing.
- **SQL table aliases**: Not applicable to this plan’s fences; if you add SQL elsewhere, no table aliases (full names only).
- **Plan Vala fences**: Do **not** use comment-only placeholders (`/* … */` or empty `return "";`) where real logic belongs — either **complete, compilable Vala** in the fence or **prose-only** steps (no fake stub block).

### Phase 5

**Files:** `liboccoder/Task/Tool.vala`.

**Replace** (`Tool.reference_contents()`) — after **`load_files(this.references)`** in **`run()`**, build executor reference markdown. **Start with** **`this.parent.reference_contents(MarkdownPhase.EXECUTION)`** — that is the **task-wide** **`references` + `shared_references`** aggregate with full bodies (**same code path** as refinement, different **`stage`**). **Then** append **`this.parent.link_content(link, MarkdownPhase.EXECUTION)`** for **per-run examination** links on **`this.references`** only (links from **`build_exec_runs`** examination slice that **`Details.reference_contents`** does not walk). **Do not** double-emit links already inside **`parent.reference_contents`** (e.g. skip **`foreach`** bodies for links that are only **shared** when they are already in the parent string, or match against **`parent.exam_references`** and append **only** those examination links). Adjust **`MarkdownPhase`** if a more specific enum value fits executor review; it must **not** be **`REFINEMENT`**.

```vala
// Tool.vala — full replacement of reference_contents() (private)
private string reference_contents()
{
	var md = this.parent.reference_contents(MarkdownPhase.EXECUTION);
	foreach (var link in this.references) {
		bool is_exam = false;
		foreach (var e in this.parent.exam_references) {
			if (link.href == e.href && link.path == e.path) {
				is_exam = true;
				break;
			}
		}
		if (!is_exam) {
			continue;
		}
		if (link.scheme == "http" || link.scheme == "https") {
			continue;
		}
		var block = this.parent.link_content(link, MarkdownPhase.EXECUTION);
		if (block != "") {
			md += block;
		}
	}
	return md;
}
```

```vala
// Tool.vala — full replacement of run()
public async void run() throws GLib.Error
{
	yield this.fill_model();
	this.chat_call.tools.clear();
	var definition = this.parent.skill_manager.fetch(this.parent);
	this.use_write_executor_template = false;
	if (this.tool_call != null && this.tool_call.function.name == "write_file") {
		this.use_write_executor_template = true;
	} else if (this.tool_call == null && this.parent.tools.size == 0
			&& definition.tools.contains("write_file")) {
		this.use_write_executor_template = true;
	}
	var tool_output = "";
	if (this.tool_call != null) {
		var tool_impl = this.parent.runner.session.manager.tools.get(
				this.tool_call.function.name) as OLLMchat.Tool.BaseTool;
		this.tool_run_result = yield tool_impl.execute(this.parent.chat(), this.tool_call, true);
		tool_output = this.tool_call_details();
	}
	yield this.parent.runner.load_files(this.references);
	var reference_content = this.reference_contents();
	var executor_input = tool_output + reference_content;
	this.add_message(new OLLMchat.Message("ui",
		"Tool run finished. Reviewing Tool Output"));
	var response_text = "";
	var last_issues = "";
	for (var try_count = 0; try_count < 5; try_count++) {
		this.synthesized_write_arguments = null;
		var tpl = this.executor_prompt(executor_input, response_text, last_issues);
		var messages = new Gee.ArrayList<OLLMchat.Message>();
		messages.add(new OLLMchat.Message("system", tpl.filled_system));
		messages.add(new OLLMchat.Message("user", tpl.filled_user));
		this.session.messages.add(new OLLMchat.Message("system", tpl.filled_system));
		this.session.messages.add(new OLLMchat.Message("user", tpl.filled_user));
		var model_label = this.session.model_usage.model != "" ?
			this.session.model_usage.display_name_with_size() : "";
		var model_part = model_label != "" ? " with " + model_label : "";
		this.add_message(new OLLMchat.Message("ui",
			OLLMchat.Message.fenced(
				"markdown.oc-frame-info.collapsed Reviewing Tool Output" + model_part,
				tpl.filled_user)));
		this.add_message(new OLLMchat.Message("ui-waiting", "Waiting for response"));
		try {
			var response = yield this.chat_call.send(messages, null);
			response_text = (response != null) ? response.message.content : "";
		} catch (GLib.Error e) {
			last_issues = e.message;
			if (try_count < 4) {
				this.add_message(new OLLMchat.Message("ui", OLLMchat.Message.fenced(
					"text.oc-frame-warning.collapsed Executor try %d failed".printf(try_count + 1),
					last_issues)));
				continue;
			}
			this.parent.issues += "\n" + "Executor failed after 5 tries: " + last_issues;
			throw e;
		}
		var parser = new ResultParser(this.parent.runner, response_text);
		var exec_ok = parser.exec_extract(this);
		if (this.parent.runner.in_replay) {
			((OLLMchat.Call.ReplayChat) this.chat_call).report_replay_outcome(parser.issues);
		}
		if (exec_ok && this.synthesized_write_arguments != null) {
			var impl = this.parent.runner.session.manager.tools.get("write_file") as OLLMchat.Tool.BaseTool;
			if (impl == null) {
				this.parent.issues += "\nwrite_file tool not registered.";
				throw new GLib.IOError.INVALID_ARGUMENT("write_file missing");
			}
			var func = new OLLMchat.Response.CallFunction.with_values(
				"write_file", this.synthesized_write_arguments);
			var tc = new OLLMchat.Response.ToolCall.with_values("", func);
			this.tool_run_result = yield impl.execute(this.parent.chat(), tc, true);
		}
		if (exec_ok) {
			return;
		}
		this.parent.runner.replay_step("exec_parse_issues",
			response_text + "\n\nParse issues:\n" + parser.issues);
		last_issues = parser.issues.strip();
		if (try_count < 4) {
			this.add_message(new OLLMchat.Message("ui", OLLMchat.Message.fenced(
				"text.oc-frame-warning.collapsed Executor try %d".printf(try_count + 1),
				last_issues)));
		}
	}
	this.parent.issues += "\n" + "Executor failed after 5 tries: " + last_issues;
	throw new GLib.IOError.INVALID_ARGUMENT("Task executor: " + last_issues);
}
```

```vala
// Tool.vala — full replacement of executor_prompt()
private OLLMcoder.Skill.PromptTemplate executor_prompt(
	string executor_input,
	string previous_analysis = "",
	string previous_issues = "") throws GLib.Error
{
	var definition = this.parent.skill_manager.fetch(this.parent);
	var project = this.parent.runner.sr_factory.project_manager.active_project;
	var project_description = (project == null) ? "" : project.project_description();
	var tpl = OLLMcoder.Skill.PromptTemplate.template(
		this.use_write_executor_template
			? "task_execution_write.md"
			: "task_execution.md");
	tpl.system_fill(0);
	tpl.fill(6,
		"what_is_needed", this.parent.task_data.get("what is needed").to_markdown(),
		"skill_definition", definition.execute,
		"project_description", project_description,
		"executor_input", executor_input,
		"executor_previous_analysis", previous_analysis.strip() == "" ? "" :
			tpl.header_raw("Previous analysis", previous_analysis),
		"executor_retry_issues", previous_issues.strip() == "" ? "" :
			tpl.header_raw("Issues with it", previous_issues));
	return tpl;
}
```
