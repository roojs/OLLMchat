# 1.30 — Chat input composer (compact → expand)

> **Do not update `docs/plans/1.0-summary.md` for this plan until it is done and archived.**

**Status:** proposed

**Pointer:** `docs/guide-to-writing-plans.md` — **Checklist for plans**; proposed Vala follows **`docs/coding-standards.md`**

---

## Purpose

- **🔷** Replace the paned, user-draggable expandable input with a **content-sized** composer.
- **🔷** Empty / idle / after a finished turn: **single-line** TextView, placeholder **`Give me a task`**.
- **🔷** Composer sits in a row **above the current footer** (`ChatBar`).
- **🔷** Send uses a **play** icon on a **normal** primary button (current subtle rounded corners — **not** a circle); while running, footer shows **square stop** icon **and “Stop”** text.
- **🔷** Compact: play sits **flush on the right of the text field**; footer send is **hidden**.
- **🔷** When text needs more than one line (**newline** or **horizontal overflow** at compact width): **same TextView grows** (no widget flip); play **hides** on the side so the field goes **full width**; play/send moves to the **footer** (`ChatBar`); cap ~**50%** of the chat area then **scroll**.
- **🔷** **🚫** Do **not** flip `Gtk.Entry` ↔ `Gtk.TextView` — Android keyboard thrashing; height/resize is already handled by `ScrolledView`.
- **🔷** Keep current keys: **Ctrl+Enter** sends; **Enter** newline.
- **🔷** Remove the tools menu button on the footer (Manage Tool Availability).
- **🔷** Model dropdown: **flat** — same background as the bar; looks like text + chevron, still clickable.
- **🔷** While a reply is running: **hide the composer** (same as today); Stop stays on the footer.
- **ℹ️** Phase 1–4 landed; **2026-07-17** superseded dual-widget flip with single expanding TextView + side→footer play.

---

## Current layout (reference)

- **ℹ️** **Before Phase 1:** vertical paned = chat view | `input_column` (`above_input` + `ChatInput`); below paned = `lower_box` (`ChatPermission` + `ChatBar`). Input height via paned (~40–60%).
- **ℹ️** **After Phase 1:** vertical box = chat view | `lower_box` (`input_column`, permission, `ChatBar`); no paned composer sizing.
- **ℹ️** `ChatBar`: model `DropDown`, tools `MenuButton`, labelled Send/Stop (`suggested-action` / `destructive-action`).
- **ℹ️** Streaming: hides input column; Send → Stop on the bar.

---

## Proposed behaviour

### Compact (empty or single-line)

- **🔷** Composer strip uses the **same gray as the footer** (one continuous band — not a different gray).
- **🔷** Vertical rhythm on the gray strip (around the white entry):
  - **3–4 px** gray below the chat output (above the white entry)
  - **white** text entry (+ play on the right in compact)
  - **3–4 px** gray below the white entry, then the footer row
- **🔷** Text entry itself is **white** (distinct from the gray strip).
- **🔷** White entry has **subtle** rounded corners (match existing button / control radius — not a fully pill shape).
- **🔷** Horizontal margin on the white entry: **~5–10 px** left and right (so it does not flush to the window edges).
- **🔷** Those sizes are **design targets only** — apply via **CSS and/or widget `margin_*` / padding**, not Vala `const` / named size constants.
- **🔷** Text + **play** (normal primary button chrome) on the **right**, **no gap** between field and play.
- **🔷** Placeholder when empty: **`Give me a task`**.
- **🔷** Footer action button **hidden** in this mode (play is on the composer row).
- **🔷** After send clears / chat finishes and input is empty again → back to compact.

### Expanded (text wraps past one line)

- **🔷** Enter expanded chrome on **either**:
  - an explicit **line break** (`\n` / Enter inserting a newline), **or**
  - text that **no longer fits one line** at compact width (parent minus side-play width — avoids hide/show oscillation)
- **🔷** **One** `Gtk.TextView` in `ScrolledView` — height follows content; **no** Entry/TextView visibility flip.
- **🔷** Expanded: field is **full width** (side play hidden); play/send on **`ChatBar`**.
- **🔷** Height grows until ~**50%** of chat height, then **scroll** (pinned to bottom while typing).
- **🔷** No user-draggable paned expander for the input.
- **🚫** Do **not** flip two widgets (Android keyboard).

### Footer / controls

- **🔷** Action control: **play** icon on a normal primary button using app
  blue **`#3584E4`** (same as `.oc-user-sent-frame` — class `chat-composer-send`).
  **🚫** Do not use `suggested-action` for composer/footer send — it follows
  the desktop accent (often orange). **🚫** not `destructive-action`. While
  running: square stop icon + “Stop” via `icon_name` + `label` (no `set_child`).
- **🔷** Remove **`tools_menu_button`** (Manage Tool Availability) from `ChatBar`.
- **🔷** Model `DropDown`: remove button chrome (flat / transparent / background matching bar) so only label + arrow read.

### Streaming

- **🔷** Hide the composer while running (same as today — not visible-but-disabled).
- **🔷** Footer shows **■ Stop** (square icon + “Stop” label) while streaming (compact inline play is not shown while the composer is hidden).

## Phases (implementation order after approval)

### Phase 1 — Layout: drop paned input sizing

- **🔷** `✔️` Stop using `Gtk.Paned` end-child as a resizable input band for composer height.
- **🔷** `✔️` Put `input_column` (`above_input` + `ChatInput`) in the bottom stack **above** `ChatBar` (order: chat view → column → permission → bar); chat view takes remaining space.
- **🔷** `✔️` Keep **`above_input`** (progress strip API) as sibling above the composer in that column.
- **🔷** `✔️` Remove paned position restore / 40–60% clamp / drag-lock; disconnect typing → paned resize.
- **🔷** `✔️` Streaming still **hides** `input_column` + disables input; no paned position tricks.
- **ℹ️** Compact/expanded dual widgets and CSS strip polish are Phase 2 / 4 — Phase 1 leaves today’s `ChatInput` in the new stack (no paned growth).
- **ℹ️** Implemented in `ChatWidget.vala` / `ChatInput.vala` (awaiting user **✅**).

### Phase 2 — Compact / expand composer

- **🔷** `✔️` Two entities: compact `Gtk.Entry` + expanded `Gtk.TextView`; flip visibility (not resize).
- **🔷** `✔️` Sync text both ways on flip; after flip, **Idle until mapped**, then focus shown widget and **cursor at end**.
- **🔷** `✔️` **Mode is driven by value**, not keypress: `compact_entry.changed` + `buffer.changed` call **`update_entry`**. Covers typing, paste, and programmatic `update_entry` from ChatWidget.
- **🔷** `✔️` Expand when value has **`\n`** or **horizontal overflow** (layout width vs entry allocation); shrink when neither holds.
- **🔷** `✔️` `update_entry` / `changed` use **`syncing`** so assignments do not recurse (`update_entry` returns early if already syncing; handlers skip when syncing).
- **🔷** `✔️` Grow expanded via `ScrolledWindow.propagate_natural_height` + `max_content_height` (~half of `chat_view` height from ChatWidget); scroll inside after that.
- **🔷** `✔️` Inline play on compact row; footer action visible only when expanded (or streaming — Stop).
- **🔷** `✔️` Placeholder **`Give me a task`** on compact `Entry`.
- **🔷** `✔️` Keyboard (inline in ctor only): **Ctrl+Enter** → send. Compact **Enter** only **writes** value `entry.text + "\n"` via `update_entry` (Entry cannot hold `\n`); mode still decided inside `update_entry` from that value. Expanded **Enter** is normal TextView newline → `buffer.changed`.
- **🔷** `✔️` Plan-named methods: **`update_entry`**, **`text()`**, **`editable(bool)`**, **`focus_idle()`** (Idle callback — wait mapped, focus + caret at end). **`GLib.Idle.add` stays inside `update_entry`**. Footer send visibility: ChatWidget sets **`ChatBar.composer_expanded`** + **`action_button.visible`** inline (no `sync_composer_*` helper). No `on_key_pressed` / `clear` / `show_*`.
- **ℹ️** Apply **after Phase 1**. Phase 2 **`ChatInput`** constructor supersedes Phase 1 §7.

Edits are **Remove** / **Replace with** / **Add** from the tree; verify surrounding context before applying.

		 *
		 * @param text Full composer text
		 */
		public void update_entry(string text)
		{
			if (this.syncing) {
				return;
			}

			var want_expanded = text.contains("\n");
			if (!want_expanded && this.compact_entry.get_allocated_width() > 0) {
				/* Measure only — do not assign Entry.text (fights user action / undo). */
				var layout = this.compact_entry.create_pango_layout(text);
				Pango.Rectangle ink;
				Pango.Rectangle logical;
				layout.get_pixel_extents(out ink, out logical);
				want_expanded = logical.width > this.compact_entry.get_allocated_width();
			}

			if (want_expanded && this.is_expanded) {
				Gtk.TextIter cur_start;
				Gtk.TextIter cur_end;
				this.buffer.get_start_iter(out cur_start);
				this.buffer.get_end_iter(out cur_end);
				if (this.buffer.get_text(cur_start, cur_end, false) == text) {
					return;
				}
				this.syncing = true;
				this.buffer.delete(ref cur_start, ref cur_end);
				this.buffer.insert(ref cur_start, text, -1);
				this.syncing = false;
				return;
			}
			if (!want_expanded && !this.is_expanded) {
				if (this.compact_entry.text == text) {
					return;
				}
				this.syncing = true;
				this.compact_entry.text = text;
				this.syncing = false;
				return;
			}

			if (want_expanded) {
				this.syncing = true;
				Gtk.TextIter start_iter;
				Gtk.TextIter end_iter;
				this.buffer.get_start_iter(out start_iter);
				this.buffer.get_end_iter(out end_iter);

					this.scrolled.max_content_height = this.expanded_max_height;
				}
				this.syncing = false;
				this.expanded_changed(true);
				GLib.Idle.add(this.focus_idle);
				return;
			}

			this.syncing = true;
			this.compact_entry.text = text;
			Gtk.TextIter start_iter;
			Gtk.TextIter end_iter;
			this.buffer.get_start_iter(out start_iter);
			this.buffer.get_end_iter(out end_iter);
			this.buffer.delete(ref start_iter, ref end_iter);

			this.compact_row.visible = true;
			this.is_expanded = false;
			this.syncing = false;
			this.expanded_changed(false);
			GLib.Idle.add(this.focus_idle);

				this.text_view.scroll_to_iter(end_iter, 0.0, false, 0.0, 1.0);

			}
			if (!this.compact_entry.get_mapped()) {
				return true;
			}
			this.compact_entry.grab_focus();
			this.compact_entry.set_position(-1);
			return false;
		}
