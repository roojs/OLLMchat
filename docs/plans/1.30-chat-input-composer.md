# 1.30 — Chat input composer (compact → expand)

> **Do not update `docs/plans/1.0-summary.md` for this plan until it is done and archived.**

**Status:** proposed

**Pointer:** `docs/guide-to-writing-plans.md` — **Checklist for plans**; proposed Vala follows **`docs/coding-standards.md`**

---

## Purpose

- **🔷** Replace the paned, user-draggable expandable input with a **content-sized** composer.
- **🔷** Empty / idle / after a finished turn: **single-line** entry, placeholder **`Give me a task`**.
- **🔷** Composer sits in a row **above the current footer** (`ChatBar`).
- **🔷** Send uses a **play** icon on a **normal** primary button (current subtle rounded corners / `suggested-action` — **not** a circle); while running, footer shows **square stop** icon **and “Stop”** text.
- **🔷** Compact: play sits **flush on the right of the text field**; footer send is **hidden**.
- **🔷** When text needs more than one line (**newline** or **horizontal overflow** at field width): **flip** to the expanded widget (two entities), grow **upward**, keep the **caret line at the bottom**, cap ~**50%** of the chat area then **scroll**.
- **🔷** Expanded: input is **full width**; play/send moves to the **footer** (`ChatBar`).
- **🔷** After each widget flip: wait until the shown control is ready, then **focus** it with the **cursor at end**.
- **🔷** Keep current keys: **Ctrl+Enter** sends; **Enter** newline (both compact and expanded).
- **🔷** Remove the tools menu button on the footer (Manage Tool Availability).
- **🔷** Model dropdown: **flat** — same background as the bar; looks like text + chevron, still clickable.
- **🔷** While a reply is running: **hide the composer** (same as today); Stop stays on the footer.
- **⏳** Phases 2–4 concrete Vala fences — after Phase 1 lands / remaining design confirms.
- **ℹ️** Today: `ChatInput` + paned end child in `ChatWidget`; footer is `ChatBar` in `lower_box`. CSS: `resources/style.css`; primary blue already used (`#3584E4` in user-message frames).

## Current layout (reference)

- **ℹ️** **Before Phase 1:** vertical paned = chat view | `input_column` (`above_input` + `ChatInput`); below paned = `lower_box` (`ChatPermission` + `ChatBar`). Input height via paned (~40–60%).
- **ℹ️** **After Phase 1:** vertical box = chat view | `lower_box` (`input_column`, permission, `ChatBar`); no paned composer sizing.
- **ℹ️** `ChatBar`: model `DropDown`, tools `MenuButton`, labelled Send/Stop (`suggested-action` / `destructive-action`).
- **ℹ️** Streaming: hides input column; Send → Stop on the bar.

## Proposed behaviour

### Compact (empty or single-line)

- **🔷** Composer strip uses the **same gray as the footer** (one continuous band — not a different gray).
- **🔷** Vertical rhythm on the gray strip (around the white entry):
  - **3–4 px** gray below the chat output (above the white entry)
  - **white** text entry (+ play on the right in compact)
  - **3–4 px** gray below the white entry, then the footer row
- **🔷** Text entry itself is **white** (distinct from the gray strip).
- **🔷** White entry has **subtle** rounded corners (match existing button / control radius — not a fully pill shape).
- **🔷** Horizontal margin on the white entry: **~5–10 px** left and right (so it does not flush to the window edges).
- **🔷** Those sizes are **design targets only** — apply via **CSS and/or widget `margin_*` / padding**, not Vala `const` / named size constants.
- **🔷** Text + **play** (normal primary button chrome) on the **right**, **no gap** between field and play.
- **🔷** Placeholder when empty: **`Give me a task`**.
- **🔷** Footer action button **hidden** in this mode (play is on the composer row).
- **🔷** After send clears / chat finishes and input is empty again → back to compact.

### Expanded (text wraps past one line)

- **🔷** Flip compact → expanded on **either**:
  - an explicit **line break** (`\n` / Enter inserting a newline), **or**
  - text that **no longer fits one visual line horizontally** (too long for the field width)
- **🔷** Horizontal overflow is easy to miss: compact mode may **not** word-wrap (clip / ellipsize / horizontal scroll — whatever GTK does for a single-line field). Expand when content **would need** more than one line at the current width, not only when a wrap already happened.
- **🔷** **Two widgets**, not one resized box: compact entity and expanded entity; show one / hide the other (visibility flip). Copy text across on flip.
- **🔷** Focus is the hard part: after the hide/show flip, **wait** (idle / short delay until the new widget is realizable / mapped), then **grab focus** on the shown widget and place the **cursor at the end** of the text.
- **🔷** Multi-line (expanded) field is **full width** of the input column.
- **🔷** Expanded height follows content; **current line stays at the bottom**; top grows until ~**50%** of available chat height, then **scroll** inside the field (still pinned to bottom while typing).
- **🔷** Play/send moves to **`ChatBar`** (footer); inline play is with the compact entity only.
- **🔷** No user-draggable paned expander for the input — height is **content-driven** on the expanded widget only.
- **🚫** Do not implement compact/expand by resizing a single text box.
- **💩** Exact pair (`Gtk.Entry` + `Gtk.TextView`, or two differently configured views) — settle in Phase 2 fences.

### Footer / controls

- **🔷** Action control: **play** icon on a **standard** `suggested-action` button (subtle rounded corners like the rest of the UI — **🚫 not circular**); while running, **square stop** icon **plus “Stop”** text, `destructive-action` OK.
- **🔷** Remove **`tools_menu_button`** (Manage Tool Availability) from `ChatBar`.
- **🔷** Model `DropDown`: remove button chrome (flat / transparent / background matching bar) so only label + arrow read.

### Streaming

- **🔷** Hide the composer while running (same as today — not visible-but-disabled).
- **🔷** Footer shows **■ Stop** (square icon + “Stop” label) while streaming (compact inline play is not shown while the composer is hidden).

## Phases (implementation order after approval)

### Phase 1 — Layout: drop paned input sizing

- **🔷** `⏳` Stop using `Gtk.Paned` end-child as a resizable input band for composer height.
- **🔷** `⏳` Put `input_column` (`above_input` + `ChatInput`) in the bottom stack **above** `ChatBar` (order: chat view → column → permission → bar); chat view takes remaining space.
- **🔷** `⏳` Keep **`above_input`** (progress strip API) as sibling above the composer in that column.
- **🔷** `⏳` Remove paned position restore / 40–60% clamp / drag-lock; disconnect typing → paned resize.
- **🔷** `⏳` Streaming still **hides** `input_column` + disables input; no paned position tricks.
- **ℹ️** Compact/expanded dual widgets and CSS strip polish are Phase 2 / 4 — Phase 1 leaves today’s `ChatInput` in the new stack (no paned growth).

Edits are **Remove** / **Replace with** / **Add** from the tree; verify surrounding context before applying.

#### 5. `ChatWidget.vala` — `streaming_state`: hide composer only

**Why:** Hide `input_column` while running; no paned height save/restore.

**Where:** private method `streaming_state`.

**Depends on:** §1, §3.

##### Remove

```vala
		/** Updates streaming state: when running, hide text area (paned position = full + input not visible); when stopped, restore position. */
		private void streaming_state(bool streaming)
		{
			this.streaming = streaming;
			this.chat_input.visible = !streaming;
			this.input_column.visible = !streaming;
			this.chat_input.set_input_editable(!streaming);
			this.chat_input.set_input_sensitive(!streaming);
			this.chat_bar.sync_streaming(streaming);

			if (streaming) {
				this.paned.set_cursor(new Gdk.Cursor.from_name("default", null));
				GLib.Idle.add(() => {
					if (!this.streaming) {
						return false;
					}
					var h = this.paned.get_allocated_height();
					if (h <= 0) {
						return true;
					}
					var pos = this.paned.get_position();
					this.saved_bottom_height = (int) (h - pos);
					if (this.saved_bottom_height < this.min_bottom_size) {
						this.saved_bottom_height = this.min_bottom_size;
					}
					this.locking_paned_position = true;
					this.paned.set_position(h);
					GLib.Idle.add(() => {
						this.locking_paned_position = false;
						return false;
					});
					return false;
				});
				return;
			}
			this.paned.set_cursor(null);
			GLib.Idle.add(() => {
				int h = this.paned.get_allocated_height();
				int want_40 = (int)(h * 0.4);
				int want_60 = (int)(h * 0.6);
				int want = this.saved_bottom_height > 0 ? this.saved_bottom_height : want_40;
				if (want < this.min_bottom_size) {
					want = this.min_bottom_size;
				}
				if (want < want_40) {
					want = want_40;
				}
				if (want > want_60) {
					want = want_60;
				}
				if (h > want) {
					this.paned.set_position(h - want);
				}
				/* GLib.debug("scroll_to_bottom_caller reason=streaming_stop_paned_idle"); */

			});
```
