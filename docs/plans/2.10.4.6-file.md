# 2.10.4.6 — `File` (client and server)

**Status:** PROPOSAL — after 2.10.4.5.

**Parent:** [`2.10.4.1`](2.10.4.1-ollmfilesd-rpc-api.md)

---

## Client — `libocfiles/V2/File.vala`

RPC via `RpcClient`. Namespace **`OLLMfiles`**.

**Client-only (no RPC):**

- `File.buffer`, `buffer_provider`, `create_buffer` — after `File.read` returns content
- `File.new_fake` (id &lt; 0) until **`File.register`** succeeds
- `tree_factory` / tree-sitter — local on buffer, or deferred `File.parse.ast`

---

## Server — `ollmfilesd`

`OLLMfiles.File` + disk I/O; **`DeleteManager.remove`** for delete path.

**Daemon-internal:** `FileHistory.cleanup_old_backups`; `VectorMetadata.cleanup_all_deleted` after delete.

---

## Methods — today’s code → wire

| `File.{method}` | Today | Notes |
| --------------- | ----- | ----- |
| `get` | `ProjectManager.get_file_from_active_project` | Resolve path → `filebase` — object TBD (`File` vs `ProjectManager`) |
| `read` | buffer load / `reload_file_from_disk` | `params.path`; returns `filebase` + `content` |
| `write` | `write_buffer_to_disk`, `on_file_contents_change` | `params.content` — queues scan/index in daemon |
| `activate` | `ProjectManager.activate_file` | Track active file — **`File.activate`** vs `ProjectManager`? |
| `metadata.set` | `on_file_metadata_change` | cursor, scroll — no reindex |
| `changed.check` | `check_active_file_changed` | external change vs buffer |
| `register` | `convert_fake_file_to_real` | fake → real row |
| `delete` | `DeleteManager.remove` | `params.path` |
| `index` | per-file index queue | with `BackgroundScan` |

```json
→ {"method":"File.read","params":{"path":"/home/alan/proj/foo/src/Ap.php"}}
→ {"method":"File.write","params":{"path":"...","content":"..."}}
```

**Open:** top-level `params.path` only vs nested `params.file`; whether `get` stays on `ProjectManager.get_folder_at_path`.
