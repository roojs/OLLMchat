### Chat bar (model dropdown)

_(none)_

### Chat rendering

- **🔷** `⏳` GtkSourceView font not monospace on device
  - Code blocks / SourceView text renders in a proportional or wrong font on Android; desktop uses `resources/style.css` `.source-view { font-family: monospace; }` — likely needs Android font fallback or explicit `Pango.FontDescription`

### Tools (Android POC)

Tools button is empty because Android never calls `OLLMtools.Registry.fill_tools()` — not a `ChatBar` bug.

**Could work now (minimal registry):**

| Tool | Verdict | Notes |
|------|---------|-------|
| `web_fetch` | ✅ | HTTP/TLS on device |
| `session_fetch` | ✅ | Current session / history refs |
| `google_search` | ⚠️ | Only if user configures API key (Custom Search; no public free tier) |

**Disable for POC (no project workspace):**

| Tool | Verdict | Notes |
|------|---------|-------|
| `read_file` | 🚫 | Needs `ProjectManager` / workspace paths |
| `write_file` | 🚫 | Same |
| `edit_mode` | 🚫 | Same |

**Not on device:**

| Tool | Verdict | Notes |
|------|---------|-------|
| `run_command` | 🚫 | No useful shell on Android |
| `codebase_search` | 🚫 | Vectors + projects — out of scope |
| Wrapped `.tool` defs | 🚫 | Out of scope |
| MCP (HTTP / stdio) | 🚫 | HTTP could work on device; blocked on **MCP management UI** — desktop still hand-edits `mcp.json` ([`docs/mcp-settings.md`](../mcp-settings.md)); Android waits on that

**Later (not this phase):**

- **🔷** `run_command` (or equivalent) via **RPC** on hosted `ollmfilesd` — needs RPC connected first; remote execution on your machine, not the phone
- **🔷** **MCP** — after desktop gets settings UI for servers (today: edit `mcp.json` only); then HTTP MCP on Android is plausible; stdio MCP less so on device

### Session lifecycle

- **🔷** `⏳` Cold start opens fresh `EmptySession`
  - User expects last chat back — sidebar may list history but boot does not restore
- **🔷** `⏳` Verify saves after `force-stop`
  - Check `files/.../history/` after send + kill (`Session.save_async`)
- **🔷** `⏳` Decide: new chat on restart vs restore last `fid`
- **💩** `⏳` Pause/stop flush + resume restore
  - Android-only — needs approval for shared `Manager` / `ChatWidget`

### Settings

  - **ℹ️** Ref: `ModelsPage.vala`, `ModelRow.vala`
