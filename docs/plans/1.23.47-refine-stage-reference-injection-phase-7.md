## Coding standards checklist

Copied from **`.cursor/rules/CODING_STANDARDS.md`** — “Checklist for all plans”. If a bullet conflicts with **§ Conventions** in the [parent plan](1.23.40-refine-stage-reference-injection.md), **the standards file wins**.

- **Nullable types**: Design new APIs/properties to avoid nullable types where possible (default objects / flags).
- **Null checks**: Avoid generic null checks; only where the design explicitly requires null (see standards for the narrow exceptions).
- **String interpolation**: No `@"..."` except multi-line usage/help/documentation; use `+` concatenation otherwise.
- **Temporary variables**: No one-use temporaries; no trivial aliases (allowed exception: long property chains **4+** steps).
- **Brace placement**: Line breaks before `{` for namespace/class/method; **inline** `{` for `if`/`for`/`while`/`switch`; never a one-line `if` with body on the same line.
- **`this.` prefix**: Use for instance member access in new/modified Vala.
- **GLib prefix & `using`**: Fully qualify `GLib.*`; do not add `using` for new code.
- **Property initialization**: Prefer `get; set; default = …` (or field defaults), not constructor assignment for those defaults.
- **Line length & breaking**: Break long method calls, concatenations, docblocks, and comments (docblock/comment line length per full standards).
- **StringBuilder**: Avoid `GLib.StringBuilder` unless building strings in loops with **hundreds** of iterations; use `+` and `string.joinv()`.
- **String building in loops**: Never accumulate with `+=` inside a loop; use `string.nfill()` / `replace()`, `string.joinv()`, etc.
- **ArrayList for strings**: Do not use `Gee.ArrayList<string>` only to join; use `string[]` with **`string[] name = {}`** only for empty init (not `new string[0]`).
- **Loops and nesting**: In loops, use **`continue`** to skip cases and keep the body flat; avoid **`else`** and nested **`if`** inside loops.
- **Character looping**: Do not iterate characters unless there is truly no alternative; prefer string methods and `GLib.Regex`.
- **File info try/catch**: Wrap `query_info()`-style calls only when file existence is unknown — not right after a successful read of the same path.
- **Try/catch scope**: Wrap only the minimal expression(s) that throw; keep setup outside `try`.
- **Underscore prefix**: No leading `_` on variables, fields, or properties.
- **`get_*` methods**: Prefer properties or verb-less/action names over `get_foo()` (see standards for exceptions).
- **Docblocks**: Multiline `/** … */` with `@param` / `@return` where relevant; not one-liners for real behavior.
- **Debug / warning messages**: No timestamps or monotonic time in `GLib.debug()` / `GLib.warning()`; no class/method prefixes in the message (file/line come from the log).
- **Gee.HashMap**: Use `.set()` / `.get()` / `.has_key()` / `.unset()` — not `[]`.
- **Gee.ArrayList**: Use `.get()` / `.set()` / `.add()` / `.remove_at()` — not `[]`.
- **Reducing nesting**: Prefer early `return` / `continue`; avoid `else` where flow stays clear; prefer `switch` over long `if` / `else if` chains when editing.
- **SQL table aliases**: Not applicable to this plan’s fences; if you add SQL elsewhere, no table aliases (full names only).
- **Plan Vala fences**: Do **not** use comment-only placeholders (`/* … */` or empty `return "";`) where real logic belongs — either **complete, compilable Vala** in the fence or **prose-only** steps (no fake stub block). Use **`var`** for locals when the initializer fixes the type (**`bool`**, **`int`**, **`string`**, etc.); keep explicit types only where inference does not apply.

### `FileBuffer` / **`DummyFileBuffer`** / **`GtkSourceFileBuffer`** — trimmed multi-line search (**`libocfiles`** + **`liboccoder`**)

**Goal:** **`search_text`** matching is **not** implemented with **`get_text()`** + **`string.index_of`** in **`WriteChange`** or **`Request`**. Add **`OLLMfiles.FileBuffer`** API so **both** buffer implementations **search the same way**: line-block match with **`strip()`** on each line so **indentation may differ** between the excerpt and the file.

**Algorithm (shared semantics):**

1. Split **`needle`** (**`search_text`**) into lines with **`"\n"`** (same as **`DummyFileBuffer`** line cache split).
2. Let **`needle_lines[]`** be those lines (at least one line after split; trim comparisons below).
3. Scan buffer start lines **`i`** from **0** to **`get_line_count() - needle_lines.length`** (inclusive range so all following lines exist).
4. **Anchor:** **`get_line(i).strip() == needle_lines[0].strip()`** — find the **first line** of the excerpt in the buffer using **trim** on both sides (indent in the file and in **`search_text`** may differ).
5. **Continuation:** for each **`k`** from **1** to **`needle_lines.length - 1`**, require **`get_line(i + k).strip() == needle_lines[k].strip()`**.
6. Each complete match increments a **non-overlapping** forward count (after a match at **`i`**, next candidate start is **`i + 1`**, or **`i + needle_lines.length`** if you only want block-disjoint matches — **pick one rule** and use it in **both** **`DummyFileBuffer`** and **`GtkSourceFileBuffer`**; document in Valadoc). **`count_trimmed_search_matches (needle)`** returns that count; callers require **exactly 1**.

**`DummyFileBuffer`:** Implement against the cached **`lines`** array (same logic as today’s line storage): walk indices, **`strip()`** compare — no full-buffer **`index_of`**.

```vala
		{
			this.validate_structure ();
			if (this.issues.strip () != "") {
				return;
			}
			var project_manager = this.project_manager;
			var has_ast = (this.ast_path.strip () != "");
			var has_lines = (this.start_line >= 1 && this.end_line >= this.start_line);
			var norm = "";
			if (has_ast || has_lines || this.search_text.strip () != "") {
				norm = this.file_path;
				if (project_manager.active_project != null) {
					if (!GLib.Path.is_absolute (norm)) {
						norm = GLib.Path.build_filename (project_manager.active_project.path, norm);
					}
				}
				if (!GLib.FileUtils.test (norm, GLib.FileTest.IS_REGULAR)) {
					this.issues += "\nwrite_file: file does not exist (required for ast_path / line_numbers / search_text mode)";
					return;
				}
			}
			if (this.search_text.strip () == "" && !has_ast) {
				return;
			}
			if (has_ast) {
				var file = project_manager.get_file_from_active_project (norm);
				if (file == null) {
					file = new OLLMfiles.File.new_fake (project_manager, norm);
				}
				project_manager.buffer_provider.create_buffer (file);
				if (!file.buffer.is_loaded) {
					try {
						yield file.buffer.read_async ();
					} catch (GLib.Error e) {
						this.issues += "\nwrite_file: failed to read file: " + e.message;
						return;
					}
				}
				var change = new OLLMfiles.FileChange (file) {
					ast_path = this.ast_path.strip (),
					replacement = this.content
				};
				yield change.resolve_ast_path ();
				if (change.has_error || (change.result != "" && change.result != "applied")) {
					this.issues += "\nwrite_file: "
						+ (change.result != "" ? change.result : "AST path did not resolve");
					return;
				}
				return;
			}
			var file = project_manager.get_file_from_active_project (norm);
			if (file == null) {
				file = new OLLMfiles.File.new_fake (project_manager, norm);
			}
			project_manager.buffer_provider.create_buffer (file);
			if (!file.buffer.is_loaded) {
				try {
					yield file.buffer.read_async ();
				} catch (GLib.Error e) {
					this.issues += "\nwrite_file: failed to read file: " + e.message;
					return;
				}
			}
			var n = file.buffer.count_trimmed_search_matches (this.search_text);
			if (n == 0) {
				this.issues += "\nwrite_file: search_text not found in file";
				return;
			}
			if (n > 1) {
				this.issues += "\nwrite_file: search_text must match exactly once";
				return;
			}
		}
```
