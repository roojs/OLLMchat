# Plan 1.24.2: OpenAI API — Generate (all libraries)

## Overview

Phase 2 integration of OpenAI-compatible **generate** for **all libraries** that use it. Parent plan: **1.24** (OpenAI API chat and embed compatibility). Ollama has `/api/generate`; OpenAI has `/v1/completions` (different shape, no conversation). This plan covers every generate usage across the codebase.

## Scope

**Method:** Generate only.

**Libraries:** libollmchat (Client) — currently the only consumer. `Client.generate()` uses `Call.Generate` (Ollama `/api/generate`), returns `Response.Generate`.

**Refer to plan 1.24 for:** rationale, API differences.

## Code that would need changing (generate only — all libraries)

| Library   | File(s)       | Current usage | Change for v1 |
|-----------|---------------|----------------|---------------|
| **libollmchat** | `Client.vala` | `generate()`: `Call.Generate`, `/api/generate`, `Response.Generate`. | Decide: (a) add `Call.Completions` + v1 path, map to `Response.Generate` or adapter, or (b) leave Generate as Ollama-only for now. |

## Notes

- Generate is a small surface; no other libraries use it today. Optional migration.
- OpenAI `/v1/completions` is a different API (prompt → completion, no messages array). If we migrate, we need a request/response mapping or a dedicated v1 completions type.
