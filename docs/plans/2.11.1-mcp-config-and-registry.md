# 2.11.1. MCP Config and Base Registry

## Overview

Phase 1 of the MCP loader: config types for `mcp.json` and the base `OLLMmcp.Registry` class. No transport implementations, no Loader, no meson — code is not built until 2.11.4.

Parent: [2.11-mcp-loader-tool.md](2.11-mcp-loader-tool.md)

## Status

✅ **Code in place** – Config.vala and Registry.vala added under libocmcp/. Not yet in meson (2.11.4); linter will show missing deps until then.

## Scope

- **Dedicated MCP config file:** `mcp.json` is **just an array** (root = JSON array of server entries). No wrapper object.
- **Config class = single service:** One **`OLLMmcp.Config`** class representing a single MCP server entry: `id`, `enabled`, `transport` ("stdio" | "http"); for stdio: `command`, `args`, `env`, optional **`network`**; for http: `url`. See parent plan for full schema.
- **Registry loads MCP and gets array of Config:** Registry (or a loader helper) loads `~/.config/ollmchat/mcp.json` and deserializes it into an **array of `OLLMmcp.Config`**. Each Config is one service. Later, `fill_tools()` will iterate this array, create a Client per (enabled) Config via factories, and register tools for each client.
- **Registry:** `OLLMmcp.Registry` in `libocmcp/Registry.vala` with:
  - `init_config()` — no-op or minimal (no tool config types to register yet if we don’t add MCP to Config2 in this phase).
  - `setup_config_defaults(OLLMchat.Settings.Config2 config)` — no-op or ensure an optional MCP section in config if we add it later.
  - `fill_tools(Manager manager, ProjectManager? project_manager)` — stub: no factories yet, so do nothing (or just log). Same signature as the other registries. When implemented: load MCP file → array of Config → for each enabled Config create Client (via factory) → register all tools from that client.

## Files

All under **libocmcp/** (not yet in meson):

- `libocmcp/Config.vala` – namespace `OLLMmcp`. **`Config`** class = one MCP server (single service): id, enabled, transport, command, args, env, url, network. Provide a way to **load** `~/.config/ollmchat/mcp.json` and return an **array/list of `Config`** (file root is JSON array).
- `libocmcp/Registry.vala` – namespace `OLLMmcp`. Class `Registry` with `init_config()`, `setup_config_defaults(Config2)`, `fill_tools(Manager, ProjectManager?)` (stub). When filled in: load MCP file → array of Config → create clients and register tools (via factories) for all enabled Configs.

## Out of scope (later phases)

- Loader, Client, Factory (Mcp/Factory), Tool (Mcp/Tool).
- Stdio/Http implementations.
- Adding libocmcp to meson or app (2.11.4).

## Completion

- Config can be loaded and parsed; struct/class for server config is clear.
- Registry exists and can be instantiated; its three methods are callable and match the pattern of tools/vector registries.
- Code compiles only when libocmcp is added in 2.11.4; this phase just adds the source files.
