# 1.4. Client Configuration Setup

## Overview

Client configuration system for setting up and managing server connections, API keys, and basic client settings. Includes bootstrap dialog for initial setup, configuration persistence, and version management. Focuses on the connection layer between the application and LLM servers.

## Implementation Details

### Phase 1: Bootstrap Dialog ‚úÖ DONE

#### Bootstrap Dialog UI ‚úÖ
- **Framework**: Uses libadw (Adwaita) preferences dialog ‚úÖ
- **Structure**: Single header/preferences page ‚úÖ
- **Header Title**: "Set up initial connect" ‚úÖ
- **Description**: 
  > First step is to connect to a ollama or openai server (this is really designed for locally hosted LLMs, however you should be able to use online LLMs if you have an account) ‚úÖ

#### Dialog Fields ‚úÖ
1. **Host Field** ‚úÖ
   - Label: "Host"
   - Placeholder: `http://127.0.0.1:11434/api`
   - Input type: Text entry
   - Required: Yes

2. **API Key Field** ‚úÖ
   - Label: "API Key"
   - Description: "(optional) only need for online services or if you serve via nginx proxy"
   - Input type: Text entry
   - Required: No

3. **Next Button** ‚úÖ
   - Action: Validate and test connection
   - Behavior:
     - Button locks (disabled) when clicked ‚úÖ
     - Shows Adwaita spinner while checking server ‚úÖ
     - Calls version endpoint to verify connection ‚úÖ
     - On success: Save config and proceed ‚úÖ
     - On failure: Show error, unlock button ‚úÖ

#### Connection Testing ‚úÖ
- **Method**: Uses GET `/api/version` endpoint ‚úÖ
- **Purpose**: Verify server is reachable and responding ‚úÖ
- **Reference**: https://docs.ollama.com/api-reference/get-version
- **Response**: Expects `{ "version": "x.x.x" }` format ‚úÖ
- **Error Handling**: 
  - Network errors (connection refused, timeout) ‚úÖ
  - Invalid response format ‚úÖ
  - Authentication errors (if API key required) ‚úÖ

#### Client API Support ‚úÖ
- **Method**: `version()` method added to `Client` class ‚úÖ
- **Implementation**: 
  - `Call.Version` class created ‚úÖ
  - Returns version string from server ‚úÖ
  - Handles errors appropriately ‚úÖ
- **Usage**: Called during bootstrap to verify connection ‚úÖ

#### Bootstrap Flow ‚úÖ
1. First run: Show bootstrap dialog (libadw preferences dialog) ‚úÖ
2. User enters host (with default placeholder) ‚úÖ
3. User optionally enters API key ‚úÖ
4. User clicks "Next" button ‚úÖ
5. Button locks, spinner shows ‚úÖ
6. System calls `GET /api/version` to test connection ‚úÖ
7. On success: Save config as first default client, close dialog ‚úÖ
8. On failure: Show error message, unlock button for retry ‚úÖ
- **Scope**: Bootstrap handles initial client setup ‚úÖ

#### Files Implemented ‚úÖ
- `ollmchat/BootstrapDialog.vala` - Bootstrap dialog implementation ‚úÖ
- `libollmchat/Call/Version.vala` - Version API call ‚úÖ
- `libollmchat/Client.vala` - `version()` method ‚úÖ
