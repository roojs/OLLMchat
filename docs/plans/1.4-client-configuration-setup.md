### Features

- Configuration for setting up client options
- Bootstrap dialog for initial setup:
  - Enter IP address or address of the API
  - Check whether it needs an API key
  - Store in config as the first default client
- Modify config to support multiple clients (related to 1.3)

### Bootstrap Dialog Flow
1. First run: Show bootstrap dialog
2. User enters API address/IP
3. System checks if API key is needed
4. If needed, prompt for API key
5. Test connection
6. Store as first default client in config
