# 7.14.15 — Progress collapse header: active row title

**Status:** proposed

**Parent:** `docs/plans/7.14.14-progress-tree-collapse-header.md` (**✅** collapse chrome)

**Pointer:** `.cursor/rules/CODING_STANDARDS.md`, `docs/guide-to-writing-plans.md`

---

## Goals

- 🔷 Collapsed header shows **`ProgressItem.title`** for the row the runner is driving; **“Skill activity”** when argument is **`null`**.
- 🔷 **`ProgressList`**: one signal — `active_item_changed(ProgressItem? item)`.
- 🔷 Callers **`emit`** at slice entry; **`emit(null)`** when idle.
- 🔷 **`ProgressView`**: connect in **`set_runner`** with a **lambda** — no named handler, no disconnect / handler id (view lifetime matches the strip).

---

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| Signal + emit sites (live + replay) + view handler | `active_item` property, `set_active`, tree heuristics |

---

## Emit contract

- **`active_item_changed(pi)`** — `pi` is the row whose **title** column should appear in the collapse header.
- **`active_item_changed(null)`** — idle (**“Skill activity”**).
- 🔷 Prefer **emit on entry**; avoid **null** between back-to-back slices in the same stack if the next slice emits immediately (e.g. **refine** → **run_exec** on same **`Details`**).

## Acceptance criteria

- Live: header tracks **Task Creation** → **detail title** → **tool** as emits fire.
- Replay: header follows **#17–20** during restore.
- Idle: **#3**, **#6**, **#21**, and error exits → **“Skill activity”**.
- Grep: no `watch_progress_model` / `notify_items` in **`ProgressView`**.
