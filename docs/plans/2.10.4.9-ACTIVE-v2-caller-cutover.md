## Procedure (🔷)

1. **Copy** — duplicate the shipping file into `libocfiles/V2/` (same class name, namespace `OLLMfiles`).
2. **Transport**
  - `RpcClient.vala`, `RpcClientBoot.vala`, `Copyable.vala` stay at **repo root**
  - **🚫** not under `V2/`
  - shared infrastructure, not parallel replacements
  - future V2 Meson list **includes** them
  - **🚫** no `V2/` copy step
3. **Commit** — land the verbatim copy on its own commit (user-driven).
4. **Reshape** — one file per change set
  - RPC-backed list models
  - **🚫** no in-process scan/sandbox
  - match [`done/2.10.4.9.1`](done/2.10.4.9.1-DONE-design-project-files-and-client-api.md)
  - **plus** client `*Files` copies below
  - extends “no client `project_files`” — copies become thin RPC facades, not daemon index
5. **Callers**
  - duplicate into **`<lib>/V2/`** (or **`liboctools/*V2/`** for tools)
  - same relative path under the copy dir
  - same class name and namespace
  - reshape there for review
  - **🚫** do not edit shipping caller sources until Meson flip
  - same rule as **`libocfiles/V2/`**
6. **Meson cutover (later)** — **⏳ Not started**
  - flag toggles each library's source list (`ocfiles_src`, `occoder_src`, `octools_src`, …)
  - shipping paths → **`V2/*`**
  - spawn `ollmfilesd` from app startup

**Per-file status (🔷):**

- **Copy** — verbatim duplicate from shipping (`cp` / `diff -q` clean) **before** any reshape edits on that file
- **Add** — 🆕 net-new file; no shipping source; skip copy entirely
- **Reshape** — RPC / API edits on a copied file only (shipping stays frozen)
- **🚫** do not author a new `V2/` file from scratch — you lose the shipping→V2 diff

Legend (use only what applies):

- copy ✔️ / copy 📋 — duplicate from shipping
- add ✔️ / add 📋 — 🆕 net-new file in tree / still owed
- reshape ⏳ / reshape ✔️ — on a **copy** only

ℹ️ `V2 PM` in reshape notes means `libocfiles/V2/ProjectManager.vala` (thin RPC client)

- 🚫 not a copy under `liboctools`
- tools still take `OLLMfiles.ProjectManager`
- reshape swaps shipping calls (`get_file_from_active_project`, `project_files`, …) for V2 API (`fetch_file`, `fetch_files`, …)

### B-10 — lazy file list (dropdown)

**🔷** Paged **`Folder.fetch_files`** for **`FileDropdown`** only. **🚫** not semantic search (**`Codebase.search`**).

**RPC budget**

| When | Calls |
|------|-------|
| Open dropdown / project change | **1** — first page |
| Scroll near bottom | **+1** page if **`offset < total`**, not already **`loading`** |
| Search text settled | **1** debounced **`refresh(query)`** |
| Open file | **`File.read`** in **`open_file`** — not list RPC |

**Wire — `Folder.fetch_files`**

Type: **`ollmfilesd/CallParam.vala`** → **`FolderParams`** (shared by all **`Folder.*`** RPC).

**Existing fields (unchanged — already on wire today)**

- **`path`** — project root for this call.
  - Used today by **`Folder.fetch`**, **`Folder.fetch_files`**, **`Folder.fetch_pending_approvals`**, **`Folder.project_description`** (client sets **`path = project.path`**).
- **`project_path`** — which project owns the row.
  - Used today only by **`Folder.contains_folder`** together with **`path`** = directory to check (**`project_path`** = project root, **`path`** = dir path).

**New fields (B-10 — add to `FolderParams`; only `fetch_files` reads them)**

- **`offset`** — skip rows; default **`0`**.
- **`limit`** — page size; default **`50`** (one screen of dropdown rows).
- **`query`** — dropdown filter string; default **`""`** (browse all).
  - **🚫** not semantic search — that stays on **`VectorParams.query`** → **`Codebase.search`**.

Other **`Folder.*`** handlers ignore **`offset`**, **`limit`**, **`query`** if present.

**Request (client → daemon)**

- Method: **`Folder.fetch_files`** (renamed from **`fetch_file_list`** in 9.3.1 relay).
- Params: **`path`** = active project root; **`offset`**, **`limit`**, **`query`** as above.
- **🚫** do not send **`project_path`** for this call — same as today.

**Response — today vs B-10**

| | Today (shipped relay) | B-10 |
|---|----------------------|------|
| **`result`** | every file in project | one page of **`File[]`** metadata only (**🚫** no content) |
| **`msg`** | unused on success | total count as decimal string |
| pagination | none — full dump | client stops when **`offset + page.size >= int.parse(msg)`** |

- **`msg`** meaning: project file count when **`query`** empty; match count when filtering.
- **🚫** no **`has_more`** flag — derive from total vs **`offset`**.

**Browse order (server default — only consumer is dropdown)**

1. Recent (**`last_viewed`** within 24h) first

### B-12 — `ProjectFiles.refresh` redesign

**Problem (current interim code):** every **`refresh(query)`** wipes the in-memory list and re-fetches page 0. That means:

- Debounced search throws away pages already loaded for the previous query.
- Re-open dropdown / project change / index notification re-downloads rows we may still have cached.
- Full **`items_changed(0, old, new)`** replaces the whole model — list view flickers and rebuilds bindings unnecessarily.

**🔷** Redesign **`libocfiles/V2/ProjectFiles.refresh`** so it does not always clear-and-reload:

- Same **`query`**, list still valid → skip RPC or patch rows only.
- **`query`** changed → reset pagination state; still prefer minimal **`items_changed`** ranges where possible.
- Index change (new file, delete, approval) → row-level add/remove/update, not full list nuke.
- **`load_more`** keeps append-only tail behaviour.

Wire (**`fetch_files`**) and daemon **`cached_search`** stay; fix is client list lifecycle only.

### B-13 — ValidateLink async RPC

**Problem:** V2 **`ValidateLink.file()`** must call **`fetch_file`** / **`contains_folder`** when **`file_cache`** misses.

**Mainline (done):** **`ValidateLink`** — async signatures only; **logic unchanged**; callers **`yield validate_all`**.

**V2 (Meson flip):** **`V2/Task/ValidateLink.vala`** — replace **`file()`** logic with RPC; flip with **`libocfiles`** V2 API.
