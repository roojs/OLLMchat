## Procedure (рџ”·)

1. **Copy** вЂ” duplicate the shipping file into `libocfiles/V2/` (same class name, namespace `OLLMfiles`).
2. **Transport**
  - `RpcClient.vala`, `RpcClientBoot.vala`, `Copyable.vala` stay at **repo root**
  - **рџљ«** not under `V2/`
  - shared infrastructure, not parallel replacements
  - future V2 Meson list **includes** them
  - **рџљ«** no `V2/` copy step
3. **Commit** вЂ” land the verbatim copy on its own commit (user-driven).
4. **Reshape** вЂ” one file per change set
  - RPC-backed list models
  - **рџљ«** no in-process scan/sandbox
  - match [`done/2.10.4.9.1`](done/2.10.4.9.1-DONE-design-project-files-and-client-api.md)
  - **plus** client `*Files` copies below
  - extends вЂњno client `project_files`вЂќ вЂ” copies become thin RPC facades, not daemon index
5. **Callers**
  - duplicate into **`<lib>/V2/`** (or **`liboctools/*V2/`** for tools)
  - same relative path under the copy dir
  - same class name and namespace
  - reshape there for review
  - **рџљ«** do not edit shipping caller sources until Meson flip
  - same rule as **`libocfiles/V2/`**
6. **Meson cutover (later)** вЂ” **вЏі Not started**
  - flag toggles each library's source list (`ocfiles_src`, `occoder_src`, `octools_src`, вЂ¦)
  - shipping paths в†’ **`V2/*`**
  - spawn `ollmfilesd` from app startup

**Per-file status (рџ”·):**

- **Copy** вЂ” verbatim duplicate from shipping (then reshape on the copy)
- **Add** вЂ” рџ†• net-new file; no shipping source; skip copy entirely
- **Reshape** вЂ” RPC / API edits on a copied file (shipping stays frozen)

Legend (use only what applies):

- copy вњ”пёЏ / copy рџ“‹ вЂ” duplicate from shipping
- add вњ”пёЏ / add рџ“‹ вЂ” рџ†• net-new file in tree / still owed

**New fields (B-10 вЂ” add to `FolderParams`; only `fetch_files` reads them)**

- **`offset`** вЂ” skip rows; default **`0`**.
- **`limit`** вЂ” page size; default **`50`** (one screen of dropdown rows).
- **`query`** вЂ” dropdown filter string; default **`""`** (browse all).
  - **рџљ«** not semantic search вЂ” that stays on **`VectorParams.query`** в†’ **`Codebase.search`**.

Other **`Folder.*`** handlers ignore **`offset`**, **`limit`**, **`query`** if present.

**Request (client в†’ daemon)**

- Method: **`Folder.fetch_files`** (renamed from **`fetch_file_list`** in 9.3.1 relay).
- Params: **`path`** = active project root; **`offset`**, **`limit`**, **`query`** as above.
- **рџљ«** do not send **`project_path`** for this call вЂ” same as today.

**Response вЂ” today vs B-10**

| | Today (shipped relay) | B-10 |
|---|----------------------|------|
| **`result`** | every file in project | one page of **`File[]`** metadata only (**рџљ«** no content) |
| **`msg`** | unused on success | total count as decimal string |
| pagination | none вЂ” full dump | client stops when **`offset + page.size >= int.parse(msg)`** |

- **`msg`** meaning: project file count when **`query`** empty; match count when filtering.
- **рџљ«** no **`has_more`** flag вЂ” derive from total vs **`offset`**.

**Browse order (server default вЂ” only consumer is dropdown)**

1. Recent (**`last_viewed`** within 24h) first
2. Recent: **`last_viewed`** desc
3. Tie-break: **`path`** asc

**Search (`query` non-empty):** basename/path filter + wildcards (shipping rules); match-rank sort; **`msg`** = number of matches (not project total).

### B-12 вЂ” `ProjectFiles.refresh` redesign

**Problem (current interim code):** every **`refresh(query)`** wipes the in-memory list and re-fetches page 0. That means:

- Debounced search throws away pages already loaded for the previous query.
- Re-open dropdown / project change / index notification re-downloads rows we may still have cached.
- Full **`items_changed(0, old, new)`** replaces the whole model вЂ” list view flickers and rebuilds bindings unnecessarily.

**рџ”·** Redesign **`libocfiles/V2/ProjectFiles.refresh`** so it does not always clear-and-reload:

- Same **`query`**, list still valid в†’ skip RPC or patch rows only.
- **`query`** changed в†’ reset pagination state; still prefer minimal **`items_changed`** ranges where possible.
- Index change (new file, delete, approval) в†’ row-level add/remove/update, not full list nuke.
- **`load_more`** keeps append-only tail behaviour.

Wire (**`fetch_files`**) and daemon **`cached_search`** stay; fix is client list lifecycle only.
