## Procedure (рЯФЈ)

1. **Copy** вАФ duplicate the shipping file into `libocfiles/V2/` (same class name, namespace `OLLMfiles`).
2. **Transport** вАФ `RpcClient.vala`, `RpcClientBoot.vala`, `Copyable.vala` stay at **repo root** (not under `V2/`). They are shared infrastructure, not parallel replacements. The future V2 Meson list **includes** them; there is **no** `V2/` copy step.
3. **Commit** вАФ land the verbatim copy on its own commit (user-driven).
4. **Reshape** вАФ one file per change set: RPC-backed list models, no in-process scan/sandbox, match [`done/2.10.4.9.1`](done/2.10.4.9.1-DONE-design-project-files-and-client-api.md) **plus** client `*Files` copies below (extends the вАЬno client `project_files`вАЭ bullet вАФ copies become thin RPC facades, not daemon index).
5. **Callers** вАФ duplicate into **`<lib>/V2/`** (or **`liboctools/*V2/`** for tools) вАФ same relative path under the copy dir, same class name and namespace; reshape there for review. **рЯЪЂ** do not edit shipping caller sources until Meson flip вАФ same rule as **`libocfiles/V2/`**.
6. **Meson cutover (later)** вАФ flag toggles each library's source list (`ocfiles_src`, `occoder_src`, `octools_src`, вА¶) from shipping paths to **`V2/*`**; spawn `ollmfilesd` from app startup. **вП≥ Not started.**
