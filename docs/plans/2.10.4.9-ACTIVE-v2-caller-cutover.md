## Procedure (🔷)

1. **Copy** — duplicate the shipping file into `libocfiles/V2/` (same class name, namespace `OLLMfiles`).
2. **Transport** — `RpcClient.vala`, `RpcClientBoot.vala`, `Copyable.vala` stay at **repo root** (not under `V2/`). They are shared infrastructure, not parallel replacements. The future V2 Meson list **includes** them; there is **no** `V2/` copy step.
3. **Commit** — land the verbatim copy on its own commit (user-driven).
4. **Reshape** — one file per change set: RPC-backed list models, no in-process scan/sandbox, match [`done/2.10.4.9.1`](done/2.10.4.9.1-DONE-design-project-files-and-client-api.md) **plus** client `*Files` copies below (extends the “no client `project_files`” bullet — copies become thin RPC facades, not daemon index).
5. **Callers** — modules outside `libocfiles` are edited **in place** (no `V2/` subfolder) **except bwrap tools** — see **Bwrap tool V2 copies** below.
6. **Meson cutover (later)** — flag toggles `ocfiles_src` between shipping list and V2 list; spawn `ollmfilesd` from app startup. **⏳ Not started.**
