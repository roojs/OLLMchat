# 1.3.11 — Config pane field layout (description under control)

> **Do not update `docs/plans/1.0-summary.md` for this plan.**

**Status:** in progress

**Pointer:** `docs/guide-to-writing-plans.md` — **Checklist for plans**; proposed Vala follows **`docs/coding-standards.md`**

---

## Purpose

- **🔷** Settings / config rows use **`Adw.ActionRow`**: label + subtitle left, control right.
- **🔷** Subtitle (help text) should sit **under the control on the right**, not under the label on the left.
- **✅** Phase 1 done — **`ConnectionAdd`** bootstrap / add-connection dialog. See **`docs/plans/done/1.3.11.1-DONE-connection-add-field-layout.md`**
- **🔷** `⏳` Phase 2 — **`AddModelDialog`** (connection, model, size rows)
- **🔷** `⏳` Phase 3 — **`AndroidSettingsDialog`** models page (connection + model rows)
- **🔷** `⏳` Phase 4 — settings **`Rows/*`** (tool / model option rows from **`pspec` blurbs)
- **ℹ️** **`ModelUsage`** expander subtitle and **`ConnectionRow`** inner rows (no blurbs today) — out of scope unless review shows the same squeeze

---

## Pattern (copy from Phase 1)

- Drop **`subtitle`** on the **`Adw.ActionRow`**.
- Build a vertical suffix box (**`halign = END`**, spacing **4**).
- Append the control, then a **`Gtk.Label`** for the description.
- Label: **`wrap = true`**, **`css_classes = {"dim-label"}`**, **`xalign = 1.0`**, **`max_width_chars = 45`**.
- **🚫** No new private helper methods — inline at each call site (**`docs/coding-standards.md`** — method names and new methods).

## Phase 2 — `AddModelDialog`

**File:** `ollmapp/SettingsDialog/AddModelDialog.vala`  
**Where:** **`AddModelDialog()`** constructor — connection, model, and size rows.

### 1. `AddModelDialog.vala` — connection row

**Why:** Same left-stacked subtitle as bootstrap dialog.

**Where:** **`AddModelDialog()`**, block that builds **`connection_row`**.

#### Remove

```vala
			this.connection_row = new Adw.ActionRow() {
				title = "OpenAPI / Ollama server",
				subtitle = "Select the server connection to use"
			};
			this.connection_row.add_suffix(this.connection_dropdown);
```

#### Replace with

```vala
			var connection_suffix = new Gtk.Box(Gtk.Orientation.VERTICAL, 4) {
				halign = Gtk.Align.END
			};
			connection_suffix.append(this.connection_dropdown);
			connection_suffix.append(new Gtk.Label(
				"Select the server connection to use"
			) {
				wrap = true,
				wrap_mode = Pango.WrapMode.WORD,
				xalign = 1.0f,
				justify = Gtk.Justification.RIGHT,
				css_classes = {"dim-label"},
				max_width_chars = 45
			});
			this.connection_row = new Adw.ActionRow() {
				title = "OpenAPI / Ollama server"
			};
			this.connection_row.add_suffix(connection_suffix);
```

### 7. Each `Rows/*` subclass — wrap suffix control(s)

**Where:** **`setup_widget()`** in each row class, immediately before **`this.add_suffix(...)`**.

**Depends on:** §6.

Apply the Phase 1 suffix-box pattern to the widget (or **`button_box`**) passed to **`add_suffix`**. Description text: **`this.pspec.get_blurb()`** — skip the label when blurb is empty.

**Files:**

- `ollmapp/SettingsDialog/Rows/String.vala` — **`entry`**
- `ollmapp/SettingsDialog/Rows/Bool.vala` — switch widget
- `ollmapp/SettingsDialog/Rows/Connection.vala` — dropdown
- `ollmapp/SettingsDialog/Rows/Model.vala` — dropdown / pulldown
- `ollmapp/SettingsDialog/Rows/IntRow.vala` — **`button_box`** (Auto + spin + clear)
- `ollmapp/SettingsDialog/Rows/FloatRow.vala` — **`button_box`**

**Example — `String.vala` `setup_widget()` suffix only:**

#### Remove

```vala
			this.add_suffix(this.entry);
```

#### Replace with

```vala
			var suffix = new Gtk.Box(Gtk.Orientation.VERTICAL, 4) {
				halign = Gtk.Align.END
			};
			suffix.append(this.entry);
			var blurb = this.pspec.get_blurb();
			if (blurb != null && blurb.strip() != "") {
				suffix.append(new Gtk.Label(blurb) {
					wrap = true,
					wrap_mode = Pango.WrapMode.WORD,
					xalign = 1.0f,
					justify = Gtk.Justification.RIGHT,
					css_classes = {"dim-label"},
					max_width_chars = 45
				});
			}
			this.add_suffix(suffix);
```

Repeat the same shape for the other §7 files (swap the appended control widget).
