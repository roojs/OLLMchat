## How calls work

- Transport: one JSON object per line on `~/.local/share/ollmchat/ollmfilesd.sock` (NDJSON).
- Protocol: JSON-RPC 2.0. Responses may be out of order; use `id`.
- **Object-based:** every call names a target with `params.object` and uses `method` = **`{kind}.{verb}`** (e.g. `project.activate`, `file.read`).
- **Identity:** `path` is the id (absolute path). Same as today’s `path_map` / `child_map`. No opaque handles in v1.
- **Payload:** rows are **`filebase`** objects — same fields as `OLLMfiles.FileBase` / the DB. Indexed code elements use **`vector_metadata`** rows (`OLLMfiles.SQT.VectorMetadata` / `vector_metadata` table). Client hydrates thin handles; daemon holds FAISS + full graphs.
- **Vector layer:** semantic index/search lives in **`libocvector`** (`Indexer`, `Database`, `Search.Search`, `VectorMetadata`). Daemon embeds that stack; RPC exposes a **`vector`** object (scoped by project path), not only `project.index` / a minimal search stub.
- **Reads vs writes:** `*.get`, `*.read`, `*.snapshot` = query (no scan/index side effects). Other verbs = action (may queue work, write disk, bump `revision`).
- **Push:** server sends notifications as JSON-RPC messages with `method` under `event.*` (no `id`).

**Connect:**

```json
→ {"jsonrpc":"2.0","id":1,"method":"daemon.hello","params":{"protocol":1,"client":"ollmchat"}}
← {"jsonrpc":"2.0","id":1,"result":{"protocol":1,"server":"ollmfilesd","ready":true}}
```

**Typical `object` values:**

```json
{"kind":"daemon"}
{"kind":"projects"}
{"kind":"project","path":"/home/alan/proj/foo"}
{"kind":"file","path":"/home/alan/proj/foo/src/Ap.php"}
{"kind":"path","path":"/home/alan/proj/foo/src"}
```

**`filebase` row** (all file/folder results use this shape; omit GObject refs like `parent`, `manager`, `buffer`):

```json
{
  "id": 1001,
  "path": "/home/alan/proj/foo/src/Ap.php",
  "parent_id": 42,
  "base_type": "f",
  "last_modified": 1717000000,
  "last_vector_scan": 1717000100,
  "is_project": false,
  "is_text": true,
  "delete_id": 0
}
```

**`vector_metadata` row** (one indexed element — `OLLMfiles.SQT.VectorMetadata`; used in search results and `file.vector.list`):

```json
{
  "vector_id": 501,
  "file_id": 1001,
  "start_line": 10,
  "end_line": 42,
  "element_type": "method",
  "element_name": "read_async",
  "description": "Reads file content from disk into buffer",
  "ast_path": "OLLMfiles.File-read_async"
}
```

**UI memory:** client does not load the whole `Folder.children` tree — use `project.snapshot` for a flat list (dropdown/search) and materialize `File` instances only for open tabs + paths tools touch. See parent plan.

**Multi-window (not v1):** later, `client_id` on params and per-client active project; design so optional fields can be added without breaking protocol 1.

## `daemon` — service

### `daemon.hello`

First message after connect.

```json
→ {"jsonrpc":"2.0","id":1,"method":"daemon.hello","params":{"protocol":1,"client":"ollmchat"}}
← {"jsonrpc":"2.0","id":1,"result":{"protocol":1,"server":"ollmfilesd","ready":true}}
```

### `daemon.get`

Service health, global active project/file, per-project scan + index queues.

```json
→ {"jsonrpc":"2.0","id":2,"method":"daemon.get","params":{"object":{"kind":"daemon"}}}
← {"jsonrpc":"2.0","id":2,"result":{"uptime_s":3600,"active":{"project":"/home/alan/proj/foo","file":"/home/alan/proj/foo/src/Ap.php"},"projects":{"/home/alan/proj/foo":{"revision":42,"filesystem":{"scanning":true,"current_path":"/home/alan/proj/foo/src"},"index":{"paused":false,"queue_size":12,"current_file":"/home/alan/proj/foo/src/Ap.php"}}}}}
```

### `daemon.migrate`

First-run / missing DB (`ProjectMigrate` in app today).

```json
→ {"jsonrpc":"2.0","id":3,"method":"daemon.migrate","params":{"object":{"kind":"daemon"}}}
← {"jsonrpc":"2.0","id":3,"result":{"migrated":true,"projects":3}}
```

### `daemon.shutdown`

```json
→ {"jsonrpc":"2.0","id":4,"method":"daemon.shutdown","params":{"object":{"kind":"daemon"},"force":false}}
← {"jsonrpc":"2.0","id":4,"result":{"ok":true}}
```

## `project` — one codebase root (`Folder` with `is_project`)

### `project.activate`

Set active project; optionally queue filesystem scan (`read_dir`). `skip_scan: true` = today’s `disable_initial_scan`.

```json
→ {"jsonrpc":"2.0","id":20,"method":"project.activate","params":{"object":{"kind":"project","path":"/home/alan/proj/foo"},"skip_scan":false}}
← {"jsonrpc":"2.0","id":20,"result":{"revision":42,"scan_queued":true}}
```

### `project.get`

Revision and flags for one project (subset of `daemon.get`).

```json
→ {"jsonrpc":"2.0","id":21,"method":"project.get","params":{"object":{"kind":"project","path":"/home/alan/proj/foo"}}}
← {"jsonrpc":"2.0","id":21,"result":{"revision":42,"filesystem":{"scanning":false},"index":{"paused":false,"queue_size":0}}}
```

### `project.snapshot`

Flat project file list for UI — **not** a nested directory tree.

```json
→ {"jsonrpc":"2.0","id":22,"method":"project.snapshot","params":{"object":{"kind":"project","path":"/home/alan/proj/foo"},"since_revision":41}}
← {"jsonrpc":"2.0","id":22,"result":{"revision":43,"unchanged":false,"filebase":[{"id":1001,"path":"/home/alan/proj/foo/src/Ap.php","base_type":"f","last_modified":1717000000}]}}
```

```json
→ {"jsonrpc":"2.0","id":23,"method":"project.snapshot","params":{"object":{"kind":"project","path":"/home/alan/proj/foo"},"since_revision":43}}
← {"jsonrpc":"2.0","id":23,"result":{"revision":43,"unchanged":true}}
```

### `project.scan`

Full filesystem rescan for this project.

```json
→ {"jsonrpc":"2.0","id":24,"method":"project.scan","params":{"object":{"kind":"project","path":"/home/alan/proj/foo"}}}
← {"jsonrpc":"2.0","id":24,"result":{"queued":true}}
```

### `project.index`

Queue semantic indexing (`BackgroundScan` → `Indexer.index_filebase`). Optional: `force`, `project_summary_only` (CLI `--project-summary`).

```json
→ {"jsonrpc":"2.0","id":25,"method":"project.index","params":{"object":{"kind":"project","path":"/home/alan/proj/foo"},"force":false}}
← {"jsonrpc":"2.0","id":25,"result":{"queued":true,"files_enqueued":130}}
```

### `project.index.pause` / `project.index.resume`

Vector scan banner (`BackgroundScan` pause).

```json
→ {"jsonrpc":"2.0","id":26,"method":"project.index.pause","params":{"object":{"kind":"project","path":"/home/alan/proj/foo"}}}
← {"jsonrpc":"2.0","id":26,"result":{"paused":true}}
```

## `vector` — semantic index + search (`libocvector`)

Scoped to a project: `object` = `{"kind":"vector","project":"/home/alan/proj/foo"}`.

Daemon owns `OLLMvector.Database` (FAISS file), `vector_metadata` SQL, embed/analyze Ollama clients (`CodebaseSearchToolConfig`), and `Indexing.Indexer` / `Search.Search`. UI/tool code calls here instead of opening FAISS or `files.sqlite` for vector tables.

### `vector.status`

Startup health: dimension, vector count, models available (`Database.check_required_models_available`, `init_databases`).

```json
→ {"jsonrpc":"2.0","id":70,"method":"vector.status","params":{"object":{"kind":"vector","project":"/home/alan/proj/foo"}}}
← {"jsonrpc":"2.0","id":70,"result":{"ready":true,"dimension":1024,"vector_count":18432,"embed_model":"bge-m3","analyze_model":"qwen3-coder:30b"}}
```

### `vector.search`

Full semantic search — maps `OLLMvector.Search.Search.execute` + `RequestCodebaseSearch` (not a toy hit list). Returns `vector_metadata` + snippets / tool-formatted text.

```json
→ {"jsonrpc":"2.0","id":71,"method":"vector.search","params":{"object":{"kind":"vector","project":"/home/alan/proj/foo"},"query":"where is file reading implemented","max_results":10,"language":"","element_type":"method","category":"","only_file":"","format":"tool"}}
```

```json
← {"jsonrpc":"2.0","id":71,"result":{"results":[{"vector_id":501,"distance":0.42,"metadata":{"file_id":1001,"element_type":"method","element_name":"read_async","start_line":10,"end_line":42},"file_path":"/home/alan/proj/foo/libocfiles/File.vala","snippet":"..."}],"formatted":"..."}}
```

<!-- `format`: `tool` = citation string for LLM tool; `json` = structured only (CLI --json) -->

### `vector.list`

All `vector_metadata` rows for one file (`oc-vector-search --show-info`).

```json
→ {"jsonrpc":"2.0","id":72,"method":"vector.list","params":{"object":{"kind":"vector","project":"/home/alan/proj/foo"},"file_path":"/home/alan/proj/foo/libocfiles/File.vala"}}
← {"jsonrpc":"2.0","id":72,"result":{"items":[{"vector_id":501,"element_type":"method","element_name":"read_async","start_line":10,"end_line":42}]}}
```

### `vector.get`

Reconstruct one embedding or dump floats (`oc-vector-search --dump-vector` / debug).

```json
→ {"jsonrpc":"2.0","id":73,"method":"vector.get","params":{"object":{"kind":"vector","project":"/home/alan/proj/foo"},"ast_path":"OLLMfiles.File-read_async"}}
← {"jsonrpc":"2.0","id":73,"result":{"vector_id":501,"dimension":1024,"values":[0.01,-0.02]}}
```

<!-- `values` optional or separate `vector.dump` if payload huge -->

### `vector.project.describe`

Project-level summary from `ProjectAnalysis` / `vector_metadata` on project root (`Folder` project description, `--project-summary`).

```json
→ {"jsonrpc":"2.0","id":74,"method":"vector.project.describe","params":{"object":{"kind":"vector","project":"/home/alan/proj/foo"}}}
← {"jsonrpc":"2.0","id":74,"result":{"description":"...","dependencies":"..."}}
```

### `vector.reset`

Wipe FAISS + vector metadata (`Indexer.reset_database` / CLI `--reset-database`).

```json
→ {"jsonrpc":"2.0","id":75,"method":"vector.reset","params":{"object":{"kind":"vector","project":"/home/alan/proj/foo"},"confirm":true}}
← {"jsonrpc":"2.0","id":75,"result":{"ok":true}}
```

**Daemon-internal (no extra RPC if covered above):** `VectorBuilder`, `Analysis.analyze_tree`, `ImageAnalyzer`, `FolderAnalysis`, FAISS `save_index` during `file.index` queue; `VectorMetadata.cleanup_all_deleted` on delete.

## `file` — one file (`File` / `filebase` row)

### `file.get`

Resolve path to `filebase` (`get_file_from_active_project`).

```json
→ {"jsonrpc":"2.0","id":40,"method":"file.get","params":{"object":{"kind":"file","path":"/home/alan/proj/foo/src/Ap.php"}}}
← {"jsonrpc":"2.0","id":40,"result":{"found":true,"filebase":{"id":1001,"path":"/home/alan/proj/foo/src/Ap.php","base_type":"f"}}}
```

### `file.read`

Load disk content into client buffer.

```json
→ {"jsonrpc":"2.0","id":41,"method":"file.read","params":{"object":{"kind":"file","path":"/home/alan/proj/foo/src/Ap.php"}}}
← {"jsonrpc":"2.0","id":41,"result":{"filebase":{"id":1001,"base_type":"f","last_modified":1717000000},"encoding":"utf-8","content":"<?php\n..."}}
```

### `file.write`

Save from editor — daemon writes disk + DB + queues scan/index. No separate “file saved” RPC.

```json
→ {"jsonrpc":"2.0","id":42,"method":"file.write","params":{"object":{"kind":"file","path":"/home/alan/proj/foo/src/Ap.php"},"content":"<?php\n// edited\n"}}
← {"jsonrpc":"2.0","id":42,"result":{"filebase":{"id":1001,"last_modified":1717001000},"revision":45}}
```

### `file.metadata.set`

Cursor / scroll — no reindex.

```json
→ {"jsonrpc":"2.0","id":43,"method":"file.metadata.set","params":{"object":{"kind":"file","path":"/home/alan/proj/foo/src/Ap.php"},"cursor_line":10,"cursor_offset":4,"scroll_position":1200}}
← {"jsonrpc":"2.0","id":43,"result":{"ok":true}}
```

### `file.changed.check`

External change vs buffer (`check_active_file_changed` / banner).

```json
→ {"jsonrpc":"2.0","id":44,"method":"file.changed.check","params":{"object":{"kind":"file","path":"/home/alan/proj/foo/src/Ap.php"},"buffer_dirty":true,"last_known_mtime":1717000000}}
← {"jsonrpc":"2.0","id":44,"result":{"status":"changed_on_disk"}}
```

## Errors

```json
← {"jsonrpc":"2.0","id":4,"error":{"code":-32001,"message":"project not found","data":{"path":"/missing"}}}
```

Codes: `-32600` bad request, `-32601` unknown method, `-32602` bad params, `-32001` not found, `-32002` busy, `-32003` io error.

## Deferred (not v1)

- `file.read.chunk` — large files
- `vector.get` full float dump streaming — huge payloads
- `path.scan.overlay` — agent sandbox `Sandbox.Scan`
- `file.parse.ast` — `tree_factory` / ResolveLink (or keep local tree-sitter on `file.read` content)
- `project.files.recent` — recent dropdown sort
- `session.restore` — one-shot session open
- `file.git.blame` — interactive git UI
- `client_id` / `window_id` — multi-window active project
- Per-row `filebase` deltas in `project.snapshot` (only `since_revision` unchanged flag in v1)

## Client-only (no RPC)

- Editor `File.buffer` and `buffer_provider`
- Gtk signals re-emitted locally after successful RPC
- Chat / session / history
- Fake `File` (id &lt; 0) until `file.register` succeeds
