## `PhaseEnum` → stage column (`to_human()`) — reference

| `PhaseEnum` | Pango label (bold active where shown) | Where set |
|-------------|----------------------------------------|-----------|
| `NONE` | *(blank)* | queued / not running |
| `TOOLS_RUNNING` | **Running Tool** | `Tool.run` — `tool_impl.execute` |
| `EXECUTION` | **Review output** | `Tool.run` — after tool / preload / executor LLM (incl. post-LLM validate/write slice; **no** separate `EXEC_VALIDATE` **status** on `Tool`) |
| `EXECUTION_RETRY` | **Retry Review output** | `Tool.run` — retry loop `try_count > 0` |
| `EXEC_WRITE` | **Writing Files** | `Tool.run` — before `foreach` over `writes` (`write_file` apply) |
| `POST_EXEC` | **Post review** | **`Details.run_post_exec`** only (multi-run synthesis — not in `Tool.run`) |
| `COMPLETED` | grey **✓** | pass finished |

Wire / replay still uses `exec_validate` in transcripts; **`to_human(EXEC_VALIDATE)`** is blank (not shown in UI). Do not assign **`EXEC_VALIDATE`** to **`Tool.status`** for progress.

## Class + `ProgressItem`

```vala
	public class Tool : OLLMchat.Agent.Base, ProgressItem
```

```vala
	private PhaseEnum status_value = PhaseEnum.NONE;

	public PhaseEnum status {
		get { return this.status_value; }
		set {
			if (this.status_value == value) {
				return;
			}
			this.status_value = value;
			this.notify_property("status_str");
			// Do not notify_property("status") — by design; bind status-str only.
		}
	}

	public string title {
		owned get {
			return this.name != "" ? this.name : "Tool";
		}
	}

	public string status_str {
		owned get { return this.status.to_human(); }
	}

	public GLib.ListModel children { get; default = new GLib.ListStore(typeof(ProgressItem)); }
```
