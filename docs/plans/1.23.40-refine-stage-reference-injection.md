## Coding standards checklist

Copied from **`.cursor/rules/CODING_STANDARDS.md`** — “Checklist for all plans”. If a bullet conflicts with **§ Conventions** in this document, **the standards file wins**.

- **Nullable types**: Design new APIs/properties to avoid nullable types where possible (default objects / flags).
- **Null checks**: Avoid generic null checks; only where the design explicitly requires null (see standards for the narrow exceptions).
- **String interpolation**: No `@"..."` except multi-line usage/help/documentation; use `+` concatenation otherwise.
- **Temporary variables**: No one-use temporaries; no trivial aliases (allowed exception: long property chains **4+** steps).
- **Brace placement**: Line breaks before `{` for namespace/class/method; **inline** `{` for `if`/`for`/`while`/`switch`; never a one-line `if` with body on the same line.
- **`this.` prefix**: Use for instance member access in new/modified Vala.
- **GLib prefix & `using`**: Fully qualify `GLib.*`; do not add `using` for new code.
- **Property initialization**: Prefer `get; set; default = …` (or field defaults), not constructor assignment for those defaults.
- **Line length & breaking**: Break long method calls, concatenations, docblocks, and comments (docblock/comment line length per full standards).
- **StringBuilder**: Avoid `GLib.StringBuilder` unless building strings in loops with **hundreds** of iterations; use `+` and `string.joinv()`.
- **String building in loops**: Never accumulate with `+=` inside a loop; use `string.nfill()` / `replace()`, `string.joinv()`, etc.
- **ArrayList for strings**: Do not use `Gee.ArrayList<string>` only to join; use `string[]` with **`string[] name = {}`** only for empty init (not `new string[0]`).
- **Loops and nesting**: In loops, use **`continue`** to skip cases and keep the body flat; avoid **`else`** and nested **`if`** inside loops.
- **Character looping**: Do not iterate characters unless there is truly no alternative; prefer string methods and `GLib.Regex`.
- **File info try/catch**: Wrap `query_info()`-style calls only when file existence is unknown — not right after a successful read of the same path.
- **Try/catch scope**: Wrap only the minimal expression(s) that throw; keep setup outside `try`.
- **Underscore prefix**: No leading `_` on variables, fields, or properties.
- **`get_*` methods**: Prefer properties or verb-less/action names over `get_foo()` (see standards for exceptions).
- **Docblocks**: Multiline `/** … */` with `@param` / `@return` where relevant; not one-liners for real behavior.
- **Debug / warning messages**: No timestamps or monotonic time in `GLib.debug()` / `GLib.warning()`; no class/method prefixes in the message (file/line come from the log).
- **Gee.HashMap**: Use `.set()` / `.get()` / `.has_key()` / `.unset()` — not `[]`.
- **Gee.ArrayList**: Use `.get()` / `.set()` / `.add()` / `.remove_at()` — not `[]`.
- **Reducing nesting**: Prefer early `return` / `continue`; avoid `else` where flow stays clear; prefer `switch` over long `if` / `else if` chains when editing.
- **SQL table aliases**: Not applicable to this plan’s fences; if you add SQL elsewhere, no table aliases (full names only).
- **Plan Vala fences**: Do **not** use comment-only placeholders (`/* … */` or empty `return "";`) where real logic belongs — either **complete, compilable Vala** in the fence or **prose-only** steps (no fake stub block).

### Conventions (read first)

**§ Coding standards checklist** (above) applies to **every** new or edited Vala fragment in this plan. **§ Helpers** and the bullets below are **plan-specific** add-ons; they must not contradict **`CODING_STANDARDS.md`**.

**Lists / maps — `.clear()` only when justified**

- **Do not** call **`.clear()`** on **`Gee.ArrayList`**, **`Gee.HashMap`**, or similar **by habit** or because you **assume** something might be left inside. If the collection is **empty**, **newly constructed**, or you are about to **replace** it with a **new** instance or a **fresh** value from another API (so no stale elements remain), **do not** add **`.clear()`** “to be safe.” **Decide:** did a **prior** step **actually** put data in there that **must** be discarded before the next step? If **no**, skip. If **yes**, **`.clear()`** or **assign `new Gee.ArrayList<T>()`** (or rebuild the map) — whichever matches how that field is owned elsewhere in the class.
- **`Details.refine()`** retry loop: **`this.tools.clear()`** / **`this.code_blocks.clear()`** at the **start** of each attempt remain correct because a **failed** parse can **leave** tool calls and fenced blocks in those lists that **must not** leak into the next iteration — that is **required** clearing, not ceremony.
- **`fill_task_data`** / reference link lists: assign **only** from **`this.task_data.get(key).links()`** (new lists from the block) — **no** leading **`.clear()`**; assignment **replaces** the reference — **Phase 1** **`Replace (Details.fill_task_data())`** fence.
- **`Definition.tools`** is a special case (property default; **`load()`** fills an empty new **`Definition`** — see below).
- **`chat_call.tools`** is the LLM tool registry, not task-owned **`Gee.ArrayList`** fields; clear only when the **send/API contract** requires it — do not conflate with task **`tools`** / **`code_blocks`**.

```vala
// List.vala — full replacement of to_key_map() (signatures + behaviour contract)
public Gee.Map<string, Block> to_key_map(string key_mode = "")
{
	var result = new Gee.HashMap<string, Block>();
	foreach (var node in this.children) {
		var item = node as ListItem;
		if (item == null) {
			continue;
		}
		var v = new Block(FormatType.PARAGRAPH);
		var k = item.key_value(v).strip();
		if (k == "") {
			continue;
		}
		k = key_mode == GLib.CharacterSet.a_2_z ? k.down() : k;
		result.set(k, v);
	}
	return result;
}
```

```vala
// Details.vala — full replacement of fill_task_data()
/**
 * Ensure expected task_data keys exist (placeholder block if missing), then refresh
 * fields derived from task_data (e.g. references, shared_references, exam_references).
 * Called from the ctor and update_props after task_data changes — the name alone
 * does not spell out that dual role.
 */
private void fill_task_data()
{
	this.requires_user_approval = this.task_data.has_key("requires user approval");
	var empty = new Markdown.Document.Block(Markdown.FormatType.PARAGRAPH);
	string[] keys_to_fill = {
		"what is needed",
		"skill",
		"references",
		"expected output",
		"shared references",
		"examination references" };
	foreach (var key in keys_to_fill) {
		if (!this.task_data.has_key(key)) {
			this.task_data.set(key, empty);
		}
		switch (key) {
			case "references":
				this.references = this.task_data.get(key).links();
				break;
			case "shared references":
				this.shared_references = this.task_data.get(key).links();
				break;
			case "examination references":
				this.exam_references = this.task_data.get(key).links();
				break;
			default:
				break;
		}
	}
}
```

{

```vala
				continue;
			}
			task.code_blocks.add(block);

		found_list = true;

		}
```

```vala
{
```

```vala
// ResultParser.exec_extract(Tool ex) — shape: branch, then one write path
public bool exec_extract(Tool ex)
{
	if (!this.document.headings.has_key("result-summary")) {
		this.issues += "\nThis task's executor output must include a \"Result summary\" section.";
		return false;
	}
	ex.summary = this.document.headings.get("result-summary");
	ex.document = this.document;
	if (!ex.use_write_executor_template) {
		return true;
	}
	ex.synthesized_write_arguments = null;
	if (!this.document.headings.has_key("change-details")) {
		this.issues += "\nWrite executor output must include ## Change details.";
		return false;
	}
	var cd = this.document.headings.get("change-details");
	var doc = this.document;
	var idx = -1;
	for (var i = 0; i < doc.children.size; i++) {
		if (doc.children.get(i) == cd) {
			idx = i;
			break;
		}
	}
	if (idx < 0 || idx + 1 >= doc.children.size) {
		this.issues += "\nChange details: expected a list immediately after the heading.";
		return false;
	}
	var list_node = doc.children.get(idx + 1);
	if (!(list_node is Markdown.Document.List)) {
		this.issues += "\nChange details: first block after heading must be a bullet list.";
		return false;
	}
	var kv = ((Markdown.Document.List) list_node).to_key_map(GLib.CharacterSet.a_2_z);
	var mode = kv.has_key("output_mode")
		? kv.get("output_mode").to_markdown().strip().down() : "";
	string content = "";
	if (mode == "fenced") {
		for (var j = idx + 2; j < doc.children.size; j++) {
			var n = doc.children.get(j);
			if (n is Markdown.Document.Block) {
				var b = (Markdown.Document.Block) n;
				if (b.kind == Markdown.FormatType.FENCED_CODE_QUOTE
						|| b.kind == Markdown.FormatType.FENCED_CODE_TILD) {
					content = b.code_text;
					break;
				}
			}
		}
		if (content == "") {
			this.issues += "\nWrite output: expected a fenced code block after Change details.";
			return false;
		}
	} else if (mode == "next_section") {
		string[] chunks = {};
		for (var j = idx + 2; j < doc.children.size; j++) {
			chunks += doc.children.get(j).to_markdown();
		}
		content = string.joinv("\n\n", chunks).strip();
	} else {
		this.issues += "\nChange details: output_mode must be fenced or next_section.";
		return false;
	}
	var args = new Json.Object();
	if (kv.has_key("file_path")) {
		args.set_string_member("file_path", kv.get("file_path").to_markdown().strip());
	}
	if (kv.has_key("ast_path")) {
		args.set_string_member("ast_path", kv.get("ast_path").to_markdown().strip());
	}
	if (kv.has_key("location")) {
		args.set_string_member("location", kv.get("location").to_markdown().strip());
	}
	if (kv.has_key("start_line")) {
		args.set_int_member("start_line",
			int.parse(kv.get("start_line").to_markdown().strip()));
	}
	if (kv.has_key("end_line")) {
		args.set_int_member("end_line",
			int.parse(kv.get("end_line").to_markdown().strip()));
	}
	if (kv.has_key("complete_file")) {
		var s = kv.get("complete_file").to_markdown().strip().down();
		args.set_boolean_member("complete_file", (s == "true" || s == "1"));
	}
	if (kv.has_key("overwrite")) {
		var s = kv.get("overwrite").to_markdown().strip().down();
		args.set_boolean_member("overwrite", (s == "true" || s == "1"));
	}
	args.set_string_member("content", content);
	ex.synthesized_write_arguments = args;
	return true;
}
```
