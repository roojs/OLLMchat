## Templates (4 wrappers, `resources/skill-prompts/`)

Skill body from `resources/skills/*.md` still merged in; only the wrapper file changes.

| # | File | When to load |
| - | ---- | ------------ |
| 1 | `task_refinement.md` | Skill has **≥1 tool** after **write-class tools** are stripped from the skill’s tool list. |
| 2 | `task_refinement_references.md` *(new)* | **Else** (no remaining tools — write-only or no tools). |
| 3 | `task_execution.md` | **Standard** executor — tool runs (results + body). |
| 4 | `task_execution_write.md` *(new)* | **Write** runs — apply / file output / structured change text. |

**Write-class tool list:** one definition in code; those tools **do not** push the task onto row **1**.

**Exec row 3 vs 4:** pick **4** when the run is write/apply output; **3** otherwise (independent of which refine ran).

**Post-execution (`task_post_exec.md`):** run **only** when **more than one** execution run was produced for the task (e.g. multiple tool runs or one run per examination reference). **Skip** when there was exactly **one** run. On the **write** path (template **4**), post-exec is **not** used; **Result summary** is the completion signal.

Task creation / task list iteration wrappers unchanged by this plan.

### Summary

| # | Focus |
| - | ----- |
| 1 | **Tool refinement** — **## Tool Calls** + whether a **shared reference** is needed. |
| 2 | **Reference refinement** — same **`## Task`** list as `task_refinement.md`, plus **two** extra keys only: **Shared references**, **Examination references**. Same link format as **References** (flat line: **Key** then one or more `[Title](/absolute/path)` links — see `task_refinement.md` / **Reference link types**). **No nested lists.** Omit any key that would have no links. |
| 3 | **Tool execution** — **Result summary** + **body** (current standard). |
| 4 | **Write execution** — **Result summary** matches `docs/skills-format.md` (prose + in-doc `#` links, not a key-value list). **Change details** = **Key** value lines + **`output_mode`** (`next_section` | `fenced`) + `write_file`-aligned keys (**§ Phase 7**). |

**Reference refinement → runner (two paths only):**

- **Examination references** present (≥1 link) → **split**: one run **per examination link**. For each run the runner **does not** pass the whole examination list into the executor — it **maps** that run’s precursor so **References** resolves to **exactly one** of those links (the current slice). **Shared references** (if any) are the same shared precursor for every slice; the executor never sees sibling examination links in that run.
- **Examination references** omitted or empty → **single** run; precursor from **References** only (may list multiple links on one line — normal standard).
- Omit **References** in refinement output when every link lives on **Examination** / **Shared** only; the runner still fills **References** per run when it builds each slice (synthetic one-link **References** for split runs).
- **Expected output** (refined **## Task**) must match what the executor can actually do: on split paths, say **per-run** scope (e.g. summation / findings for **this** file only), not “analyze all examination files” in one shot — that text will be copied into templates as the good example.
