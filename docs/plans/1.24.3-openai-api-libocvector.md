# Plan 1.24.3: OpenAI API — Chat (all libraries)

## Overview

Phase 2 integration of OpenAI-compatible **chat** for **all libraries** that use it. Parent plan: **1.24** (OpenAI API chat and embed compatibility). We use the v1 endpoint (`/v1/chat/completions`) and either map responses into `Response.Chat` (adapter) or switch to v1 types. This plan covers every chat usage across the codebase.

## Scope

**Method:** Chat only.

**Libraries:** libollmchat (Client, Agent, History, Tools), libollmchatgtk (ChatWidget, ChatView), libocvector (VectorBase), liboctools (EditMode, Child/Factory, ReadFile/Tool), examples (oc-test-cli), ollmapp (ollmchat-cli).

**Refer to plan 1.24 for:** rationale, API differences, request/response shapes, adapter vs single-type options.

## Code that would need changing (chat only — all libraries)

| Library | File(s) | Current usage | Change for v1 |
|---------|---------|----------------|---------------|
| **libollmchat** | `Client.vala` | `chat_execute(Call.Chat)`, `chat()`: build `Call.Chat`, return `Response.Chat`. | Add v1 path: build `Call.ChatCompletions`, call `exec()`/`exec_stream()`, map to `Response.Chat` (adapter) or return v1 types. |
| **libollmchat** | `Agent/Base.vala`, `Interface.vala`, `Factory.vala` | `chat_call = new Call.Chat`, `chat()`, `configure_tools(Call.Chat)`, signals `stream_chunk(Response.Chat)`, `message_completed(Response.Chat)`. | When v1: construct `Call.ChatCompletions`, stream via `exec_stream()`, feed adapter into existing signals or extend for v1 types. |
| **libollmchat** | `History/Session.vala`, `SessionBase.vala`, `Manager.vala` | `handle_stream_chunk(Response.Chat)`, `finalize_streaming(Response.Chat)`, signals. | Receive adapter or v1 type; same flow. |
| **libollmchat** | `Tool/RequestBase.vala`, `BaseTool.vala`, `ChatContentInterface.vala` | `on_stream_chunk(Response.Chat)`, `on_message_completed(Response.Chat)`, `chat()`. | Accept adapter or common type. |
| **libollmchatgtk** | `ChatWidget.vala`, `ChatView.vala` | `on_stream_chunk_handler(Response.Chat)`, `append_assistant_chunk`, `finalize_assistant_message`, `process_new_chunk`. | Consume adapter or v1 type. |
| **libocvector** | `VectorBase.vala` | `new Call.Chat`, `chat.send()`, `Response.Chat`. | Use `Call.ChatCompletions` + exec/exec_stream when v1; map response. |
| **liboctools** | `EditMode/Stream.vala`, `Request.vala`, `Child/Factory.vala`, `ReadFile/Tool.vala` | `Response.Chat`, `on_stream_chunk(Response.Chat)`, `configure_tools(Call.Chat)`, `Call.Chat chat_call`. | Accept adapter or common type. |
| **examples** | `oc-test-cli.vala` | `new Call.Chat`, `write_stats(Response.Chat)`. | v1 path; `write_stats` takes adapter or both types. |
| **ollmapp** | `ollmchat-cli.vala` | `write_stats(Response.Chat)`. | Same as examples: adapter or overload. |

## Notes

- Client is the main entry point for chat; add v1 path there so all libraries can use it.
- Option A: keep `Response.Chat` in signals/APIs and adapt v1 → `Response.Chat`. Option B: switch all to v1 types. See plan 1.24 (and former 1.24.1 background) for adapter vs cutover.
- Embed is in plan 1.24.1; generate is in plan 1.24.2.
