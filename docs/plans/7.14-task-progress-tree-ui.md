## Purpose (shared)

- **`Runner`** owns **`this.progress`** (**`ProgressList`**). Updates use **`clear_pending()`** / **`add_pending()`** / **`add_completed()`**; **`rebuild()`** only when a full resync is required (**`7.14.5`**). **`ProgressList`** ctor takes **`Runner`**.
- **`ProgressView`**: **`Gtk.ColumnView`** + scroll — **no new `Gtk.TreeView`**.
- **`ProgressList`** holds **`Details`** rows only (completed + pending segments). **Task-list create** / **iterate** top rows are **not** entries in **`ProgressList.items`** — state on **`Runner`** (or **`7.14.2`** UI model). **`Tool`** rows live under **`Details`** via **`ToolList`**.
- **`ProgressItem`:** **`status`** (**`PhaseEnum`**), **`title`**, **`status_str`** (**`PhaseEnum.to_human()`** — **`7.14.1-phase1-core`**; column binds **`status-str`** so Gtk updates when **`status`** changes). Same enum as markdown / wire / validation where values overlap — progress-only values (**`TOOLS_RUNNING`**, **`COMPLETED`**, …) live in **`7.14.1-phase1-core`**.

## Stage column — UI contract (normative)

**Typography**

- **Active** work: **bold**, **plain English**, **≤ 2 words** (e.g. “Creating”, “Retrying”).
- **Idle / complete:** **not bold**.
- **Not started:** **blank** in the stage column.
- **Done** (where a check fits): **grey Unicode tick** (e.g. `✓` with muted foreground) — use for **tool** row completion line in the table below; other row kinds use **empty** when finished as noted.

**Task list creation (top-level row)**

| State | Stage column |
|-------|----------------|
| In progress | **Creating** |
| Retry loop | **Retrying** |
| Finished | **empty** |

**`Details` row**

| State | Stage column |
|-------|----------------|
| (typical) | **Refine** → … → **empty** when idle for that slice |
| Retry | **Retrying** |
| Optional merge | **Post review** (if we merge that concept) |

Use **`Refine`**, **`Retrying`**, **`Post review`** as short labels consistent with **`PhaseEnum.to_human()`** output (**`7.14.1-phase1-core`**).

**`Tool` row**

| State | Stage column |
|-------|----------------|
| Running | **Executing** |
| Review | **Review output** |
| Retry | **Retrying** |
| Done | grey **✓** (tick) |
| Not started | **blank** |

Exact markup strings finalized in **`7.14.1-phase1-core`** (**`PhaseEnum.to_human()`**); must stay **≤ 2 words** where marked bold-active.

## `PhaseEnum` and **`ProgressItem.status_str`**

- **`ProgressItem.status`** (**`PhaseEnum`**) is the row’s progress slice; **`ProgressItem.status_str`** = **`status.to_human()`** (**`7.14.1-phase1-core`**) — **Pango** markup for **inactive / active / done** (see enum doc). Bind **`status-str`** so the stage column updates when **`status`** changes.
- **`is_active`** drives **bold** vs normal for the active slice; **`NONE`** → blank stage (**`to_human()`** returns empty).
- Task-list **create** / **iterate** top rows (**7.14.2**) use **`Runner`** state, not necessarily **`ProgressList.items`**; same typography rules apply when those rows are shown.
