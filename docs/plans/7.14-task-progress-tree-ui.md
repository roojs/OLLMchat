## Purpose (shared)

- **`Runner`** owns **`this.progress`** (**`ProgressList`**). Updates go through **`replacePending()`**, **`addComplete()`**, etc. **`ProgressList`** ctor takes **`Runner`**.
- **`ProgressView`**: **`Gtk.ColumnView`** + scroll — **no new `Gtk.TreeView`**.
- **`Runner` adapter:** **`Runner`** appears in **multiple row shapes** (create root, iterate child, …). Use **adapters** (small types owned by **`ProgressList`**) so the tree model does not special-case **`Runner`** everywhere — see **`7.14.1`**.
- **`ProgressItem`:** shaped for **what the column renderers need** — mostly **bindable** properties (getters/setters + **`notify`**), not a fat domain dump.

## Stage column — UI contract (normative)

**Typography**

- **Active** work: **bold**, **plain English**, **≤ 2 words** (e.g. “Creating”, “Retrying”).
- **Idle / complete:** **not bold**.
- **Not started:** **blank** in the stage column.
- **Done** (where a check fits): **grey Unicode tick** (e.g. `✓` with muted foreground) — use for **tool** row completion line in the table below; other row kinds use **empty** when finished as noted.

**Task list creation (top-level row)**

| State | Stage column |
|-------|----------------|
| In progress | **Creating** |
| Retry loop | **Retrying** |
| Finished | **empty** |

**`Details` row**

| State | Stage column |
|-------|----------------|
| (typical) | **Refine** → … → **empty** when idle for that slice |
| Retry | **Retrying** |
| Optional merge | **Post review** (if we merge that concept) |

Use **`Refine`**, **`Retrying`**, **`Post review`** as short labels consistent with **`to_human()`** output.

**`Tool` row**

| State | Stage column |
|-------|----------------|
| Running | **Executing** |
| Review | **Review output** |
| Retry | **Retrying** |
| Done | grey **✓** (tick) |
| Not started | **blank** |

Exact strings finalized in **`7.14.3`**; must stay **≤ 2 words** where marked bold-active.

## `PhaseEnum` and display

- Prefer storing **logical status** as **`PhaseEnum`** (extend the enum if gaps appear — e.g. creation / iteration-specific values).
- Add **`to_human()`** (or rename existing human string helper if one exists) for **stable** short labels used in the stage column and tooltips.
- Renderers map **enum + row kind + active flag** → bold vs empty vs tick.

## Visualization hierarchy & collapse

(Same as before — see **`7.14.1`** for hooks.)

- **Task list create** — top-level.
- **Task list iterate** — child of **`Details`** (after post-exec).
- **`Details`** / **`Tool`** — as already specified in phase docs.
- **Collapse tools** when advancing to next **`Details`** (exec) or when **iteration** completes.

## Height & placement

- **Height** = **`ProgressView`** / **`ScrolledWindow`** height (show **all tools** for active detail + that detail + **two prior `Details`** with tools **collapsed** — not row filtering).
- **Placement:** **`ProgressView`** **between** main **chat** and **text entry** (above input).
