# 8.0 вЂ” libollmchat local GGUF backend

**Status:** proposed

**Pointer:** `.cursor/rules/CODING_STANDARDS.md` вЂ” Checklist for all plans В· **Layout:** `docs/guide-to-writing-plans.md`

---

## Purpose

- **рџ”·** Add a Vala-facing binding to a local inference library that is easily available on Debian and Ubuntu.
- **рџ”·** The local library must load GGUF models.
- **рџ”·** The local library must support text inference and embeddings.
- **рџ”·** Implement this as an alternative backend for `libollmchat`.
  - **рџ”·** End-user code should keep using the standard `libollmchat` API.
  - **рџ”·** `libocvector` should not need a one-off local-vectorizer path.
- **рџ”·** Replace vectorization so documents can be embedded without Ollama or OpenAI.
- **рџ”·** Add code to download the BGE embedding model into `~/.local/share/ollmchat/models/`.

---

## Backend target

- **рџ”·** The backend must use a distro-available inference library.
- **рџ’©** Use `llama.cpp` / `libllama` as the first target.
  - **вЏі** Debian and Ubuntu package it as `llama.cpp`, with development files in `libllama-dev`.
  - **вЏі** It supports GGUF model loading.
  - **вЏі** It supports embedding mode through the `libllama` C API.
- **рџ’©** Create a local VAPI for the subset of `llama.h` used by OLLMchat.
  - **вЏі** Keep the binding narrow.
  - **вЏі** Include only model load, context creation, tokenization, decode, sampling, embedding extraction, and cleanup.
- **рџ’©** Add a small C wrapper if direct Vala binding to structs or callbacks becomes brittle.
  - **вЏі** Prefer stable Vala-facing functions over exposing every `llama.cpp` detail.

## libollmchat API shape

- **рџ”·** Callers keep using existing `libollmchat` APIs.
  - **вЏі** `Client.embed()` and `Client.embed_array()` return `Response.Embed`.
  - **вЏі** `Call.Embeddings.exec_embedding()` returns `Response.Embed`.
  - **вЏі** `Client.generate()` returns `Response.Generate`.
  - **вЏі** `Client.chat_execute()` continues to accept `Call.ChatBase`.
- **рџ”·** **`Settings.Connection` keeps existing fields** вЂ” see [`8.0.2`](8.0.2-backend-abstraction-in-libollmchat.md).
  - **рџ”·** **`url` starting with `/`** вЂ” local model directory (Unix).
  - **рџ”·** **`url` matching `^[A-Za-z]:[/\\]`** вЂ” local model directory (Windows drives **AвЂ“Z**).
  - **вЏі** **`url` otherwise** вЂ” HTTP / Ollama / OpenAI-compatible (current behaviour).
- **рџ”·** Connection factory methods + **`CallLocal.*`** overrides вЂ” designed in Phase 2; implemented Phase 3+ (see subplan).
  - **вЏі** e.g. `connection.ChatCompletions(model)`, `connection.Embeddings(model)`.
  - **вЏі** No new Connection helper methods.
  - **в„№пёЏ** Phase 1 `Local/` probes temporary; remove when absorbed into `CallLocal`.
- **рџљ«** Separate **`Backend.*`** layer вЂ” rejected; see [`8.0.2`](8.0.2-backend-abstraction-in-libollmchat.md).

## Model layout

- **рџ”·** Download BGE into `~/.local/share/ollmchat/models/`.
- **рџ’©** Store GGUF weights and metadata side by side under that directory.
  - **вЏі** `~/.local/share/ollmchat/models/bge-m3/`
    - **вЏі** `model.gguf`
    - **вЏі** `manifest.json`
    - **вЏі** `sha256`
  - **вЏі** Keep existing `Response.Model.get_cache_path()` JSON cache behavior for remote model metadata.
- **рџ’©** Use a local model manifest rather than hardcoding one filename throughout the codebase.
  - **вЏі** Alias: `bge-m3:local`.
  - **вЏі** Family: `bge`.
  - **вЏі** Capabilities: `embedding`.
  - **вЏі** Dimension: detected from the first embedding and stored after verification.
  - **вЏі** Source URL and checksum: reviewed before implementation.
- **рџ’©** Treat a changed embedding model or dimension as an index invalidation event.
  - **вЏі** Existing FAISS indexes should be rebuilt when the local BGE model changes dimension.

## Vectorization cutover

- **рџ”·** `libocvector` should use the standard `libollmchat` API for local embeddings.
- **рџ’©** Keep `libocvector` calling `Call.Embeddings`.
  - **вЏі** The configured `codebase_search.embed.connection` points to a local GGUF connection.
  - **вЏі** `Database.embed_dimension()` still probes by running a test embedding.
  - **вЏі** `VectorBuilder` and `DocumentationVectorBuilder` still receive `Response.Embed`.
  - **вЏі** `Search.execute()` still embeds the query through the same call path.
- **рџ’©** Remove Ollama-specific temp-model customization from local embedding calls.
  - **вЏі** `Response.Model.customize()` remains for HTTP / Ollama.
  - **вЏі** Local BGE uses model manifest options instead.
- **рџ’©** Decide normalization deliberately.
  - **вЏі** `Call.Embed` normalizes embeddings today.
  - **вЏі** `Call.Embeddings` currently does not normalize.
  - **вЏі** Local backend should match the vector index metric chosen for FAISS.

## Local inference behavior

- **рџ”·** The backend must support loading a GGUF and running inference.
- **рџ”·** The backend must support embeddings.
- **рџ’©** Deliver embeddings first.
  - **вЏі** This unblocks document vectorization without Ollama or OpenAI.
  - **вЏі** It is the lowest-risk path because responses map cleanly to `Response.Embed`.
- **рџ’©** Add non-streaming generate next.
  - **вЏі** Map `Client.generate()` / `Call.Generate` onto local decode.
  - **вЏі** Return `Response.Generate`.
- **рџ’©** Add chat after generate.
  - **вЏі** Start with non-streaming `Call.ChatCompletions` or `Call.Chat`.
  - **вЏі** Reuse `Response.Chat` and existing message conversion.
  - **вЏі** Tool calling remains HTTP-only until JSON/tool-call formatting is proven locally.
- **рџ’©** Streaming local chat is a later phase.
  - **вЏі** It needs token callbacks to emit `Call.ChatBase.stream_chunk`.

## Download and model management

- **рџ”·** Add code to download the BGE model into `~/.local/share/ollmchat/models/`.
- **рџ’©** Add a local model manager in `libollmchat`.

  - **вЏі** Local `models()` returns installed local manifests.
  - **вЏі** Missing default BGE may appear as downloadable in the settings UI.
- **рџ’©** Wire settings UI after the backend works.
  - **вЏі** Add a connection type for local GGUF.
  - **вЏі** Show installed BGE state.
  - **вЏі** Offer a download action for the default BGE model.

## Phased delivery

### Phase 1 вЂ” dependency spike and VAPI вњ…

**Subplan:** [`8.0.2-backend-abstraction-in-libollmchat.md`](8.0.2-backend-abstraction-in-libollmchat.md)

- **вњ…** Factories on Connection (`Models()`, `Embeddings()`, вЂ¦) + `CallLocal.*` overrides вЂ” documented for Phase 3+.
- **рџљ«** No `backend` enum, no `model_dir` field, no skeleton classes in Phase 2.

### Phase 3 вЂ” local models list + embeddings

- **рџ”·** `вЏі` **`CallLocal.Models`** first вЂ” list GGUF models under `connection.url`; runnable test.
- **рџ”·** `вЏі` **`CallLocal.Embeddings`** next вЂ” local BGE without Ollama or OpenAI; runnable test.
- **рџ’©** `вЏі` Return `Response.Embed` from local calls.
- **рџ’©** `вЏі` Preserve dimension probing.
- **рџ’©** `вЏі` Add vector dump comparison tests for repeatability.
- **вЏі** Connection factory methods and `is_local()` helper land with first `CallLocal.*` class, not before.

### Phase 4 вЂ” BGE download

- **рџ”·** `вЏі` Download BGE into `~/.local/share/ollmchat/models/`.
- **рџ’©** `вЏі` Add manifest, checksum verification, and partial-download handling.
- **рџ’©** `вЏі` Expose install state through `Client.models()` and settings UI.

### Phase 5 вЂ” libocvector cutover

- **рџ”·** `вЏі` Configure codebase/document vectorization to use the local backend.
- **рџ’©** `вЏі` Keep `libocvector` call sites on `Call.Embeddings`.
- **рџ’©** `вЏі` Rebuild or invalidate FAISS indexes when model identity or dimension changes.

### Phase 6 вЂ” local generate and chat

- **рџ”·** `вЏі` Support local GGUF text inference through the standard API.
- **рџ’©** `вЏі` Implement non-streaming `Client.generate()` first.
- **рџ’©** `вЏі` Add non-streaming chat after generate.
- **рџ’©** `вЏі` Treat streaming and tool calling as follow-up work unless explicitly promoted.
