# 1.24.7 — Gate temp models (`customize`) on Ollama-native API

> **Do not update `docs/plans/1.0-summary.md` for this plan.** Sub-plans stay off the summary until **done** (move to `done/` + rename). No in-progress listing in 1.0 — ever.

**Status:** proposed

**Parent:** `docs/plans/1.24-DONE-openai-api-chat-compatibility.md`

**Approval:** User direction 2026-05-29 (detection + UI hide). No code until explicit go-ahead.

## Purpose

- **`Model.customize()`** calls Ollama **`POST /api/create`** when **`num_ctx`** is set — wrong on OpenAI-only backends.
- **🔷** Set **`ollama_native`** once per connection (**`-1`** / **`0`** / **`1`**) — in memory for the session; serializes when config is saved elsewhere (not this plan’s job).
- **🔷** Skip **`Call.Create`** / temp-model naming when the connection is not Ollama-native.
- **🔷** Settings options UI: **hide** **`num_ctx`** on non-Ollama connections — **`visible = false`**, row stays in list and config.
- **🚫** URL-based Ollama detection — not part of this plan (was LLM draft noise).

**ℹ️** Replaces the old URL-only **`supports_ollama_native`** bool sketch in this plan’s first draft.

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| **`Connection.ollama_native`** stored int (**`-1`** / **`0`** / **`1`**) | Re-probing on every chat or settings page open |
| **`detect_ollama()`** — idempotent; call anytime (no-op when already set) | Removing **`Call.Create`** or **`Call.Version`** |
| Detection at **connection add** and **config load** (**`check_connections`**) | Dynamic re-hide when usage **connection dropdown** changes (verify only; not expected) |
| **`Model.customize()`** early exit when not Ollama-native | Disabling **`temperature`** / OpenAI-supported rows |
| Options UI: **`visible = false`** on **`num_ctx`** (**🔷**); optional other Ollama rows (**💩**) | Removing rows from **`rows`** list or config serialization |
| **`ollmchat-cli`** / **`oc-test-cli`** **`--ctx`** guard (**💩**) | New user-facing “Ollama mode” toggle |
| **🚫** URL-based Ollama detection | |

## 🔷 Stored flag (normative)

| **`ollama_native`** | Meaning |
| ------------------- | ------- |
| **`-1`** | Not checked yet (legacy configs, new connection before probe) |
| **`0`** | Not Ollama-native (**`customize()`** / temp models off; Ollama-only UI rows hidden) |
| **`1`** | Ollama-native (**`/api/create`**, **`num_ctx`** temp models, Ollama-only UI rows visible) |

**🔷** Use **`connection.ollama_native == 1`** (or **`!= 1`** to hide) — no wrapper property.

**🔷** **`detect_ollama()`** owns the “already set?” check — **`async void`**, no return value.

---

## 🔷 Detection (normative)

**Call **`yield connection.detect_ollama()`** whenever a connection is verified** — no caller guard. If **`ollama_native`** is already **`0`** or **`1`**, the method **`return`**s immediately (no HTTP).

**🚫** Detection is **not** based on connection URL shape — no **`api/`** suffix checks, no URL short-circuit before version.

| Step | Check | Result |
| ---- | ----- | ------ |
| **0 — Idempotent entry** | **`ollama_native != -1`** | **`return`** (no HTTP) |
| **1 — Version** | **`Call.Version.exec_version()`** (`GET /api/version`, Ollama-only) | Success → **`1`**; any **`Error`** → **`0`** |

**Where to call (always, after models succeed)**

| When | Call site |
| ---- | --------- |
| Bootstrap / **Add connection** | **`ConnectionAdd.test_and_save()`** |
| App / CLI load, settings reconnect | **`Config2.check_connections()`** |

**ℹ️** **`Call.Models`** remains the separate **reachability** check (existing behaviour). **`detect_ollama()`** uses **version only**.

**🚫** **`check_connections`** does **not** call **`Config2.save()`** for **`ollama_native`** — setup / user save paths handle persistence later.

---

## 🔷 Options UI — hide **`num_ctx`** (normative)

When **`connection != null`** and **`connection.ollama_native != 1`**, set **`row.visible = false`** on the **`num_ctx`** row. Row **stays** in **`Options.rows`** and in saved JSON.

**🔷** Use **`row.visible = false`** — **not** **`sensitive = false`**, **not** dropping rows from the container. Hidden rows do not need live updates after first render.

## Concrete code proposals

Intro: hunks are **Remove** / **Replace with** / **Add** from the tree; verify surrounding context before applying.

### 1. `libollmchat/Settings/Connection.vala` — stored flag + detection

**Why:** Single persisted source for Ollama-native capability.

**Where:** After **`is_working`** field; new async method before **`load_models()`**.

**Depends on:** none.

#### Add — after **`is_working`** field

Serializable tri-state only — use **`ollama_native == 1`** at call sites; no wrapper property.

```vala
		/**
		 * Ollama-native API support: -1 = not checked, 0 = no, 1 = yes.
		 * Set once by detect_ollama() while still -1.
		 */
		public int ollama_native { get; set; default = -1; }

```

#### Add — before **`load_models()`** method

Idempotent Ollama detector — no-op when already set.

```vala
		/**
		 * Ollama detector: GET /api/version when ollama_native is still -1.
		 * Safe to call anytime; no-op when already set.
		 */
		public async void detect_ollama()
		{
			if (this.ollama_native != -1) {
				return;
			}

			try {
				var version_call = new OLLMchat.Call.Version(this);
				yield version_call.exec_version();
				this.ollama_native = 1;
			} catch (Error e) {
				GLib.debug(
					"Connection %s: version probe failed (%s), not Ollama-native",
					this.url,
					e.message
				);
				this.ollama_native = 0;
			}
		}

```

**ℹ️** **`ollama_native`** uses default **`Json.Serializable`** serialization — no **`serialize_property`** case add.

### 2. `libollmchat/Settings/Config2.vala` — detect on load

**Why:** Legacy configs may have **`ollama_native == -1`**; resolve on startup check.

**Where:** **`check_connections()`** method.

**Depends on:** §1.

#### Remove

```vala
		public async void check_connections()
		{
			foreach (var entry in this.connections.entries) {
				var connection = entry.value;
				try {
					// Test connection by calling models endpoint directly with short timeout
					var original_timeout = connection.timeout;
					connection.timeout = 5;  // 5 seconds - connection check should be quick
					try {
						var models_call = new OLLMchat.Call.Models(connection);
						yield models_call.exec_models();
						connection.is_working = true;
					} finally {
						connection.timeout = original_timeout;
					}
				} catch (Error e) {
					connection.is_working = false;
					GLib.debug("Connection %s is not working: %s", connection.url, e.message);
				}
			}
		}
```

#### Replace with

```vala
		public async void check_connections()
		{
			foreach (var entry in this.connections.entries) {
				var connection = entry.value;
				// Test connection by calling models endpoint directly with short timeout
				var original_timeout = connection.timeout;
				connection.timeout = 5;  // 5 seconds - connection check should be quick
				try {
					var models_call = new OLLMchat.Call.Models(connection);
					yield models_call.exec_models();
					connection.is_working = true;
					yield connection.detect_ollama();
				} catch (Error e) {
					connection.is_working = false;
					GLib.debug("Connection %s is not working: %s", connection.url, e.message);
				} finally {
					connection.timeout = original_timeout;
				}
			}
		}
```

### 5. `ollmapp/SettingsDialog/Rows/Options.vala` — hide rows when not Ollama-native

**Why:** User must not configure **`num_ctx`** when **`customize()`** will ignore it; rows remain for storage.

**Where:** End of **`init_rows()`** after all rows added. **Do not** add a **`Connection`** ctor parameter — resolve from existing **`config`** + **`dialog.app.config.connections`**.

**Depends on:** §1, §6 (ModelRow must set **`connection`** on dummy **`ModelUsage`**).

**Design:** **`Options`** already receives **`dialog`** and **`config`**. When **`config`** is **`ModelUsage`**, **`connection`** is the URL key into **`Config2.connections`** (same as **`Rows.Connection`** dropdown). Lookup there; no extra parameter threading. Ctor and **`init_rows`** signature unchanged.

#### Keep

```vala
			this.rows.add(new Int(dialog, options, options_class.find_property("seed")) {
				min_value = -1.0,
				max_value = 2147483647.0,
				step_value = 1.0,
				digits = 0,
				default_value = -1.0
			});
		}
```

#### Add — immediately before that closing **`}`** of **`init_rows`**

**🔷** Hide **`num_ctx`** when not Ollama-native. **💩** Uncomment extra names if that section is approved.

```vala
			if (!(this.config is OLLMchat.Settings.ModelUsage)) {
				return;
			}
			var mu = (OLLMchat.Settings.ModelUsage) this.config;
			if (mu.connection == "" ||
					!dialog.app.config.connections.has_key(mu.connection)) {
				return;
			}
			var connection = dialog.app.config.connections.get(mu.connection);
			if (connection.ollama_native == 1) {
				return;
			}

			string[] hide_when_not_ollama = { "num_ctx" };
			// 💩 if approved, add: top_k, min_p, typical_p, repeat_last_n, repeat_penalty
			foreach (var row in this.rows) {
				var name = row.pspec.get_name();
				foreach (var hide_name in hide_when_not_ollama) {
					if (name == hide_name) {
						row.visible = false;
						break;
					}
				}
			}

```
