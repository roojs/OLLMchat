# 1.24.7 — Gate temp models (`customize`) on Ollama-native API

> **Do not update `docs/plans/1.0-summary.md` for this plan.** Sub-plans stay off the summary until **done** (move to `done/` + rename). No in-progress listing in 1.0 — ever.

**Status:** proposed

**Parent:** `docs/plans/1.24-DONE-openai-api-chat-compatibility.md`

**Depends on:** **`1.24.3.2`** ( **`ChatCompletions`** default chat path)

**Pointer:** `.cursor/rules/CODING_STANDARDS.md` — **Checklist for all plans**

**Approval:** No code until the user approves this plan.

---

## Purpose

- **`Model.customize()`** always calls Ollama **`POST /api/create`** when **`num_ctx`** is set.
- With **`ChatCompletions`** on OpenAI-only connections, that call is wrong — it fails and silently falls back today.
- **🔷** Detect Ollama-native API support before **`customize()`** / temp-model creation.
- **🔷** Disable **context size (`num_ctx`)** in settings UI when the connection cannot create temp models.

**ℹ️** **`1.24-DONE-openai-api-chat-compatibility.md`** mentions temp models in one table row only — no implementation task until this plan.

---

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| **`Connection`** capability flag (URL-based) | Runtime probe of **`/api/create`** on every chat |
| **`Model.customize()`** early exit when unsupported | Removing **`Call.Create`** |
| Settings UI: disable **`num_ctx`** row per connection | Disabling all Ollama-only option rows (**`top_k`**, etc.) — follow-up if wanted |
| **`ollmchat-cli`** / **`oc-test-cli`** **`--ctx`** guard | New user-facing connection config toggle |

---

## 🔷 Detection rule (normative)

**Ollama-native APIs** (including **`/api/create`** for temp models): connection URL normalised to end with **`api/`** (same convention as local Ollama: **`http://host:11434/api`**).

| **`supports_ollama_native`** | Temp model / **`customize()`** | **`num_ctx`** UI |
| ----------------------------- | ------------------------------ | ---------------- |
| **true** (`…/api/`) | Allowed when **`num_ctx != -1`** | Editable |
| **false** (e.g. **`https://api.openai.com/v1`**) | Skip — return **`this.name`** | Disabled (auto only) |

**🚫 Do not** call **`Call.Create`** when **`supports_ollama_native`** is false.

---

## Acceptance criteria

- [ ] **`Connection.supports_ollama_native`** property matches the URL rule above.
- [ ] **`Model.customize()`** returns **`this.name`** without **`Call.Create`** when unsupported (even if **`num_ctx`** is set).
- [ ] Models settings (**`ModelRow`**) and usage settings (**`Rows.ModelUsage`**) disable the **`num_ctx`** row when connection is not Ollama-native.
- [ ] **`ollmchat-cli`** / **`oc-test-cli`**: **`--ctx`** ignored (or warned once) on unsupported connections — no **`Call.Create`** attempt.
- [ ] Local Ollama connection: temp model creation still works.
- [ ] **`ninja -C build`** clean.

## Concrete code proposals

Intro: hunks are **Remove** / **Replace with** / **Add** from the tree; verify surrounding context before applying.

### 1. `libollmchat/Settings/Connection.vala` — `supports_ollama_native` property

**Why:** Single source of truth for “can this server do **`/api/create`**?”.

**Where:** After **`url`** property (or before **`init()`**).

**Depends on:** none.

#### Add — read-only property; true when URL ends with **`api/`** (normalise trailing **`/`** first).

```vala
		/**
		 * True when this connection targets Ollama-style native APIs (`/api/...`),
		 * including model create for temp `num_ctx` variants.
		 */
		public bool supports_ollama_native {
			get {
				var conn_url = this.url;
				if (conn_url == "") {
					return false;
				}
				if (!conn_url.has_suffix("/")) {
					conn_url += "/";
				}
				return conn_url.has_suffix("api/");
			}
		}

```

### 6. `ollmapp/ollmchat-cli.vala` — guard `--ctx` override

**Why:** CLI must not set **`num_ctx`** then rely on **`customize()`** on OpenAI connections.

**Where:** Both blocks that set **`model_usage.options.num_ctx = opt_ctx_num`** (search **`opt_ctx_num`**).

**Depends on:** §1.

#### Add — before assigning **`num_ctx`**, only when connection supports Ollama-native create.

Pattern (apply at each **`opt_ctx_num`** site):

```vala
			if (opt_ctx_num > 0) {
				if (this.manager.session.model_usage.connection != "" &&
						this.manager.config.connections.has_key(this.manager.session.model_usage.connection)) {
					var conn = this.manager.config.connections.get(this.manager.session.model_usage.connection);
					if (conn.supports_ollama_native) {
						if (this.manager.session.model_usage.options == null) {
							this.manager.session.model_usage.options = new OLLMchat.Call.Options();
						}
						this.manager.session.model_usage.options.num_ctx = opt_ctx_num;
					} else {
						GLib.warning("--ctx ignored: connection does not support Ollama temp models");
					}
				}
			}
```

*(Replace the existing unconditional **`num_ctx = opt_ctx_num`** blocks with this shape — exact **Keep**/**Remove** per site when implementing.)*
