## Wire naming (search only)

**Why not `vector.search`?**

- RPC convention maps **`{Object}.{method}`** → Vala class + **`rpc_{method}`** signal
- **`OLLMfilesd.Vector`** is already a **namespace** (`ollmfilesd/Vector/*` pipeline)
- **🚫** no top-level **`Vector`** RPC handler class

**Why `Codebase`?**

- matches **`CodebaseSearch`** tool naming
- **`OLLMfilesd.Vector`** stays the indexing namespace — **🚫** not an RPC class

**Agreed mapping (search only):**

| Layer | Name |
| ----- | ---- |
| Wire method | **`Codebase.search`** |
| Register prefix | **`"Codebase"`** |
| Vala handler class | **`Codebase`** (`OLLMfilesd.Codebase`) |
| Signal | **`rpc_search`** |
| Params | **`VectorParams`** — **unchanged** |

**Unchanged (not this tranche):**

- **`vector.list`**, **`vector.get`**, **`vector.reset`**
- **`Folder.project_description`** — ✔️ on existing **`Folder`** handler (not **`vector.*`**)
- **`VectorParams`** fields and type name
- **`ollmfilesd/Vector/*`** indexing pipeline namespace

## Wire contract

### Request

```json
{
  "method": "Codebase.search",
  "params": {
    "path": "/abs/project/root",
    "query": "authentication logic",
    "language": "",
    "element_type": "",
    "category": "",
    "max_results": 10,
    "format": "tool"
  }
}
```

- **`path`** — project root (client sends **`active_project.path`**)
- **`query`** — required on client; daemon may assume non-empty
- **`language`**, **`element_type`**, **`category`**, **`max_results`** — optional filters (same semantics as shipping)
- **`format`**
  - **`tool`** — **🔷** required for CodebaseSearch cutover
  - **`json`** — **🚫** not in this tranche

**Params on `VectorParams` not used by CodebaseSearch caller today:**

- **`only_file`**, **`file_path`**, **`ast_path`**
  - **🚫** do not wire special cases unless user promotes them to **🔷**

### Response

- **`format=tool`**
  - **`Response.msg`** = full markdown string for the LLM
  - same shape as shipping **`format_results`** / **`SearchResult.to_markdown`**
- Hard failures (project missing, DB unavailable, embed/FAISS error)
  - **`Response.error`** with message
  - client throws on **`response.error`**
- Empty-result cases (no files, no vectors after filter, zero FAISS hits)
  - **🔷** **`Response.msg`** with the same LLM-facing strings shipping returns
  - **🚫** not **`Response.error`** for “no matches”

## Current behaviour (shipping reference)

**Orchestration:** `libocvector/Tool/RequestCodebaseSearch.vala`

1. Resolve **`active_project`**
2. **`project_files.get_ids(language)`** → file id list
3. SQL filter on **`vector_metadata`** → **`filtered_vector_ids`**
   - function/method → **`IN ('function','method')`**
   - category → doc/section filter (same SQL as shipping)
4. **`OLLMvector.Search.Search.execute()`** → **`SearchResult`** list
5. Load file buffers → **`format_results`** → markdown

**FAISS:** `libocvector/Search/Search.vala` (uses **`OLLMvector.Database`**)

**Formatting:** `libocvector/Search/SearchResult.vala` → **`to_markdown()`**

**Daemon today:** has **`OLLMvector2.Database`** + **`OLLMvector2.Search`**
- returns **`SearchHit`** (FAISS + metadata)
- **🚫** no **`SearchResult`**, **🚫** no **`to_markdown`**

**Prior plan note:** [`done/2.10.4.11-DONE-libocvector2.md`](done/2.10.4.11-DONE-libocvector2.md) listed **`ollmfilesd/Vector/SearchResult.vala`** as a later snippet layer — **not copied yet** ([`done/2.10.4.12`](done/2.10.4.12-DONE-ollmfilesd-vector-implementation.md)).

## Proposed behaviour (daemon)

### Registration

- New class **`ollmfilesd/Codebase.vala`** — **`OLLMfilesd.Codebase`**
- Register in **`Application.vala`**
  - **`OLLMrpc.Request.register("Codebase", …, VectorParams)`**
  - same pattern as **`Folder`**, **`File`**
- Signal: **`rpc_search`** (from wire **`Codebase.search`**)

### Search steps (must match shipping)

1. **`project_manager.project_root(params.path)`**
   - error if missing
2. Ensure project file index loaded if needed
   - **`yield project.load_files_from_db()`** when **`project_files`** empty
   - **💩** exact load trigger — confirm vs shipping
3. **`project_files.get_ids(params.language)`**
   - empty → LLM message in **`msg`** (same strings as shipping)
4. SQL **`filtered_vector_ids`**
   - port from **`RequestCodebaseSearch`** verbatim
   - use **`OLLMvector2.SQT.VectorMetadata`**
5. **`OLLMvector2.Search`**
  - **`project_manager.vector_db`**
  - **`project_manager.db`**
  - **`app.config`**
  - **`filter_vector_ids`** from step 4
6. Format hits → markdown in **`Response.msg`**
   - **see Design fork**

### Snippet source on daemon

- **💩 Option A:** daemon **`File.buffer`** via **`buffer_provider`**
  - mirrors shipping **`get_code_snippet`**
- **💩 Option B:** read line range from disk
  - no buffer dependency
- **🔷** Pick one in review — default recommendation: **Option A**
  - daemon already has **`FileBuffer`** / **`buffer_provider`**
  - matches shipping citation lines

## Design fork (approve before coding)

**Where does search + format code live?**

| Option | What | Pros | Ris |
| ------ | ---- | ---- | --- |
| **A** | Copy/adapt **`SearchResult.vala`** → **`ollmfilesd/Vector/SearchResult.vala`** + **`Codebase`** relay with **`rpc_search`** | Matches [`2.10.4.11`](done/2.10.4.11-DONE-libocvector2.md) stub; formatting isolated | New file; adapt from **`libocvector`**, not **`libocvector2`** |
| **B** | All logic inline in **`Codebase`** **`rpc_search`** handler | Fewer files | Easy to drift from shipping; hard to review |
| **C** | Extend **`libocvector2`** with **`SearchResult`/`to_markdown`** | Shared library | Changes public API; **`ollmfilesd`-only concern today |

**Recommendation for review:** **Option A**

- **`ollmfilesd/Vector/SearchResult.vala`** — adapt from **`libocvector/Search/SearchResult.vala`**
  - **`file_id` → path** via **`project_files.get_by_id`**
  - snippets via daemon buffer (Option A above)
- **`ollmfilesd/Codebase.vala`** — **`rpc_search`** only
  - SQL filter block ported from **`RequestCodebaseSearch`**
  - delegates FAISS to **`OLLMvector2.Search`**
  - delegates markdown to **`SearchResult.to_markdown`**
- **🚫** do **not** link **`libocvector`** into **`ollmfilesd`**
- **🚫** do **not** put orchestration only in **`Application.vala`**
- **🚫** do **not** rename **`VectorParams`** or other **`vector.*`** wire names
