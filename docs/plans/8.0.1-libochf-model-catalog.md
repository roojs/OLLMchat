## Purpose

- **🔷** Local inference ([`8.0`](8.0-libollmchat-local-gguf-backend.md)) needs a reliable way to **find** and **fetch** GGUF weights.
  - **ℹ️** Ollama registry / `libollamaweb` discovery is useful for Ollama pulls but flaky for our local-GGUF path.
  - **ℹ️** Hugging Face Hub exposes **public JSON search** and **unauthenticated file resolve** for most open models.
- **🔷** Add a new standalone library **`libochf`** — Vala namespace **`OLLMhf`**, **no** dependency on `libollmchat` / `ollmapp`.
- **🔷** Hub **JSON metadata** via **`OLLMrpc.Client.call()`** + **`Bin.Json(AUTO)`** — not ad-hoc libsoup + **`Json.Parser`** in **`libochf`**.
  - **🔷** Response: parse JSON → **`json_to_bin(obj, bin, typeof(OLLMhf.Model))`** → **`bin.parse()`**.
- **🔷** GGUF **binary download** — **`libochf.Download.Service`** only (libsoup stream-to-disk); **not** `libocrpc`.
- **🔷** Mirror the **`libollamaweb` API surface** closely enough that UI and download code can swap catalog backends later.
  - **⏳** `OLLMhf.Search.Session` — debounced search coordinator (app owns debounce/cancel).
  - **⏳** `OLLMhf.Model` + `OLLMhf.ModelFile` — per-repo catalog on disk under a catalog directory.
  - **⏳** `OLLMhf.Search.Service.search()` — live Hub search.
  - **⏳** `OLLMhf.Search.Service.fetch_detail()` — repo metadata + GGUF sibling list.
  - **⏳** `OLLMhf.Download.Service` — stream one resolved file into `~/.local/share/ollmchat/models/` (feeds [`8.0` Phase 4](8.0-libollmchat-local-gguf-backend.md#phase-4--bge-download)).
- **🔷** `⏳` Phase 0 — land [`8.2.1`](8.2.1-libocrpc-auto-json-and-http-client.md) (blocks everything else here).
- **🔷** `⏳` Phase 1 — **`libochf`** + fixture tests (no app wiring).
- **🔷** `⏳` Phase 2 — BGE-m3 default download for [`8.0` Phase 4](8.0-libollmchat-local-gguf-backend.md#phase-4--bge-download).

## API investigation (2026-07-09)

Probed live against `https://huggingface.co` without auth. Hub speaks **JSON** over HTTP (no HTML scraping).

### Base URL and auth

- **ℹ️** Base: `https://huggingface.co`
- **ℹ️** OpenAPI spec: `https://huggingface.co/.well-known/openapi.md`
- **ℹ️** Hub docs: `https://github.com/huggingface/hub-docs/blob/main/docs/hub/api.md`
- **ℹ️** **No token required** for public model metadata and open-repo file resolve.
- **ℹ️** **Gated repos** (`"gated": true`) return **401** on resolve unless authenticated — out of scope for Phase 1–2 defaults.

### Rate limits (observed)

- **ℹ️** `GET /api/models…` — **500 requests / 300 s** window (unauthenticated).
- **ℹ️** `GET …/resolve/…` — **3000 requests / 300 s** window.

### Search — `GET /api/models`

- **ℹ️** Returns a **JSON array** (not `{ "models": … }`).
- **ℹ️** Pagination via **`Link` header** (`rel="next"`) + **`cursor=`**.
- **ℹ️** Key query params: `search`, `author`, `filter` (e.g. `gguf`), `sort`, `direction=-1`, `limit` (≤100), `full=true`.

**Example (verified):**

```text
GET /api/models?search=gguf&filter=gguf&limit=20&sort=downloads&direction=-1
→ 200 application/json, Link: …; rel="next"
```

### Model detail — `GET /api/models/{repo_id}`

- **ℹ️** `{repo_id}` = `author/name`.
- **ℹ️** **`siblings[]`**: `{ "rfilename", "size", "lfs" }` — filter `.gguf` for local inference.

### Download resolve — `GET /{repo_id}/resolve/{rev}/{path}`

- **ℹ️** Not under `/api/` — 302/307 to CDN; stream bytes to disk.
- **ℹ️** Headers: `Accept-Ranges`, `x-linked-size`, `ETag`, `x-repo-commit`.

### Local layout ([`8.0` § Model layout](8.0-libollmchat-local-gguf-backend.md#model-layout))

- **🔷** `~/.local/share/ollmchat/models/{alias}/` — `model.gguf`, `manifest.json`, `sha256`.
- **ℹ️** Probe candidate for BGE: **`groonga/bge-m3-Q4_K_M-GGUF`** / **`bge-m3-q4_k_m.gguf`** (~418 MB) — confirm before locking default.

## `libochf` library shape

Meson target **`libochf`** — depends on **`libocrpc`** (+ **gee**, **gio**). HTTP + JSON live in **`libocrpc`**, not in **`libochf`** sources.

### Namespace and wire types (`OLLMhf.*`)

- **🔷** Vala namespace: **`OLLMhf`** — library / Meson target: **`libochf`**.
- **⏳** Wire types implement **`OLLMrpc.Bin.Serializable`** + **`rpc_register()`**.
- **⏳** **`OLLMhf.Model`**
  - **⏳** `repo_id` — Hub `id` / `modelId`; canonical key (like `OllamaWeb.Model.slug`).
  - **⏳** `author`, `downloads`, `likes`, `tags`, `pipeline_tag`, `library_name`.
  - **⏳** `gated`, `private`, `disabled`, `refined`.
  - **⏳** `files` — `Gee.ArrayList<OLLMhf.ModelFile>`.
  - **⏳** `list_markup`, `display` — GTK row strings.
  - **⏳** `load` / `save` / `path` / `exists` under catalog dir.
- **⏳** **`OLLMhf.ModelFile`** (maps to `OllamaWeb.ModelVariant`)
  - **⏳** `filename`, `size_bytes`, `quant_label`.
  - **⏳** `resolve_path(revision)` → Hub `/{repo_id}/resolve/{rev}/{filename}`.

### Hub decode (per call)

- **⏳** Search **`GET /api/models`** — Hub returns top-level **`[…]`** (upstream of **`json_to_bin`**). Sort out in HTTP client / **`Service.search()`** when implementing — flag, wrap, or loop; TBD.
- **⏳** Detail **`GET /api/models/{repo}`** — object root → **`json_to_bin(obj, bin, typeof(OLLMhf.Model))`** → **`bin.parse()`**.

### Search stack (`OLLMhf.Search.*`)

Parallel API to `libollamaweb/Search/`; implementation uses **`libocrpc` HTTP client**:

- **⏳** **`Client.vala`** — HTTP-mode **`OLLMrpc.Client`**; **`Service`** builds **`Request.method`** (path + query via **`+`**).
- **⏳** **`Service.vala`** — `search()` (HF array — upstream, TBD); `fetch_detail()` (object → **`json_to_bin`**).
- **⏳** **`Session.vala`** — `model_dir`, `search()`, `refine()`, `rows_ready`, `cancel()`.

**Swap-friendly with `libollamaweb`:**

- **⏳** `Session.search(non_empty_query, category)` → `Gee.ArrayList<OLLMhf.Model>`.
- **⏳** `Session.refine()` — background detail for slugs missing `refined` / empty `files`.

### Download stack (`OLLMhf.Download.*`)

- **⏳** **`Download.Service.vala`** — resolve URL → stream GGUF into [`8.0` model layout](8.0-libollmchat-local-gguf-backend.md#model-layout).

### Tests (offline fixtures)

- **⏳** Meson suite **`ochf`** — no live network in CI.
- **⏳** Fixtures under `tests/data/ochf/` — recorded Hub JSON bodies.
- **⏳** **`new Bin.Json(AUTO)`** → **`json_to_bin(fixture_obj, bin, typeof(OLLMhf.Model))`** → **`bin.parse()`** → assert fields.
