# 7.14.14 — Task progress tree: collapse header (“Skill activity”)

**Status:** proposed

**Parent:** `docs/plans/done/7.14-DONE-task-progress-tree-ui.md`

**Pointer:** `.cursor/rules/CODING_STANDARDS.md` (**Checklist for all plans**), `docs/guide-to-writing-plans.md`

---

## Goals

- 🔷 **Collapse / expand** the task progress tree (`ProgressView` / `Gtk.ColumnView`).
- 🔷 **Collapsed state** shows a single header row only:
  - 🔷 **Expander control on the left** — same interaction as markdown **oc-frame** blocks (`go-next-symbolic` collapsed, `go-up-symbolic` expanded; see `libocmarkdowngtk/RenderSourceView.vala`).
  - 🔷 **One expander, then label text** — horizontal row: button, then **“Skill activity”** (plain label).
  - 🔷 **No coloration** on that header — no `oc-frame-*` theme classes, no Pango bold/markup on the title.
- 🔷 **Header sits above the grid** — not inside `ColumnView`.
- 🔷 **Remove the column title row** (`Title` / `Stage` / `Idx` headers) whenever the grid is visible; the collapse header replaces that chrome.
- 🔷 **Default collapsed** — tree hidden on first show (strip is a one-line “Skill activity” bar until expanded).
- 🔷 **Stays collapsed when data arrives** — binding `Runner.progress`, restore/replay rebuild, and live `ProgressList` updates must **not** open the revealer; only the header expander toggles visibility.

---

## Interaction / lifecycle

- 🔷 **`ProgressView()`** — `body_revealer.reveal_child = false`; expander `go-next-symbolic` / “Expand”.
- 🔷 **`Factory.activate` → `set_runner`** — new `SingleSelection` + `TreeListModel` on `runner.progress`; **re-apply collapsed** (revealer false, expander icon reset) so session switch / re-activation does not leave the grid open.
- 🔷 **While collapsed** — rows may still update in the model (restore, replay, live tasks); user sees **header only** until they expand.
- 🔷 **User expand/collapse** — toggle only via header button (same as markdown frames); no other code path sets `reveal_child`.
- 🚫 **Do not** auto-expand when `progress.get_n_items() > 0`, first row added, or `set_runner` completes.
- 🚫 **Do not** change `TreeListModel` auto-expand (`false, true` in ctor / `set_runner`) — that is row depth inside the grid, not strip visibility.

---

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| `liboccoder/Task/ProgressView.vala` — header row + `Gtk.Revealer` around existing `ScrolledWindow` / `ColumnView` | Row-level tree collapse (`TreeExpander` / `Details` / `Tool` behaviour) |
| `resources/style.css` — hide `columnview > header`; optional compact header spacing | Changing `ProgressList` data, `Runner`, replay, or session bind (**`7.14.5`**, **`7.14.7`**) |
| Preserve existing row factories, selection, keyboard nav, click → chat scroll; **288** px `min_content_height` when expanded | Revisiting strip height / content-driven sizing |

---

## Acceptance criteria

- Skills agent active: progress strip shows **“Skill activity”** + expander; **no** `ColumnView` column header row visible.
- **Default:** grid **hidden** (collapsed); one short header line only.
- After **`set_runner`** (including restore while Skills agent active): grid still **hidden**; expander still “Expand” / `go-next`.
- Restore / replay / live progress filling rows while collapsed: strip height stays **header-only** (no jump to 288 px) until user expands.
- Click expander: grid visible; click again: grid hidden.
- Expander icons/tooltips match markdown frames (`go-next` / Expand, `go-up` / Collapse).
- Row click / Up-Down selection / `scroll_to_message` unchanged when grid is expanded.
- No new public API on `ProgressView` beyond widget structure (no new methods unless user approves).

## Verification

- Activate **Skills** agent → one-line **Skill activity** strip, grid hidden.
- Restore a session with many progress rows **without** expanding → strip stays one line; expand manually → rows appear.
- Switch Skills session → `set_runner` → still collapsed (not left expanded from prior session).
- Expand → rows + stages visible; **no** column title labels.
- Collapse → grid gone; strip returns to one line.
- Click a progress row (expanded) → chat still scrolls to message.
