## Purpose

- **🔷** **`.gitignore` / `path_is_ignored`** belongs **only** in **`ollmfilesd`** — the sole production filesystem scanner.
- **🔷** Remove duplicate **`GitProvider`** (libgit2) from **`liboccoder`** and orphan copy under **`libocfiles`**.
- **🔷** Keep **`GitProviderBase`** stub in **`libocfiles`** / **`ollmfilesd`** for the interface; only **`ollmfilesd`** gets a real **`GitProvider`**.

---

## Is this a major change? Will it break production?

**No — not major; low production risk.**

| Area | Impact |
|------|--------|
| **`ollmapp`** | **None** — no `GitProvider` / `Ggit` usage in Vala today; `libgit2-glib` in `ollmapp/meson.build` is **leftover** (can drop in same tranche) |
| **`ollmfilesd`** | **Keep** — `ollmfilesd/GitProvider.vala` is the **canonical** implementation |
| **`libocfiles` in-process scan** | **Behaviour change only for direct callers** — `git_provider` stays **`GitProviderBase`** stub → no gitignore (correct: don't scan filesystem outside daemon) |
| **`examples/oc-test-files`** | **One line** — stop assigning `OLLMcoder.GitProvider`; test tool loses in-process gitignore (use daemon or accept stub) |
| **`libocvector` / `oc-vector-index`** | **None** — already RPC/daemon; namespace comment is stale doc only |

**Won't break:** desktop app, daemon RPC path, vector index via `ollmfilesd`.

**Won't fix by itself:** vector queue slowness on OLLMchat until **`ollmfilesd` GitProvider** is built and deployed (already in tree, unverified queue count post-fix).
