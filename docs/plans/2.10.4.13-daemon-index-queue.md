## Purpose

- **🔷** Wire semantic indexing in **`ollmfilesd`** — queue, drain, `Indexer.index_filebase`.
- **🔷** **`BackgroundScan` is not threaded in the daemon** — no UI to protect; single `ProjectManager`; async queue on default `MainLoop`.
- **🔷** **`scan_update` → RPC notification** — replaces in-process GObject signal; client banner at cutover via `RpcClient.notification`.
- **🚫** No `libocfiles` / `libocvector` edits.
- **🚫** No `vector.*` RPC **requests** (search, reset, …).
- **🚫** No client `VectorScanBanner` relay — cutover ([`2.10.4.9`](2.10.4.9-v2-wiring-and-cutover.md)).

## Step 1 — Simplify `Vector/BackgroundScan.vala` ✅

**🔷 Remove worker thread**

- drop `GLib.Thread`, `worker_context`, `worker_loop`, `main_context.invoke`
- async queue on daemon default `MainContext`

**🔷 Remove `worker_project_manager`**

- ctor takes daemon's single `OLLMfilesd.ProjectManager`

**🔷 Ctor**

- drop `OLLMvector.Tool.CodebaseSearchTool`
- take `OLLMvector2.Database`, `SQ.Database`, `OLLMfilesd.ProjectManager`, `GitProviderBase`, `Config2`
- drop `embedding_client` if unused

**🔷 Keep**

- file queue + sequential drain (`startQueue`)
- wait for fs scan (`pm.scanning.size > 0`)
- mtime vs `last_vector_scan` gate
- `Indexer.index_filebase`
- config change handling (simplify — no thread restart)

## Step 3 — Progress → RPC notification ⏳

**🔷 Remove** GObject `signal void scan_update`

**🔷 Emit** server→client notification at same points as today:

| When | `queue_size` | `current_file` |
| ---- | ------------ | -------------- |
| start of each file | remaining in queue | file path |
| queue empty | `0` | `""` |

**Wire:**

```json
{"jsonrpc":"2.0","method":"event.vector.scan_update","object-type":"Vector","params":{"queue_size":12,"current_file":"/path/to/file.vala"}}
```

**Delivery:** `StdioConnection.write(OLLMrpc.Notification)` — broadcast to connected clients

**💩 ⏳** extend `OLLMrpc.Notification` with `params` if needed (today: `method` + `object_type` + `id` only)
