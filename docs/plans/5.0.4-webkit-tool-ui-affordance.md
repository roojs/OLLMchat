## Purpose

- **🔷** Tools that need chrome implement a **GTK-free** interface: **icon name**, **tooltip text**, **view widget**, and a **`show_view`** signal.
- **🔷** **`ChatBar`** builds the **toggle button** from icon + tooltip (shell decides button chrome later).
- **🔷** Application wires: pack toggles, host views, listen to bar toggle + tool `show_view`.
- **🔷** **Toggle / visibility owned by the UI** — tool only emits `show_view` when the user must see the host (e.g. CF).
- **🔷** First consumer: **`browser`** (`BrowserStack` as view).
- **🔷** Desktop Linux first; Android uses the same contract via a **public content stack** on `ChatWidget`.
- **⏳** Multi-tool mutual exclusion — design only; do **not** implement until a second UI tool exists.
- **ℹ️** Replaces parent Phase **6** hand-waving with fences.

---

## Design locks

### Interface `OLLMchat.Tool.UiWidgets` (GTK-free)

- **🔷** Opt-in: only tools with chrome implement it (no null “optional widget” API).
- **🔷** Properties (non-nullable for v1 — sole implementer is `browser`; revisit if a second tool differs):
  - **`icon_name`** — `string` (e.g. `web-browser-symbolic`)
  - **`tooltip_text`** — `string` (e.g. `Browser`)
  - **`view_widget`** — `GLib.Object` (shell casts to `Gtk.Widget`)
- **🔷** Signal **`show_view()`** — tool emits when the user must see the view (CF / challenge). Shell shows the host and sets the matching toggle active.
- **🚫** No `get_button_widget` / tool-owned `Gtk.Button` — bar owns button construction.
- **🚫** Do not make interface members nullable “just in case” until a real second implementer needs it.

### Who builds / who toggles

- **🔷** **`ChatBar`** creates `Gtk.ToggleButton` from `icon_name` + `tooltip_text`, packs it in a public tool-button area **leftmost** (before the model dropdown).
- **🔷** **`ChatBar`** emits **`tool_toggle(string tool_name, bool active)`** when the user toggles.
- **🔷** **Application** listens, shows/hides the hosted view (desktop right pane / Android content stack).
- **🔷** Tool does **not** show/hide / set stack `visible_child`.
- **🔷** Tool **does** emit **`show_view`** when it needs the host visible; app handles like “toggle on”.

### ChatWidget surfaces (public)

- **🔷** **`view_stack`** — `Gtk.Stack` wrapping the transcript (`"chat"` = existing `ChatView`); app/`Android` can `add_named` tool views and switch `visible_child_name`.
- **🔷** **`chat_bar`** already public — app calls bar API to add toggles; bar owns the footer button strip.
- **🔷** Making the stack **public** is enough for Android (no separate window-only stack required for tool views). Composer / permission / bar stay outside the stack.

### Where the view is parented

- **🔷** **Desktop:** `WindowPane.tab_view` (same place as source / agent widgets) + `schedule_pane_update`.
- **🔷** **Android:** `ChatWidget.view_stack` (chat ↔ tool); lower strip stays put.
- **ℹ️** Desktop may still add the view only to `tab_view` and leave `"chat"` as the only child of `view_stack` until Android needs the swap — but **land the public stack in `ChatWidget` now** so both shells share one surface.

### Multi-tool (later)

- **🔷** `⏳` When a second tool has UI: showing one view turns others off.
- **🚫** Do not implement that loop in the first pass.

```vala
namespace OLLMchat.Tool
{
	/**
	 * Optional chrome for a tool: toggle metadata, host view, and show request.
	 *
	 * Implemented only by tools that need UI. {@link view_widget} is
	 * {@link GLib.Object} so {@link OLLMchat} stays GTK-free; the shell
	 * casts to {@code Gtk.Widget}. The shell / {@code ChatBar} owns the
	 * toggle button and visibility — the tool emits {@link show_view}
	 * when the user must see the host.
	 *
	 * == Example ==
	 *
	 * {{{
	 * public class MyTool : BaseTool, UiWidgets {
	 *     public string icon_name { get { return "web-browser-symbolic"; } }
	 *     public string tooltip_text { get { return "Browser"; } }
	 *     public GLib.Object view_widget { get { return this.host; } }
	 *
	 *     void on_challenge() {
	 *         this.show_view();
	 *     }
	 * }
	 * }}}
	 */
	public interface UiWidgets : GLib.Object
	{
		/**
		 * Icon name for the {@code ChatBar} toggle (e.g. web-browser-symbolic).
		 */
		public abstract string icon_name { get; }

		/**
		 * Tooltip for the {@code ChatBar} toggle.
		 */
		public abstract string tooltip_text { get; }

		/**
		 * Host view for this tool.
		 *
		 * Shell casts to {@code Gtk.Widget} and parents it into the desktop
		 * right pane or {@code ChatWidget.view_stack}.
		 */
		public abstract GLib.Object view_widget { get; }

		/**
		 * Tool needs the host visible (e.g. Cloudflare challenge).
		 *
		 * Shell shows the view and sets the matching toggle active.
		 */
		public signal void show_view();
	}
}
```

##### Part 3 — `add_tool_toggle` / `toggle_active_tool` (named in this plan)

#### Add — after constructor / before `sync_streaming`

```vala
		/**
		 * Create and pack a tool toggle from icon + tooltip.
		 *
		 * @param tool_name stable id ({@link OLLMchat.Tool.BaseTool.name})
		 * @param icon_name symbolic icon for the toggle
		 * @param tooltip_text tooltip for the toggle
		 */
		public void add_tool_toggle(string tool_name, string icon_name, string tooltip_text)
		{
			var button = new Gtk.ToggleButton() {
				icon_name = icon_name,
				tooltip_text = tooltip_text,
			};
			button.set_data<string>("tool-name", tool_name);
			button.toggled.connect(() => {
				this.tool_toggle(tool_name, button.active);
			});
			this.tool_button_box.append(button);
		}

		/**
		 * Set a tool toggle active without the caller holding the button.
		 *
		 * Used when a tool emits {@link OLLMchat.Tool.UiWidgets.show_view}.
		 *
		 * @param tool_name {@link OLLMchat.Tool.BaseTool.name}
		 * @param active desired toggle state
		 */
		public void toggle_active_tool(string tool_name, bool active)
		{
			for (var child = this.tool_button_box.get_first_child(); child != null; child = child.get_next_sibling()) {
				var button = child as Gtk.ToggleButton;
				if (button == null) {
					continue;
				}
				if (button.get_data<string>("tool-name") != tool_name) {
					continue;
				}
				if (button.active == active) {
					return;
				}
				button.active = active;
				return;
			}
		}
```

**ℹ️** `add_tool_toggle` / `toggle_active_tool` / `tool_toggle` / public `tool_button_box` are **named in this plan**.

### 7. `ollmapp/Window.vala` — desktop wire after `fill_tools`

**Why:** app discovers `UiWidgets`, asks bar to render toggles, hosts views, relays toggle + `show_view`.

**Where:** immediately after `app.mcp_registry.fill_tools(...)`.

**Depends on:** §4, §5, §6.

#### Add — after both `fill_tools` calls

```vala
			foreach (var tool in this.history_manager.tools.values) {
				var ui = tool as OLLMchat.Tool.UiWidgets;
				if (ui == null) {
					continue;
				}
				var widget_id = tool.name;
				var view = (Gtk.Widget) ui.view_widget;
				this.chat_widget.chat_bar.add_tool_toggle(
					widget_id, ui.icon_name, ui.tooltip_text);
				ui.show_view.connect(() => {
					this.chat_widget.chat_bar.toggle_active_tool(widget_id, true);
				});
			}
			this.chat_widget.chat_bar.tool_toggle.connect((tool_name, active) => {
				if (!this.history_manager.tools.has_key(tool_name)) {
					return;
				}
				var ui = this.history_manager.tools.get(tool_name) as OLLMchat.Tool.UiWidgets;
				if (ui == null) {
					return;
				}
				var view = (Gtk.Widget) ui.view_widget;
				if (!active) {
					this.window_pane.schedule_pane_update(false);
					return;
				}
				if (this.window_pane.tab_view.get_child_by_name(tool_name) == null) {
					this.window_pane.tab_view.add_named(view, tool_name);
				}
				view.visible = true;
				this.window_pane.tab_view.set_visible_child_name(tool_name);
				this.window_pane.schedule_pane_update(true);
			});
```

**ℹ️** `toggle_active_tool(…, true)` fires `toggled` → `tool_toggle` → pane show (same path as a user click).

**🚫** Do **not** `new OLLMwebkit.BrowserStack()` in `Window` / `ChatWidget`.
