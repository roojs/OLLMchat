## Purpose

- **рџ”·** Keep **ollama.com live search** (`libollamaweb`) for discovery вЂ” search quality is good; **do not** replace it with a registry catalog API.
- **рџ”·** Use **catalog data we already fetch** (`OllamaWeb.Model` from search + tags-page refine) to start **local** model pulls.
- **рџ”·** Define the **pull reference** string passed to `POST /api/pull` on the userвЂ™s selected Ollama connection.
- **в„№пёЏ** Prior investigation (registry API) confirmed: no public JSON catalog replaces tags-page refine; this plan assumes **refined per-slug JSON** under `{data_dir}/ollamaweb-models/` stays the detail source.

### Backlog (acceptance)

- **рџ”·** `вЏі` **Pull reference** built from **`OllamaWeb.Model.slug`** + selected **`ModelVariant.name`**, not display `name`.
- **рџ”·** `вЏі` **Start Download** blocked until the selected row has **refined tags** (size dropdown populated from refine).
- **рџ”·** `вЏі` **Cloud-only variants** not offered for local pull (tags whose `name` contains `cloud` or `size == "-"`).
- **рџ’©** `вЏі` **Connection guard:** disable Start Download (or warn) when selected connection is not Ollama-native (`connection.ollama_native != 1`).
- **рџ”·** `вЏі` Manual smoke: search в†’ pick model в†’ pick size в†’ pull completes on local Ollama; model appears on Models page after `model_complete`.

## Current behaviour

- **в„№пёЏ** `SearchResults` + `OllamaWeb.Search.Session` already provide shallow search rows and background **refine** (tags HTML в†’ per-slug JSON in `data_dir/ollamaweb-models/`).
- **в„№пёЏ** `AddModelDialog` wires search, size dropdown, and **Start Download** в†’ `PullManager.start_pull(model_name, connection)` (`ollmapp/SettingsDialog/AddModelDialog.vala`).
- **в„№пёЏ** Pull execution is unchanged from **1.3.4**: `Call.Pull` в†’ `POST {connection.url}/pull` with `{"name": "<pull reference>", "stream": true}`; `PullManager` tracks progress in `data_dir/loading.json`.
- **рџЊ—** Start Download uses **`selected_model.name`** + `":"` + **`tag.name`** вЂ” usually matches `slug`, but **`slug` is the canonical registry key** (especially for `author/model` derivatives).
- **рџЊ—** No guard when **`tags` is empty** (refine not finished) вЂ” user can start pull with no variant or stale data.
- **рџЊ—** Size dropdown sorts cloud tags last but **still lists them** вЂ” local `ollama pull` on a cloud tag will fail or pull the wrong artefact.

## Where models land (download location)

Two different вЂњlocationsвЂќ вЂ” only one is under OLLMchat control:

### Pull target (OLLMchat controls)

- **рџ”·** User picks a **connection** in Add Model (default Ollama server URL, e.g. `http://127.0.0.1:11434/api`).
- **рџ”·** OLLMchat sends **`Call.Pull`** to that connectionвЂ™s **`/api/pull`** endpoint.
- **в„№пёЏ** `PullManager` / `PullManagerThread` already own this path вЂ” **no new download manager**.

### On-disk weights (Ollama controls)

- **в„№пёЏ** The **local Ollama daemon** stores pulled blobs under its model directory (`OLLAMA_MODELS` or platform default, e.g. `~/.ollama/models/`).
- **рџљ«** OLLMchat does **not** choose or configure that directory in this plan (contrast **`8.0`** local GGUF under `~/.local/share/ollmchat/models/`).
- **в„№пёЏ** During pull, Ollama resolves the reference against **`registry.ollama.ai`** (OCI manifests). OLLMchat never calls the registry directly.

### Pull reference в†’ registry (mapping)

| Catalog field | Pull `name` | Registry path (resolved by Ollama, not OLLMchat) |
|---------------|-------------|--------------------------------------------------|
| `slug` = `llama3`, tag `8b` | `llama3:8b` | `registry.ollama.ai/v2/library/llama3/manifests/8b` |
| `slug` = `zerocopia/gemini-3-flash-preview`, tag `latest` | `zerocopia/gemini-3-flash-preview:latest` | `registry.ollama.ai/v2/zerocopia/gemini-3-flash-preview/manifests/latest` |

- **рџ”·** **Rule:** `pull_reference = slug` when no variant selected; else `slug + ":" + variant.name`.
- **в„№пёЏ** `ModelVariant.name` from tags HTML is the **tag suffix** only (`8b`, `latest`, `4b-it-q4_K_M`) вЂ” matches what `ollama pull` expects after the model prefix.

## Proposed behaviour

### Data flow (search в†’ pull)

```mermaid
sequenceDiagram
  participant UI as AddModelDialog
  participant SR as SearchResults
  participant SS as Search.Session
  participant Disk as ollamaweb-models/*.json
  participant PM as PullManager
  participant Ollama as Local Ollama /api/pull
  participant Reg as registry.ollama.ai

  UI->>SR: queue_search(q)
  SR->>SS: search + refine
  SS->>Disk: save refined Model JSON
  SS-->>UI: OllamaWeb.Model rows + tags
  UI->>UI: user picks connection, variant
  UI->>PM: start_pull(pull_reference, connection)
  PM->>Ollama: POST /api/pull name=pull_reference
  Ollama->>Reg: fetch manifests/layers
  Ollama-->>PM: stream progress
  PM-->>UI: progress_updated / model_complete
```

### UI rules

- **рџ”·** **Search / refine:** unchanged вЂ” still `libollamaweb` HTML search + tags refine (`4.8` / `4.8.1`).
- **рџ”·** **Size dropdown:** only **local-pullable** variants (`ModelVariant` where tag name does not contain `cloud` and `size != "-"`).
- **рџ”·** **Start Download:**
  - disabled when `selected_model.slug == ""`
  - disabled when `selected_model.tags.size == 0` or `!selected_model.refined`
  - disabled when size dropdown has no local variants
  - **рџ’©** disabled when `connection.ollama_native != 1` (after `detect_ollama` on dialog open)
- **рџ”·** On click: `pull_reference` from **`slug` + variant** в†’ `PullManager.start_pull(pull_reference, connection)` (existing API).

### What we reuse (no duplicate fetch)

| Data | Source | Used for |
|------|--------|----------|
| Search hits | `Session.search` | Pulldown rows |
| Tag names, sizes, context | `Session.refine` в†’ disk JSON | Size dropdown, pull suffix |
| `slug` | search row + disk | Pull reference prefix |
| `features` | search + tags page | List markup only (not pull) |
| Pull + progress | `PullManager` (**1.3.4**) | Download execution |

- **рџљ«** No second HTTP fetch at pull time (no registry client in OLLMchat).
- **рџљ«** No replacement of tags-page refine with `registry.ollama.ai` or `ollama.com/api/tags` (cloud-only).

## Concrete code proposals

Intro: hunks are **Remove** / **Replace with** / **Add**; verify context in tree before applying.

### 1. `ollmapp/SettingsDialog/AddModelDialog.vala` вЂ” Start Download: `slug`-based pull reference

**Why:** `slug` is the stable registry key; `name` is display-only and can diverge for derivatives.

**Where:** `start_download_button.clicked` handler (build `model_name` before `start_pull`).

**Depends on:** none.

#### Replace with вЂ” pull reference construction (inside clicked handler, after `selected_model.slug` check):

```vala
				var pull_reference = this.selected_model.slug;

				var size_index = this.size_dropdown.selected;
				if (size_index != Gtk.INVALID_LIST_POSITION &&
				    (int)size_index < this.size_list_store.get_n_items()) {
					var tag = this.size_list_store.get_item((uint)size_index) as OllamaWeb.ModelVariant;
					if (tag != null && tag.name != "") {
						pull_reference = pull_reference + ":" + tag.name;
					}
				}

				this.dialog.pull_manager.start_pull(pull_reference, connection);
```

#### Remove вЂ” old block that used `selected_model.name` and duplicate `start_pull` call:

```vala
				var model_name = this.selected_model.name;

				var size_index = this.size_dropdown.selected;
				if (size_index != Gtk.INVALID_LIST_POSITION &&
				    (int)size_index < this.size_list_store.get_n_items()) {
					var tag = this.size_list_store.get_item((uint)size_index) as OllamaWeb.ModelVariant;
					if (tag != null) {
						model_name = model_name + ":" + tag.name;
					}
				}
				
				// Start pull operation
				this.dialog.pull_manager.start_pull(model_name, connection);
				// this.dialog.pull_manager.start_pull(model_name, connection);
```

### 2. `ollmapp/SettingsDialog/AddModelDialog.vala` вЂ” guard pull until refine + filter cloud tags

**Why:** Pull must use refined tag list; cloud variants are not local-pullable.

**Where:** `update_size_dropdown()` вЂ” tag filter loop; `start_download_button.sensitive` updates.

**Depends on:** В§1.

#### Replace with вЂ” `update_size_dropdown()` tag loop: skip cloud / remote-only rows

In the `foreach (var tag_obj in sorted_tags)` loop, **before** `this.size_list_store.append(tag_obj)`:

```vala
				if (tag_obj.name.down().contains("cloud") || tag_obj.size == "-") {
					continue;
				}
```

#### Add вЂ” at end of `update_size_dropdown()`, toggle Start Download

After `this.size_row.visible = true;`:

```vala
			this.start_download_button.sensitive =
				this.selected_model.refined
				&& this.size_list_store.get_n_items() > 0;
```

#### Add вЂ” in clicked handler, early return when not pullable

Immediately after `selected_model.slug == ""` check:

### 3. `ollmapp/SettingsDialog/AddModelDialog.vala` вЂ” **рџ’©** Ollama-native connection guard

**Why:** OpenAI-compatible endpoints do not implement `/api/pull`.

**Where:** `load()` after connections populated; `connection_dropdown` `notify::selected`.

**Depends on:** В§2.

#### Add вЂ” helper sensitivity refresh (inline in `load()`, end of `item_selected` / connection change)

When connection selection changes, set:

```vala
			var conn_index = (int)this.connection_dropdown.selected;
			var can_pull = false;
			if (conn_index >= 0 && conn_index < this.connection_urls.size) {
				var url = this.connection_urls.get(conn_index);
				if (this.dialog.app.config.connections.has_key(url)) {
					can_pull = this.dialog.app.config.connections.get(url).ollama_native == 1;
				}
			}
			if (!can_pull) {
				this.start_download_button.sensitive = false;
			}
```

- **в„№пёЏ** Call this after `update_size_dropdown()` and when connection dropdown changes.
- **рџ’©** Optional tooltip on disabled button: вЂњPull requires an Ollama server connectionвЂќ.
