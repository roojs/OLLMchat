# 3.4. Session Status Indicators

## Overview

UI improvements for session status: indicators when a session is running, and highlighting for available content.

## Status

✅ **DONE** - Implemented in HistoryBrowser with spinner and unread indicators.

## Implementation Details

### Features
- **Running Session Indicator** - Rotating indicator/spinner when a session is actively running
- **Content Available Highlight** - Red highlight or indicator to show that new content is available
- Visual feedback for session state

### UI Components
- Session status indicator (rotating spinner/icon)
- Content available indicator (red highlight/badge)
- Integration with ChatView or HistoryBrowser

### Files Created/Modified
- ✅ `libollmchatgtk/HistoryBrowser.vala` - Added spinner widget and unread label
- ✅ `libollmchat/History/SessionBase.vala` - Added `is_running` property and `css_classes` property
- ✅ `resources/style.css` - Added `.oc-has-unread` styling for unread sessions

### Implementation Notes
- ✅ Spinner widget placed in HistoryBrowser secondary_box (between info and date)
- ✅ Spinner visibility and spinning state bound to `session.is_running` property
- ✅ Unread label widget added to HistoryBrowser with red styling
- ✅ Unread label visibility bound to `session.has_unread` property
- ✅ Row background highlight via `css_classes` property (applies `oc-has-unread` class when `unread_count > 0`)
- ✅ `is_running` property set in `Session.send()` and cleared in `Session.finalize_streaming()`
- ✅ Indicators are visible but not intrusive (spinner only shows when running, unread label only when unread_count > 0)
