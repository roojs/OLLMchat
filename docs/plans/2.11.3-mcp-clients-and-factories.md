# 2.11.3. MCP Clients (Stdio + Http)

## Overview

Phase 3 of the MCP loader: concrete transport implementations. Add `libocmcp/Stdio/Client.vala` and `libocmcp/Http/Client.vala`. Registry creates the Client (switch on config.transport); no Stdio.Factory or Http.Factory.

Parent: [2.11-mcp-loader-tool.md](2.11-mcp-loader-tool.md)

## Scope

- **Stdio:** `libocmcp/Stdio/Client.vala` – `OLLMmcp.Stdio.Client` extends `OLLMmcp.Client`. Run subprocess under bwrap when available; respect `Config.network`. Spawn from Config (command, args, env); JSON-RPC over stdin/stdout; connect, disconnect, list_tools, call_tool. No Stdio.Factory — Registry creates it (switch on transport == "stdio").
- **Http:** `libocmcp/Http/Client.vala` – `OLLMmcp.Http.Client` extends `OLLMmcp.Client`. Connect to Config.url; JSON-RPC over HTTP; connect, disconnect, list_tools, call_tool. No Http.Factory — Registry creates it (switch on transport == "http").
- **Registry:** Add `create_client(Config)` (or inline in fill_tools): switch on config.transport → `new Stdio.Client(config)` or `new Http.Client(config)`.

## Files

Under **libocmcp/** (still not in meson):

- `libocmcp/Stdio/Client.vala` – Stdio.Client extends Client; stdio subprocess + JSON-RPC.
- `libocmcp/Http/Client.vala` – Http.Client extends Client; HTTP JSON-RPC to url.
- `libocmcp/Registry.vala` – add create_client(Config) or switch in fill_tools to instantiate Stdio.Client or Http.Client. No Stdio/Factory.vala or Http/Factory.vala.
