# 6.0 - Unified Task File Format Consolidation

## Status

⏳ PENDING

## Purpose

Consolidate the prompt, goals, completed, and proposed sections into a single unified task file format that serves as the source of truth across all plan files, summary index, and archive structure.

## Problem Summary

The project currently maintains three complementary formats: (1) individual plan files in `docs/plans/`, (2) a summary index in `docs/plans/1.0-summary.md`, and (3) an archive in `docs/plans/done/`. While these already form a cohesive system, standardization is needed to eliminate duplication and ensure consistent `Status:` values across all files.

## Scope

- **In scope:**
  - Define unified format with `Status:` as source of truth
  - Standardize all existing plan files to use valid `Status:` values (`done`, `needs_testing`, `on_hold`, `proposed`, `rejected`)
  - Archive completed plans to `docs/plans/done/` with `DONE-` prefix
  - Automate summary index generation from plan file `Status:` lines
  - Update [`docs/guide-to-writing-plans.md`](docs/guide-to-writing-plans.md) to formalize the `Status:`-based pattern

- **Out of scope:**
  - Implementation code proposals (defer to **plan_code** step)
  - UI changes or automation scripts (defer to follow-up tasks)

## Acceptance criteria

- [ ] All plan files use valid `Status:` values from the defined set
- [ ] Completed plans are archived in `docs/plans/done/` with `DONE-` prefix and `Status: done`
- [ ] Summary index (`1.0-summary.md`) reflects plan file `Status:` values with checkbox indicators
- [ ] [`docs/guide-to-writing-plans.md`](docs/guide-to-writing-plans.md) updated to formalize `Status:` line as source of truth
- [ ] Automated script documented for future summary generation

## 1. Define Unified Format Requirements

### Issue

Current plan files use inconsistent `Status:` values and section structures.

### Goal

Define a single unified format with mandatory `Status:` line and standardized sections.

### Scope

- Update [`docs/guide-to-writing-plans.md`](docs/guide-to-writing-plans.md) to specify valid `Status:` values
- Ensure all new plans follow the unified format
- Migrate existing plans to the new format

### Considerations

- Summary index must auto-generate from plan files

## 2. Migration Path

### Issue

Existing plan files need to be standardized.

### Goal

Move all completed plans to archive and update `Status:` values across all files.

### Scope

- Archive all `Status: done` plans to `docs/plans/done/` with `DONE-` prefix
- Update `Status:` values in active plans to match valid set
- Ensure `1.0-summary.md` checkbox indicators reflect plan file `Status:` values

### Considerations

- Migration should be automated where possible
- Maintain link references between summary and individual plan files
