# 7.14.1 — `ProgressList` + `Runner.progress` + Runner adapters (phase 1)

**Status:** proposed

**Parent:** `docs/plans/7.14-task-progress-tree-ui.md`

**Checklist:** `.cursor/rules/CODING_STANDARDS.md` — **Checklist for all plans**.

**Layout:** `docs/guide-to-writing-plans.md`

---

## Purpose

- Introduce **`ProgressList`** (**`Runner`** holds **`this.progress`**; **`ProgressList`** holds **`weak Runner`** to avoid **`Runner` ↔ `ProgressList` ref cycle).
- **`Runner`** calls **`replace_pending()`**, **`add_complete()`**, **`rebuild_from_runner()`** (or equivalent — names fixed in patch) at transitions.
- **Adapters only for `Runner`:** **`ProgressRunnerCreateAdapter`** (top-level **task-list create** row) and **`ProgressRunnerIterateAdapter`** (**task-list iterate** row, child of a **`Details`** in the tree). **`Details`** and **`Tool`** **implement `ProgressItem` directly** — **no** adapters for them.
- Phase **1** may use **stub** strings for **`ProgressItem`** stage/title on **`Details`** / **`Tool`** until **`7.14.3`** (**`to_human()`**, real labels).

**No** GTK in **`liboccoder`** — model only (**`7.14.2`** adds **`ProgressView`**).

---

## Scope

| In scope | Out of scope |
|----------|----------------|
| New **`Task/`** types + **`meson.build`** + **`Runner`** hooks | **`Gtk.ColumnView`** (**`7.14.2`**) |
| **`ProgressItem`** on **`Details`** / **`Tool`** (stubs OK) | **`to_human()`** / bold / tick rendering (**`7.14.3`**) |

---

## Acceptance criteria

1. **`Runner.progress`** is non-null after construction; **`ProgressList`** uses **`weak Runner`**.
2. **`replace_pending()`** / **`rebuild_from_runner()`** invoked when **`pending`** is replaced (e.g. after **`parse_task_list_iteration`** success) so stale rows are dropped.
3. **Runner-only** adapters exist for create + iterate; **`Details`**/**`Tool`** satisfy **`ProgressItem`** without adapters.

namespace OLLMcoder.Task
{

/**
 * Top-level row: initial task-list creation / retry loop (Runner.send_async).
 * Owned by ProgressList; reads try_count / phase from ProgressList or Runner via callbacks.
 */
public class ProgressRunnerCreateAdapter : GLib.Object, ProgressItem
{
	public weak OLLMcoder.Skill.Runner runner { get; private set; }

	/**
	 * 0 = first attempt; >0 means retrying parse.
	 */
	public int try_index { get; set; default = 0; }

	/**
	 * True while waiting on LLM for task_list_parse response.
	 */
	public bool awaiting_response { get; set; default = false; }

	/**
	 * True after successful parse (issues empty) for this attempt.
	 */
	public bool parse_succeeded { get; set; default = false; }

	public ProgressRunnerCreateAdapter(OLLMcoder.Skill.Runner r)
	{
		this.runner = r;
	}

	public ProgressRowKind progress_row_kind {
		get { return ProgressRowKind.TASK_LIST_CREATE; }
	}

	public string progress_title {
		owned get { return "Task list"; }
	}

	public string progress_stage {
		owned get {
			if (this.parse_succeeded) {
				return "";
			}
			if (this.awaiting_response) {
				return this.try_index > 0 ? "Retrying" : "Creating";
			}
			return "";
		}
	}

	public bool progress_stage_active {
		get { return this.awaiting_response && !this.parse_succeeded; }
	}
}

/**
 * Child under a Details: task-list iteration (Runner.run_task_list_iteration).
 * Owned by ProgressList.
 */
public class ProgressRunnerIterateAdapter : GLib.Object, ProgressItem

	/**
	 * The Details whose post-exec triggered iteration (parent row in tree).

	public weak Details? for_detail { get; private set; }

	public int try_index { get; set; default = 0; }

	public bool awaiting_response { get; set; default = false; }

	public bool parse_succeeded { get; set; default = false; }

	public ProgressRunnerIterateAdapter(OLLMcoder.Skill.Runner r, Details? detail)

### 3. `liboccoder/Task/ProgressList.vala` — new

#### Add

```vala
/*
 * Copyright (C) 2026 Alan Knowles <alan@roojs.com>
 * SPDX-License-Identifier: LGPL-3.0-or-later
 */

namespace OLLMcoder.Task
{

/**
 * Central owner of progress row objects for the skill Runner.
 * GTK ColumnView binds here in 7.14.2 — phase 1 exposes flat lists + rebuild hooks.
 */
public class ProgressList : GLib.Object
{
	public weak OLLMcoder.Skill.Runner runner { get; private set; }

	/**
	 * Single create-row adapter; recreated or reset in rebuild_from_runner().
	 */
	public ProgressRunnerCreateAdapter? create_row { get; private set; default = null; }

	/**
	 * Current iteration row if any (child of a Details in the logical tree).
	 */
	public ProgressRunnerIterateAdapter? iterate_row { get; private set; default = null; }

	/**
	 * Flat snapshot of all ProgressItem rows for simple iteration (tree shape in 7.14.2).
	 */
	public Gee.ArrayList<ProgressItem> items { get; private set; default = new Gee.ArrayList<ProgressItem>(); }

	public ProgressList(OLLMcoder.Skill.Runner r)
	{
		this.runner = r;
		this.create_row = new ProgressRunnerCreateAdapter(r);
		this.rebuild_from_runner();
	}

	/**
	 * Call when pending List is replaced (e.g. after successful parse_task_list_iteration).
	 */
	public void replace_pending()
	{
		this.rebuild_from_runner();
	}

	/**
	 * Call when a step moves completed → pending bookkeeping (optional hook).
	 */
	public void add_complete()
	{
		this.rebuild_from_runner();
	}

	/**
	 * Full resync from runner.completed + runner.pending — may clear iterate_row, repopulate Details/Tool items.
	 */
	public void rebuild_from_runner()
	{
		this.items.clear();
		if (this.create_row != null) {
			this.items.add(this.create_row);
		}
		if (this.iterate_row != null) {
			this.items.add(this.iterate_row);
		}
		var r = this.runner;
		if (r == null) {
			this.notify_property("items");
			return;
		}
		this.append_lists(r.completed, r.pending);
		this.notify_property("items");
	}

	private void append_lists(OLLMcoder.Task.List completed, OLLMcoder.Task.List pending)
	{
		foreach (var step in completed.steps) {
			foreach (var d in step.children) {
				this.items.add(d);
				this.append_tools(d);
			}
		}
		foreach (var step in pending.steps) {
			foreach (var d in step.children) {
				this.items.add(d);
				this.append_tools(d);
			}
		}
	}

	private void append_tools(Details d)
	{
		foreach (var t in d.exec_runs) {
			this.items.add(t);
		}
	}

	/**
	 * Task-list creation: set state before chat_call.send (try loop).
	 */
	public void task_list_create_attempt(int try_index)
	{
		if (this.create_row == null) {
			this.create_row = new ProgressRunnerCreateAdapter(this.runner);
		}
		this.create_row.try_index = try_index;
		this.create_row.awaiting_response = true;
		this.create_row.parse_succeeded = false;
		this.notify_property("items");
	}

	public void task_list_create_parse_done(bool success)
	{
		if (this.create_row != null) {
			this.create_row.awaiting_response = false;
			this.create_row.parse_succeeded = success;
		}
		this.notify_property("items");
	}

	/**
	 * Iteration: bind for_detail before run_task_list_iteration loop.
	 */
	public void iteration_begin(Details? for_detail)
	{
		this.iterate_row = new ProgressRunnerIterateAdapter(this.runner, for_detail);
		this.iterate_row.try_index = 0;
		this.iterate_row.awaiting_response = true;
		this.iterate_row.parse_succeeded = false;
		this.rebuild_from_runner();
	}

	public void iteration_attempt(int try_index)
	{
		if (this.iterate_row != null) {
			this.iterate_row.try_index = try_index;
			this.iterate_row.awaiting_response = true;
			this.iterate_row.parse_succeeded = false;
		}
		this.notify_property("items");
	}

	public void iteration_parse_done(bool success)
	{
		if (this.iterate_row != null) {
			this.iterate_row.awaiting_response = false;
			this.iterate_row.parse_succeeded = success;
		}
		this.notify_property("items");
	}

	public void iteration_end()
	{
		this.iterate_row = null;
		this.rebuild_from_runner();
	}

	public void clear()
	{
		this.items.clear();
		this.create_row = null;
		this.iterate_row = null;
		this.notify_property("items");
	}
}

}
```

## Implementation notes (read before merge)

1. **`ProgressList.rebuild_from_runner()`** must **not** duplicate **`create_row`** entries — keep **one** create adapter, append **`Details`**/**`Tool`** in visual order; **dedupe** **`items`** vs sketch above.
2. **`iteration_begin(null)`** — replace with real **`Details`** reference when **`run_task_list_iteration`** caller has the triggering task (**`List.vala`** / **`Runner`** — spike).
3. **`on_replay`** / session load: **`progress.clear()`** or **`rebuild_from_runner()`** — **`7.14.5`**; optional **`progress.clear()`** at start of **`send_async`** if needed.
4. Run **`meson compile`** after reordering **`meson.build`** — resolve any circular **`ProgressList`** ↔ **`Tool`** order by moving file lines only.
