**Relevant checklist (topics):**

- **StringBuilder usage** — avoid **`GLib.StringBuilder`** unless building in loops with **hundreds** of iterations; prefer **`+`** and **`string.joinv()`** where appropriate.
- **Building strings in loops** — follow **§ Building strings in loops**; do not assume **`+=`** in a tight loop without matching the doc’s allowed patterns.
- **Nullable types / null checks** — prefer **flags** (**`widgets_created`**, **`content_rendered`**) over nullable widgets as the **primary** contract.
- **`this.` prefix** — use **`this.`** on instance members in new/modified Vala.
- **New methods** — avoid unless this plan or the user explicitly approves.

## 👎 Do not do / rejected

Items here are **out of scope** — **do not** implement.

- **👎** **Internal pending storage** inside **`RenderSourceView`** — **no** second **`string`**, **`StringBuilder`**, or **“pending”** field to accumulate chunks while **`code_text`** is deferred. **No** “flush from pending” at **`end_code_block`**.
- **👎** **Optional reconcile** pass in **`end_code_block`** when **not streaming** — **unless** a **real** bug proves buffers **diverged** from **`code_text`** (then fix the **cause**, not a standing reconcile).
- **👎** **Optional (product)** first expand = **source** only — **not** in scope unless product asks again.

## Placeholder vs empty **`body_revealer`** (**ℹ️** investigation)

- **ℹ️** **Gtk 4** — **`Gtk.Revealer.set_child`** accepts **`null`** (**child** “can be **NULL**” per **GTK** docs). So the **revealer** can have **no** body child until **first expand**, then **one** **`set_child(this.scrolled_window)`** — **no** dummy **`Gtk.Box`**, **no** swap/reparent of a placeholder (**avoids** detach bugs).
- **ℹ️** **Today** (**`RenderSourceView.vala`** ~378–386) — **`body_revealer.set_child(this.scrolled_window)`** immediately in **ctor**. **Lazy** plan: build **header** + **frame** + **`body_revealer`** with **`set_child(null)`**, materialize **scrolled** + **stack** + **GtkSource** on expand → **single** **`set_child`**.
- **ℹ️** **Resize** paths use **`body_revealer.get_allocated_width()`** (~491) — with **no** child, width may be **0** until **first** show; **guard** **`for_width`** (existing pattern when collapsed) — **verify** in implementation, **no** placeholder **required** for layout.
- **👎** **Do not** add a **placeholder** widget **only** to rip it out and attach the **real** body later — **prefer** **null** child **or** **never** create **revealer**’s child until expand (**one** attach).

## Purpose

- **⚠️** **History load** and **restore** replay large assistant bodies through **`MarkdownGtk.Render`** → **`RenderSourceView`**. Today the frame **eagerly** builds **GtkSource** + **nested** **`MarkdownGtk.Render`** — and **`add_code_text`** feeds **buffers** even when **hidden**.
- **Outcome:** **Ctor** + **`Message.is_stream`** defer **body** widget creation and align **`code_text`** rules (**§ Proposed behaviour**). **flags** over nullable-only contracts (**Avoiding Nullable Types**).
- **⚠️** **Streaming** (**`Message.is_stream`**): **`add_code_text`** does **not** grow **`code_text`** while the fence is **open**; **`end_code_block`** or **wipe** assigns **`code_text`**. Live text in **GtkSource** / nested **`Render`** **when those widgets exist**.
- **⚠️** **Not streaming:** **`code_text`** from **first** **`add_code_text`** (today **`code_content`**).
- **⚠️** **Streaming** + **lazy** **`.collapsed`** (body widgets **not** created yet): **`RenderSourceView`** does **not** retain chunks (**§ 👎**). **Upstream** (**`ChatView`** / **`Render`**) must **not** feed **`add_code_text`** until the body is materialized **or** must **buffer** the fence segment **outside** this class — **product/contract** for that path is **TBD** with **§ Approved** ctor behaviour.

## Acceptance criteria

- **⚠️** **`RenderSourceView` ctor** accepts **`Message.is_stream`** (or **`bool` is_stream**) and **defers** **body** widget creation when the plan requires (**shell** only until expand).
- **⚠️** **Restore / load:** **Initially collapsed** fence does **not** run **nested** **`MarkdownGtk.Render.add`** / **GtkSource** insert until **expand** (or **min:** **source** view).
- **⚠️** **Streaming:** **`add_code_text`** does **not** append **`code_text`** while **open**; **`end_code_block`** / **snapshot before wipe** sets **`code_text`**. **👎** **no** internal **pending** buffer (**§ 👎**).
- **⚠️** **Not streaming:** **`code_text`** from **start** of fence (**`+=`** per **`add_code_text`** as today **`code_content`**).
- **⚠️** **Thinking / `.collapsed-on-done`:** **Streaming** + **revealed** → **buffers** only; **collapse** / **fold** → **snapshot** **`code_text`** **then** **clear** buffer **text**; **widgets** stay (**§ Proposed behaviour** — **collapse**). **Not streaming** → **`code_text`** from **start**; **keep** **shell** / **`widgets_created`**.
- **⚠️** **`widgets_created`**, **`content_rendered`** — explicit **state** inside the frame (**§ Proposed behaviour**); **not** **`source_view == null`** alone.

## Wiring **`is_stream`** into **`Render`** (**ℹ️** — why this is not “pass **`Message`**”)

- **ℹ️** **`MarkdownGtk.Render`** (**`libocmarkdowngtk`**) **cannot** take **`OLLMchat.Message`** — wrong layer (**no** **`OLLMchat`** dependency in that library). **`Render.on_code_block(true, lang)`** only receives the **fence lang** string from the parser.
- **ℹ️** **`ChatView`** (**`libollmchatgtk`**) **has** **`Message`** (**`append_complete_assistant_message`**) and knows **live chunk** vs **bulk** (**`append_assistant_chunk`** vs complete message).
- **ℹ️** **Pattern:** add **`bool is_stream`** (or **`streaming`**) **on** **`MarkdownGtk.Render`**, **set from** **`ChatView`** whenever an assistant body is being fed (**e.g.** **`message.is_stream`** before **`process_new_chunk_direct`**, and the matching value for **live** stream). **`Render.on_code_block`** reads **`this.is_stream`** and passes **`this.is_stream`** into **`new RenderSourceView(this, lang, this.is_stream)`**. **No** **`Message`** in **`Render.vala`**.

## Proposed behaviour (contract)

- **⚠️** **`Message.is_stream`** / **`Render.is_stream`** — **approved** at **ctor** (**§ Wiring**). Drives **initial** policy: **streaming** vs **bulk** **`code_text`** rules and whether **body** widgets may be **deferred**.
- **ℹ️** **Inside the frame** (after **ctor**), **ongoing** behaviour is driven mainly by **`widgets_created`** and **`content_rendered`** — e.g. after **collapse**: **buffers** cleared, **`content_rendered == false`**, **`widgets_created == true`** → **re-open** **refills** from **`code_text`**. **Ctor** **`is_stream`** sets **live vs bulk** **policy**; it does **not** replace these **flags** for **collapse** / **expand** / **replay**.
- **⚠️** **`code_text` — streaming:** Set **only** at **`end_code_block`** (from GtkSource/nested) or **wipe** snapshot — **not** **`+=`** in **`add_code_text`** while **open**.
- **⚠️** **`code_text` — not streaming:** **`+=`** each **`add_code_text`** (today **`code_content`**).
- **⚠️** **`.collapsed` + streaming + no body yet:** **👎** **no** in-frame chunk queue — coordinate **upstream** (**§ Purpose**).
- **⚠️** **Collapse** (user hides body, **or** **`.collapsed-on-done`** **fold** after streaming **finished** rendering): **First** snapshot GtkSource + nested → **`code_text`** (if **streaming** and not already set for this segment). **Then** **clear** **text** in GtkSource + nested **`Render`** (**buffers** empty); **`content_rendered = false`**. **`widgets_created`** stays **true** — **do not** destroy **GtkSource** / **stack** / nested **`Render`**. **On** **re-open** / **expand**: **refill** buffers from **`code_text`** (**replay**), set **`content_rendered = true`**.
- **⚠️** **Expand** (first time): create **body** if **`!widgets_created`**; **replay** **`code_text`** when **`!content_rendered`**.

## Concrete code proposals

Hunks are **Remove** / **Replace with** / **Add**; verify context in tree.

### 1. `libocmarkdowngtk/RenderSourceView.vala` — ctor, flags, **`code_text`**

#### Add

```vala
		private bool widgets_created = false;
		private bool content_rendered = false;
		private string code_text = "";
```

- **✅** **Ctor** — add **`bool is_stream`** (from **`Message.is_stream`**). When **`is_stream`** **and** **lazy** body path, **do not** allocate GtkSource / stack / nested **`Render`** until **first expand** (or **`§ 3`** materialization).
- **Replace with** — migrate **`code_content`** **`StringBuilder`** → **`code_text`** **`string`** per **`.cursor/rules/CODING_STANDARDS.md`**; **not streaming** → **`code_text +=`** in **`add_code_text`**; **streaming** → **`code_text`** only **`end_code_block`** / wipe (**👎** **no** **pending** field).

---

### 2. `libocmarkdowngtk/RenderSourceView.vala` — `add_code_text`

- **Not streaming:** **`this.code_text += text`**; **buffers** per **collapsed** / **revealer** rules.
- **Streaming:** **no** **`code_text +=`**. If **body** exists: GtkSource + nested. If **`.collapsed`** **and** **no** body widgets yet: **`return`** immediately — **upstream** must not rely on hidden storage here (**§ 👎**).

---

### 2b. `libocmarkdowngtk/RenderSourceView.vala` — `end_code_block`

- **Streaming:** **`this.code_text =`** GtkSource/nested read **once**; **👎** **not** from a **pending** buffer.
- **Not streaming:** **`code_text`** already accumulated.

---

### 3. `libocmarkdowngtk/RenderSourceView.vala` — **shell** vs **deferred body**

- **✅** **Ctor:** **header** + **frame** + **`body_revealer`**. **Deferred body:** **`body_revealer.set_child(null)`** until materialize — **§ Placeholder vs empty `body_revealer`** (**no** placeholder **Gtk.Box** swap).
- **Body** (**scrolled** + **stack** + **GtkSource** + nested **`Render`**) **only** when **`widgets_created`** becomes **true** — **one** **`set_child`**, not **detach/replace**.

---

### 4. Collapse / expand — **§ Proposed behaviour**

- **Collapse:** snapshot **`code_text`** → **clear** GtkSource + nested **text** → **`content_rendered = false`** → **`widgets_created`** unchanged.
- **Expand:** if **`!content_rendered`**, **replay** **`code_text`** into buffers; **`content_rendered = true`**.

---

### 5. `libocmarkdowngtk/Render.vala` — `on_code_block`

#### Replace with

- **`this.childview = new RenderSourceView(this, lang, this.is_stream);`** — **`this.is_stream`** on **`Render`** (**§ Wiring**); **set** from **`ChatView`** before markdown for that assistant turn is fed.
