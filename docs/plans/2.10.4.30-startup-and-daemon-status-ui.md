## Purpose

- **🔷** Extend the **startup blocking overlay** (today **`CheckingConnectionDialog`**) so it stays up through **Ollama verification** and **`ollmfilesd` RPC connect**, with **wording that changes per step**, and dismisses only when the **main shell is stable** (agent dropdown populated, chat shell wired).
- **🔷** Repurpose **`ActivityBanner`** (renamed from **`VectorScanBanner`**) as a **non-blocking daemon-activity status bar** for work that happens **after** the overlay dismisses — project list load, project activate / filesystem scan, vector index queue.
- **🔷** Produce an explicit **inventory** of what status text we can drive **today** vs what needs **new daemon notifications**.
- **🔷** Add a formal **`disable_filesd`** switch so builds that **cannot** spawn **`ollmfilesd`** (Android) or **do not need** project workspace (chat-only) skip daemon boot cleanly — document what breaks without the daemon.
- **🔷** Final polish tranche for the **2.10.4 RPC migration** — UI feedback for daemon boot and early project work that is invisible today.

## `disable_filesd` — flag and implications

### Android — your assumption is correct

- **🔷** The Android POC **cannot** usefully spawn **`ollmfilesd`** on-device (no packaged daemon binary, no Unix socket boot path, sandbox limits).
- **🔷** Android **does not need** local **`ollmfilesd`** for the current POC — chat + **`web_fetch`** / **`session_fetch`** / **`google_search`** only ([`9.0-android-poc-summary.md`](9.0-android-poc-summary.md) **Tools** table).
- **✅** Today: **`initialize_client`** on Android skips daemon entirely — LLM connect → agents → chat. **No** **`ProjectManager`** RPC client.

**ℹ️** **Later** Android could connect to a **remote** **`ollmfilesd`** on the dev machine ([`9.0`](9.0-android-poc-summary.md) *Later* bullet) — that is **not** `disable_filesd`; it is a different transport target.

### Proposed flag shape

| Mechanism | Where | Default |
| --------- | ----- | ------- |
| **Meson** **`files_daemon`** | **`meson_options.txt`** — `false` for Android APK target | `true` on Linux desktop |
| **Compile-time** | **`libocfiles`** / **`ollmapp`** — when `false`, **🚫** link **`ClientBoot`** spawn path; optional stub PM | — |
| **Runtime** (💩 optional) | **`config.2.json`** key e.g. **`files_daemon.enabled`** | Desktop `true` |

**🔷** Minimum for this tranche: **Meson / platform** flag so **`initialize_client`** branches like Android today — **not** a user-facing Settings toggle unless requested.

**💩** Env override **`OLLM_DISABLE_FILESD=1`** for dev chat-only desktop runs — confirm before build.

### Overlay / status bar when `disable_filesd`

| Layer | With daemon | Without daemon |
| ----- | ----------- | -------------- |
| Overlay step 3–4 | *Starting / connecting filesystem daemon…* | **Skipped** |
| Overlay step 5 | *Loading workspace…* (`load_projects_from_db` + wire) | *Preparing chat…* (agents + tools only) |
| Status bar (Layer 2) | Project load, scan, index | **Hidden / unused** — no daemon RPC, no **`scan_update`** |

### What works without `ollmfilesd`

| Feature | Works? | Notes |
| ------- | ------ | ----- |
| LLM chat (**`chatter`** agent) | **✅** | Same as Android POC |
| History / sessions | **✅** | Local SQLite under app data dir |
| **`web_fetch`**, **`session_fetch`**, **`google_search`** | **✅** | No project paths required |
| Settings (connections, models) | **✅** | Unchanged |
| Agent dropdown | **✅** | Only agents registered without PM — today **`chatter`** only on Android |

### What does **not** work without `ollmfilesd`

Everything that goes through **`libocfiles.ProjectManager`** RPC or daemon-owned **`files.sqlite`**:

| Feature | Broken because |
| ------- | -------------- |
| **Project list** / create / remove | **`ProjectManager.load_projects_from_db`**, **`create_project`**, … — daemon DB |
| **Activate project** / file tree | **`activate_project`** → daemon **`read_dir`** |
| **Coding assistant** (**`CodeAssistant`** / **`SourceView`**) | **`AgentFactory`** needs real PM — **`restore_active_state`**, editor buffers via **`File.*`** |
| **Skill runner** (Conductor) | Same — project-scoped skills |
| **File tools** — **`read_file`**, **`write_file`**, **`edit_mode`** | Need workspace paths + daemon I/O |
| **`codebase_search`** | Daemon FAISS + vector pipeline |
| **`run_command`** / sandbox (future **`Bubble.*`**) | Daemon execution + overlay scan |
| **Approvals** / pending file history | **`Folder.fetch_pending_approvals`**, daemon SQL |
| **File-change banner** | Compares buffer to daemon-backed file state |
| **Vector / scan status bar** | No **`event.vector.scan_update`** source |
| **Session `project_path` restore** | **`AgentDropdown.session_selection`** → **`activate_project`** — no-op or error without PM |

**🔷** In short: **without the file daemon you lose the project workspace** — not just “project manager UI”, but **all** coding-agent, file-tree, vector, and filesystem-backed tool surfaces. Pure chat remains fine.

### Desktop today vs target

| | Desktop **`Window.vala`** today | Target with **`disable_filesd`** |
| - | ------------------------------- | -------------------------------- |
| **`rpc.connect`** | **Required** — failure → **`tool_error_banner`**, **return** | **Skipped** when flag set |
| **`project_manager`** | Real **`OLLMfiles.ProjectManager`** + RPC | **Null** or no-op stub (mirror Android) |
| Agents registered | **`chatter`** + **`CodeAssistant`** + **SkillRunner** | **`chatter`** only (💩 unless remote daemon added later) |
| Tools | Full **`tools_registry.fill_tools(pm)`** | Subset registration (Android pattern) |

### Touch points (flag)

| File | Change |
| ---- | ------ |
| **`meson_options.txt`** / **`ollmapp/meson.build`** | **`files_daemon`** option; Android build sets **`false`** |
| **`ollmapp/Window.vala`** | Branch **`initialize_client`** — daemon overlay steps + **`rpc.connect`** only when enabled |
| **`libocfiles/ProjectManager.vala`** | **💩** **`files_daemon`** compile guard or stub interface — avoid linking spawn on Android |
| **`ollmapp/android/OllmchatWindow.vala`** | Align with shared overlay API; keep implicit **`disable_filesd`** |

**🚫** Do **not** fail boot when **`disable_filesd`** and user picks **Coding assistant** — hide or disable that agent when PM unavailable (same as Android today: only **`chatter`** in dropdown).

## Status inventory — what we can show

### A. Client-local (no new wire) — **available now**

These are **`yield`** boundaries or explicit calls in **`ollmapp/Window.vala`** / **`libocfiles/ProjectManager.vala`**. Label text is set **before** the call and cleared **after** (or on error).

| Step | Trigger (client) | Suggested label | Layer |
| ---- | ---------------- | --------------- | ----- |
| Ollama probe | **`config.check_connections()`** | *Connecting to LLM server…* | Overlay |
| Model verify | **`initialize_model`**, **`ensure_model_usage`** | *Verifying chat model…* | Overlay |
| Required models | **`ensure_required_models`** | *Checking tool models…* | Overlay |
| Daemon boot | **before** **`yield rpc.connect(hello)`** | *Starting filesystem daemon…* | Overlay |
| Daemon hello | **during** **`yield rpc.connect(hello)`** | *Connecting to filesystem daemon…* | Overlay |
| Project list (window boot) | **before/after** **`yield load_projects_from_db`** | *Loading project list…* | Bar (or overlay step 5) |
| Tool / agent wire | **before** **`agent_dropdown.wire()`** | *Preparing agents…* | Overlay step 5 |
| Overlay dismiss | **after** **`setup_chat_widget`** + first **`agent_dropdown.wire()`** | — | Overlay hides |
| Project list (agent pane) | **`AgentFactory.initialize_widget`** → **`load_projects_from_db`** | *Loading project list…* | Bar |
| Filesystem scan start | **`event.project.scan_start`** (Phase 3) | *Scan starting for \<project name\>…* | Bar |
| Filesystem scan end | **`event.project.scan_end`** (Phase 3) | *Scan ended — \<N\> files* | Bar → brief pause → hide |
| Vector index queue | **`event.vector.scan_update`** | *Indexing: \<file\> — X/Y* (today's text) | Bar ✅ |

**🚫** **No** per-step filesystem progress on the wire — scan is **~12s** today; **start + end** notifications only (**🔷** user requirement).

**🚫** Drop client-local *Opening project…* on **`activate_project`** send if daemon **`scan_start`** covers it.

## Phase 1 — Startup overlay ✔️

**🔷** Single overlay for the whole boot sequence — **`OLLMapp.BusyDialog`** (`ollmapp/BusyDialog.vala`) with public **`status_label`**; callers use **`present`/`close`** and set **`status_label.label`**.

### Step labels (ordered)

1. *Connecting to LLM server…*
2. *Verifying models…* (skip sub-step if fast / unchanged connection)
3. *Starting filesystem daemon…* / *Connecting to filesystem daemon…* (split if spawn vs socket distinguishable in **`ClientBoot`**)
4. *Loading workspace…* — covers **`load_projects_from_db`** + agent registration + **`agent_dropdown.wire()`**

### Dismiss rule

- **🔷** Hide overlay when **`initialize_client`** reaches **post-`wire()`** milestone (agents visible, **`new_chat_button.sensitive = true`**).
- **🔷** On **`rpc.connect`** failure — overlay closes, existing **`tool_error_banner`** (or alert) unchanged.
- **🚫** Do **not** wait for session restore or coding-agent **`restore_active_state`** — that is **Layer 2**.

### Touch points

| File | Change |
| ---- | ------ |
| **`ollmapp/Initialize.vala`** | Pass overlay / callback into **`run`** for Ollama + model steps (today show/hide around **`check_connections`** only) |
| **`ollmapp/Window.vala`** | Own overlay for **`initialize_client`** daemon + workspace steps |
| **`libocrpc/Client.vala`** / **`ClientBoot.vala`** | **💩** Optional stage signal (*spawning* vs *connecting*) — only if labels need split without guessing |

## Phase 2 — Repurpose status bar

**🔷** **`ActivityBanner`** is the daemon-activity bar (renamed from **`VectorScanBanner`**).

**🚫** No wrapper API — same rule as **`BusyDialog`**: public GTK children, callers set them from outside.

### Public surface (already exists today)

| Member | Caller use |
| ------ | ---------- |
| **`label`** | **`banner.label.label = "Loading project list…"`** |
| **`progress_bar`** | **`fraction`**, **`pulse`** (indeterminate while duration unknown) |
| **`revealer`** | Slide-down transition; use **`show()`** / **`hide()`** for visibility |

**ℹ️** **`ActivityBanner.update_scan_status`** / **`hide()`** exist today for vector index math — Phase 2 may **inline** that logic in **`Window.vala`** notification handlers and drop those methods, not add replacements.

**🚫** Do **not** add **`set_indeterminate`**, **`set_progress`**, **`clear`**, or other façade methods.

### Wiring

| Source | Handler |
| ------ | ------- |
| **`Window.initialize_client`** | Bar: *Loading project list…* around **`load_projects_from_db`** |
| **`rpc.notification`** | **`event.project.scan_start`** / **`event.project.scan_end`** (Phase 3) |
| **`rpc.notification`** | **`event.vector.scan_update`** — inline in **`Window.vala`**; labels *Indexing…* |
| **`AgentFactory.initialize_widget`** | Bar during second **`load_projects_from_db`** + **`restore_active_state`** |

**🔷** **Auto-hide:** **`ActivityBanner.hide()`** arms a fixed **2s** idle timer, then slides the revealer closed. **`show()`** cancels a pending hide. Callers set **`label`** / **`progress_bar`** directly; use **`show()`** when work starts and **`hide()`** when it ends.

**🔷** When vector index is active, scan-end message may show briefly then yield to index progress or hide on idle.

## Phase 3 — Project filesystem scan start / end (daemon)

**🔷** **`read_dir`** on project activate is **~12s** today — **no** mid-scan progress. Two notifications only: **start** when scan begins, **end** when it finishes with **file count**.

**🚫** Do **not** stream the file tree or emit per-folder / every-100-step updates.

### Wire shape

Reuse **`OLLMrpc.Notification.message`** (same as **`event.vector.scan_update`**):

| `method` | `object_type` | When | `message` |
| -------- | ------------- | ---- | --------- |
| **`event.project.scan_start`** | **`Project`** | Before **`read_dir`** in **`activate_project`** | `"<project_path>"` |
| **`event.project.scan_end`** | **`Project`** | After **`read_dir`** completes (before RPC reply) | `"<file_count> <project_path>"` |

Examples:

```json
{"jsonrpc":"2.0","method":"event.project.scan_start","object-type":"Project","message":"/home/alan/proj/myapp"}
```

```json
{"jsonrpc":"2.0","method":"event.project.scan_end","object-type":"Project","message":"1842 /home/alan/proj/myapp"}
```

**ℹ️** **`file_count`** — count from **`project.project_files`** (or equivalent) after **`update_from`** at end of scan; not a live walk counter.

### Client labels (`Window.vala` — public banner widgets)

| Event | **`label.label`** | **`progress_bar`** |
| ----- | ----------------- | ------------------ |
| **`scan_start`** | *Scan starting for \<basename\>…* | **`pulse = true`** (optional) |
| **`scan_end`** | *Scan ended — \<N\> files* | **`fraction = 1.0`** or hide bar track |
| After **2s** idle | — | **`hide()`** (unless **`show()`** arrived) |

### Daemon touch points

| File | Change |
| ---- | ------ |
| **`ollmfilesd/ProjectManager.vala`** | **`activate_project`**: broadcast **`scan_start`** before **`read_dir`**; **`scan_end`** after with file count |
| **`ollmfilesd/Application.vala`** | **`broadcast`** (existing) |

**🚫** No changes inside **`Folder.read_dir`** loop for progress — emit only at activate boundaries.

### Client touch points

| File | Change |
| ---- | ------ |
| **`ollmapp/Window.vala`** | Handle **`scan_start`** / **`scan_end`** in **`rpc.notification`**; **`activity_banner.show()`** / **`hide()`** |
| **`ollmapp/ActivityBanner.vala`** | **`show()`**, **`hide()`** with fixed **2s** idle auto-hide |

Register both methods in [`2.10.4.1-ollmfilesd-rpc-api.md`](2.10.4.1-ollmfilesd-rpc-api.md) **`event.*`** row when implemented.

## LLM implementer guardrails

- **🚫** No new RPC request methods for status — use notifications + client hooks only.
- **🚫** No per-client routing in this tranche — broadcast **`scan_update`** behaviour unchanged.
- **🚫** Do not block **`initialize_client`** on vector indexing or filesystem scan completion.
- **🚫** Do not merge overlay and bar into one widget — two layers per **Target UX**.
- **🚫** Do not require **`ollmfilesd`** when **`disable_filesd`** — no spawn attempt, no error banner for missing daemon.
- **🚫** Do not register **Coding assistant** / **SkillRunner** / file tools without a live PM + daemon.
- **🚫** Filesystem scan — **start + end only**; no **`filesystem_scan_update`** or per-step throttle.
- **🚫** Status bar — **public widgets only**; no new setter/helper methods on the banner class.
