## Purpose

- **🔷** Port **`libocwebkit`** (from **5.0**) to **Android** (WebView + accessibility + globe toggle in the Android bottom bar).
- **🔷** Same tool contract as 5.0: `help` / `fetch` / `search` / `press` / current-page → accessibility markdown; one browser per chat; session cookies.
- **⏳** Start after 5.0 has a stable **`libocwebkit`** API (proven via **`libocwebkit/examples/oc-test-webkit`** on Linux), so Android implements against that surface rather than inventing a parallel one.

---

## Platform notes

- **🔷** **5.0** is **Linux-first** (WebKitGTK a11y spike).
- **🔷** **Windows** a11y (webview2-gtk work) is **[`5.0.1`](5.0.1-windows-webkit-accessibility.md)**.
- **🔷** This plan is **Android only** — do not block 5.0 Linux on Android WebView work.
- **💩** Prefer an Android backend inside **`libocwebkit`** (same tool + **`Browser` / `BrowserStack`** API as desktop) rather than a second tool name or an Android-only tool class.

---

## Likely work (backlog)

- **⏳** **🔷** Android WebView host owned by the chat session (Pixiewood / JNI as needed).
- **⏳** **🔷** Accessibility tree → same markdown contract as 5.0 (`AccessibilityNodeInfo` or equivalent).
- **⏳** **🔷** `search`: a11y fill Google + submit (no JS SERP scrape).
- **⏳** **🔷** Globe toggle on Android bottom bar next to tool selector; show WebView on CF / challenge.
- **⏳** **🔷** Session cookie jar per chat on Android.
- **⏳** **🔷** CF / challenge handoff adapted to Android WebView response hooks (Snappr Linux/Windows paths are not copy-paste).
- **⏳** **🔷** Register / enable `browser` tool on Android agent tool sets once host works.
