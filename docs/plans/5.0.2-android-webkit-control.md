## Purpose

- **🔷** Wire **`libocwebkit`** on Android against **`webkitgtk-android`** (not a hand-rolled Pixiewood/JNI WebView inside chat).
- **🔷** Same tool contract as Linux **5.0**: `help` / `fetch` / `search` / `press` / `download` / `whereami` → a11y markdown; one browser per chat; session cookies.
- **🔷** Same chrome contract as **5.0.4**: `UiWidgets` + `ChatWidget.view_stack` globe toggle in the Android shell.
- **🔷** Download parity with **[`done/5.0.5`](done/5.0.5-DONE-webkit-download.md)** using the backend’s WebKit-shaped `Download` API (**1.1**).

---

## Split of responsibility (locked)

| Layer | Owns | Does **not** own |
| ----- | ---- | ---------------- |
| **`webkitgtk-android`** | `WebView` widget, host JNI, a11y dump/fill/press primitives, cookies, download *engine*, phone-sized background viewport | Tool wire, markdown parse, `ChatPermission`, activity banner, globe UI |
| **This plan (OLLMchat)** | `#if ANDROID` in `libocwebkit`, `A11y` / CF / settle call sites, Registry / `AndroidToolsRegistration`, `OllmchatWindow` §8 wire, permission + Downloads folder policy | Native WebView embed |

**🔷** Pattern mirrors Snappr / **5.0.1**: `#if ANDROID` → `using WebKitGtkAndroid;` else WebKit / webview2-gtk.
