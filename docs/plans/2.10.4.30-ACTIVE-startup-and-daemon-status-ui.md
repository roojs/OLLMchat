## Purpose

- **рҹ”·** Extend the **startup blocking overlay** (today **`CheckingConnectionDialog`**) so it stays up through **Ollama verification** and **`ollmfilesd` RPC connect**, with **wording that changes per step**, and dismisses only when the **main shell is stable** (agent dropdown populated, chat shell wired).
- **рҹ”·** Repurpose **`ActivityBanner`** (renamed from **`VectorScanBanner`**) as a **non-blocking daemon-activity status bar** for work that happens **after** the overlay dismisses вҖ” project list load, project activate / filesystem scan, vector index queue.
- **рҹ”·** Produce an explicit **inventory** of what status text we can drive **today** vs what needs **new daemon notifications**.
- **рҹ”·** Add a formal **`disable_filesd`** switch so builds that **cannot** spawn **`ollmfilesd`** (Android) or **do not need** project workspace (chat-only) skip daemon boot cleanly вҖ” document what breaks without the daemon.
- **рҹ”·** Final polish tranche for the **2.10.4 RPC migration** вҖ” UI feedback for daemon boot and early project work that is invisible today.

## `disable_filesd` вҖ” flag and implications

### Android вҖ” your assumption is correct

- **рҹ”·** The Android POC **cannot** usefully spawn **`ollmfilesd`** on-device (no packaged daemon binary, no Unix socket boot path, sandbox limits).
- **рҹ”·** Android **does not need** local **`ollmfilesd`** for the current POC вҖ” chat + **`web_fetch`** / **`session_fetch`** / **`google_search`** only ([`9.0-android-poc-summary.md`](9.0-android-poc-summary.md) **Tools** table).
- **вң…** Today: **`initialize_client`** on Android skips daemon entirely вҖ” LLM connect вҶ’ agents вҶ’ chat. **No** **`ProjectManager`** RPC client.

**в„№пёҸ** **Later** Android could connect to a **remote** **`ollmfilesd`** on the dev machine ([`9.0`](9.0-android-poc-summary.md) *Later* bullet) вҖ” that is **not** `disable_filesd`; it is a different transport target.

### Proposed flag shape

| Mechanism | Where | Default |
| --------- | ----- | ------- |
| **Meson** **`files_daemon`** | **`meson_options.txt`** вҖ” `false` for Android APK target | `true` on Linux desktop |
| **Compile-time** | **`libocfiles`** / **`ollmapp`** вҖ” when `false`, **рҹҡ«** link **`ClientBoot`** spawn path; optional stub PM | вҖ” |
| **Runtime** (рҹ’© optional) | **`config.2.json`** key e.g. **`files_daemon.enabled`** | Desktop `true` |

**рҹ”·** Minimum for this tranche: **Meson / platform** flag so **`initialize_client`** branches like Android today вҖ” **not** a user-facing Settings toggle unless requested.

**рҹ’©** Env override **`OLLM_DISABLE_FILESD=1`** for dev chat-only desktop runs вҖ” confirm before build.

### Overlay / status bar when `disable_filesd`

| Layer | With daemon | Without daemon |
| ----- | ----------- | -------------- |
| Overlay step 3вҖ“4 | *Starting / connecting filesystem daemonвҖҰ* | **Skipped** |
| Overlay step 5 | *Loading workspaceвҖҰ* (`load_projects_from_db` + wire) | *Preparing chatвҖҰ* (agents + tools only) |
| Status bar (Layer 2) | Project load, scan, index | **Hidden / unused** вҖ” no daemon RPC, no **`scan_update`** |

### What works without `ollmfilesd`

| Feature | Works? | Notes |
| ------- | ------ | ----- |
| LLM chat (**`chatter`** agent) | **вң…** | Same as Android POC |
| History / sessions | **вң…** | Local SQLite under app data dir |
| **`web_fetch`**, **`session_fetch`**, **`google_search`** | **вң…** | No project paths required |
| Settings (connections, models) | **вң…** | Unchanged |
| Agent dropdown | **вң…** | Only agents registered without PM вҖ” today **`chatter`** only on Android |

### What does **not** work without `ollmfilesd`

Everything that goes through **`libocfiles.ProjectManager`** RPC or daemon-owned **`files.sqlite`**:

| Feature | Broken because |
| ------- | -------------- |
| **Project list** / create / remove | **`ProjectManager.load_projects_from_db`**, **`create_project`**, вҖҰ вҖ” daemon DB |
| **Activate project** / file tree | **`activate_project`** вҶ’ daemon **`read_dir`** |
| **Coding assistant** (**`CodeAssistant`** / **`SourceView`**) | **`AgentFactory`** needs real PM вҖ” **`restore_active_state`**, editor buffers via **`File.*`** |
| **Skill runner** (Conductor) | Same вҖ” project-scoped skills |
| **File tools** вҖ” **`read_file`**, **`write_file`**, **`edit_mode`** | Need workspace paths + daemon I/O |
| **`codebase_search`** | Daemon FAISS + vector pipeline |
| **`run_command`** / sandbox (future **`Bubble.*`**) | Daemon execution + overlay scan |
| **Approvals** / pending file history | **`Folder.fetch_pending_approvals`**, daemon SQL |
| **File-change banner** | Compares buffer to daemon-backed file state |
| **Vector / scan status bar** | No **`event.vector.scan_update`** source |
| **Session `project_path` restore** | **`AgentDropdown.session_selection`** вҶ’ **`activate_project`** вҖ” no-op or error without PM |

**рҹ”·** In short: **without the file daemon you lose the project workspace** вҖ” not just вҖңproject manager UIвҖқ, but **all** coding-agent, file-tree, vector, and filesystem-backed tool surfaces. Pure chat remains fine.

### Desktop today vs target

| | Desktop **`Window.vala`** today | Target with **`disable_filesd`** |
| - | ------------------------------- | -------------------------------- |

| **`project_manager`** | Real **`OLLMfiles.ProjectManager`** + RPC | **Null** or no-op stub (mirror Android) |
| Agents registered | **`chatter`** + **`CodeAssistant`** + **SkillRunner** | **`chatter`** only (рҹ’© unless remote daemon added later) |
| Tools | Full **`tools_registry.fill_tools(pm)`** | Subset registration (Android pattern) |

### Touch points (flag)

| File | Change |
| ---- | ------ |
| **`meson_options.txt`** / **`ollmapp/meson.build`** | **`files_daemon`** option; Android build sets **`false`** |
| **`ollmapp/Window.vala`** | Branch **`initialize_client`** вҖ” daemon overlay steps + **`rpc.connect`** only when enabled |
| **`libocfiles/ProjectManager.vala`** | **рҹ’©** **`files_daemon`** compile guard or stub interface вҖ” avoid linking spawn on Android |
| **`ollmapp/android/OllmchatWindow.vala`** | Align with shared overlay API; keep implicit **`disable_filesd`** |

**рҹҡ«** Do **not** fail boot when **`disable_filesd`** and user picks **Coding assistant** вҖ” hide or disable that agent when PM unavailable (same as Android today: only **`chatter`** in dropdown).

## Phase 3 вҖ” Project filesystem scan start / end (daemon)

**рҹ”·** **`read_dir`** on project activate is **~12s** today вҖ” **no** mid-scan progress. Two notifications only: **start** when scan begins, **end** when it finishes with **file count**.

**рҹҡ«** Do **not** stream the file tree or emit per-folder / every-100-step updates.

### Wire shape

Reuse **`OLLMrpc.Notification.message`** (same as **`event.vector.scan_update`**):

| `method` | `object_type` | When | `message` |
| -------- | ------------- | ---- | --------- |
| **`event.project.scan_start`** | **`Project`** | Before **`read_dir`** in **`activate_project`** | `"<project_path>"` |
| **`event.project.scan_end`** | **`Project`** | After **`read_dir`** completes (before RPC reply) | `"<file_count> <project_path>"` |

Examples:

```json
{"jsonrpc":"2.0","method":"event.project.scan_start","object-type":"Project","message":"/home/alan/proj/myapp"}
```

```json
{"jsonrpc":"2.0","method":"event.project.scan_end","object-type":"Project","message":"1842 /home/alan/proj/myapp"}
```

**в„№пёҸ** **`file_count`** вҖ” count from **`project.project_files`** (or equivalent) after **`update_from`** at end of scan; not a live walk counter.

### Client labels (`Window.vala` вҖ” public banner widgets)

| Event | **`label.label`** | **`progress_bar`** |
