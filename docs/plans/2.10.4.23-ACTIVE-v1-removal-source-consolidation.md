## Purpose

**2.10.4.21** flipped Meson to compile **V2 copies** under parallel paths (`libocfiles/V2/*`, `liboccoder/V2/*`, `*V2/` tool dirs, `ollmapp/V2/Window.vala`) while **shipping V1** remained on disk for **`-Dv2testing=false`**. That was intentional scaffolding.

**рџ”·** This plan **removes scaffolding**:

1. **Promote V2 sources** to their **canonical paths** (no more `V2/` / `*V2/` suffix dirs for production code).
2. **Delete all unused V1** sources (in-process SQLite PM, **`Sandbox/*`**, shipping **`Window.vala`**, full **`libocvector`** FAISS tree, вЂ¦).
3. **Remove the `v2testing` Meson option** and every `if get_option('v2testing')` branch.
4. **Install as `libocfiles.so`** again (not **`libocfiles2.so`** alongside v1).
5. **App / CLI / packaging cleanup** вЂ” dead options, docs, debian split names if needed.

**в„№пёЏ** After this plan, there is **no** way to build the old in-process file/index stack from this repo.

### Display vs domain split

- **Daemon (`ollmfilesd/`)** вЂ” derived checks as **methods**, not GObject computed **properties**
- **Client (`libocfiles/`)** вЂ” keep computed **properties** for Gtk binding (`path_basename`, `icon_name`, вЂ¦)

| Surface | Client | Daemon | Wire |
|---------|--------|--------|------|
| `path_basename` | computed property | method / inline | **рџљ«** not sent |
| `icon_name` | computed property | **рџљ«** not on type | **рџљ«** not sent |
| `display_with_indicators` | computed property | **рџљ«** not on type | **рџљ«** not sent |
| `is_alias` | computed property | **вЏі** method | **рџљ«** not sent |
