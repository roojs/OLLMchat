## Design revisions (approved at merge)

The original sub-plan forbade helpers on **`Base`**, **`meson.build`** changes, and any wiring. At review we **approved** the following deviations — they are intentional, not drift:

### **`Action.Base` extends `OLLMchat.Agent.Base`**

- **🔷** Resolves **`protected fill_model`** / **`chat_call`** for **`PostExamMerge.run`** and future **`run()`** wiring.
- **🔷** Constructor: **`base(task.runner.sr_factory, task.session)`** then **`replace_chat(task.chat())`** so the action shares the task’s live chat, not a orphan **`ChatCompletions`**.
- **ℹ️** Instantiating an Action only to call **`extract_result`** / **`extract`** still runs this constructor — acceptable overhead; the Action shell matches what full **`run()`** dispatch will use later.

### **Executor extraction on `Action.Base` (partial move from `ResultParser`)**

- **🔷** **`Base.extract_result(parser, ex)`** — ref-only / default executor parse (was **`ResultParser.exec_extract`** without the **`write_file`** branch).
- **🔷** **`Base.extract_tool(parser)`** — legacy/test synthetic exec run (was **`ResultParser.extract_exec`**).
- **🔷** **`WriteExec.extract_result`** — override for **`write_file`** Change-details + Path 2 (was the **`write_file`** branch inside **`exec_extract`**).
- **🔷** **`PostExamMerge.extract(parser)`** — post-exec synthesis parse (was **`ResultParser.exec_post_extract`**).
- **🔷** **`Base.fill_model`** — skill **`model:`** header override (same logic as **`Details.fill_model`**; **`PostExamMerge.run`** uses **`this.fill_model()`** / **`this.chat_call`**).

**Call sites wired (extraction only):**

| Caller | Dispatch |
| --- | --- |
| **`Task.Tool.run`** | **`WriteExec`** vs **`RefOnly.extract_result`** by **`write_file`** in skill |
| **`Details.run_post_exec`** | **`PostExamMerge.extract`** (body of **`run_post_exec`** still on **`Details`**) |
| **`Skill.Runner`** replay | same Action extract dispatch as live |
| **`examples/oc-test-skill-agent.vala`** | **`RefOnly.extract_tool`** |

**Removed from `ResultParser`:** **`exec_extract`**, **`exec_post_extract`**, **`extract_exec`**. Planning/refinement parsers unchanged.

### **Build**

- **🔷** All six Action sources added to **`liboccoder/meson.build`** (after **`Skill/Runner.vala`**, before **`Task/ResultParser.vala`**).

### **Still not done ( → 7.16.1b )**

- **⏳** Duplicate **`run_exec`** / **`run_post_exec`** / **`post_exec_prompt`** bodies remain on **`Details`**.

- **🔷** Lift **`run_exec`** / **`run_post_exec`** / **`post_exec_prompt`** bodies verbatim; replace **`this.`** with **`this.task.`** where the field lived on **`Details`**.

- **🔷** **`ExamSequence`** vs **`RunTools`** vs **`RefOnly`** vs **`WriteExec`** — each runner keeps its own **`run()`** copy for dispatch clarity at wire time.

## Runners (what was copied)

### `Base`

- **`protected Task.Details task`**
- **`public abstract async void run () throws Error`**
- **`extract_result`**, **`extract_tool`**, **`fill_model`** — see **Design revisions**

### `PostExamMerge`

- Copied from **`Details.run_post_exec`** + **`Details.post_exec_prompt`** (private on this class).
- **`extract(parser)`** — post-exec response parse (also called from **`Details.run_post_exec`** until full wire).

### `ExamSequence` — path **A**

- Copied from **`Details.run_exec`** (exam queue built by **`build_run_queue`** before dispatch — still on **`Details`**).

### `RunTools` — path **B** (legacy)

- Same **`run_exec`** copy as **`ExamSequence`**; dispatches to **`PostExamMerge`** when **`tools().size > 1`**.

### `RefOnly` — path **C**

- Same **`run_exec`** copy; happy path is single lone **`exec`** row.

### `WriteExec` — path **D**

- Same **`run_exec`** copy; **`write_file`** parse in **`extract_result`** override; tool execution stays in **`Tool.run`**.
