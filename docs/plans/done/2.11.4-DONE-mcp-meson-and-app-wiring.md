# 2.11.4. MCP Meson and App Wiring

Parent: [2.11-DONE-mcp-loader-tool.md](2.11-DONE-mcp-loader-tool.md)

**Status:** ✅ done (archived 2026-05-26). Build completed with [2.11.6](2.11.6-DONE-mcp-first-compile-vala-fixes.md).

**Checklist:** `docs/coding-standards.md` — *Checklist for all plans* (minimal diff, no speculative helpers, match existing registry/meson patterns).

## Purpose

- Move the **bubblewrap / overlay / seccomp** stack from **`liboctools/RunCommand/`** into **`libocfiles/Sandbox/`** as namespace **`OLLMfiles.Sandbox`** (project-files shaped; shared by **`run_command`** and MCP stdio).
- Compile **`libocmcp/`** for the first time (phases 2.11.1–2.11.3 and 2.11.5 source is on disk but not in meson).
- Wire **`OLLMmcp.Registry`** into **`ollmapp`** and **`ollmchat-cli`** the same way as **`OLLMtools.Registry`** and **`OLLMvector.Registry`**.
- After this phase, a full **`meson compile`** succeeds and MCP tools from **`~/.config/ollmchat/mcp.json`** appear in the agent tool list when servers are enabled.

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| **`libocfiles/Sandbox/`** — move **`Bubble`**, **`Overlay`**, **`Scan`**, **`RunSeccomp`**; namespace **`OLLMfiles.Sandbox`**; extend **`libocfiles/meson.build`** (seccomp, gio-unix, `seccomp-fd-pass.c`) | MCP settings UI in Config2 / ToolsPage |
| Trim **`liboctools/meson.build`** — remove moved sources; **`RunCommand.Request`** uses **`OLLMfiles.Sandbox.Bubble`** | Changing Loader / Client / Registry / sandbox policy logic |
| New **`libocmcp/meson.build`** — **`libollmchat` + `libocfiles` only** (no **`liboctools`**) | Unit tests (manual integration per **Completion**) |
| **`libocmcp/Client/Stdio.vala`** — **`OLLMfiles.Sandbox.*`** instead of **`OLLMtools.RunCommand.*`** | |
| Root **`meson.build`**: `subdir('libocmcp')`, wrapper **`LD_LIBRARY_PATH`** | |
| **`ollmapp/meson.build`**: link **`ocmcp`** for **`ollmchat`** and **`ollmchat-cli`** | |
| **`Application.vala`**, **`Window.vala`**, **`ollmchat-cli.vala`** — MCP registry wiring | |
| **`examples/oc-test-bubble.vala`** + **`examples/meson.build`** — **`OLLMfiles.Sandbox`**; drop **`octools`** dep if unused | |
| **`docs/meson.build`** — valadoc paths for moved Sandbox sources under **`libocfiles`** | |

## Acceptance criteria

- `meson setup build && meson compile -C build` succeeds with **`libocmcp.so`** produced.
- **`libocmcp`** links **`libollmchat`** + **`libocfiles`** only — **not** **`liboctools`** (sandbox types come from **`ocfiles`** vapi).
- **`liboctools`** still builds **`run_command`** via **`OLLMfiles.Sandbox`** (same behaviour as before the move).
- **`ollmchat`** and **`ollmchat-cli`** link against **`ocmcp`** (vapi + rpath).
- App constructs **`mcp_registry`**, calls **`init_config()`** before loading config, **`setup_config_defaults(config)`** when config is loaded (same order as tools/vector).
- **`Window.initialize_client`** calls **`app.mcp_registry.fill_tools(history_manager, project_manager)`** immediately after tools and vector **`fill_tools()`**.
- **`ollmchat-cli`** calls **`mcp_registry.fill_tools()`** in the same places as tools/vector (**`run_test`**, **`run_agent_tool`**).
- With a valid **`mcp.json`** (e.g. Chrome stdio, **`network: true`**), MCP tools show as **`mcp:{server_id}:{tool_name}`** and at least one **`tools/call`** works.

## Dependency tree (meson + types)

**Short answer:** Sandbox lives in **`libocfiles`** as **`OLLMfiles.Sandbox`**. **`libocmcp` depends on `libollmchat` + `libocfiles` only** (plus GLib/Gee/JSON/Soup; seccomp via **`libocfiles`**). **`liboctools` does not depend on `libocmcp`**. **`liboctools` still depends on `libocfiles`** for **`run_command`** sandbox.

### Meson link order (this phase)

- **`libocfiles.so`** (extended)
  - existing: sqlite, tree-sitter, …
  - **add:** gio-unix, libseccomp, **`vapi/seccomp-fd-pass.c`**
  - **sources:** `Sandbox/Scan.vala`, `Sandbox/Overlay.vala`, `Sandbox/Bubble.vala`, `Sandbox/RunSeccomp.vala`

- **`liboctools.so`** (trimmed)
  - **`libollmchat`**, **`libocfiles`** (gains Sandbox vapi)
  - **remove:** RunCommand sandbox `.vala` + seccomp C from this library
  - **keep:** `RunCommand/Tool.vala`, `RunCommand/Request.vala` (Request calls **`OLLMfiles.Sandbox.Bubble`**)

- **`libocmcp.so`**
  - **`libollmchat`**, **`libocfiles`** (stdio: **`OLLMfiles.Sandbox.Bubble`** / **`RunSeccomp`**)
  - **not linked:** `liboctools`, `libocvector`, `liboccoder`

- **`ollmchat` / `ollmchat-cli`**
  - existing libs + **add `libocmcp`**

### `OLLMfiles.Sandbox` (moved from `OLLMtools.RunCommand`)

| Class | File | Role |
| ----- | ---- | ---- |
| **`Bubble`** | `libocfiles/Sandbox/Bubble.vala` | bwrap argv, `exec()`, `build_bubble_args()`, `can_wrap()` |
| **`Overlay`** | `libocfiles/Sandbox/Overlay.vala` | overlay dirs; uses **`Folder.build_roots()`** |
| **`Scan`** | `libocfiles/Sandbox/Scan.vala` | post-run overlay → live FS + **`ProjectFiles`** / **`FileHistory`** |
| **`RunSeccomp`** | `libocfiles/Sandbox/RunSeccomp.vala` | NOTIFY + evidence; holds **`unowned Bubble`** |

**Stays in `OLLMtools.RunCommand`:** **`Tool`**, **`Request`** (chat tool surface; uses **`OLLMfiles.Sandbox.Bubble`** only).

**Existing `OLLMfiles` types used (unchanged location):** **`Folder`**, **`File`**, **`FileAlias`**, **`FileBase`**, **`FileHistory`**, **`ProjectFiles`**, **`ProjectManager`** (via **`Scan`**).

### What `libocmcp` uses

- **`OLLMchat`** — `History.Manager`, `Tool.BaseTool`, `Tool.RequestBase`, `Settings.Config2`
- **`OLLMfiles`** — `ProjectManager`, `Folder?` on stdio spawn
- **`OLLMfiles.Sandbox`** (**stdio only**)
  - `Bubble.can_wrap()`, `Bubble`, `RunSeccomp` — same usage as [2.11.5](2.11.5-DONE-mcp-security-sandbox-and-seccomp.md)
- **`Client.Http`** — no Sandbox

### `OLLMmcp` internal layout (unchanged)

`Registry` → `Loader` → `Client` / `Factory` / `Tool` / `Request` / `Config` / `McpTypes` (see prior plans 2.11.1–2.11.3).

### 7. `libocmcp/meson.build` — new library (no `liboctools`)

**Why:** First compile of **`libocmcp/*.vala`**; stdio sandbox via **`ocfiles`** vapi only.

**Where:** New file **`libocmcp/meson.build`**.

**Depends on:** §1–§4.

#### Add — entire file

```meson
# Build ocmcp library (libocmcp.so) - depends on libollmchat, libocfiles

valac = meson.get_compiler('vala')

ocmcp_deps = [
  dependency('gee-0.8'),
  dependency('gio-2.0'),
  dependency('glib-2.0'),
  dependency('gobject-2.0'),
  dependency('json-glib-1.0'),
  dependency('libsoup-3.0'),
  valac.find_library('posix'),
]

ocmcp_src = files([
  'McpTypes.vala',
  'Config.vala',
  'Client/Base.vala',
  'Client/Stdio.vala',
  'Client/Http.vala',
  'Factory.vala',
  'Tool.vala',
  'Request.vala',
  'Loader.vala',
  'Registry.vala',
])

lib_build_rpath = ':'.join([
  meson.current_build_dir(),
  meson.current_build_dir() / '..' / 'libocfiles',
  meson.current_build_dir() / '..' / 'libollmchat',
])

ocmcp_base_lib = library('ocmcp',
  dependencies: [
    ocmcp_deps,
    ollmchat_vapi_dep,
    ocfiles_vapi_dep,
  ],
  sources: ocmcp_src,
  link_with: [ollmchat_base_lib],
  vala_header: 'ocmcp.h',
  vala_vapi: 'ocmcp-meson.vapi',
  build_rpath: lib_build_rpath,
  include_directories: [
    include_directories('..' / 'libollmchat'),
  ],
  vala_args: [
    '--pkg=ollmchat',
    '--pkg=ocfiles',
    '--pkg=json-glib-1.0',
    '--pkg=libsoup-3.0',
    '--vapidir', meson.current_build_dir() / '..' / 'libocfiles',
    '--vapidir', meson.current_build_dir() / '..' / 'libollmchat',
  ],
  install: true
)

ocmcp_vapi = custom_target('ocmcp-vapi',
  output: 'ocmcp.vapi',
  input: ocmcp_src,
  command: [
    valac_exe,
    '-C', '--debug',
    '--target-glib=auto',
    '--vapidir', meson.current_build_dir() / '..' / 'libocfiles',
    '--vapidir', meson.current_build_dir() / '..' / 'libollmchat',
    '--vapidir', '/usr/share/vala/vapi',
    '--pkg', 'posix',
    '--pkg', 'json-glib-1.0',
    '--pkg', 'libsoup-3.0',
    '--pkg', 'ollmchat',
    '--pkg', 'ocfiles',
    '--pkg', 'gobject-2.0',
    '--pkg', 'glib-2.0',
    '--pkg', 'gio-2.0',
    '--pkg', 'gee-0.8',
    '--library', 'ocmcp',
    '--header', meson.current_build_dir() / 'ocmcp.h',
    '--vapi', '@OUTPUT@',
    '@INPUT@'
  ],
  depends: [ocmcp_base_lib],
  build_by_default: true,
  install: true,
  install_dir: get_option('datadir') / 'vala' / 'vapi'
)

ocmcp_vapi_dep = declare_dependency(
  link_args: ['-L' + meson.current_build_dir(), '-locmcp'],
  sources: [ocmcp_vapi[0]],
  dependencies: [ollmchat_vapi_dep, ocfiles_vapi_dep]
)
```

ℹ️ **`Client.Stdio`** uses gio-unix/seccomp types transitively through **`--pkg=ocfiles`** (compiled into **`libocfiles`**). No duplicate **`seccomp-fd-pass.c`** on **`libocmcp`**.
