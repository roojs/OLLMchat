# 7.14.9 Task progress row title = `BaseTool.title` only

**Status:** done

**Verified:** `liboccoder/Task/Tool.vala` — `ProgressItem.title` returns `this.parent.runner.session.manager.tools.get(key).title` with `key = this.name` or `this.tool_call.function.name`.

**Pointer — `.cursor/rules/CODING_STANDARDS.md` (`## Checklist for all plans`):** avoid defensive null branches unless justified; avoid new methods; **`docs/guide-to-writing-plans.md`** workflow.

## Purpose

- Progress tree rows use **`manager.tools[name].title`** — the registered tool’s {@link OLLMchat.Tool.BaseTool.title}.
- **⚠️** No fallback to raw function name or placeholders.
- **⚠️** No extra guards on parent/runner/session/manager: invariant is that **`Task.Tool`** rows are used only when that chain is valid.

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| Thin `title` getter: registry lookup + **`return … .title`** | Helpers, `Function.label`, `friendly_title`, etc. |
| Key = `this.name` if set, else `tool_call.function.name` | Exam / skill-list titling |

## Acceptance criteria

- **`title`** equals **`session.manager.tools.get(key).title`** where `key` resolves from **`name`** or **`tool_call.function.name`**.
- **No** branching that returns `""` for missing registry/parent unless the user later requires it.

## Implemented code (`liboccoder/Task/Tool.vala`)

```vala
		/**
		 * Progress tree label: {@link OLLMchat.Tool.BaseTool.title} for the tool registered on the session.
		 */
		public string title {
			owned get {
				var key = this.name != "" ? this.name : this.tool_call.function.name;
				return this.parent.runner.session.manager.tools.get(key).title;
			}
		}
```

## Notes

- **ℹ️** Wrapped tools: whatever **`BaseTool.title`** returns on the registered instance is what appears; registration owns the label.
- **👎** Do not re-layer null checks or `has_key` unless a real boundary test proves a crash.
