### 2 — `libocmcp/McpTypes.vala` — `JsonRpcRequest`

**🔷 Design:** `JsonRpcRequest` is a normal Vala object — **separate properties** per MCP param shape. Wire JSON still has one `"params"` key; `serialize_property` maps the right property into that key.

| Method | Property on `JsonRpcRequest` | Type |
| ------ | ---------------------------- | ---- |
| `initialize` | `params_initialize` | `InitializeParams?` |
| `tools/list` | *(leave both null)* | → `{}` on wire |
| `tools/call` | `params_tools_call` | `CallToolParams?` |

**🚫** Call sites do not assign `params` or `Json.Object`. No `params_json` parse loop.

**ℹ️** Keep GObject property `params` (private set, never assigned) so json-glib invokes `serialize_property` for wire key `"params"`. Typed fields are set instead; `params_initialize` / `params_tools_call` are **blocked** from appearing as top-level JSON keys (`return null`).

**ℹ️** `Json.Object` stays on `Factory.inputSchema`, `CallToolParams.arguments`, `Request.arguments` (dynamic JSON) — issue 4 is `set_boxed` only.

**Where:** class `JsonRpcRequest`.

##### Part 1 — replace `params` with typed properties

#### Keep

```vala
		public string method { get; set; default = ""; }
```

#### Remove

```vala
		/**
		 * Request params: any Json.Serializable (e.g. InitializeParams, CallToolParams) or Json.Object for raw.
		 * Serialization maps to/from the "params" JSON key.
		 */
		public Object? params { get; set; default = null; }
```

#### Replace with

```vala
		/** Set for `initialize` only; serialized to JSON `"params"`. */
		public InitializeParams? params_initialize { get; set; default = null; }
		/** Set for `tools/call` only; serialized to JSON `"params"`. */
		public CallToolParams? params_tools_call { get; set; default = null; }
		/** Wire key `params` only — do not assign; use typed fields above. */
		public Object? params { get; private set; default = null; }
```

##### Part 2 — `serialize_property`: `switch`, block typed fields, emit `"params"`

#### Remove

```vala
		public override Json.Node serialize_property(string property_name, Value value, ParamSpec pspec)
		{
			if (property_name == "params") {
				if (this.params == null) {
					var empty = new Json.Node(Json.NodeType.OBJECT);
					empty.set_object(new Json.Object());
					return empty;
				}
				if (this.params is Json.Object) {
					var node = new Json.Node(Json.NodeType.OBJECT);
					node.set_object((Json.Object) this.params);
					return node;
				}
				return Json.gobject_serialize(this.params);
			}
			return default_serialize_property(property_name, value, pspec);
		}
```

#### Replace with

```vala
		public override Json.Node serialize_property(string property_name, Value value, ParamSpec pspec)
		{
			switch (property_name) {
			case "params_initialize":
			case "params_tools_call":
				return null;
			case "params":
				if (this.params_initialize != null) {
					return Json.gobject_serialize(this.params_initialize);
				}
				if (this.params_tools_call != null) {
					return Json.gobject_serialize(this.params_tools_call);
				}
				var empty = new Json.Node(Json.NodeType.OBJECT);
				empty.set_object(new Json.Object());
				return empty;
			default:
				return default_serialize_property(property_name, value, pspec);
			}
		}
```

**Why `return null` on typed fields:** same pattern as `libollmchat/Response/Chunk.vala` — omit those properties from top-level JSON; only wire `"params"` (via `case "params"` + `method`) carries the payload.

**ℹ️** No `switch (this.method)` on `"params"` — call sites set only the matching typed field; `tools/list` leaves both null → `{}`.

##### Part 3 — delete `deserialize_property` override

Same **Keep** / **Remove** as before (client only sends requests; remove broken inbound `params` handler).

### 3 — `libocmcp/Client/Stdio.vala`

**Depends on:** §2. **`jrequest`** takes a built **`JsonRpcRequest`** (caller sets `id`, `method`, and the right typed property).

##### Part 1 — `jrequest`: take `JsonRpcRequest`, drop parse/build junk

#### Remove

```vala
		private async Json.Node? jrequest(string method, string? params_json = null) throws Error
		{
			Json.Object? params_obj = null;
			if (params_json != null && params_json != "") {
				var p = new Json.Parser();
				try {
					p.load_from_data(params_json, -1);
					var params_node = p.get_root();
					if (params_node != null && params_node.get_node_type() == Json.NodeType.OBJECT) {
						params_obj = params_node.get_object();
					}
				} catch (GLib.Error e) {
					params_obj = null;
				}
			}
			var req = new OLLMmcp.JsonRpcRequest() {
				id = (int) this.next_id++,
				method = method,
				params = params_obj
			};
			yield this.write(Json.to_string(Json.gobject_serialize(req), false));

			var response = yield this.read_jresponse();
			this.check_jerr(response);
			return response;
		}
```

#### Replace with

```vala
		private async Json.Node? jrequest(OLLMmcp.JsonRpcRequest req) throws Error
		{
			yield this.write(Json.to_string(Json.gobject_serialize(req), false));

			var response = yield this.read_jresponse();
			this.check_jerr(response);
			return response;
		}
```

### 7 — `libocmcp/Client/Stdio.vala` — `disconnect` SIGTERM

**Where:** `disconnect()`, `if (this.process != null)` block.

**ℹ️** `Gio.Subprocess.send_signal(int)` — `GLib.SubprocessSignal` is not in the Vala vapi. Use `Posix.Signal.TERM` (named SIGTERM). `g_subprocess_send_signal` is race-free and no-ops if the child already exited; no `try/catch` or `Posix.kill` on `get_identifier()`.

#### Remove

```vala
				this.process.send_signal(GLib.SubprocessSignal.TERM);
```

#### Replace with

```vala
				this.process.send_signal(Posix.Signal.TERM);
```

**🚫** `force_terminate()` unless SIGKILL is explicitly required.
