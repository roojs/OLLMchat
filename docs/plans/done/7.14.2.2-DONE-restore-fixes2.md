## Correct behaviour to implement

- **⚠️** Replay must **simulate the running session**: apply the **same** **`Details` / `Tool`** field updates live code applies, **at the same points** (after each successful parse / run), so **`on_replay`** is not a separate “restore policy” with end-of-step guards.
- **⚠️** After **`extract_refinement`**, if **`pr.issues == ""`**, set that **`Details`** to **`PhaseEnum.REFINED`** — same branch as **`Details.refine`** (~513).
- **⚠️** After **`exec_post_extract`**, if **`pp.issues == ""`**, set **`exec_done`** / **`status`** as **`run_post_exec`** does before return, then **`run_exec`**’s caller sets **`Details`** terminal state — match that sequence (**ℹ️** `Details.vala` **`run_post_exec`** / **`run_exec`**).
- **⚠️** After **`exec_extract(Tool)`**, if **`px.exec_extract(ex_run)`**, set **`ex_run.status = PhaseEnum.COMPLETED`** where **`Tool.run`** would after a successful extract — same as Part 3 hunks below.
- **⚠️** **Do not** add a **bulk** **`foreach`** at **`EXEC_VALIDATE`** to force **`COMPLETED`** on every **`Details`** / **`Tool`** in the step — that is a **tail guard**, not a simulation of live. If something is still **`NONE`** at **`add_completed`**, fix the **earlier** replay branches (including, when needed, **`Details.status = COMPLETED`** after the **last** tool run for that task mirrors **`run_exec`**’s single-run tail — **ℹ️** `Details.run_exec` ~802–812).

**ℹ️** Live **`add_completed`** assumes **`Step`** / **`Details`** state already matches finished work; replay should reach that state **incrementally**, not by patching at the boundary.

### 1. `liboccoder/Skill/Runner.vala` — `on_replay`

**✅** Parts 1–3 below are **applied** in tree (see **§ Landed**).

##### Part 1 — `PhaseEnum.REFINEMENT` / `content-stream`

#### Remove

```vala
					var pr = new OLLMcoder.Task.ResultParser(this, m.content);
					var st_r = this.pending.steps.get(this.replay_step_pos);
					pr.extract_refinement(st_r.children.get(this.replay_details_pos));
					this.progress.rebuild();
```

#### Replace with — Match **`Details.refine`**: **`REFINED`** when **`issues == ""`**.

```vala
					var pr = new OLLMcoder.Task.ResultParser(this, m.content);
					var st_r = this.pending.steps.get(this.replay_step_pos);
					var d_ref = st_r.children.get(this.replay_details_pos);
					pr.extract_refinement(d_ref);
					if (pr.issues == "") {
						d_ref.status = OLLMcoder.Task.PhaseEnum.REFINED;
					}
					this.progress.rebuild();
```

##### Part 2 — `PhaseEnum.POST_EXEC` / `content-stream`

#### Remove

```vala
					var pp = new OLLMcoder.Task.ResultParser(this, m.content);
					var st_p = this.pending.steps.get(this.replay_step_pos);
					pp.exec_post_extract(st_p.children.get(this.replay_details_pos));
					this.progress.rebuild();
```

#### Replace with — Match **`run_post_exec`** success: **`exec_done`** + **`COMPLETED`**.

```vala
					var pp = new OLLMcoder.Task.ResultParser(this, m.content);
					var st_p = this.pending.steps.get(this.replay_step_pos);
					var d_post = st_p.children.get(this.replay_details_pos);
					pp.exec_post_extract(d_post);
					if (pp.issues == "") {
						d_post.exec_done = true;
						d_post.status = OLLMcoder.Task.PhaseEnum.COMPLETED;
					}
					this.progress.rebuild();
```

##### Part 3 — `PhaseEnum.EXECUTION` / `content-stream`

#### Keep

```vala
					var px = new OLLMcoder.Task.ResultParser(this, m.content);
```

#### Remove

```vala
					px.exec_extract(d_exec.tools().get_at(this.replay_tool_pos));
					this.progress.rebuild();
```

#### Replace with — Match **`Tool.run`**: **`COMPLETED`** when **`exec_extract`** returns **`true`**.

```vala
					var ex_run = d_exec.tools().get_at(this.replay_tool_pos);
					if (px.exec_extract(ex_run)) {
						ex_run.status = OLLMcoder.Task.PhaseEnum.COMPLETED;
					}
					this.progress.rebuild();
```

**⏳** **`EXEC_VALIDATE` / `add_completed`:** **No** separate hunks here — keep the existing **`step`** move + **`add_completed(step)`** as today. Terminal **`Details` / `Tool`** state should already match live from **Parts 1–3** (and any follow-up needed to mirror **`run_exec`**’s final **`Details.status = COMPLETED`** for single-run tasks, if still wrong in testing).
