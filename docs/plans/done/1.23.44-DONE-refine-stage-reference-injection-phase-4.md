## Exec reference model

- **`Tool.references`** ‚Äî after **`build_exec_runs()`**, each exec **`Tool`**‚Äôs **`references`** property **aliases** **`Details.shared_references`** (**assign**, not **`clear()`** / **`add`** loops). **`Details`** owns the list; exec runs must **not** replace it with a new **`Gee.ArrayList`** or mutate it in ways that fight concurrent runs. Matches **Property Initialization** / avoid pointless collection churn in **`CODING_STANDARDS.md`**.
- **`Tool.exam_reference`** ‚Äî examination link when present. **Executor input must never omit the examination block when `exam_reference != null`:** always emit **`## Examination reference`**. If **`link_content`** resolves to **`""`** (missing file, fetch failure, empty cache, etc.), still include that heading and add an explicit line that the examination content is **empty or unavailable** ‚Äî never skip the section silently. Omitting it confuses the executor; stating emptiness is critical context.
- **`http`/`https`**: **`Details.link_content`** + **`Runner.reference_content`** same pattern as **`task`** (raw body ‚Üí **`header_fenced(..., "markdown")`**).
- **`http_cache`**: **on `Runner` only** (**`internal`**). **`Task.Tool.load_http`** fills it; **`reference_content`** reads it. **Never cache an empty string for a URL the executor will see** when the fetch failed ‚Äî the model needs the **same kind of signal as a normal `web_fetch` tool result** (i.e. keep **`ERROR: ‚Ä¶`** text from **`execute()`**, or set an explicit **`ERROR: ‚Ä¶`** line for setup failures). Silently empty body reads like ‚Äúthe page was blank,‚Äù not ‚Äúthe fetch failed.‚Äù

**Both** **`Runner`** and **`Task.Tool`** expose **`load_links(links)`** ‚Äî same idea, two call sites:

| Type | **`load_links(links)`** |
|------|-------------------------|
| **`Runner`** | **`file:`** only ‚Üí **`yield load_file`**; anything else ‚Üí **`continue`**. Used from **`Details.refine`** (no HTTP at refine). |

One **`foreach`**, **`continue`** skips, **no `else` / `else if`**. **`Runner`** has **`load_file`** + **`load_links`**; **`Tool`** has **`load_http`** + **`load_links`**.

Prefetch is not model output ‚Äî **`load_http`** builds a fake **`ToolCall`** so **`web_fetch`** uses the normal **`execute()`** path. The **first argument** to **`ToolCall.with_values(...)`** is exactly that **`id`** field (same role as **`call_abc123`** in a real assistant message). ```vala
		private async void load_http(Markdown.Document.Format link)
		{
			if (link.scheme != "http" && link.scheme != "https") {
				return;
			}
			var key = link.href != "" ? link.href : link.path;
			if (this.parent.runner.http_cache.has_key(key)) {
				return;
			}
			if (key == "") {
				this.parent.runner.http_cache.set(key,
					"ERROR: Reference URL is empty; cannot prefetch.");
				return;

			var tool_impl = this.parent.runner.session.manager.tools.get("web_fetch");
			var args = new Json.Object();
			args.set_string_member("url", key);
			args.set_string_member("format", "markdown");
			var fn = new OLLMchat.Response.CallFunction.with_values("web_fetch", args);
			var call = new OLLMchat.Response.ToolCall.with_values("http-fake-id", fn);
			var md = yield tool_impl.execute(this.chat(), call, true);
			this.parent.runner.http_cache.set(key, md);
