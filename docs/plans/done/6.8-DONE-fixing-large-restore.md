# 6.8 Fixing large restore (markdown path performance)

**Status: done**

See **`.cursor/rules/CODING_STANDARDS.md`** checklist (new methods, string building in loops, `GLib.debug`, `this.`, etc.).

---

## Purpose

Large assistant bodies make **`Parser.add()`** quadratic: the default path does **`str += c.to_string()`** for every codepoint. Track the current **TEXT run** as a **byte slice** in **`chunk`** (`text_start_pos` … `chunk_pos`) and copy into **`str`** only at boundaries.

---

## Current behaviour (reference)

- **`RenderBase.add()`** → **`Parser.add()`** scans **`chunk`** in a **`while`** loop.
- Pending **`FormatType.TEXT`** payload is accumulated only in **`str`**, grown with **`+=`** on the default branch until flush.

---

## Proposed behaviour

- Add **`text_start_pos`** (byte offset into **`chunk`**) so TEXT not yet merged into **`str`** is **`chunk.substring(text_start_pos, chunk_pos - text_start_pos)`**.
- At boundaries (newline, **`on_node(TEXT)`**, **`ref str`** handlers, end of **`add()`**), **`str = str + chunk.substring(...)`** then **`text_start_pos = chunk_pos`**.
- Default branch: advance **`chunk_pos`** only; extend slice (**`text_start_pos = saved_chunk_pos`** when a new run starts).
- **`handle_line_break`**: take **`ref text_start_pos`** and **`chunk`**; after every **`chunk_pos`** advance, **`text_start_pos = chunk_pos`**.
- No new **`Parser`** methods; merge logic stays inline in **`add()`** / **`handle_line_break()`**.

---

## Scope

| In scope | Out of scope |
| -------- | -------------- |
| **`libocmarkdown/Parser.vala`** — **`add()`**, **`handle_line_break()`** | New **`Parser`** helpers |
| **`FormatMap` / maps** — only if **`ref str`** contract requires | **`process_inline()`** unless profiled |
| Tests, **`tests/markdown/repro-gtkmd-hang.md`** | DB, 6.6 replay timing |

---

## Acceptance criteria

- No new methods unless you approve.
- Large chunk parse time ~linear in input size (no growing cost per 5k loop iterations).
- Remove temporary **`GLib.debug`** around **`parser.add`** when present.

### 1. `libocmarkdown/Parser.vala` — `add()` introduce `text_start_pos`

#### Keep (before — unchanged)

```vala
		public void add(string in_chunk, bool is_end_of_chunks = false)
		{
			var chunk = this.leftover_chunk + in_chunk; // Prepend leftover_chunk so it's processed first
			this.leftover_chunk = ""; // Clear leftover_chunk after using it
			var chunk_pos = 0;
			var saved_chunk_pos = 0;
			var escape_next = false;
			// Accumulated plain inline text since last flush (no format markers * _ ` [ <).
			// Flushed on format match, newline, need-more-input, or end of add();
			// never contains newlines.
			var str = "";
```

#### Add

```vala
			// Start of current TEXT run in chunk, not yet merged into str (bytes). Slice: [text_start_pos, chunk_pos).
			var text_start_pos = 0;
```

#### Keep (after — unchanged)

```vala
			//GLib.debug("  [str] INIT: str='%s' (empty)", str);

			while (chunk_pos < chunk.length) {
				saved_chunk_pos = chunk_pos;
				var c = chunk.get_char(chunk_pos);
```

### 2. `libocmarkdown/Parser.vala` — newline

#### Remove

```vala

					this.handle_line_break(ref chunk_pos, ref str);
					escape_next = false;
					continue;
				}
```

#### Replace with

```vala
				if (c == '\n') {
					if (saved_chunk_pos > text_start_pos) {
						str = str + chunk.substring(text_start_pos, saved_chunk_pos - text_start_pos);
						text_start_pos = saved_chunk_pos;
					}
					this.handle_line_break(ref chunk_pos, ref str, ref text_start_pos, chunk);
					escape_next = false;
					continue;
				}
```

### 5. `libocmarkdown/Parser.vala` — left delimiter: space flush

#### Remove

```vala
				// Left (opening) delimiter: must be preceded by whitespace; peek + handle
				// If str is empty and current char is space, flush str and leave space in chunk for left check

					this.renderer.on_node(FormatType.TEXT, false, str);
					str = "";
				}
```

#### Replace with

```vala
				// Left (opening) delimiter: must be preceded by whitespace; peek + handle
				// If str is empty and current char is space, flush str and leave space in chunk for left check
				if (chunk_pos > text_start_pos) {
					str = str + chunk.substring(text_start_pos, chunk_pos - text_start_pos);
					text_start_pos = chunk_pos;
				}
				if (str.length != 0 && c.isspace()) {
					this.renderer.on_node(FormatType.TEXT, false, str);
					str = "";
					text_start_pos = chunk_pos;
				}
```
