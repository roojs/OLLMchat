## Purpose

- **🔷** Phase A only: standalone **`libocvector2/`** library + meson — **no consumer wiring**.
- **🔷** **`libocfiles` is never** a consumer of `libocvector2`.
- **🔷** **`libocvector`** — **do not touch** until retirement at app cutover.
- **🔷** **`examples`**, **`ollmapp`**, **`ollmfilesd`** — not this phase (`ollmfilesd` follows the **sister relay** pattern in general daemon wiring / companion plan).
- **🔷** Keep **`SQT/VectorMetadata`** (move from `libocfiles`; same class name).
- **🔷** Keep class name **`VectorBuilder`** (not `BatchStore`); flat root, not `Indexing/`.
- **🔷** **`file_id`** on each row is opaque **`int64`** (daemon `filebase.id`); SQL lives in the vector stack.

## Boundary

**💩** `libocvector2` owns:

- FAISS + embed API
- **`SQT/VectorMetadata.vala`** — moved from `libocfiles`
- **`VectorBuilder`**, **`Search`** — bodies from `libocvector`; namespace `OLLMvector2.SQT`

**💩** `libocvector2` does not know about: `File`, `Folder`, `ProjectManager`, `filebase` schema, tree-sitter, snippet reads, scan queues, tools, RPC.

**💩** When wired (Phase B): **`ollmfilesd`** only — indexer, search assembly, `vector.*` RPC server side.

```text
Today:   libocfiles/SQT/VectorMetadata  +  libocvector → libocfiles
Phase A: libocvector2/SQT/VectorMetadata  (library exists; nothing wired yet)
Later:   ollmfilesd → libocvector2
         libocfiles  → no VectorMetadata, no link to ocvector2 (RPC to daemon)
         libocvector → retired (untouched until cutover)
```

**ℹ️** `VectorMetadata` lived in **`libocfiles`** to break a circular dep ([`1.23.18`](done/1.23.18-DONE-tail-end-issues.md)). **`libocfiles`** is not a downstream library of `libocvector2`.

**💩** Vector stack only:

**💩** What moves off `libocfiles` at cutover (daemon / RPC — not Phase A):

- **`Folder.project_description()`** → `vector.project.describe` RPC

**🚫** No second metadata type, no **`MetadataRow`** rename, no daemon SQL copy under `ollmfilesd/SQT/`.

```vala
{
	public class VectorMetadata : Object
	{
		public int64 id { get; set; default = 0; }
		public int64 vector_id { get; set; default = 0; }
		public int64 file_id { get; set; default = 0; }
		// … start_line, end_line, element_type, element_name, description,
		//    ast_path, md5_hash, category, codedoc_*, signature, namespace, parent_class …

		public static void initDB (SQ.Database db);
		public static SQ.Query<VectorMetadata> query (SQ.Database db);
		public void saveToDB (SQ.Database db, bool sync = false);
		public static Gee.ArrayList<VectorMetadata> lookup_vectors (SQ.Database db, int64[] vector_ids);
		public static Gee.ArrayList<VectorMetadata> lookup_path (SQ.Database db, int64[] file_ids, string ast_path);
```

```vala
namespace OLLMvector2
{
	public struct FaissHit { public int64 vector_id; public float distance; public int rank; }

	public abstract class VectorBase : Object { /* connection / request_analysis */ }
	public class VectorToolConfig : OLLMchat.Settings.BaseToolConfig { /* embed, analysis, vision */ }
	public class Index : Object { /* FAISS */ }

	public class Database : VectorBase
	{
		public const int DISABLE_INDEX = -1;

		public async float[] embed (string text) throws GLib.Error;

		public int64[] add_vectors (OLLMchat.Response.FloatArray vectors);
		public async FaissHit[] search (string query, uint64 k, int64[]? filter_vector_ids = null) throws GLib.Error;
		public void save_index ();
		public void reset_index ();
	}

	public class VectorBuilder : Object

		public async void process_elements (

		) throws GLib.Error;

	public class Search : VectorBase

		public string query { get; set; }

		public SQT.VectorMetadata metadata { get; construct; }
	}

```

**💩** Embed document text stays in **`VectorBuilder`** — not a new type on the row.

