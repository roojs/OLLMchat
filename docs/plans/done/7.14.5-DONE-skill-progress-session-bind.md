# 7.14.5 — Skill progress vs session switch

**Status:** done (archived) — implemented in-tree  

**Pointer:** `.cursor/rules/CODING_STANDARDS.md` — Checklist for all plans  

**Related:** `docs/plans/done/7.14-DONE-task-progress-tree-ui.md`, `docs/plans/done/7.14.4-DONE-chatview-integration.md`

---

## Purpose

- **`ProgressView`** tracks **`Runner.progress`** for the **active** chat.
- **`Skill.Factory.activate`** calls **`set_runner`** but today runs mainly when **`agent_activated`** fires (agent **type** change). Same agent, different session → **`activate`** often never runs again → stale tree.

---

## Scope

| In scope | Out of scope |
| -------- | ------------- |
| **`Session`** switch / restore path so **`factory.activate(host)`** runs when the active chat changes while **`agent_name`** is unchanged | Refactoring **`AgentFactory.activate`** performance beyond **idempotent** reuse of existing widget paths |
| **`libollmchat`** + **`ollmapp`** wiring; reuse **`Agent.Factory.activate`** | **`OLLMcoder`** type checks or **`activate_agent`** calls inside **`Window.session_activated`** (project + dropdown only) |

---

## Discussion

**ℹ️** **`Window`** already connects **`agent_activated`** → **`factory.activate.begin(this, …)`** — that is the reuse point.

**ℹ️** **`Manager.switch_to_session`** sets **`session`**, calls **`loaded_session.activate()`**, emits **`session_activated`**, then yields **`restore_messages`**. **`Session.ensure_agent_handler()`** lives inside **`restore_messages`** (or never runs if there are no messages). So **`session.agent`** may not exist at **`session_activated`** time.

**⚠️** **No defensive null guards** in **`Skill.Factory.activate`** for **`session_agent()`** — fix **ordering / ensure**, not **`if (runner == null)`** (see **Plans and defensive code** in the guide).

**👎** **`Skill.Factory`** / **`session.activate_agent`** branches in **`Window.session_activated`**.

**👎** **`if (host.session_agent() == null) return`** in **`Skill.Factory`** as the primary fix.

**🔶** **`EmptySession`** may have no handler until **`activate_agent`** — **`session_restored`** handler should **`return`** when **`session.agent == null`** before **`factory.activate`** (call-site policy in **`Window`**, not **`Skill.Factory`**).

**💩** Optional narrow **`Agent.Factory`** hook if **`OLLMcoder.AgentFactory.activate`** proves too heavy on every switch — confirm before adding.

**ℹ️** **`Agent.Factory.activate`** / **`deactivate`** are **`async void`** and **do not** declare **`throws`**. **`OLLMcoder`** **`initialize_widget()`** implementations catch **`GLib.Error`** and **`GLib.warning`** — UI failures stay inside the factory. **`factory.activate.end(res)`** has nothing useful to catch at **`Window`** unless **`activate`** is changed to **`throws`** and errors are handled meaningfully (not only **`warning`**).

**👎** **`try` / `catch` around `factory.activate.end(res)`** in **`Window`** when **`agent_activated`** already calls **`end`** without **`catch`** — duplicated noise; either **omit** **`catch`** here or add **`throws`** to **`activate`** and one real recovery path (**💩** confirm first).

**⚠️** **`get_active_agent()`** today returns **`Agent.Factory?`**. **👎** **`if (factory != null)`** around **`activate`** in **`Window`**. Resolve to **`Agent.Factory`** (non-null) with **§3** fallback.

**⚠️** If **`agent_factories`** has **no** entry for **`session.agent_name`**, **downgrade to `just-ask`**: **`GLib.warning`**, **`just-ask`** factory (**`just-ask`** is **always** registered — **👎** no fallback / **`critical`** path for missing **`just-ask`**).

**👎** **`(!)`** non-null casts — assign fallback then **`return f`** / **`factory = …get("just-ask")`** (**§1** § **§3**) so returns and **`create_agent`** use plain identifiers only.

**ℹ️** **`ensure_agent_handler()`** (**§1**): on miss, **`warning`**, **`this.agent_name = "just-ask"`**, **`just-ask`** factory, **`create_agent`** — session matches handler; **`void`**, **no** **`throws`**. **`get_active_agent()`** (**§3**) does **not** mutate **`session`**.

---

## Acceptance criteria

- Switch between two **Skills** chats with the same **`agent_name`**: progress tree matches the **current** session.
- **`session.agent`** exists before **`Skill.Factory.activate`** runs on that path (**ensure** / order, not null-branching in **`Skill.Factory`**).
- Unknown **`agent_name`** in registry → **`warning`** + **`just-ask`** (**§1** mutates **`session.agent_name`** when ensuring handler; **§3** getter does not mutate).
- No **`OLLMcoder`** imports or **`is Skill.Factory`** in **`Window.vala`** for this behaviour.
- One mechanical pattern for “call **`get_active_agent().activate`**” (no triple copy of ad-hoc blocks).

---

## Concrete code proposals

Intro: hunks are **Remove** / **Replace with** / **Add** from the tree; verify context before applying.

### 1. `libollmchat/History/Session.vala` — expose ensure + drop **`throws`**

**Goal:** **`Manager`** can create the handler **before** UI bind. Missing factory → **warn** + **downgrade `agent_name` to `just-ask`** + **`just-ask`** factory (same policy as **§3**).

##### Part A — Signature

#### Remove

```vala
		private void ensure_agent_handler() throws Error
```

#### Replace with

```vala
		public void ensure_agent_handler()
```

Strip **`@throws`** from the docblock above **`ensure_agent_handler`** (describe downgrade in prose if useful).

##### Part B — Factory lookup

#### Remove

```vala
			var factory = this.manager.agent_factories.get(agent_name);
			if (factory == null) {
				GLib.critical("Agent '%s' not found in manager", agent_name);
				throw new OllmError.INVALID_ARGUMENT("Agent '%s' not found in manager", agent_name);
			}
```

#### Replace with — Downgrade session to **`just-ask`** when **`agent_name`** is unknown (**`just-ask`** always present).

```vala
			var factory = this.manager.agent_factories.get(agent_name);
			if (factory == null) {
				GLib.warning("Agent '%s' not found in manager; downgrading session to just-ask", agent_name);
				this.agent_name = "just-ask";
				factory = this.manager.agent_factories.get("just-ask");
			}
```

### 2. `libollmchat/History/Manager.vala` — `switch_to_session`

**Goal:** After **`Session`** is active, **ensure** handler **before** **`session_activated`** so any later **`factory.activate`** sees **`session.agent`**.

#### Keep

```vala
			this.session = loaded_session;
			loaded_session.activate();

			this.session_activated(loaded_session);
```

#### Replace with — Ensure agent for persisted sessions before UI signals (**no** **`try`** — **§1**).

```vala
			this.session = loaded_session;
			loaded_session.activate();

			if (loaded_session is Session) {
				((Session) loaded_session).ensure_agent_handler();
			}

			this.session_activated(loaded_session);
```

### 3. `libollmchat/History/Manager.vala` — `get_active_agent`

**Goal:** Callers (**§4**) use **`Agent.Factory`** (non-null). Unknown **`agent_name`** → **`warning`**, assign **`just-ask`**, **`return f`** (**no** **`(!)`**). Does **not** mutate **`session`** (getter).

#### Remove

```vala
		public OLLMchat.Agent.Factory? get_active_agent()
		{
			return this.agent_factories.get(this.session.agent_name == "" ? "just-ask"
				 : this.session.agent_name);
		}
```

#### Replace with

```vala
		public OLLMchat.Agent.Factory get_active_agent()
		{
			string name = this.session.agent_name == "" ? "just-ask" : this.session.agent_name;
			var primary = this.agent_factories.get(name);
			if (primary != null) {
				return primary;
			}
			GLib.warning("Agent.Factory missing for agent_name '%s'; using just-ask", name);
			var fallback = this.agent_factories.get("just-ask");
			return fallback;
		}
```

Update the docblock **`@return`** line to match (remove “or null if not found”; note fallback).

### 4. `ollmapp/Window.vala` — rebind factories on session restore

**Goal:** Same **`factory.activate.begin(this, …)`** as **`agent_activated`**, triggered when **`session_restored`** fires (after **`restore_messages`**, **`session.agent`** already ensured for **`Session`** by §2).

Depends on **§3** (**`get_active_agent()`** non-null).

#### Keep

```vala
			this.history_manager.agent_activated.connect((factory) => {
				factory.activate.begin(this, (obj, res) => {
					factory.activate.end(res);
				});
			});
			this.history_manager.agent_deactivated.connect((factory) => {
```

#### Add — Reuse activate when the active session changes without an agent-type change.

Same shape as **`agent_activated`** (no **`catch` around `activate.end`** — see **Discussion** **`ℹ️` / `👎`**).

```vala
			this.history_manager.session_restored.connect((_session) => {
				if (_session.agent == null) {
					return;
				}
				var factory = this.history_manager.get_active_agent();
				factory.activate.begin(this, (obj, res) => {
					factory.activate.end(res);
				});
			});
```
