## 1. Purpose

When **`run_command`** uses **`Bubble`** (**`Bubble.can_wrap()`** true), tool output returned to the LLM **may** include:

1. **Network access reporting** — If **`allow_network`** is false, append **ground-truth** text **only** when user-notify observed **`socket`** / **`connect`** (parent **§ Goals** + **§ Concrete delivery**). **Do not** tell the LLM the run was “about networking” when **`RunSeccomp.network`** is empty — without NOTIFY evidence, a non-zero exit is ambiguous (missing file, syntax error, etc.).
2. **File access reporting** — When the run has a defined allowlist, append evidence for filesystem operations **outside** that set (parent **§ Goals** + **§ Concrete delivery**).
3. **Degraded NOTIFY** — If user-notify cannot be established, **append one short notice** and **still run `bwrap`** as today; **do not** fail the run for NOTIFY setup alone.

**Role split (from parent, mandatory):** **Bubblewrap** enforces isolation. **Seccomp on this path is observe-and-report only** — **`SCMP_ACT_NOTIFY`** + **`SECCOMP_USER_NOTIF_FLAG_CONTINUE`** (or equivalent non-blocking observation). **Do not** ship **`SCMP_ACT_TRAP`** / exit **159** as the product path.

## 2. Goals (proposed acceptance criteria)

1. **`liboctools`** links **libseccomp** + **`vapi/seccomp-fd-pass.c`** using the **same Meson pattern** as **`examples/meson.build`** for **`oc-test-seccomp-spike`** (see **§ 4**).
2. **Single class `RunSeccomp`** in **`liboctools/RunCommand/RunSeccomp.vala`** holds **all** seccomp-related logic listed in **§ 6**; **`Bubble.vala`** contains **no** direct **`Seccomp.*`**, **`SeccompLinux.*`**, or **`pass_unix_fd` / `receive_unix_fd`** calls.
3. **`Bubble.exec`** still launches **only** **`bwrap`** with **`build_bubble_args`** — **`argv[0]`** unchanged. **`GLib.SubprocessLauncher`** is used **only** to add **`take_fd`** + **`set_child_setup`** for NOTIFY handoff (**§ 5**).
4. **`read_subprocess_output`** appends **`RunSeccomp`** **`network`**, **`fs`**, and **`skipped`** only when each is non-empty (**§ 8** — no networking narrative without NOTIFY evidence); **`Bubble`** does **not** mirror those strings in separate fields (**§ 11**).
5. **`!Bubble.can_wrap()`** path unchanged: **no** seccomp strings, **no** new dependencies required for that path.
6. **`oc-test-seccomp-spike`** and **`oc-user-notif-exchange`** remain **examples**; logic from the latter is **folded into `RunSeccomp`** (not a second public type in **`liboctools`**).

## 3. Non-goals (this subplan does not)

- **`write_roots`** API, delimiter, permission prompt redesign, or new **`bwrap --bind`** wiring — parent owns those; here we only **consume** whatever allowlist **`Bubble`** can derive when file reporting is implemented.
- **`Request.execute_with_subprocess()`** seccomp / NOTIFY.
- Helper binaries, extra **`argv[0]`** programs, or **`Posix.fork`** in application code.
- **`GLib.Thread`** for the notify supervisor — use **`MainContext`** + **`UnixFDSource`** (or equivalent) with **`yield subprocess.wait_async`**.

## 5. Why `SubprocessLauncher` (not a second program)

**Design rule (proposed):** The **only** external program **`Bubble`** starts for the sandboxed command is **`bwrap`**, with arguments from **`build_bubble_args`** — same **`argv`** shape as today.

**`SubprocessLauncher`** is a **GLib API**, not another executable. It is proposed **only** because **`GLib.Subprocess.newv`** cannot **`take_fd`** (fixed child fd for the socket end) or **`set_child_setup`** (code that runs in the **spawn child** after the libc fork, **before** **`exec`** into **`bwrap`**). **`launcher.spawnv (argv)`** still **`exec`s** **`argv[0]`** = **`bwrap`**.

## 6. Class `RunSeccomp` — proposed responsibilities and API

**Location:** **`liboctools/RunCommand/RunSeccomp.vala`**.

**Naming rule:** **Public** method identifiers use **one to three English words** in **`snake_case`** (e.g. **`wire_launcher`**, **`finish_handshake`**). **Private** helpers are unrestricted. **There is no fixed limit** on the number of public methods — add private methods freely.

### 6.1 Proposed public methods (minimum set)

| Method | Proposed behaviour |
|--------|-------------------------|
| **`wire_launcher (SubprocessLauncher launcher, bool want_network_syscalls, bool want_fs_syscalls)`** | Create **`AF_UNIX`** **`socketpair`**. **`launcher.take_fd`** child end to a **fixed** child fd constant (e.g. **`3`**) agreed in code comments and **must not** collide with stdio. **`launcher.set_child_setup`** registers a **private static** trampoline that runs **only in the spawn child**: build **`Seccomp.Filter (SCMP_ACT_ALLOW)`**, add **`SCMP_ACT_NOTIFY`** rules for **`socket`** + **`connect`** when **`want_network_syscalls`**, **`openat`** when **`want_fs_syscalls`**, **`PR_SET_NO_NEW_PRIVS`**, **`load`**, **`notify_fd`**, **`pass_unix_fd`** to the sync socket, **`read`** sync byte **`'S'`** from parent, close socket. Store parent’s remaining socket fd in the instance. **On any failure before a safe state:** **`return`** from the trampoline without **`exit`**; **kernel caveat:** if **`load()`** with NOTIFY already succeeded and a later step fails, the PR **must** clear NOTIFY without a supervisor (e.g. **`Seccomp.Filter.reset`** + validated replacement **`load`**, or equivalent) **before** **`return`** — see parent kernel discussion. |
| **`finish_handshake ()`** | After **`spawnv`**: **`receive_unix_fd`** on parent socket; **always** **`write ('S')`** on that socket so the child never blocks waiting for sync; set **`skipped`** to a fixed one-line message if recv or write failed or fd invalid; close parent socket end; retain **`notify_fd`** when valid. |
| **`attach_notify_loop (MainContext ctx, Subprocess subprocess)`** | If **`notify_fd < 0`**, no-op. Else set **`O_NONBLOCK`**, attach **`UnixFDSource`**, dispatch **`notify_alloc` / `notify_receive` / `USER_NOTIF_FLAG_CONTINUE` / `notify_respond`** (same semantics as **`examples/oc-user-notif-exchange.vala`**). Update counters and **`openat_paths`** only — not **`network`** / **`fs`** yet. **Before** **`yield read_subprocess_output`**. |
| **`drain_notify_readable ()`** | Non-blocking drain on **`notify_fd`** (same cycle as **`attach_notify_loop`**). Call once after **`wait_async`** so late NOTIFY events are counted. |
| **`finish_evidence_formatting ()`** | Sets **`network`** / **`fs`** from counters and **`openat_paths`** with parent **§ Concrete delivery** wording (**§ 12.3**, one assembly site). Idempotent when empty. |
| **`detach_sources ()`** | Remove **`UnixFDSource`**, **`close`** **`notify_fd`**. **`Bubble.exec`** **`finally`**. |

Additional **public** methods are allowed if they keep **`Bubble`** thin (e.g. **`reset_for_run`**) provided names stay **1–3 words**.

### 6.2 Proposed public properties

| Property | Meaning |
|----------|---------|
| **`network`** | Final **or** intermediate network appendix text per parent **§ Concrete delivery**; empty if no events or **`allow_network`** true (caller passes flag; class may ignore NOTIFY rules when network allowed). |
| **`fs`** | File appendix per parent **§ Concrete delivery**; empty when no qualifying events or policy says omit (**parent § 6c** unconstrained **`allow_write`**). |
| **`skipped`** | Single-line user-visible notice when NOTIFY path did not complete; empty on success. |

### 6.3 Private implementation (must live inside this class)

- **`Seccomp.Filter`**, **`SeccompLinux.prctl`**, **`Seccomp.pass_unix_fd` / `receive_unix_fd`**, **`add_notify`** wrapping **`rule_add_array`**.
- Optional **`decode_openat_path`** (**`process_vm_readv`**) for **`openat`** paths (same idea as **`examples/oc-test-seccomp-spike.vala`**).
- Allowlist construction for **`fs`** — **must** mirror **`build_bubble_args`** / parent **§ Non-project writable surfaces**; **`Bubble`** passes resolved data into **`RunSeccomp`** via **`wire_launcher`** or a follow-up setter **documented in this plan** if the PR introduces one.
- Multi-line **`network`** / **`fs`** text is assembled **only** in **`finish_evidence_formatting`**, verbatim from parent **§ Concrete delivery**.

## 7. `Bubble.exec` — proposed control flow

1. **`var run_seccomp = new RunSeccomp ();`**
2. **`var launcher = new SubprocessLauncher (...)`** with the **same** pipe flags **`Subprocess.newv`** uses today (**`STDOUT_PIPE`**, **`STDERR_PIPE`**, **`STDIN_INHERIT`**).
3. **`run_seccomp.wire_launcher (launcher, !this.allow_network, true);`** — second bool is “observe **`openat`** for file reporting”; set **`false`** in a phased PR if file reporting ships later, **without** changing this plan’s class layout.
4. **`var subprocess = launcher.spawnv (args);`** where **`args`** is **`build_bubble_args`** output — **`args[0]`** is **`bwrap`**.
5. **`run_seccomp.finish_handshake ();`**
6. **`run_seccomp.attach_notify_loop (GLib.MainContext.get_default (), subprocess);`**
7. **`yield read_subprocess_output (subprocess, run_seccomp);`** — inside **`read_subprocess_output`**, after **`wait_async`** and stream watch cleanup, **`run_seccomp.drain_notify_readable ()`** then **`run_seccomp.finish_evidence_formatting ()`** before composing the returned string (**§ 12.4.1**).
8. In **`exec`** **`finally`**: **`run_seccomp.detach_sources ();`**.

**Overlay scan / cleanup:** unchanged **`finally`** behaviour around **`exec`**, in addition to **`run_seccomp.detach_sources`** above.

---

## 8. `Bubble.read_subprocess_output` — proposed text rules

1. On **non-zero** exit: if **`run_seccomp.network`** is non-empty, append it (observed **`socket`** / **`connect`** under **`!allow_network`**). If **`run_seccomp.network`** is empty, append **nothing** about networking — seccomp did not record network-related syscalls for this run, so a generic “networking was disabled” line would mislead the LLM.
2. Append **`run_seccomp.fs`** when non-empty (failure block; same paragraph as exit code or immediately after — pick one style in PR and document in commit).
3. On **zero** exit: if **`run_seccomp.fs`** is non-empty, append after **`ret_str`** as in **§ 12.5**.
4. If **`run_seccomp.skipped`** is non-empty, append it **once** (placement: **choose one** in PR, document once).

**Code citations for current behaviour:** today **`Bubble.vala`** always appends a networking hint after **`Exit code:`** when **`!allow_network`** (see parent citation). **This subplan removes that** whenever **`run_seccomp.network`** is empty. **Proposed Vala deltas:** **§ 12.4–12.5**.

## 11. Risks (acknowledged)

- **`process_vm_readv`** may fail under **Yama** — file path line may degrade to syscall-only text; document in PR.
- **`seccomp_reset` + second `load`** behaviour must be validated on target kernels if NOTIFY teardown is needed on failure paths.

### 11.1 LLM-facing copy (where strings live)

- **Audience:** Tool output is read by the **LLM**. **`network`** / **`fs`** paragraphs describe **observed** sandbox behaviour (NOTIFY counters), then **how to change the next `run_command`** when relevant. **Do not** narrate networking (or file-policy) when seccomp produced **no** matching evidence — the exit code and command stderr remain the signal.
- **Single owner:** All NOTIFY-derived appendices (**`network`**, **`fs`**, **`skipped`**) are **`RunSeccomp`** properties. **`Bubble.read_subprocess_output`** reads **`run_seccomp.network`** (etc.) directly; **`Bubble`** **does not** add parallel **`seccomp_*`** fields.
- **No scattered `const`:** Do **not** define the same paragraph as a file-level **`const`** in **`Bubble`** and again in **`RunSeccomp`**. **`RunSeccomp.finish_evidence_formatting`** is the assembly site for **`network`** / **`fs`**. Parent [2.22.1 § Concrete delivery](./2.22.1-DONE-run-command-seccomp-network-and-path-evidence.md) remains the **wording authority** for those appendices; if product text changes, update the **parent** plan and **`RunSeccomp`** in one PR.
- **When NOTIFY did not observe network syscalls:** **`run_seccomp.network`** stays empty — **`read_subprocess_output`** must **not** invent a networking explanation. **`run_seccomp.skipped`** may still be non-empty (NOTIFY path degraded); that line explains **observability**, not “the command hit the network.”

### 12.3 `RunSeccomp.vala` — **complete** proposed implementation (one compilation unit)

**`decode_openat_path`:** For **`openat`** NOTIFY events, reads the tracee’s pathname argument from remote memory with **`Seccomp.vm_readv`** (C **`seccomp_vm_readv`** in seccomp-fd-pass). Returns **`null`** if the event is not **`openat`**, **`notify_id_valid`** fails, the address is null, the read fails (e.g. **Yama**), or the buffer cannot be NUL-terminated safely. This is **not** a separate “try_openat_path” API — that name was plan noise; the behaviour is this private method only.

**`finish_evidence_formatting`** builds **`this.network`** and **`this.fs`** with **multiline** template strings (**`@"` … `"`** spanning lines) and **`$(…)`** interpolation for **`summary`** / **`paths_block`** — readable for long LLM copy ([CODING_STANDARDS.md](../../.cursor/rules/CODING_STANDARDS.md) exception for multi-line messages). Vala uses **`$(expr)`**, not **`${expr}`**. Escape embedded double quotes as **`\"`** inside the template. Prose must still match the parent plan **verbatim** (the **§ 12.3** sample is structural — copy exact wording from the parent in the PR).

```vala
public class RunSeccomp : GLib.Object
{
	public const int SYNC_SOCK_CHILD_FD = 3;

	public string network { get; private set; default = ""; }
	public string fs { get; private set; default = ""; }
	public string skipped { get; private set; default = ""; }

	int parent_sock = -1;
	int notify_fd = -1;
	uint unix_fd_source = 0;

	int nr_socket = -1;
	int nr_connect = -1;
	int nr_openat = -1;
	int count_socket = 0;
	int count_connect = 0;
	string[] openat_paths = {};

	static bool child_want_network;
	static bool child_want_fs;

	public RunSeccomp ()
	{
		this.nr_socket = Seccomp.syscall_resolve_name ("socket");
		this.nr_connect = Seccomp.syscall_resolve_name ("connect");
		this.nr_openat = Seccomp.syscall_resolve_name ("openat");
	}

	/**
	 * Add one SCMP_ACT_NOTIFY rule for a syscall name.
	 *
	 * @param ctx seccomp filter under construction in the child
	 * @param syscall_name syscall name understood by libseccomp
	 * @return 0 on success, negative on failure
	 */
	static int add_notify (Seccomp.Filter ctx, string syscall_name)
	{
		return ctx.rule_add_array (
			Seccomp.SCMP_ACT_NOTIFY,
			Seccomp.syscall_resolve_name (syscall_name),
			0,
			null);
	}

	/**
	 * Add NOTIFY rules for socket/connect and optionally openat.
	 */
	static int add_notify_rules (Seccomp.Filter ctx, bool want_network, bool want_fs)
	{
		int r;
		if (want_network) {
			r = RunSeccomp.add_notify (ctx, "socket");
			if (r < 0) {
				return r;
			}
			r = RunSeccomp.add_notify (ctx, "connect");
			if (r < 0) {
				return r;
			}
		}
		if (want_fs) {
			r = RunSeccomp.add_notify (ctx, "openat");
			if (r < 0) {
				return r;
			}
		}
		return 0;
	}

	static void child_seccomp_handshake ()
	{
		int sock = RunSeccomp.SYNC_SOCK_CHILD_FD;
		var ctx = new Seccomp.Filter (Seccomp.SCMP_ACT_ALLOW);
		if (RunSeccomp.add_notify_rules (ctx, RunSeccomp.child_want_network, RunSeccomp.child_want_fs) < 0) {
			return;
		}
		if (SeccompLinux.prctl (SeccompLinux.PR_SET_NO_NEW_PRIVS, 1, 0, 0, 0) != 0) {
			return;
		}
		if (ctx.load () < 0) {
			return;
		}
		int nfd = ctx.notify_fd ();
		if (nfd < 0) {
			return;
		}
		if (Seccomp.pass_unix_fd (sock, nfd) != 0) {
			Posix.close (nfd);
			return;
		}
		Posix.close (nfd);
		var sync = new uint8[1];
		if (Posix.read (sock, sync, 1) != 1 || sync[0] != 'S') {
			return;
		}
		Posix.close (sock);
	}

	public void wire_launcher (GLib.SubprocessLauncher launcher, bool want_network_syscalls, bool want_fs_syscalls)
	{
		int[] sv = { 0, 0 };
		if (Posix.socketpair (Posix.AF_UNIX, Posix.SOCK_STREAM, 0, sv) != 0) {
			this.skipped = "seccomp: socketpair failed";
			return;
		}
		this.parent_sock = sv[0];
		launcher.take_fd (sv[1], RunSeccomp.SYNC_SOCK_CHILD_FD);
		Posix.close (sv[1]);
		RunSeccomp.child_want_network = want_network_syscalls;
		RunSeccomp.child_want_fs = want_fs_syscalls;
		launcher.set_child_setup (() => {
			RunSeccomp.child_seccomp_handshake ();
		});
	}

	public void finish_handshake ()
	{
		if (this.parent_sock < 0) {
			this.skipped = "seccomp: internal error (no sync socket)";
			return;
		}
		int nfd = Seccomp.receive_unix_fd (this.parent_sock);
		if (nfd < 0) {
			this.skipped = "Seccomp user-notify was not set up for this run; syscall evidence is unavailable.";
		}
		uint8[] sync_b = { (uint8) 'S' };
		if (Posix.write (this.parent_sock, sync_b, 1) != 1) {
			this.skipped = "Seccomp user-notify handshake did not complete; syscall evidence is unavailable.";
			if (nfd >= 0) {
				Posix.close (nfd);
			}
			Posix.close (this.parent_sock);
			this.parent_sock = -1;
			this.notify_fd = -1;
			return;
		}
		Posix.close (this.parent_sock);
		this.parent_sock = -1;
		if (nfd >= 0) {
			this.notify_fd = nfd;
		} else {
			this.notify_fd = -1;
		}
	}

	/**
	 * Best-effort pathname string for an openat user-notify event.
	 *
	 * Uses process_vm_readv to copy from the tracee address space.
	 * Returns null when the event is not openat, validation fails,
	 * or the host blocks cross-process memory read.
	 *
	 * @param ev kernel notification payload
	 * @return path text or null
	 */
	string? decode_openat_path (Seccomp.Notif ev)
	{
		if (this.notify_fd < 0) {
			return null;
		}
		if (ev.sc_data.nr != this.nr_openat) {
			return null;
		}
		if (Seccomp.notify_id_valid (this.notify_fd, ev.id) != 0) {
			return null;
		}
		uint64 path_address = ev.sc_data.args[1];
		if (path_address == 0) {
			return null;
		}
		const int PATH_MAX = 4096;
		var local = new uint8[PATH_MAX];
		size_t copy_len = local.length - 1;
		var local_iov = Posix.iovector () { iov_base = local, iov_len = copy_len };
		var remote_iov = Posix.iovector () {
			iov_base = (void*) (uintptr) path_address,
			iov_len = copy_len
		};
		ssize_t n = Seccomp.vm_readv (
			(int) ev.pid,
			&local_iov,
			1,
			&remote_iov,
			1,
			0);
		if (n <= 0) {
			return null;
		}
		local[n < (ssize_t) copy_len ? (size_t) n : copy_len] = 0;
		return (string) local;
	}

	void on_notify_event (Seccomp.Notif ev)
	{
		if (ev.sc_data.nr == this.nr_socket) {
			this.count_socket++;
			return;
		}
		if (ev.sc_data.nr == this.nr_connect) {
			this.count_connect++;
			return;
		}
		if (ev.sc_data.nr != this.nr_openat) {
			return;
		}
		string? p = this.decode_openat_path (ev);
		if (p != null) {
			this.openat_paths += p;
		}
	}

	public void drain_notify_readable ()
	{
		if (this.notify_fd < 0) {
			return;
		}
		for (;;) {
			Seccomp.Notif* req_ptr;
			Seccomp.NotifResp* resp_ptr;
			if (Seccomp.notify_alloc (out req_ptr, out resp_ptr) != 0) {
				return;
			}
			int r = Seccomp.notify_receive (this.notify_fd, req_ptr);
			if (r < 0) {
				Seccomp.notify_free (req_ptr, resp_ptr);
				if (Posix.errno == Posix.EAGAIN || Posix.errno == Posix.EWOULDBLOCK) {
					return;
				}
				return;
			}
			Seccomp.Notif ev = *req_ptr;
			this.on_notify_event (ev);
			resp_ptr.id = ev.id;
			resp_ptr.val = 0;
			resp_ptr.error = 0;
			resp_ptr.flags = Seccomp.USER_NOTIF_FLAG_CONTINUE;
			Seccomp.notify_respond (this.notify_fd, resp_ptr);
			Seccomp.notify_free (req_ptr, resp_ptr);
		}
	}

	public void attach_notify_loop (GLib.MainContext ctx, GLib.Subprocess subprocess)
	{
		if (this.notify_fd < 0) {
			return;
		}
		unowned GLib.MainContext use_ctx = ctx != null ? ctx : GLib.MainContext.get_default ();
		int fl = Posix.fcntl (this.notify_fd, Posix.F_GETFL, 0);
		if (fl >= 0) {
			Posix.fcntl (this.notify_fd, Posix.F_SETFL, fl | Posix.O_NONBLOCK);
		}
		var fdsrc = new GLib.UnixFDSource (this.notify_fd, GLib.IOCondition.IN | GLib.IOCondition.HUP);
		fdsrc.set_callback (() => {
			this.drain_notify_readable ();
			return true;
		});
		this.unix_fd_source = fdsrc.attach (use_ctx);
	}

	public void finish_evidence_formatting ()
	{
		if (this.count_socket > 0 || this.count_connect > 0) {
			string summary = "socket (%d×), connect (%d×)".printf (
				this.count_socket,
				this.count_connect);
			this.network = @"---
Sandbox: networking was disabled for this run, but the command attempted network-related operations:

$(summary)

To enable networking on a later run, pass \"network\": true in run_command (user approval required).
";
		}
		if (this.openat_paths.length > 0) {
			string paths_block = string.joinv ("\n", this.openat_paths);
			this.fs = @"---
Sandbox: a write or filesystem operation targeted a path outside the directories allowed for this run.

$(paths_block)

To request access, add write_roots: a semicolon-separated list of absolute directory roots, e.g.
  \"write_roots\": \"/path/to/output;/path/to/cache\"
Avoid \"/\" or entire home unless necessary; users often reject broad paths.
";
		}
	}

	public void detach_sources ()
	{
		if (this.unix_fd_source != 0) {
			GLib.Source.remove (this.unix_fd_source);
			this.unix_fd_source = 0;
		}
		if (this.notify_fd >= 0) {
			Posix.close (this.notify_fd);
			this.notify_fd = -1;
		}
	}
}
```

### 12.4 `Bubble.vala` — **`exec`** (`SubprocessLauncher` + **`RunSeccomp`**)

**Replace `Subprocess.newv` through `read_subprocess_output`** (keep **`overlay.create`**, **`build_bubble_args`**, outer **`finally`** for **`overlay.cleanup`**). Declare **`RunSeccomp run_seccomp`** so **`finally`** can call **`run_seccomp.detach_sources ()`** even if **`spawnv`** throws. **`args[0]`** remains **`bwrap`**.

```vala

GLib.debug ("running command: %s", cmd_str);

	var launcher = new GLib.SubprocessLauncher (
		GLib.SubprocessFlags.STDOUT_PIPE |
		GLib.SubprocessFlags.STDERR_PIPE |
		GLib.SubprocessFlags.STDIN_INHERIT);

			"Failed to create bubblewrap subprocess: " + e.message);
```
