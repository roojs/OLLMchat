## What this series covers

Task progress **`ListModel`** (**`ProgressItem`** / **`ToolList`** / **`ProgressList`**), **`PhaseEnum`** extensions for row **`status`**, and **`PhaseEnum.to_human()`** (Pango markup for the stage column). **`Runner.progress`** and **`Details`** / **`Tool`** / **`List`** edits are **split into sub-plans** so each file stays reviewable; **`status`** is advanced in **`Runner`** / **`Details`** / **`Tool`** / **`List`** — see **§ Sub-plans** and the **Compile order** below.

**Shared product rules** (activity-first, `proposed_tools` vs `children`, negative guards on `items_changed`, `var` locals, `0u` counters): spelled out in **`docs/plans/done/7.14.1-DONE-phase1-core.md`** § Purpose — do not duplicate here.

**Compile order (strict):**

1. **`7.14.1-phase1-core`** — extend **`PhaseEnum`** + add **`ProgressItem.vala`** (interface only needs existing **`PhaseEnum`**).
2. **`7.14.1.3`** — minimal **`Details` implements ProgressItem** (status storage + **`notify`** on **`status_str`**) so **`ProgressList`** can cast to **`Details`** and read **`status`**.
3. **`7.14.1.1`** — **`Tool` implements ProgressItem**.
4. **`7.14.1-phase1-core`** — add **`ToolList.vala`**, **`ProgressList.vala`**, **`meson`** ( **`ProgressList`** depends on **`Details`** + **`Tool`** for casts).
5. **`7.14.1.2`** — **`Runner`** **`progress`** + hooks.
6. **`7.14.1.3`** / **`7.14.1.1`** — finish ports (**`build_run_queue`**, **`proposed_tools`**, execution **`status`** updates).
7. **`7.14.1.4`** — **`List`**.

If you prefer **one branch**: merge all sub-plans in one PR following the same order.

## Acceptance criteria (series)

1. **`Runner.progress`** non-null after construction; **`ProgressList`** holds **`weak Runner`**.
2. **`PhaseEnum`** includes progress-row values (**`TOOLS_RUNNING`**, **`COMPLETED`**, …) and **`to_human()`** returns **Pango**-safe strings for **`Gtk`** (**`use_markup`** / **CellRenderer**), with **inactive / active / done** variants per parent **`7.14`** stage rules.
3. **`ProgressList`** incremental API (**`clear_pending`** / **`add_pending`** / **`add_completed`** / **`rebuild`** / **`clear`**) matches **`7.14.1-phase1-core`** (negative guards, **`0u`** counters).
4. **`Details.children`** = execution queue; **`proposed_tools`** = refinement proposals; **`7.14.1.3`** + **`7.14.1.1`** wire **`status`** as work advances.
5. **`List`** calls **`progress.add_completed(step)`** when a step moves to **`completed`** (**`7.14.1.4`**).
