## Goal

Automated **round-trip** tests for every wire method that has a server handler today, plus explicit **negative** cases for client stubs that lack a daemon relay.

Tests must **not** touch `~/.local/share/ollmchat/` — isolated `data_dir` under the Meson build tree (same idea as `oc-test-files --test-db`, but for the whole daemon data layout via `ApplicationInterface.data_dir`).

**First pass is shell-only** — NDJSON **script files** run through `ollmfilesd --interactive --rpc-script=FILE`; assert on captured stdout with **`jq`**. No Vala test binary, no `RpcClient`, no `Folder`/`File` `rpc_register()` (typed `result` blobs stay opaque JSON until [`2.10.4.9-ACTIVE-v2-caller-cutover.md`](../2.10.4.9-ACTIVE-v2-caller-cutover.md) wires the client library).

**Not used by this suite:** Unix socket + `socat`/`nc` (production transport only).

## Harness

### Layout

```text
build/test-rpc-<pid>/
  files.sqlite          # daemon DB after shutdown (ApplicationInterface.data_dir)
  out.ndjson            # captured stdout (ready + responses, one line each)
  ollmfilesd.stderr     # captured stderr (debugging)
  projects/             # fixture tree copied from tests/rpc-fixtures/ (T1+)

tests/rpc/
  t0.script             # T0 NDJSON requests (one per line, # comments OK)
  t1.script             # T1 ProjectManager cases (planned)
  t2.script             # T2 File cases (planned)
```

### Components

| Piece | Role |
| ----- | ---- |
| **`tests/test-rpc.sh`** | Meson `test()` entry — run script, assert on NDJSON lines |
| **`tests/test-rpc-common.sh`** | Shared helpers: `run_rpc_script`, `rpc_line`, `jq_line_ok` |
| **`tests/rpc/*.script`** | Git-tracked NDJSON request scripts (one JSON-RPC object per line) |
| **`tests/rpc-fixtures/`** | Minimal project tree copied into temp dir per run (T1+) |
| **`build/ollmfilesd/ollmfilesd`** | Daemon binary under test |

**Deferred:** `tests/oc-test-rpc.vala` — only worth it once `libocrpc` + V2 types are Meson-wired and we want typed deserialize (tranche 3 / cutover).

**Not in initial pass:** `RpcClient`, `RpcClientBoot`, V2 `ProjectManager`, socket spawn/poll.

### Flow per test run

1. Create `build/test-rpc-$$/`.
2. (T1+) Copy `tests/rpc-fixtures/minimal-project/` into `{test_dir}/projects/…`.
3. **`test-rpc.sh`** runs:

   ```bash
   OLLMFILES_IS_TEST=1 ollmfilesd --interactive --data-dir="$TEST_DIR" \
     --rpc-script="$SCRIPT" >"$TEST_DIR/out.ndjson" 2>"$TEST_DIR/ollmfilesd.stderr"
   ```

   - **`--interactive`** — no fork, no socket, no pid file; stdin/stdout NDJSON.
   - **`OLLMFILES_IS_TEST=1`** — `NON_UNIQUE` so tests skip D-Bus name acquisition.
   - Script finishes → daemon calls **`Daemon.shutdown`** (last line) or quits after script → **`cleanup()`** flushes DB.

4. Line 1 of **`out.ndjson`** is always the **`Daemon.ready`** notification (reference, not a full object blob).
5. Lines 2+ are JSON-RPC **responses** in script order (one response per request line).
6. **`test-rpc.sh`** uses **`rpc_line N`** + **`jq_line_ok`** on each expected line.
7. (T1+) After script run, assert **`files.sqlite`** / filesystem if the case needs persistence.
8. Teardown: remove temp dir on success; leave on failure.

### Script file format

```text
# comments and blank lines ignored
{"jsonrpc":"2.0","id":1,"method":"Daemon.hello","params":{"protocol":1,"client":"test-rpc"}}
{"jsonrpc":"2.0","id":2,"method":"ProjectManager.load_projects_from_db","params":{}}
{"jsonrpc":"2.0","id":99,"method":"Daemon.shutdown","params":{}}
```

Assert on **JSON fields** only (`.result.path`, `.result.id`, `.result | type == "array"`, `.error.code`). No GObject deserialize.

### Shell helpers (landed)

```bash
# tests/test-rpc-common.sh

  OLLMFILES_IS_TEST=1 "$OLLMFILESD" --interactive --data-dir="$TEST_DIR" \

```

## Fire-and-forget and DB persistence

### In-memory DB, deferred disk write

`ollmfilesd` uses `SQ.Database` — **in-memory SQLite** with backup to `{data_dir}/files.sqlite`. Daemon opens DB with **`autosave: true`**: every 60s while `is_dirty`, calls `backupDB()` (coalesced, up to ~2.5s). **`cleanup()`** calls `backup_real()` (immediate). Most `saveToDB(..., sync: false)` calls set `is_dirty` only.

**Implication:** a successful RPC response does **not** mean `files.sqlite` on disk is updated yet.

### Server replies before async work finishes

Some handlers return `ok` (or a row) while work continues on the main loop:

| Wire method | RPC response | Work after reply |
| ----------- | ------------ | ---------------- |
| `File.write` | immediate `msg: ok` | `buffer.write_real.begin()` — disk + DB |
| `File.delete` | immediate `msg: ok` | `delete_manager.remove.begin()` + `cleanup.begin()` |
| `File.register` | waits | reply in `convert_fake_file_to_real` callback |
| `ProjectManager.activate_project` | waits | reply at end of `activate_project` async (can be slow if scan runs) |
| `ProjectManager.create_project` | sync | in-memory DB updated; disk backup still deferred |
| `ProjectManager.remove_project` | sync | same |

### Preferred validation pattern: script → shutdown → SQLite

For mutations that touch the DB (and for fire-and-forget server methods), end the script with **`Daemon.shutdown`** (last request line). After **`run_rpc_script`** returns, assert on **`{data_dir}/files.sqlite`** with `sqlite3` (row present/absent, `delete_id`, path, `id > 0`, …).

```text
# t1-create.script (sketch)
{"jsonrpc":"2.0","id":1,"method":"Daemon.hello","params":{"protocol":1,"client":"test-rpc"}}
{"jsonrpc":"2.0","id":2,"method":"ProjectManager.create_project","params":{"path":"/abs/fixture/path"}}
{"jsonrpc":"2.0","id":3,"method":"Daemon.shutdown","params":{}}
```

```bash
run_rpc_script "$SCRIPT" "$RPC_OUT" "$RPC_ERR"
# jq on rpc_line 2 (hello response is line 2; line 1 is Daemon.ready)
sqlite3 "$TEST_DIR/files.sqlite" "SELECT path FROM filebase WHERE delete_id = 0 AND is_project = 1"
```

**In-session asserts** (no shutdown): use **`jq`** on the matching **`rpc_line`** for the response — e.g. T1.3 reload after create in the same script, before shutdown.

### When to use RPC reload vs SQLite

| Assert | Use |
| ------ | --- |
| Wire shape, error codes, immediate `result` fields | `jq` on `rpc_line` from `out.ndjson` |
| Row persisted, soft-delete, project list in DB | `sqlite3` after script ends (**shutdown** in script flushes via H2) |
| File bytes on disk | filesystem after script ends |

### Suite structure

- **Smoke (T0):** single script; no DB assert needed.
- **Persistence (T1–T2):** script ends with **`Daemon.shutdown`**; then `sqlite3` / filesystem asserts.
- **Mixed scripts:** in-session `jq` on intermediate lines + shutdown + SQLite for the same run.

Fire-and-forget RPC handlers (`File.write`, `File.delete`) may still be in flight at shutdown — put **`shutdown`** after those calls in the script only when the handler has finished on the main loop, or assert in-session via a follow-up read RPC in the same script.

## Assertions and wire details

- Tranches **0–2:** **`--interactive --rpc-script`** + **`jq`** on `out.ndjson` + **`sqlite3`** (persistence).
- Success: `jq -e '.error == null'`. Failure: log full line; assert `.error.code`, `.error.message`.
- **`result` objects:** assert scalar JSON fields (`.result.path`, `.result.id`, array length). json-glib may emit kebab-case keys — match wire, not Vala property names.
- **Persistence:** last script line **`Daemon.shutdown`** → `run_rpc_script` returns → assert `files.sqlite` (H2).
- **`File.changed.check`**: server returns status in `msg` as int string (current draft) — test documents contract.
- **Scan side effects:** use `skip_scan: true` on activate unless explicitly testing scan (slow, flaky in CI).
- **Filesystem checks:** `test -f`, `cmp`, `sqlite3` — after script completes.
