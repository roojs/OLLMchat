## Wire shape

Flat GObject properties on **`Notification`** — progress payload in generic **`message`** (`queue_size` + space + path; receiver splits on first space):

```json
{"jsonrpc":"2.0","method":"event.vector.scan_update","object-type":"Vector","message":"12 /path/to/file.vala"}
```

Queue empty:

```json
{"jsonrpc":"2.0","method":"event.vector.scan_update","object-type":"Vector","message":"0 "}
```

### 2. `libocrpc/Transport/Listen.vala` — broadcast hook

**Why:** Server must push notifications to all clients, not only reply on a request connection.

```vala
		public virtual void broadcast(GLib.Object gobject)
		{
		}
```

---

### 3. `libocrpc/Transport/SocketListen.vala` — fan-out

**Why:** Production socket clients each need the notification line.

#### Add — before `stop()`

```vala
		public override void broadcast(GLib.Object gobject)
		{
			foreach (var connection in this.connections) {
				connection.write(gobject);
			}
		}
```

---

### 4. `ollmfilesd/Stdio.vala` — stdout broadcast

**Why:** **`--interactive`** / **`--rpc-script`** must see the same notification lines as socket clients.

#### Add — before `stop()`

```vala
		public override void broadcast(GLib.Object gobject)
		{
			if (this.connection != null) {
				this.connection.write(gobject);
			}
		}
```

---

### 5. `ollmfilesd/Application.vala` — broadcast entry point

**Why:** **`BackgroundScan`** should not reach into **`Listen`** implementations directly.

#### Add — before `cleanup()`

```vala
		public void broadcast_notification(OLLMrpc.Notification notification)
		{
			if (this.listen != null) {
				this.listen.broadcast(notification);
			}
		}
```

#### Replace with — `emit_scan_update` body

```vala
        private void emit_scan_update (int queue_size, string current_file)
        {
            GLib.debug ("scan banner update queue_size=%d file=%s",

            this.app.broadcast_notification (new OLLMrpc.Notification () {
                method = "event.vector.scan_update",

            });
        }
```
