# 6.6 DONE: Session restore: chunked UI replay and `GLib.Cancellable`

**Status: DONE**

This plan is **completed**. Implemented in `libollmchat/History/Manager.vala` (`cancel_restore`, `session_restored`, `switch_to_session`), `libollmchat/History/Session.vala` (inherited `activate()`; `internal async restore_messages(GLib.Cancellable)`), and `libollmchatgtk/ChatWidget.vala` (`session_activated` / `session_restored` wiring; removed dead `load_messages()`).

**Note on numbering:** Plan id **6.6** is also used for `docs/plans/done/6.6-DONE-write-executor-next-section-heading-validation.md` (write executor `next_section`). This file is the **session restore / chunked UI** **6.6** archive.

---

## Purpose

Restoring a large saved session blocks the GTK main loop: `Session.restore_messages()` runs synchronously inside `Session.activate()` and calls `Manager.message_added` for every stored message, each of which drives heavy chat UI work (`append_complete_assistant_message`, markdown, etc.). The UI appears to hang until the full history is rendered.

This plan defines an implementation that **yields between messages** (e.g. ~100 ms via `GLib.Timeout`) so the main loop stays responsive, and uses **`GLib.Cancellable`** to stop in-flight replay when the user switches to another session (or another operation cancels the same token). **No extra visible “loading” chrome** is required: an empty chat filling in progressively is enough feedback.

---

## Current behaviour (reference)

- `Manager.switch_to_session()` calls `loaded_session.activate()`.
- `Session.activate()` calls `restore_messages()`, which loops over `session.messages` and emits `manager.message_added(...)` for each non-transient message in one shot.
- `ChatWidget` connects to `message_added` and renders each message. **`ChatWidget.load_messages()` is dead code** (private, never called — see §4).

---

## Proposed behaviour

1. **Do not** replay the full history synchronously inside `activate()`.
2. After `activate()`, run **`async` replay** that emits one stored message, **`yield`**s (~100 ms), repeats. Optionally tune delay or batching later.
3. **Cancellation:** `Manager` owns a **`GLib.Cancellable`** used only for the current session-restore replay. At each **`switch_to_session` start**, **`cancel()`** the previous cancellable (if any), then **`new GLib.Cancellable()`** for this switch. Pass that instance into **`Session.restore_messages(cancellable)`** (async). Between chunks, if **`cancellable.is_cancelled()`**, return immediately (optionally also check **`manager.session != this`** as a defensive guard).
4. **Signals:** Emit **`session_restored(session)`** after replay completes normally, after cancel (early return), or immediately when there is nothing to replay—so `ChatWidget` can clear `restoring_session`, re-enable scroll, and scroll to bottom. Adjust **`session_activated`** so it does not clear `restoring_session` before history is on screen.

---

## Why `GLib.Cancellable` instead of a generation counter

- Standard GLib pattern for cooperative cancellation; works with other APIs that already take a `Cancellable?`.
- **`cancel()`** is explicit: starting a new switch invalidates the previous restore without maintaining a separate monotonic **`uint`**.
- Optional: still assert **`manager.session == this`** in the loop if you want belt-and-suspenders against stray main-loop continuations.

## Implementation notes (for whoever implements)

- Reuse the **streaming tail** logic at the end of the old synchronous `restore_messages()` (last `content-stream` / `think-stream` when `is_running`) unchanged in ordering; run it **after** all prior messages have been replayed.
- **`ChatWidget.load_messages()`** — **delete** in this work: it is `private`, and `rg 'load_messages\\(' --glob '*.vala'` finds **no callers** outside its definition; session switch has long used `message_added` / `Session.restore_messages` instead.
- Verify **EmptySession** and **SessionPlaceholder → Session** paths: **`restore_messages`** runs only for **`Session`**; empty **`messages`** is a no-op inside **`restore_messages`**.
- Test: very large session, switch away mid-restore, ensure cancel stops old replay and new session loads cleanly.

---

## Acceptance criteria

- Restoring a session with many messages keeps the window responsive (UI events processed between chunks).
- User can select another session while the first is still loading; in-flight replay is cancelled via **`Cancellable`** and the new session restores correctly.
- No duplicate messages; streaming mid-restore edge cases remain correct.

```vala
			// Deactivate current session
			this.session.deactivate();
			
			// Load session data if needed (no-op for already loaded sessions)
			// For SessionPlaceholder, this returns a new Session object
```

**Add** this method in its place:

```vala
		public async void switch_to_session(SessionBase session) throws Error
		{
			if (this.cancel_restore != null) {
				this.cancel_restore.cancel();
			}
			this.cancel_restore = new GLib.Cancellable();
			var cancellable = this.cancel_restore;

			this.session.deactivate();

			SessionBase? loaded_session = yield session.load();

			if (loaded_session == null) {
				throw new GLib.IOError.FAILED("Session load returned null");
			}

			this.session = loaded_session;
			loaded_session.activate();

			this.session_activated(loaded_session);
			this.agent_status_change();

			if (loaded_session is Session) {
				yield ((Session) loaded_session).restore_messages(cancellable);
			}

			this.session_restored(loaded_session);
		}
```

```vala
		private void restore_messages()
		{
			if (this.messages.size == 0) {
				return;
			}
			
			for (int i = 0; i < this.messages.size - 1; i++) {
				if (this.messages[i].role == "ui-waiting") {
					continue;  // Transient; do not restore
				}
				this.manager.message_added(this.messages[i], this);
			}
			
			var last_msg = this.messages[this.messages.size - 1];
			if (last_msg.role == "ui-waiting") {
				return;  // Transient; do not restore
			}
			
			if (!this.is_running || 
				(last_msg.role != "content-stream" && last_msg.role != "think-stream")) {
				this.manager.message_added(last_msg, this);
				return;
			}
			
			// For streaming messages, set up current_stream_message and emit through streaming infrastructure
			// This creates the block in streaming mode (not finalized) so new chunks can append to it
			this.current_stream_message = last_msg;
			this.current_stream_is_thinking = (last_msg.role == "think-stream");
			
			// Create a minimal response object for restoration
			// Set is_thinking to match the stream type so append_assistant_chunk() uses correct formatting
			var dummy_message = new OLLMchat.Message("assistant", "", "");
			var dummy_response = new Response.Chat(
					this.agent.chat().connection, this.agent.chat()) {
				message = dummy_message,
				done = false,
				is_thinking = this.current_stream_is_thinking
			};
			
			// Call base.handle_stream_chunk() directly to emit through streaming infrastructure
			// This bypasses Session's handle_stream_chunk() which would append content again
			// The message already has the full content, we just need to display it in streaming mode
			base.handle_stream_chunk(last_msg.content, this.current_stream_is_thinking, dummy_response);
		}
```

**Add** this method where `restore_messages()` was (same region of the file; this **replaces** the old sync method—same name, new async signature with **`GLib.Cancellable`**):

```vala
		/**
		 * Replays persisted messages to the UI with delays so the main loop stays responsive.
		 * Stops when `cancellable` is cancelled (e.g. user switched session) or this session is no longer active.
		 */
		internal async void restore_messages(GLib.Cancellable cancellable)
		{
			if (this.messages.size == 0) {
				return;
			}

			for (int i = 0; i < this.messages.size - 1; i++) {
				if (cancellable.is_cancelled() || this.manager.session != this) {
					return;
				}
				if (this.messages[i].role == "ui-waiting") {
					continue;
				}
				this.manager.message_added(this.messages[i], this);
				GLib.Timeout.add(100, () => {
					this.restore_messages.callback();
					return false;
				});
				yield;
			}

			if (cancellable.is_cancelled() || this.manager.session != this) {
				return;
			}

			var last_msg = this.messages[this.messages.size - 1];
			if (last_msg.role == "ui-waiting") {
				return;
			}

			if (!this.is_running ||
				(last_msg.role != "content-stream" && last_msg.role != "think-stream")) {
				this.manager.message_added(last_msg, this);
				return;
			}

			this.current_stream_message = last_msg;
			this.current_stream_is_thinking = (last_msg.role == "think-stream");

			var dummy_message = new OLLMchat.Message("assistant", "", "");
			var dummy_response = new Response.Chat(
					this.agent.chat().connection, this.agent.chat()) {
				message = dummy_message,
				done = false,
				is_thinking = this.current_stream_is_thinking
			};

			base.handle_stream_chunk(last_msg.content, this.current_stream_is_thinking, dummy_response);
		}
```

**Remove** the entire method and its doc comment in full (lines as in the tree when you implement):

```vala
		/**
		 * Loads and renders messages from the current session.
		 * 
		 * Renders messages from session.messages with filtering for UI display.
		 * Display special session types: "think-stream", "content-stream", "user-sent", "ui"
		 * Handle "end-stream" message: when encountered, flag to ignore the next message if it's a "done" message from streaming
		 * Skip certain chat message types: "system" (not displayed in UI), "tool" (already handled), "user" (use "user-sent" instead)
		 */
		private void load_messages()
		{
			// Scrolling is already disabled in switch_to_session() before this is called
			bool ignore_next_done = false;
			int total_messages = this.manager.session.messages.size;
			int visible_count = 0;
			int message_index = 0;
			
			GLib.debug("ChatWidget.load_messages: Starting to load %d messages from session", total_messages);
			
			foreach (var msg in this.manager.session.messages) {
				message_index++;
				// Use is_ui_visible to filter messages
				if (!msg.is_ui_visible) {
					// Handle special case: "end-stream" flags next message to ignore
					if (msg.role == "end-stream") {
						ignore_next_done = true;
					}
					continue;
				}
				
				// Reset ignore flag for visible messages
				ignore_next_done = false;
				visible_count++;
				
				// Truncate content for debug output (show first 20 chars)
				string content_preview = msg.content.length > 20 ? msg.content.substring(0, 20) + "..." : msg.content;
				GLib.debug("ChatWidget.load_messages: Adding message %d/%d (role=%s, content='%s')", 
					message_index, total_messages, msg.role, content_preview);
				 
				// Display message based on role (only UI-visible messages reach here)
				// Use manager.session for rendering (load_messages is called when switching sessions)
				var session = this.manager.session;
				switch (msg.role) {
					case "ui":
						// Use same path as live so code blocks get frames with titles
						var ui_msg = new OLLMchat.Message("assistant", msg.content, msg.thinking);
						this.chat_view.append_complete_assistant_message(ui_msg, session);
						break;
					case "ui-warning":
						var warning_msg = new OLLMchat.Message("assistant", "⚠️ " + msg.content, msg.thinking);
						this.chat_view.append_complete_assistant_message(warning_msg, session);
						break;
					case "think-stream":
						// For think-stream, content is the thinking text
						var stream_msg = new OLLMchat.Message("assistant", "", msg.content);
						this.chat_view.append_complete_assistant_message(stream_msg, session);
						break;
					case "content-stream":
					case "content-non-stream":
						// Render streaming/non-streaming messages as assistant messages
						var stream_msg = new OLLMchat.Message("assistant", msg.content, msg.thinking);
						this.chat_view.append_complete_assistant_message(stream_msg, session);
						break;
					default:
						// Should not reach here if is_ui_visible is working correctly
						break;
				}
			}
			
			GLib.debug("ChatWidget.load_messages: Finished loading %d visible messages out of %d total", visible_count, total_messages);
			
			// Don't re-enable scrolling here - keep it disabled after loading history
			// Scrolling will be re-enabled when new messages arrive (in on_message_created)
		}
```

### 6. Behaviour summary (post-patch)

| Step | What happens |
| ---- | ------------ |
| User switches session | `ChatWidget` clears chat, sets `restoring_session`; chat area repopulates as messages replay. |
| `Manager.switch_to_session` | Cancels previous **`cancel_restore`**, allocates a new one, loads, **`loaded_session.activate()`** (inherited **`SessionBase.activate()`** for **`Session`**), **`session_activated`** + **`agent_status_change`**, **`yield ((Session) loaded_session).restore_messages(cancellable)`** when `loaded_session is Session`, then **`session_restored`**. |
| Mid-restore switch | New switch **`cancel()`**s the old cancellable; old **`restore_messages`** returns at next check; new switch clears UI and runs a new replay. |
| End | **`session_restored`** clears **`restoring_session`**, re-enables scroll, scrolls to bottom. |
