## Purpose (shared)

- **`Runner`** owns **`this.progress`** (**`ProgressList`**). Updates use **`clear_pending()`** / **`add_pending()`** / **`add_completed()`**; **`rebuild()`** only when a full resync is required — **`docs/plans/done/7.14.5-DONE-restore-replay.md`**. **`ProgressList`** ctor takes **`Runner`**.
- **`ProgressView`**: **`Gtk.ColumnView`** + scroll — **no new `Gtk.TreeView`**.
- **`ProgressList`**: **`Details`** (completed + pending) + one **`ProgressRunner`** row (**`docs/plans/done/7.14.1.2-DONE-runner-vala.md`**). **`Tool`** under **`Details`** via **`ToolList`**.
- **`ProgressItem`:** **`status`** (**`PhaseEnum`**), **`title`**, **`status_str`** (**`PhaseEnum.to_human()`** — **`docs/plans/done/7.14.1-DONE-phase1-core.md`**; column binds **`status-str`** so Gtk updates when **`status`** changes). Same enum as markdown / wire / validation where values overlap — progress-only values (**`TOOLS_RUNNING`**, **`COMPLETED`**, …) live in **`docs/plans/done/7.14.1-DONE-phase1-core.md`**.

## Stage column — UI contract (normative)

**Typography**

- **Active** work: **bold**, **plain English**, **≤ 2 words** (e.g. “Creating”, “Retrying”).
- **Idle / complete:** **not bold**.
- **Not started:** **blank** in the stage column.
- **Done** (where a check fits): **grey Unicode tick** (e.g. `✓` with muted foreground) — use for **tool** row completion line in the table below; other row kinds use **empty** when finished as noted.

**`ProgressRunner` row — two columns** (archived **`7.14.1.2`** + **`7.14.2`** — see **`docs/plans/done/7.14.1-2-DONE-task-progress.md`**)

| Left column (`ProgressItem.title` or dedicated bindable) | Right column (`status_str` / stage) |
|------------------------------------------------------------|--------------------------------------|
| **Task Creation** — tries suffix **empty** on first attempt; during retries show **`(k/n)`** (e.g. `(2/5)`) so retry success is visible | **Creating** \| **Retrying** \| grey **✓** when this slice finished |
| **Reviewing Task List** — same tries suffix rules | **Reviewing** \| **Retrying** \| grey **✓** when this slice finished |

Tries suffix: **omit** when `k == 0` / first send; **show** once in a retry loop (**`7.14.3`** — **`docs/plans/done/7.14.3-DONE-data-stage-refinement.md`**; same optional **`(k/n)`** pattern for **`Details`** / **`Tool`** retry rows).

**`Details` row**

| State | Stage column |
|-------|----------------|
| (typical) | **Refine** → … → **empty** when idle for that slice |
| Retry | **Retrying** |
| Optional merge | **Post review** (if we merge that concept) |

Use **`Refine`**, **`Retrying`**, **`Post review`** as short labels consistent with **`PhaseEnum.to_human()`** output (**`docs/plans/done/7.14.1-DONE-phase1-core.md`**).

**`Tool` row** (`PhaseEnum.to_human()` — **`docs/plans/done/7.14.1-DONE-phase1-core.md`**)

| `PhaseEnum` / slice | Stage column |
|---------------------|----------------|
| `TOOLS_RUNNING` | **Running Tool** |
| `EXECUTION` | **Review output** |
| `EXECUTION_RETRY` | **Retry Review output** |
| `COMPLETED` | grey **✓** (tick) |
| `NONE` | **blank** |

`POST_EXEC` / **Post review** applies to **`Details.run_post_exec`**, not `Tool.run` — see **`docs/plans/done/7.14.1.1-DONE-tool-vala.md`** grid.

## `PhaseEnum` and **`ProgressItem.status_str`**

- **`ProgressItem.status`** (**`PhaseEnum`**) is the row’s progress slice; **`ProgressItem.status_str`** = **`status.to_human()`** (**`docs/plans/done/7.14.1-DONE-phase1-core.md`**) — **Pango** markup for **inactive / active / done** (see enum doc). Bind **`status-str`** so the stage column updates when **`status`** changes.
- **`is_active`** drives **bold** vs normal for the active slice; **`NONE`** → blank stage (**`to_human()`** returns empty).
- **`ProgressRunner`** is a **`ProgressList`** row (**`docs/plans/done/7.14.1.2-DONE-runner-vala.md`**); **`title`** + **`status_str`** bind the two columns above.
