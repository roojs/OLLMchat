# 7.14.6 Progress tree → chat scroll (`message.idx` binding)

**Status:** done (archived)

**Archived:** 2026-05-13 — **`ProgressItem`** carries **`message`**, **`idx_notify_id`**, default **`assign_message`**, and **`msg_idx_txt`** via **`msg_idx_to_string()`** (span **`idx_first`–`idx_last`**). ## Purpose

- 🔷 Stop storing duplicate `msg_idx` on progress rows.
- 🔷 Make `message.idx` the only source of row Idx truth.
- 🔷 Let each row react to `message.notify["idx"]` only when needed.
- 🔷 Keep Idx column updates automatic and centralized.

## Scope

| In scope | Out of scope |
| --- | --- |
| 🔷 `ProgressItem` owns `message` + `idx_notify_id` contract | 🚫 Any parse/overlay/walk to infer row ids |
| 🔷 Replace row-local `msg_idx` with `message`; keep **`msg_idx_txt`** as the computed label (`a ? b : c`) | 🚫 Scroll viewport behavior changes |
| 🔷 Use object signal API (`this.message.disconnect(id)`) for watcher lifecycle | 🚫 Progress tree hit-test / expander refactors |
| 🔷 Replace manual row id assignment with `assign_message(m)` calls | 🚫 Extra manager/helper classes |

## Acceptance criteria

- ℹ️ **As implemented:** indices are **`Message.idx_first` / `idx_last`** (span); watcher is **`notify["idx-last"]`**, not a single **`notify["idx"]`** as in some fences below.

- 🔷 `ProgressItem` has abstract `message`, `idx_notify_id`, `msg_idx_txt`, and a **non-abstract** default **`assign_message(...)`** on the interface.
- 🔷 `ProgressItem` no longer has stored `msg_idx` (only `msg_idx_txt`, computed from `message.idx`).
- 🔷 `assign_message(...)` always notifies `msg_idx_txt` immediately.
- 🔷 If `message.idx < 0`, row subscribes to `message.notify["idx"]`.
- 🔷 If `message.idx >= 0`, row does not subscribe.
- 🔷 Re-assignment disconnects previous watcher with `this.message.disconnect(this.idx_notify_id)`.
- 🔷 All direct `*.msg_idx = ...` sites are removed from this flow.

## Proposed behavior

- 🔷 Row carries the message reference, not a copied idx number.
- 🔷 Idx column text stays **`msg_idx_txt`**, same ternary style as today: `(message != null && message.idx >= 0) ? message.idx.to_string() : "—"`.
- 🔷 Watcher is short-lived and conditional (unset idx only).
- 🚫 Do not add parsing, scanning, or lookup overlays.

### 1. `liboccoder/Task/ProgressItem.vala` — interface contract ⏳

#### Remove

```vala
	/** Last **`Message.idx`** for this row (**continue-from** / **`scroll_to_idx`**); **-1** if unset. */
	public abstract int msg_idx { get; set; }
```

#### Add — message-first contract with watcher id (**keep `msg_idx_txt` name**).

```vala
	public abstract OLLMchat.Message? message { get; set; }

	public abstract ulong idx_notify_id { get; set; }
```

#### Add — **`assign_message` on the interface as a default implementation** (not `abstract`), so implementors only declare **`message`**, **`idx_notify_id`**, and **`msg_idx_txt`**.

```vala
	public void assign_message(OLLMchat.Message m)
	{
		if (this.message != null && this.idx_notify_id != 0) {
			this.message.disconnect(this.idx_notify_id);
			this.idx_notify_id = 0;
		}
		this.message = m;
		this.notify_property("msg_idx_txt");
		if (m.idx >= 0) {
			return;
		}
		this.idx_notify_id = m.notify["idx"].connect(() => {
			this.notify_property("msg_idx_txt");
			if (m.idx >= 0 && this.idx_notify_id != 0) {
				m.disconnect(this.idx_notify_id);
				this.idx_notify_id = 0;
			}
		});
	}
```

#### Keep (unchanged signature — implementation derives idx text from `message.idx`)

```vala
	/** Label text for the Idx column (**digits** or **—**). */
	public abstract string msg_idx_txt { owned get; }
```
