## Premise: one daemon, many lightweight clients

- **Daemon:** one `OLLMfiles.ProjectManager` — SQLite, FAISS queues, filesystem scan, canonical `projects` / caches.
- **Each UI window:** its own **`V2/ProjectManager`** + its own **`RpcClient`** (separate socket sessions).
- V2 PM is thin — holds `active_project` / `active_file`, Gtk signals, and **`RpcClient`** only.
- **`RpcClient.connect()`** runs lazily on first `call()` (not in V2 PM constructor).

---

## Phase 1 — Bootstrapping ✅

**`RpcClientBoot.ensure_daemon()`** — pid file + socket under `~/.local/share/ollmchat/`:

- Socket accepts → done.
- Pid alive, socket dead → SIGTERM, grace, remove stale socket/pid, respawn.
- Else → remove stale socket/pid, spawn `ollmfilesd` (daemon forks to background).
- Poll socket after spawn (`startup_wait`, default 5s); up to `max_attempts` (default 3).

**`RpcClient.connect()`:** `RpcClientBoot.ensure_daemon` → Unix connect + **`Daemon.hello`**.

**Not wired:** Meson; systemd `prefer_systemd` (out of scope).

## Phase 2 — Client `libocfiles/V2/ProjectManager.vala` ✅

Thin client — no direct filesystem / DB / scan.

### Local (no RPC)

| Piece | Notes |
| ----- | ----- |
| `active_project`, `active_file` | Per-window UI state; signals emitted local-first |
| `buffer_provider` | Local / no-op base |
| `disable_initial_scan` | Passed as `params.skip_scan` on `activate_project` |
| `restore_active_state` | Client-local orchestration (no RPC composite) |
| `on_file_metadata_change` | Deprecated stub — metadata stays client-only |

### PM tranche — RPC relay ✅

| V2 method | Wire | Client pattern |
| --------- | ---- | -------------- |
| `load_projects_from_db` | `ProjectManager.load_projects_from_db` | await `call`; hydrate `projects` from `Folder[]` |
| `create_project` | `ProjectManager.create_project` | await `call`; append to `projects` |
| `remove_project` | `ProjectManager.remove_project` | local state first; fire-and-forget `call` |
| `activate_project` | `ProjectManager.activate_project` | local state first; fire-and-forget `call`; `skip_scan` param |

**Result types:** `Folder` / `Folder[]` via `result_type` + `Rpc.types` (not `Rpc.ProjectList` — deferred).

## Phase 3 — Server `ollmfilesd/ProjectManager.vala` + dispatch ✅

**`Listen.ProjectManager`** — wire object name matches dispatch (`ProjectManager.*`).

**Relay pattern** — `rpc_{method}` signals in `construct`:

| Wire method | Handler |
| ----------- | ------- |
| `ProjectManager.load_projects_from_db` | `yield load_projects_from_db(request)` → `Folder[]` |
| `ProjectManager.create_project` | `create_project(path)` → `Folder` |
| `ProjectManager.remove_project` | `remove_project(path_map[path])` → `ok` |
| `ProjectManager.activate_project` | set `disable_initial_scan` from `skip_scan`; `yield activate_project(request, folder)` → `ok` |

Business logic stays in server PM — relay is param mapping + `session.reply` only.

**Multi-window:** server `active_project` is last-writer (single shared PM). Per-window ownership deferred to [`2.10.4.7`](2.10.4.7-active-state-client-only.md).

## Decisions (resolved)

1. **Wire result types** — `Folder` / `Folder[]` with `result_type` string; `Rpc.ProjectList` not added. `Folder.rpc_register()` / `File.rpc_register()` referenced from `RpcClient` static construct — add when Meson wires V2 types.
2. **`ensure_daemon` spawn** — `ollmfilesd` on `PATH`; daemon forks; boot polls socket. RPC integration tests use shell spawn ([`2.10.4.10`](2.10.4.10-rpc-tests.md)), not `RpcClientBoot`.
3. **Server `active_project`** — last activate wins until [`2.10.4.7`](2.10.4.7-active-state-client-only.md).
4. **Wire method names** — full Vala names (`load_projects_from_db`, not `list`).
5. **`Listen.ProjectManager`** — PascalCase property matches wire `ProjectManager` prefix.
