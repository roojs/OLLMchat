## Open issues

### I1 — Resolved: T0.1 test expected wrong wire shape

**Actual wire (plain **`Json.Serializable`** — correct for landed §2):**

```json
{"jsonrpc":"2.0","method":"Daemon.ready","object-type":"Daemon"}
```

**Was wrong:** jq expected **`params.object_type`** nested under JSON-RPC **`params`**. json-glib emits GObject properties at top level; underscore names → kebab-case (**`object-type`**).

**Fix:** T0.1 jq — **`.["object-type"] == "Daemon"`**. No **`Notification`** serialize overrides.

### I2 — Resolved: NDJSON multi-line from **`gobject_to_data`**

**Was:** `Json.gobject_to_data` hardcodes **`pretty=TRUE`** in json-glib → ready line spanned 5 lines → **`rpc_line 1`** read only `{`.

**Fix landed:** **`Connection.write(GLib.Object)`** / **`Client.write`** use **`Json.Generator`** with **`set_pretty(false)`**. Line 1 is now single-line compact.

### I3 — Resolved: script dispatch crash

**Was:** **`dispatch()`** without **`params_node`** → empty **`CallParam`** → segfault in **`Daemon.rpc_hello`**.

**Fix landed:** **`run_script`** passes **`parser.get_root().get_object().get_member("params")`** into **`dispatch`**.

## Notification model (reference)

Server→client lines are **hints**, not object transfer.

| Field | Role |
| ----- | ---- |
| **`method`** | What happened (e.g. **`Daemon.ready`**, later **`event.file.changed`**) |
| **`object_type`** | Wire type name (e.g. **`Daemon`**, **`File`**) — same namespace as **`Response.result_type`** / **`OLLMrpc.register`** |
| **`id`** | Object id when the type has one; **`0`** for singletons like **`Daemon`** |

**Flow:** notification → client decides → **`Client.call`** (e.g. **`Daemon.hello`**, **`File.read`**) to load state.

**T0 ready line:** **`Daemon.ready`** + **`object_type: Daemon`** — not a full **`Daemon`** blob. Script still runs **`Daemon.hello`** requests for round-trip proof.

**Future** ([`2.10.4.8`](2.10.4.8-per-client-project-notifications.md)): **`event.file.changed`** with **`object_type: File`**, **`id`**, maybe **`path`** string — client refetches; no **`File`** row in notification.

**💩** Separate **`NotificationKind`** enum (int) — confirm if **`method`** + **`object_type`** is enough for T0.

## Root cause (request path)

Call sites invoke **`dispatch()`** without **`params_node`** → empty **`CallParam`** → crash in **`Daemon.rpc_hello`**.

---

## Proposed behaviour (requests)

Order: **`gobject_from_data`** from line string (try/catch) → **`Parser`** for **`params`** only → **`connection`** → **`dispatch(params)`**.

If **`gobject_from_data`** succeeds, the line is valid JSON object wire — no **`root`** null/type checks on **`Parser`**.

```vala
OLLMrpc.Request? request = null;
try {
	request = Json.gobject_from_data(
		typeof(OLLMrpc.Request),
		line,
		-1
	) as OLLMrpc.Request;
} catch (GLib.Error e) {
	GLib.warning("parse error: %s", e.message);
	continue;
}

var parser = new Json.Parser();
parser.load_from_data(line, -1);

request.connection = this;
request.dispatch(
	parser.get_root().get_object().get_member("params")
);
```

### 2. `libocrpc/Notification.vala` — reference fields (no `CallParam`)

**Why:** Notifications are references; drop **`CallParam param`** and custom **`params`** object deserialize.

**Where:** properties; remove **`serialize_property`** / **`deserialize_property`** overrides if plain properties suffice.

**Depends on:** none.

#### Remove

```vala
		public CallParam param { get; set; default = new CallParam(); }
```

#### Replace with

```vala
		public string object_type { get; set; default = ""; }
		public int id { get; set; default = 0; }
```

#### Remove — entire `serialize_property` override

```vala
		public override Json.Node serialize_property(
			string property_name,
			Value value,
			ParamSpec pspec
		) {
			switch (property_name) {
				case "param":
					return Json.gobject_serialize(this.param);
				default:
					return default_serialize_property(property_name, value, pspec);
			}
		}
```

#### Remove — entire `deserialize_property` override

```vala
		public override bool deserialize_property(
			string property_name,
			out Value value,
			ParamSpec pspec,
			Json.Node property_node
		) {
			switch (property_name) {
				case "params":
					this.param = Json.gobject_deserialize(
						typeof(CallParam), property_node
					) as CallParam;
					value = Value(typeof(CallParam));
					value.set_object(this.param);
					return true;
				default:
					return default_deserialize_property(
						property_name, out value, pspec, property_node
					);
			}
		}
```

**🔷** Verified wire (json-glib default): top-level **`object-type`** (kebab-case); **`id`** omitted when **`0`**. T0.1 jq matches this — no serialize overrides.
