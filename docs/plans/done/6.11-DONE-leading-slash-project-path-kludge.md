# 6.11 Leading slash on project-relative paths: `Format.resolve` + `is_dir` fixup

**Status: done**

**Implemented in:** `libocmarkdown/document/Format.vala` (`resolve`, `is_dir`), `liboccoder/Task/ValidateLink.vala` (`file`: `link.resolve` before `resolved_path`), `liboccoder/Task/ResolveLink.vala` (`preload_file`, `preload_links` file branch, `file`).

See **`.cursor/rules/CODING_STANDARDS.md`** — **Checklist for all plans** (new methods only where named here, **`this.`** / **`GLib.`**, multiline docblocks, no **`@"`**, etc.).

**Related:** [6.3 DONE — reference links allow relative paths](6.3-DONE-reference-links-allow-relative-paths.md) (`scheme` / `path` / `is_relative`, `abspath`).

---

## Purpose

Models emit **`/liboccoder/...`** style paths; Unix treats them as absolute from **`/`**, so **`abspath(project_root)`** does not join the project. **Silent fix:** **`Markdown.Document.Format.resolve(project_root)`** mutates **`path`** / **`href`** (and **`is_relative`** when fixing) **only when** a concrete **file or directory exists** at a corrected path under the project. **`is_dir`** repeats the same “try raw, then try leading-slash strip under project” probe for directories **without** calling **`resolve`** (no double mutation). **`ValidateLink.file()`** and **`ResolveLink`** call **`resolve`** before path lookup so validation and injection agree.

---

## Scope

| In scope | Out of scope |
| -------- | ------------ |
| **`Format.resolve(string project_root)`** — mutates link only when fix path exists on disk | Parser changes in **`Render.on_a`** (optional follow-up) |

---

## Acceptance criteria

1. **`[x](lib/foo.vala)`** (relative) unchanged when **`resolve`** runs; **`[/lib/foo.vala]`** with project **`/tmp/p`** and real file **`/tmp/p/lib/foo.vala`** ends with **`link.path`** (and **`href`**) pointing at that absolute path after **`resolve`**.
2. **`/etc/passwd`**-style paths: **no** mutation unless that path exists; if not, **`resolve`** leaves **`path`** unchanged.
3. Paths already under project as real absolutes (prefix of **`project_root`**) are **not** double-joined.
4. **`is_dir(project_root)`** returns **true** for a directory reachable either as given or after the same leading-slash-under-project probe.
5. **`ValidateLink`** and **`ResolveLink`** both see the same post-**`resolve`** **`link.path`** for **`file:`** links.
6. **Negative (no fix on disk):** **`path`** **`/lib/nope.vala`** when **`/lib/nope.vala`** does not exist **and** **`{project_root}/lib/nope.vala`** does not exist — **`resolve`** returns without mutating; **`is_dir`** is false.

### 1. `libocmarkdown/document/Format.vala` — insert `resolve`

#### Keep (before — unchanged)

```vala
		public string abspath(string to_path)
		{
			if (this.scheme != "file" || this.path == "" || GLib.Path.is_absolute(this.path)) {
				return this.path;
			}
			var base_file = GLib.File.new_for_path(to_path);
			var resolved = base_file.resolve_relative_path(this.path);
			var result = resolved.get_path();
			return result != null ? result : "";
		}

```

#### Add

Insert **after** the **`abspath`** method above, **before** **`is_dir`**. Brief **`//`** lines on non-obvious guards (see **`.cursor/rules/CODING_STANDARDS.md`** — comment line length).

```vala
		/**
		 * Correct a file link path when the model used a leading slash so it was parsed
		 * as absolute from the filesystem root. Only mutates {{{path}}} and {{{href}}} when
		 * a file or directory exists at the corrected path under {{{project_root}}}.
		 *
		 * @param project_root active project root directory
		 */
		public void resolve (string project_root)
		{
			if (this.scheme != "file" || this.path == "" || project_root == "") {
				return;
			}
			var raw = this.path;
			var gf = GLib.File.new_for_path (raw);
			// Target already exists as written (no rewrite).
			if (gf.query_exists ()) {
				return;
			}
			// Absolute path already under project; joining again would be wrong.
			if (GLib.Path.is_absolute (raw) && (raw == project_root || raw.has_prefix (project_root + "/"))) {
				return;
			}
			// Strip-from-root heuristic only for paths that look like /foo (not relative).
			if (!raw.has_prefix ("/")) {
				return;
			}
			var candidate = GLib.Path.build_filename (project_root, raw.substring (1));
			// Keep fix inside project: allow project_root itself, else require
			// project_root + "/" so "/tmp" does not prefix-match "/tmp2/...".
			if (candidate != project_root && !candidate.has_prefix (project_root + "/")) {
				return;
			}
			var cf = GLib.File.new_for_path (candidate);
			// Corrected path must exist before we rewrite link fields.
			if (!cf.query_exists ()) {
				return;
			}
			this.path = candidate;
			this.href = this.path + (this.hash != "" ? "#" + this.hash : "");
			this.is_relative = false;
		}

```

### 2. `libocmarkdown/document/Format.vala` — `is_dir` leading-slash probe

#### Remove

```vala
		/**
		 * True if this `file:` link resolves to an existing directory on disk
		 * (path resolved from [[project_root]] for relative links).
		 */
		public bool is_dir (string project_root)
		{
			if (this.scheme != "file") {
				return false;
			}
			var rp = this.is_relative ? this.abspath (project_root) : this.path;
			if (rp == "") {
				return false;
			}
			var gf = GLib.File.new_for_path (rp);
			if (!gf.query_exists ()) {
				return false;
			}
			return gf.query_file_type (GLib.FileQueryInfoFlags.NONE) == GLib.FileType.DIRECTORY;
		}

```

#### Replace with

Do **not** call **`resolve`** from **`is_dir`** (avoid double mutation). **Duplicate** only the “strip leading slash + **`build_filename(project_root, …)`** + directory” probe. Second probe: **one negative guard per line**, **`return false`** early (same style as **`resolve`**); end with explicit **`if (!cf.query_exists ())`** / not-directory **`return false`**, then **`return true`**. Use brief **`//`** on non-obvious guards (first vs second probe, under-project checks).

```vala
		/**
		 * True if this `file:` link resolves to an existing directory on disk
		 * (path resolved from [[project_root]] for relative links). If the path looks
		 * like a mistaken absolute-from-root project path, checks the same path under
		 * [[project_root]].
		 */
		public bool is_dir (string project_root)
		{
			if (this.scheme != "file") {
				return false;
			}
			var rp = this.is_relative ? this.abspath (project_root) : this.path;
			if (rp == "") {
				return false;
			}
			var gf = GLib.File.new_for_path (rp);
			// First probe: directory exactly at rp (as given or relative-resolved).
			if (gf.query_exists ()
				&& gf.query_file_type (GLib.FileQueryInfoFlags.NONE) == GLib.FileType.DIRECTORY) {
				return true;
			}
			// Second probe: mistaken absolute-from-root (/lib/...) → under project_root.
			if (!GLib.Path.is_absolute (rp)) {
				return false;
			}
			if (!rp.has_prefix ("/")) {
				return false;
			}
			if (project_root == "") {
				return false;
			}
			// Already under project: strip fix does not apply (first probe said not a dir).
			if (rp == project_root || rp.has_prefix (project_root + "/")) {
				return false;
			}
			var candidate = GLib.Path.build_filename (project_root, rp.substring (1));
			// Same under-project guard as resolve(); see resolve() for why.
			if (candidate != project_root && !candidate.has_prefix (project_root + "/")) {
				return false;
			}
			var cf = GLib.File.new_for_path (candidate);
			if (!cf.query_exists ()) {
				return false;
			}
			// Exists but is a file: not a directory target.
			if (cf.query_file_type (GLib.FileQueryInfoFlags.NONE) != GLib.FileType.DIRECTORY) {
				return false;
			}
			return true;
		}

```

## Tests

- Project root **`/tmp/proj`**, file **`lib/x.vala`**. Link **`/lib/x.vala`**: after **`resolve`**, **`path`** is **`/tmp/proj/lib/x.vala`** and **`query_exists`**. **`is_dir`** is false for that file path.
- Directory **`lib/d`** on disk at **`/tmp/proj/lib/d`**. Link **`/lib/d`**: after **`resolve`**, **`path`** is **`/tmp/proj/lib/d`** when that path exists; **`is_dir`** is true (via strip probe if raw did not resolve as a dir first).
- **Negative:** project **`/tmp/proj`**, link **`/lib/missing.vala`**, and **neither** **`/lib/missing.vala`** nor **`/tmp/proj/lib/missing.vala`** exists — **`resolve`** leaves **`path`** as **`/lib/missing.vala`**; **`is_dir`** is false (second probe also finds nothing).
- **`/etc/hosts`**: **`resolve`** does not rewrite unless that path exists (if it exists at root, early return; no mutation to project).
