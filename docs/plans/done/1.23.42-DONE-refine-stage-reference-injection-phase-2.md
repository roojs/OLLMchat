## Coding standards checklist

Copied from **`.cursor/rules/CODING_STANDARDS.md`** — “Checklist for all plans”. If a bullet conflicts with **§ Conventions** in the [parent plan](1.23.40-DONE-refine-stage-reference-injection.md), **the standards file wins**.

- **Nullable types**: Design new APIs/properties to avoid nullable types where possible (default objects / flags).
- **Null checks**: Avoid generic null checks; only where the design explicitly requires null (see standards for the narrow exceptions).
- **String interpolation**: No `@"..."` except multi-line usage/help/documentation; use `+` concatenation otherwise.
- **Temporary variables**: No one-use temporaries; no trivial aliases (allowed exception: long property chains **4+** steps).
- **Brace placement**: Line breaks before `{` for namespace/class/method; **inline** `{` for `if`/`for`/`while`/`switch`; never a one-line `if` with body on the same line.
- **`this.` prefix**: Use for instance member access in new/modified Vala.
- **GLib prefix & `using`**: Fully qualify `GLib.*`; do not add `using` for new code.
- **Property initialization**: Prefer `get; set; default = …` (or field defaults), not constructor assignment for those defaults.
- **Line length & breaking**: Break long method calls, concatenations, docblocks, and comments (docblock/comment line length per full standards).
- **StringBuilder**: Avoid `GLib.StringBuilder` unless building strings in loops with **hundreds** of iterations; use `+` and `string.joinv()`.
- **String building in loops**: Never accumulate with `+=` inside a loop; use `string.nfill()` / `replace()`, `string.joinv()`, etc.
- **ArrayList for strings**: Do not use `Gee.ArrayList<string>` only to join; use `string[]` with **`string[] name = {}`** only for empty init (not `new string[0]`).
- **Loops and nesting**: In loops, use **`continue`** to skip cases and keep the body flat; avoid **`else`** and nested **`if`** inside loops.
- **Character looping**: Do not iterate characters unless there is truly no alternative; prefer string methods and `GLib.Regex`.
- **File info try/catch**: Wrap `query_info()`-style calls only when file existence is unknown — not right after a successful read of the same path.
- **Try/catch scope**: Wrap only the minimal expression(s) that throw; keep setup outside `try`.
- **Underscore prefix**: No leading `_` on variables, fields, or properties.
- **`get_*` methods**: Prefer properties or verb-less/action names over `get_foo()` (see standards for exceptions).
- **Docblocks**: Multiline `/** … */` with `@param` / `@return` where relevant; not one-liners for real behavior.
- **Debug / warning messages**: No timestamps or monotonic time in `GLib.debug()` / `GLib.warning()`; no class/method prefixes in the message (file/line come from the log).
- **Gee.HashMap**: Use `.set()` / `.get()` / `.has_key()` / `.unset()` — not `[]`.
- **Gee.ArrayList**: Use `.get()` / `.set()` / `.add()` / `.remove_at()` — not `[]`.
- **Reducing nesting**: Prefer early `return` / `continue`; avoid `else` where flow stays clear; prefer `switch` over long `if` / `else if` chains when editing.
- **SQL table aliases**: Not applicable to this plan’s fences; if you add SQL elsewhere, no table aliases (full names only).
- **Plan Vala fences**: Do **not** use comment-only placeholders (`/* … */` or empty `return "";`) where real logic belongs — either **complete, compilable Vala** in the fence or **prose-only** steps (no fake stub block).

## Code changes — line-range file fragments (`#L12-L30`)

**Status:** **`OLLMfiles.File.contents(int start_or_count = -1, int end_line = -1)`** and **`line_count()`** are **implemented** in **`libocfiles/File.vala`**. **`Details.header_file`** / **`link_content`** implement this section’s fences in **`liboccoder/Task/Details.vala`** (**regex** on **`link.hash`**; **`header_file(..., start = -1, end = -1)`**).

### Goal

Allow **file** reference links to cite a **slice of a document by line number**, in addition to existing **heading** (`#my-section`) and **AST** (`#Namespace-Class-method`) fragments. Example:

`[Slice](/abs/path/to/File.vala#L12-L30)` — inject **lines 12–30 inclusive** ( **1-based** line numbers, same idea as GitHub’s **`#L…`** line anchors).

### Syntax

- **Only supported form:** **`#L<start>-L<end>`** — **both** line numbers use a literal **`L`** prefix (case-insensitive), e.g. **`#L12-L30`** = lines **12–30** inclusive (**1-based**). A **single** line is **`#L5-L5`**, not **`#L5`** alone.
- **Not supported** (ignore or treat as non–line-range): **`#L12`**, **`#L12-30`**, **`#-L1`**, **`#12-L30`**, or any fragment that does not match **`L<digits>-L<digits>`**.

**Precedence:** If the fragment **matches** this **exact** pattern, treat it as a **line range**; otherwise it may be a **heading** slug or **AST** path (different shapes).

### Previous behaviour (before this section landed)

- **`Markdown.Document.Format`** exposes **`link.hash`** for the fragment.
- **`link_content`** previously called **`header_file`** without line-range args, so **`#L12-L30`** was ignored. **Implemented:** **`link.hash`** matching **`^[Ll](\d+)-[Ll](\d+)$`** passes **1-based** **`start`** / **`end`** into **`header_file`**; **`header_file`** uses **`File.contents`** per the table below.

**Reference — `OLLMfiles.File.contents` (see `libocfiles/File.vala`):** **range** mode when **`end_line != -1`** (**1-based** inclusive). **Head** mode when **`end_line == -1`** (**`start_or_count ≤ 0`** ⇒ whole file; **`N > 0`** ⇒ first **`N`** lines). **`header_file`** uses **range** for **refinement** and for **non-refinement** fragments; **non-refinement** full file uses **`int.max(-1, start)`** (**`-1`** when **`start == -1`**) with **`end_line == -1`** — head, whole file (no **`line_count()`** in the call).

| Call | Meaning |
| ---- | ------- |
| **`file.contents(rs, re)`** (**`re != -1`**) | **Range:** lines **`rs`**…**`re`** (**1-based** inclusive). |
| **`contents(-1, -1)`** / **`contents(0, -1)`** | **Head:** whole file (per **`File`** — **`start_or_count ≤ 0`** when **`end_line == -1`**). |
| **Examples** (see fence below) | **REFINEMENT** no fragment **`contents(1, 20)`**; **EXECUTION** no fragment **`contents(int.max(-1, start), …)`** ⇒ **`contents(-1, -1)`**; **`#L…`** **EXECUTION** **`contents(start, end)`**. |

**Line totals:** **`file.line_count()`** for abbreviation checks.

### Parsing **`#L…`**: **regex in `link_content`** (no separate **`try_parse_*`**)

Fixed pattern **`^[Ll](\\d+)-[Ll](\\d+)$`** (**`CASELESS`**). Match on **`link.hash`** (fragment without **`#`**). **No `try`/`catch` around `match`**. **`int.parse(mi.fetch(n))`** inline in the **`header_file`** call — **no** temp locals for the captures, **no** validation of order or EOF (**`File.contents`** / **`buffer.get_text`** own bounds). **`new GLib.Regex(...)`** is written **inline** at the **`match`** call (pattern and flags visible there — **no** **`static`** field or **`static construct`**).

### `liboccoder/Task/Details.vala` — **`header_file`** (replace)

**Inputs** (what **`link_content`** passes): **full file** = **`start == -1`** and **`end == -1`**. **Line range** = **`start`** / **`end`** are **1-based** inclusive line numbers from **`#L…`** (e.g. **`1`**, **`35`**).

**What refinement should load** (same inputs, **preview** limits):

| Meaning | Input (`start`, `end`) | **Refinement** (effect) |
| -------- | ------------------------ | ------------------------- |
| Full file | **`-1`**, **`-1`** | **`contents(1, 20)`** — ask for lines **1–20**; **`File.contents`** / **`buffer.get_text`** clamp if the file is shorter (**no** **`min(…, line_count())`** in **`header_file`**). |
| Range | **`1`**, **`35`** | **`contents(1, min(35, 1 + 29))`** = **`contents(1, 30)`** — last line **`min(end, start + 29)`** (**30** lines along the fragment, **never** past **`end`**). |

**Not refinement** (execution, etc.): **full file** → **head** mode **`contents(int.max(-1, start), start == -1 ? -1 : end)`** ⇒ **`contents(-1, -1)`** (**`end_line == -1`**, **`start_or_count ≤ 0`** ⇒ whole file — see **`File.contents`**). **Range** → **range** mode **`contents(start, end)`**.

**Two `file.contents` calls only** — outer branch is **`PhaseEnum`**. **Refinement:** always **range** mode — **`int.max(start, 1)`**, last line **`start == -1 ? 20 : int.min(end, start + 29)`**. **Not refinement:** **`file.contents(int.max(-1, start), start == -1 ? -1 : end)`** — **`int.max(-1, start)`** matches **`int.max(start, 1)`**-style clamping (**`-1`** → whole-file head; **`start ≥ 1`** → fragment). No **`file.line_count()`** in **`contents`**.

```text
content = stage == PhaseEnum.REFINEMENT
	? file.contents(int.max(start, 1), start == -1 ? 20 : int.min(end, start + 29))
	: file.contents(int.max(-1, start), start == -1 ? -1 : end);
```

**Abbreviation (refinement only):** one **`if`** — **`PhaseEnum.REFINEMENT`** and (**no fragment** and file **>** 20 lines, **or** **fragment** and **`end > start + 29`**). Same message in both cases: **how many lines in the file** (**`file.line_count()`**) — no separate “requested range” wording.

```vala
internal string header_file(string line, OLLMfiles.File file, PhaseEnum stage,
	int start = -1, int end = -1)
{
	var content = stage == PhaseEnum.REFINEMENT
		? file.contents(int.max(start, 1), start == -1 ? 20 : int.min(end, start + 29))
		: file.contents(int.max(-1, start), start == -1 ? -1 : end);
	if (stage == PhaseEnum.REFINEMENT
			&& (
				(start == -1 && file.line_count() > 20)
				|| (start != -1 && end > start + 29)
			)) {
		content += "\n\n**This has been abbreviated.** The full content has "
			+ file.line_count().to_string() + " lines.\n";
	}
	var fence = (content.index_of("\n```") >= 0 || content.has_prefix("```")) ? "~~~~" : "```";
	return line + "\n\n"
		+ fence
		+ (file.language != "" ? file.language + "\n" : "\n")
		+ content + "\n"
		+ fence + "\n\n";
}
```
