## Approved scope (user sign-off)

ЁЯФ╖ What was approved тАФ implement only this (Phases 0тАУ2):

- ЁЯФ╖ `Agent.send_async` enqueues **chat** then **paired summarize**
  `PendingMessage` rows (same `done` promise) тАФ replaces
  `(new Summarizer(this)).run.begin(cancellable)`.
- ЁЯФ╖ **All queuing** lives on `Agent`:

- ЁЯФ╖ **All queued-item actions** live on `PendingMessage`:
  - `run` relays by `is_chat`

ЁЯФ╖ Layer contract (mandatory тАФ do not mix):

- ЁЯФ╖ `Agent` owns:

  - `send_async` enqueue + inline drain `while`

- ЁЯФ╖ `Agent` must **not** own:
  - chat deliver body, summarize body

- ЁЯФ╖ `PendingMessage` owns:
  - `is_chat`
  - `run` (relay + status flags + chat deliver)

- ЁЯФ╖ `PendingMessage` must **not** own `pending_messages`

## Problem

- ЁЯФ╖ Fire-and-forget `Summarizer.run.begin(cancellable)` after main chat тАФ
  overlapping summarizers; follow-up before prior summary exists.
- тД╣я╕П Shared main-turn `cancellable` with summarize тАФ Stop can cancel summary.

---

## Purpose (approved slice)

- ЁЯФ╖ FIFO on `Agent` тАФ chat and summarize entries; head runs one at a time.
- ЁЯФ╖ `send_async` enqueues chat + summarize per user send; `yield done` when both finish.
- ЁЯФ╖ Summarize uses its **own** `GLib.Cancellable`.
- ЁЯФ╖ No edits to `Session`, `Manager`, `ChatWidget`.

---

## Queue model

- ЁЯФ╖ **chat** (`is_chat = true`):
  - Agent: `yield entry.run(this)`
  - relays to chat deliver (`is_running` true)
- ЁЯФ╖ **summarize** (`is_chat = false`):
  - Agent: `yield entry.run(this)`
  - relays to `run_summarize` (`summarizing` true)

```
send_async:  enqueue chatтВБ тЖТ enqueue summarizeтВБ (same done)
             while (pending not empty): yield head.run(this)  // run relays by is_chat
```

---

## Current behaviour (pre-queue)

- тД╣я╕П Was: `Chatter.Agent.send_async()` тАФ inline chat deliver, then
  `(new Summarizer(this)).run.begin(cancellable)` fire-and-forget.
- тЬФя╕П Now: queue + paired summarize in tree (see phase pointers below).

---

## Proposed behaviour (approved)

- ЁЯФ╖ `send_async()` тАФ enqueue chat + summarize; when idle, inline `while`
  drains queue; `yield entry.done.future.wait_async()`.
- ЁЯФ╖ Drain loop тАФ pull head; always `yield head.run(this)` (`run`
  relays by `is_chat`); `run_summarize` resolves shared `done` on
  success; Agent `done.set_exception` on failure; drop paired summarize row on
  chat failure.
- тД╣я╕П No extra `ui-waiting` from `Agent` тАФ `Session.send` already
  emits `"waiting for тАж to reply"` before `send_async`; queue is agent-internal.

```vala
namespace OLLMchat.Chatter
{
	/**
	 * One queued chat or summarize step. {@link run} relays by {@link is_chat};
	 * queue drain lives on {@link Agent.send_async}.
	 */
	public class PendingMessage : GLib.Object
	{
		public bool is_chat { get; construct; default = true; }
		public Message? message { get; construct; }
		public GLib.Cancellable? cancellable { get; construct; }
		public Gee.Promise<bool> done { get; construct; }

		public PendingMessage(
			Message? message,
			GLib.Cancellable? cancellable,
			bool is_chat,
			Gee.Promise<bool>? done)
		{
			Object(
				message: message,
				cancellable: cancellable,
				is_chat: is_chat,
				done: done != null ? done : new Gee.Promise<bool>()
			);
		}

		/**
		 * Relay entry: chat deliver when {@link is_chat}, else
		 * {@link run_summarize}. Sets {@code is_running} / {@code summarizing}
		 * on {@link Agent} for the active phase.
		 *
		 * @param agent the Chatter agent supplying session and chat call
		 */
		public async void run(Agent agent) throws GLib.Error
		{
			// Summarize row: no message/cancellable; relay to run_summarize.
			if (!this.is_chat) {
				agent.summarizing = true;
				try {
					yield this.run_summarize(agent);
				} finally {
					agent.summarizing = false;
				}
				return;
			}

			agent.session.is_running = true;
			agent.session.manager.agent_status_change();
			agent.session.messages.add(this.message);

			var since_summary = agent.create_summary();
			Message? active_summary = null;
			if (since_summary.size > 0
				&& since_summary.get(0).role == "summary") {
				active_summary = since_summary.get(0);
				since_summary.remove_at(0);
			}

			var factory = (Factory) agent.session.manager.agent_factories.get(
				agent.session.agent_name);
			var outbound = new Gee.ArrayList<Message>();
			var tpl = factory.load_prompt(
				active_summary != null ? "chatter_followup.md" : "chatter_initial.md");
			outbound.add(new Message("system", tpl.system_fill(
				"conversation_summary",
				active_summary != null ? active_summary.content : "",
				"environment", factory.build_environment(agent.session))));

			foreach (var msg in since_summary) {
				outbound.add(msg);
			}

			try {
				yield agent.fill_model();
				yield agent.chat().send(outbound, this.cancellable);
			} finally {
				agent.session.is_running = false;
				agent.session.manager.agent_status_change();
			}
		}

		/**
		 * Summarize the completed chat turn (own cancellable тАФ not main-turn token).
		 *
		 * @param agent the Chatter agent for this session
		 */
		public async void run_summarize(Agent agent) throws GLib.Error
		{
			// Own cancellable тАФ Stop on main chat must not abort background summary.
			yield (new OLLMchat.Agent.Summarizer(agent)).run(
				new GLib.Cancellable());
			// Turn complete тАФ chat + summarize share this done promise.
			this.done.set_value(true);
		}
	}
}
```

#### Coding standards тАФ `PendingMessage`

- ЁЯФ╖ `run` relays тАФ `is_chat` branch sets `is_running`; `!is_chat` sets
  `summarizing` and yields `run_summarize`; Agent drain always
  `yield head.run(this)`.
- ЁЯФ╖ Actions throw тАФ `Agent.send_async` drain `while` catches and
  `done.set_exception`.
- ЁЯФ╖ No queue access тАФ no `pending_messages`, no chain.
- ЁЯФ╖ `run_summarize` resolves `done` тАФ success only; chat step does not.
- тД╣я╕П `try-catch-scope` тАФ chat path: outbound assembly outside `try`;
  `try`/`finally` around `fill_model` + `chat.send` (clears `is_running`).
  Model customize `catch` stays in `OLLMchat.Agent.Base.fill_model`.
  Summarize relay: `try`/`finally` only around `run_summarize`.

#### Replace with тАФ `send_async` (enqueue + inline drain)

```vala
		/**
		 * Enqueues chat + paired summarize; drains queue when idle; waits until
		 * both steps for this message complete.
		 *
		 * @param message API user message (`user-sent` / `ui` already from Session)
		 * @param cancellable optional cancel token for the main/tool request
		 */
		public override async void send_async(
			Message message,
			GLib.Cancellable? cancellable = null) throws GLib.Error
		{
			var entry = new PendingMessage(
				message, cancellable, true, new Gee.Promise<bool>());
			this.pending_messages.add(entry);
			// Paired summarize row тАФ same done promise; enqueued before drain starts.
			this.pending_messages.add(
				new PendingMessage(null, null, false, entry.done));
			// Another send_async is already draining; wait on this turn's promise.
			if (this.pending_processing) {
				yield entry.done.future.wait_async();
				return;
			}
			this.pending_processing = true;
			while (this.pending_messages.size > 0) {
				var head = this.pending_messages.remove_at(0);
				try {
					yield head.run(this);
				} catch (GLib.Error e) {
					head.done.set_exception(e);
					// Chat failed тАФ skip the paired summarize row already at queue head.
					if (head.is_chat && this.pending_messages.size > 0) {
						this.pending_messages.remove_at(0);
					}
				}
			}
			this.pending_processing = false;
			yield entry.done.future.wait_async();
		}
```
