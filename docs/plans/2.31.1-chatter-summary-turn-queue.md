## Problem

- **🔷** Chatter post-turn summarizer does not behave as a true background task.
  Users report:
  - Text entry stays locked (or feels locked) until **Stop**.
  - Multiple “summarizing conversation” / summary frames for what feels like one
    message — no error shown.
  - Follow-up prompts can fire before the summary is ready.
- **ℹ️** Evidence: `~/.local/share/ollmchat/history/2026/07/13/07-25-41.json`
  (agent `chatter`, 4 turns, 4 summaries).
  - Turn 3 (`user-sent` @54): model said it would download but **never called
    tools** — summary @61 still ran (“execution in progress”).
  - User sent **“continue”** (turn 4) — summary @75 repeats turn-3 opening
    (still about `[user-54]`, not “continue”).
  - Persisted JSON has **one summary per turn** in all recent chatter sessions —
    confusion is **UI/runtime**, not duplicate rows in storage.
  - `07-14-32.json` same day — 1 turn, 1 summary: normal path.

## Root causes

- **ℹ️** Fire-and-forget: `Summarizer.run.begin(cancellable)` after main chat —
  overlapping summarizers when user sends again; stale `create_summary()` on
  follow-up.
- **ℹ️** Shared `cancellable` with main turn — Stop can abort in-flight summary.
- **ℹ️** `message_added` on `response.done` **before** validation — ghost summary
  frames in `ChatView` on retry.
- **ℹ️** Validation / network failures: `GLib.warning` only — no `ui-warning`.
- **ℹ️** Lock perception: overlapping `ui-waiting` rows + shared cancellable,
  not always `is_running` stuck true.

---

## Purpose

- **🔷** Chatter work is **fully FIFO-queued** on `Chatter.Agent` — chat and
  summarize entries on one list; only the head runs at a time.
- **🔷** A turn is **complete** only after main stream + **all tool rounds**
  (`yield chat_call.send()` returns) + **`yield Summarizer.run()`** — then the
  next queued turn may start.
- **🔷** Text entry stays live while a turn is **queued** or in **background
  summary** (`is_running` false for those phases).
- **🔷** Summary uses its **own** `GLib.Cancellable` — not the main-turn token;
  Stop does not abort background summary.
- **🔷** Summarizer surfaces validation / network failures as **`ui-warning`**
  in chat — not only `GLib.warning`.
- **🔷** No edits to core history — **`Session.vala`**, **`Manager`**, **`ChatWidget`**
  stay unchanged.
- **⏳** Manual verification on Chatter agent (see **Suggested order** §5).
- **ℹ️** RPC precedent is **shape only** (`Gee.Promise` + FIFO list + chain) —
  see **Queue model** below. This is **not** a queue of `chat.send` calls.

---

## Queue model (not an RPC call queue)

**Wrong mental model:** each `send_async` is a queued RPC call; summarize is a
tail on that call.

**Correct model:** one serialized FIFO (`pending_messages`) holds **two entry
kinds**:

| Entry | `is_chat` | Handler | What runs |
|-------|-----------|---------|-----------|
| **chat** | `true` | {@link PendingMessage.run} | main + all tool rounds |
| **summarize** | `false` | {@link PendingMessage.run_summarize} | `yield Summarizer.run()` |

- **🔷** `send_async` enqueues a **chat** entry (with `Gee.Promise<bool> done`).
- **🔷** After chat deliver succeeds, {@link run} **enqueues a summarize entry**
  (insert at head) that shares the chat entry's `done` promise — not inline
  summarize inside {@link run}.
- **🔷** {@link run_summarize} resolves that `done` promise and chains the next
  head (`run` or `run_summarize` per `is_chat`).
- **🔷** User chat entries already in the queue wait behind the summarize entry
  — summarize is a first-class queued step, not a tail on the chat call.

```
chat₁ (pull) → run deliver → enqueue summarize₁ → run_summarize → chat₂ → …
                                    ▲ queue barrier (serialized entry)
```

---

## Current behaviour

- **ℹ️** `Chatter.Agent.send_async()` runs main chat, clears `is_running`, then
  `(new Summarizer(this)).run.begin(cancellable)` — fire-and-forget.
- **ℹ️** Concurrent user sends can start overlapping summarizers.
- **ℹ️** Follow-up `create_summary()` may run before the prior `summary` row
  exists.
- **ℹ️** Main-turn `cancellable` is passed to `Summarizer.run` — Stop can cancel
  summary.
- **ℹ️** `Summarizer.handle_stream_chunk()` calls `message_added` on `response.done`
  **before** link validation — failed drafts leave ghost frames in `ChatView`.
- **ℹ️** Validation retry and `catch` paths log `GLib.warning` only — no
  `ui-warning`.

---

## Proposed behaviour

- **🔷** `send_async()` enqueues a **chat** `PendingMessage` (`is_chat = true`).
  When idle, pulls head and kicks `run.begin` or `run_summarize.begin` per flag;
  `yield entry.done.future.wait_async()` until deliver **and** queued summarize
  for that turn finish.
- **🔷** {@link PendingMessage.run} — chat deliver only; on success inserts a
  **summarize** entry (`is_chat = false`) at queue head and chains.
- **🔷** {@link PendingMessage.run_summarize} — `yield Summarizer.run()`; resolves
  the linked chat `done` promise; chains next head.
- **🔷** When `pending.size > 1`, replace `ui-waiting` with
  `"queued — waiting for previous turn"` via `manager.message_added`.
- **🔷** `Summarizer` emits `message_added` for `summary` only **after**
  validation succeeds.
- **🔷** Failed validation (after retry) and `catch` → `ui-warning` + draft
  removed from `session.messages`.

#### Add — new file (full contents)

```vala

namespace OLLMchat.Chatter
{
	/**
	 * One queued chat deliver or summarize step.
	 *
	 * Chat entries ({@link is_chat} {@code true}) use {@link run}. Summarize
	 * entries ({@link is_chat} {@code false}) use {@link run_summarize} and
	 * resolve the chat entry's {@link done} promise.
	 */
	public class PendingMessage : GLib.Object
	{
		public bool is_chat { get; construct; default = true; }
		public Message? message { get; construct; }
		public GLib.Cancellable? cancellable { get; construct; }
		public Gee.Promise<bool> done { get; construct; }

		public PendingMessage(
			Message? message,
			GLib.Cancellable? cancellable,
			bool is_chat,
			Gee.Promise<bool>? done)
		{
			Object(
				message: message,
				cancellable: cancellable,
				is_chat: is_chat,
				done: done != null ? done : new Gee.Promise<bool>()
			);
		}

		/**
		 * Chat deliver: main chat and all tool rounds for {@link message}.
		 *
		 * On success enqueues a summarize entry (head insert) and chains. On
		 * failure resolves {@link done} with exception and chains. Caller must
		 * remove this entry from {@link Agent.pending_messages} before
		 * {@link run.begin}.
		 *
		 * @param agent the Chatter agent owning the queue
		 */
		public async void run(Agent agent)
		{
			agent.session.messages.add(this.message);

			Message? active_summary = null;
			var since_summary = new Gee.ArrayList<Message>();
			foreach (var msg in agent.session.messages) {
				if (msg.role == "summary") {
					since_summary.clear();
					active_summary = msg;
					continue;
				}
				if (msg.role != "user"
					&& msg.role != "assistant"
					&& msg.role != "tool") {
					continue;
				}
				since_summary.add(msg);
			}
			if (active_summary != null) {
				since_summary.insert(0, active_summary);
			}
			if (since_summary.size > 0
				&& since_summary.get(0).role == "summary") {
				active_summary = since_summary.get(0);
				since_summary.remove_at(0);
			}

			var factory = (Factory) agent.session.manager.agent_factories.get(
				agent.session.agent_name);
			var outbound = new Gee.ArrayList<Message>();

			var prompt_file = active_summary != null
				? "chatter_followup.md"
				: "chatter_initial.md";
			var tpl = factory.load_prompt(prompt_file);
			outbound.add(new Message("system", tpl.system_fill(
				"conversation_summary",
				active_summary != null ? active_summary.content : "",
				"environment", factory.build_environment(agent.session)
			)));

			foreach (var msg in since_summary) {
				outbound.add(msg);
			}

			var chat = agent.chat();
			var usage = agent.session.model_usage;
			if (usage.model_obj == null) {
				chat.model = usage.model;
			} else {
				try {
					var connection = agent.session.manager.config.connections.get(
						usage.connection);
					chat.model = yield usage.model_obj.customize(
						connection, chat.options);
				} catch (Error e) {
					agent.session.add_message(new Message("ui-warning",
						"You selected this model, however we were unable to use it. Using the default model instead."));
					chat.model = usage.model;
				}
			}

			agent.session.is_running = true;
			agent.session.manager.agent_status_change();
			try {
				yield chat.send(outbound, this.cancellable);
			} catch (GLib.Error e) {
				this.done.set_exception(e);
				if (agent.pending_messages.size > 0) {
					var next = agent.pending_messages.remove_at(0);
					if (next.is_chat) {
						next.run.begin(agent);
					} else {
						next.run_summarize.begin(agent);
					}
				} else {
					agent.pending_processing = false;
				}
				return;
			} finally {
				agent.session.is_running = false;
				agent.session.manager.agent_status_change();
			}

			agent.pending_messages.insert(0, new PendingMessage(
				null, null, false, this.done));
			if (agent.pending_messages.size > 0) {
				var next = agent.pending_messages.remove_at(0);
				if (next.is_chat) {
					next.run.begin(agent);
				} else {
					next.run_summarize.begin(agent);
				}
			} else {
				agent.pending_processing = false;
			}
		}

		/**
		 * Queued summarize step for the completed chat turn.
		 *
		 * Resolves the linked {@link done} promise (from the chat entry) and
		 * chains the next head.
		 *
		 * @param agent the Chatter agent owning the queue
		 */
		public async void run_summarize(Agent agent)
		{
			agent.summarizing = true;
			try {
				yield (new OLLMchat.Agent.Summarizer(agent)).run(
					new GLib.Cancellable());
			} catch (GLib.Error e) {
				this.done.set_exception(e);
				if (agent.pending_messages.size > 0) {
					var next = agent.pending_messages.remove_at(0);
					if (next.is_chat) {
						next.run.begin(agent);
					} else {
						next.run_summarize.begin(agent);
					}
				} else {
					agent.pending_processing = false;
				}
				return;
			} finally {
				agent.summarizing = false;
			}

			this.done.set_value(true);
			if (agent.pending_messages.size > 0) {
				var next = agent.pending_messages.remove_at(0);
				if (next.is_chat) {
					next.run.begin(agent);
				} else {
					next.run_summarize.begin(agent);
				}
			} else {
				agent.pending_processing = false;
			}
		}
	}
}
```

#### Coding standards — `PendingMessage` try structure

- **`run` (chat):** scoped try on `chat.send` + `finally` for `is_running` only.
  No summarize in {@link run}. No try on `load_prompt` / `system_fill`.
- **`run_summarize`:** scoped try on `yield Summarizer.run(…)`; `agent.summarizing`
  in `finally`. Resolves linked `done` on success.
- **Chain dispatch:** after each exit, pull next head; `next.is_chat` →
  `run.begin`, else `run_summarize.begin` (inline — no Agent helper).
- **No extra methods on `Agent`:** no `deliver_pending_message`, no `send_head`.

#### Replace with — queued `send_async` (enqueue chat + dispatch head)

```vala
		/**
		 * Enqueues a chat turn and waits until deliver and queued summarize
		 * for this message complete.
		 *
		 * @param message API user message (`user-sent` / `ui` already from Session)
		 * @param cancellable optional cancel token for the main/tool request
		 */
		public override async void send_async(
			Message message,
			GLib.Cancellable? cancellable = null) throws GLib.Error
		{
			var entry = new PendingMessage(
				message, cancellable, true, new Gee.Promise<bool>());
			this.pending_messages.add(entry);
			if (!this.pending_processing) {
				this.pending_processing = true;
				var head = this.pending_messages.remove_at(0);
				if (head.is_chat) {
					head.run.begin(this);
				} else {
					head.run_summarize.begin(this);
				}
			}
			if (this.pending_messages.size > 0) {
				this.session.manager.message_added(
					new Message("ui-waiting",
						"queued — waiting for previous turn"),
					this.session);
			}
			yield entry.done.future.wait_async();
		}
```

- **🚫** **Plan only** until explicitly approved — do not implement from this doc
  without user sign-off.

  this plan.
- **🚫** Do not mirror queue onto `OLLMcoder.Agent` in this plan — separate

- **💩** Queue depth in waiting label (`"queued — N messages ahead"`) — deferred;
