## Purpose

- **🔷** Rename bridge methods so direction is obvious (`json_to_bin`, `bin_to_json`, …).
- **🔷** Flip bridge parameter order — inputs first, output last (encode methods write to `Bin.Stream` last).
- **🔷** `Bin.Json` — constructor sets `mode` only; **`json_to_bin`** / **`bin_to_json`** honour AUTO (no separate `*_auto` methods, no `read` helper).
- **🔷** `json_to_bin` — `Json.Object` root only; optional third arg `GLib.Type type` (default `GLib.Type.INVALID`) for the base object when `Mode.AUTO`.
- **🔷** No top-level JSON array support in **`Bin.Json`** or **`libocrpc` HTTP JSON helper** — object roots only (OpenAI-style; HF search arrays handled in **`libochf`**).
- **🔷** `libocrpc` HTTP client — stateless GET/HEAD for third-party JSON APIs and binary downloads.

## Existing code — `Bin.Json` today

Source: `libocrpc/Bin/Json.vala`. Used from `ollmfilesd/StdioConnection.vala` (rpc test scripts).

Instance today: `new Bin.Json()` — no options.

Today every bridge method takes `Bin.Stream` first (parameter name `ctx`). Part A renames them and flips encode methods so inputs come first and the bin stream is last.

Parameter rule on the new API:

- **Encode** (void) — JSON (or member) inputs, then `Bin.Stream bin` to write into. Root encode: optional `GLib.Type? type` third arg on `json_to_bin`.
- **Decode** (returns `Json.Node`) — `Bin.Stream bin` to read from; return value is the output. No type hint — wire type is in the bin bytes.

### Root object (full round-trip on `Bin.Stream`)

- `write(ctx, src)` → `json_to_bin(Json.Object src, Bin.Stream bin, GLib.Type type = GLib.Type.INVALID)`
  - Encode a JSON object tree as bin bytes on the bin stream.
  - Root must have `"*type": "Request"` (or another registered name) in **EXPLICIT** mode.
  - **`type`** — optional third arg; root/base object `GLib.Type` when **AUTO** and JSON has no `*type` (e.g. `typeof(OLLMhf.Model)`).
  - Strips `*`-keys before writing.

- `parse(ctx)` → `bin_to_json(Bin.Stream bin)` → `Json.Node`
  - Decode bin bytes from the bin stream into a JSON object tree.
  - No type hint — root type comes from the bin stream.
  - Restores `"*type"` on the result (EXPLICIT) or omits `*` meta (AUTO).
  - Returns json-glib nodes, not a wire GObject.

### One object body (helper — type byte already handled)

- `read_object(ctx, alias)` → `bin_to_json_object(Bin.Stream bin, string alias)` → `Json.Node`
  - Decode one object’s property stream from bin into JSON.
  - `alias` is a wire name string (`"File"`), not a `GLib.Type`.
  - Internal; called from `bin_to_json` / `bin_member_to_json`.

- `write_object(ctx, src)` → `json_to_bin_object(Json.Object src, Bin.Stream bin)`
  - Encode one JSON object’s properties as bin bytes (no root type byte).

### One property

- `read_member(ctx, type_byte)` → `bin_member_to_json(Bin.Stream bin, uint8 type_byte)` → `Json.Node`
  - Decode one tagged property from bin into a JSON node.

- `write_member(ctx, name, node)` → `json_member_to_bin(string name, Json.Node node, Bin.Stream bin)`
  - Encode one JSON node as one tagged bin property.

### Not there yet

- `Mode` (EXPLICIT / AUTO)
- `json_to_bin(..., GLib.Type? type)` — third arg for root object when AUTO

### Stdio (only caller today — the inbound pattern)

Inbound: NDJSON line → `Json.Parser` → `Json.Object` → `json_to_bin(obj, bin)` → bin in memory → `Bin.Stream.parse()` → `Request`.

Hub / AUTO inbound: `json_to_bin(obj, bin, typeof(OLLMhf.Model))` when JSON has no `*type` — **object root only**.

Outbound: `Request` → `Bin.Stream.write` → bin → `bin_to_json` → NDJSON line.

Plain JSON text never goes straight into `bin_to_json(bin)`.

### Top-level JSON arrays (upstream — not `Bin.Json`)

OpenAI and most REST APIs return an **object** root. **`json_to_bin`** takes **`Json.Object`** only — no change for **`Bin.Json`**.

HF Hub search (**`GET /api/models`**) returns a bare **`[…]`** array. That is **upstream** of the codec: HTTP client / caller layer, not **`json_to_bin`**. Exact shape TBD when we implement — e.g. a per-request flag, wrap array into a synthetic object, or raw body + caller loop. **`libochf`** sorts it out; see [`8.0.1`](8.0.1-libochf-model-catalog.md).

### Why Part A extends `json_to_bin`

Hub JSON has no `*type`. Under `Mode.AUTO`, `json_to_bin` accepts an optional `GLib.Type` hint when the object has no `*type` — same stdio steps, no new entry point. Parsing JSON text (NDJSON line, HTTP body, fixture file) stays in transport / test harness.

## Part A — Auto JSON (`Bin.Json`)

### Goal (this plan)

- **🔷** Rename bridge methods: `write`/`parse`/… → `json_to_bin`/`bin_to_json`/… (see above).
- **🔷** Flip encode parameter order: JSON inputs first, `Bin.Stream bin` last.
- **🔷** `Mode.AUTO` round-trips without `*type` in external JSON.
- **🔷** Constructor — `mode` only:
  - `Mode.EXPLICIT` (default) — stdio behaviour; `*type` required.
  - `Mode.AUTO` — omit `*` meta on encode; optional type hint on `json_to_bin` when decode JSON lacks `*type`.

### Instance API (conceptual)

- **🔷** `⏳` `new Bin.Json(Mode mode = EXPLICIT)`
- **🔷** `⏳` `json_to_bin(Json.Object src, Bin.Stream bin, GLib.Type type = GLib.Type.INVALID)`
  - **`src`** — parsed JSON root object (input).
  - **`bin`** — memory or socket bin stream to write into (output).
  - **`type`** — optional third arg; **base/root object** `GLib.Type` when `Mode.AUTO` and `src` has no `*type`. Ignored in EXPLICIT mode.
- **🔷** `⏳` `bin_to_json(Bin.Stream bin)` → `Json.Node` — no type hint; AUTO omits `*` meta on the JSON tree.

### Inbound decode (all callers — stdio, HTTP, tests)

No `read` method. Callers follow stdio:

1. Transport: parse JSON text → **`Json.Object`** (object root only).
2. `json_to_bin(obj, mem_bin, type_hint)` — `type_hint` when `Mode.AUTO` and no `*type`.
3. `Bin.Stream.parse()` on the memory bin → `Bin.Serializable`.

Top-level **`Json.Array`** (HF search): upstream HTTP layer — see **Top-level JSON arrays** above.

### Outbound encode (stdio today)

1. `Bin.Stream.write(serializable)` → bin bytes.
2. `bin_to_json(bin)` → `Json.Node` → `Json.Generator` → line.

### Tests

- **🔷** `⏳` Rename call sites (`StdioConnection`, rpc tests) to new method names.
- **🔷** `⏳` Fixture JSON under `tests/data/libocrpc/auto-json/` — no `*type` keys.
- **🔷** `⏳` Parse fixture → `json_to_bin(obj, bin, typeof(TestModel))` → `bin.parse()` → field equality.
- **🔷** `⏳` Explicit-mode stdio tests unchanged (`new Bin.Json()` default).

## Part B — HTTP client (`libocrpc`)

From [`8.2` Phase 3](8.2-full-rpc-system.md#phase-3--http-transport-bin-inside-json-outside) — **client only**.

### Goal

- **🔷** One-shot HTTP **GET** / **HEAD** per call (no persistent socket read loop).
- **🔷** JSON endpoints with **object** root: transport parses body → **`Json.Object`** → **`json_to_bin`** → **`Bin.Stream.parse()`** with **`mode == AUTO`**.
- **🔷** Binary downloads (GGUF resolve URLs) — raw byte stream to caller **`OutputStream`**.

### API shape (conceptual)

- **⏳** **`OLLMrpc.Http.Client`** (name TBD)
  - **⏳** `get_json_object_async(...)` — JSON **object** root → `json_to_bin` → `bin.parse()`
  - **⏳** Raw GET for non-object bodies (HF search array) — shape TBD (flag / wrap / caller parse)
  - **⏳** `download_async(url, dest_stream, cancellable)` — follow redirects; stream body to disk
  - **⏳** `head_async(url, cancellable)` — response headers (size probes)

### Tests

- **🔷** `⏳` Recorded responses — no live network in default CI.
- **🔷** `⏳` Redirect chain (302 → CDN) for resolve-style URLs.
