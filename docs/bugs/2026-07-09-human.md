# Vector scan dead zone — **FIXED** (2026-07-09)

Scanning and `event.vector.*` notifications are working in normal use.

---

## What was wrong

1. **Embed probe race** — `activate_project` queued vector work before `open_vector_db` finished → silent skip, no re-queue, ActivityBanner never saw vector events.
2. **Filesystem gitignore broken** — project root `is_repo=0` cached forever; `copy_from` overwrote scan `is_ignored`/`is_repo` (wrong except property names); ~50k vendored/ignored files walked and queued.

---

## Fixes

| Area | Change |
| ---- | ------ |
| `ProjectManager` | `yield open_vector_db()` before `queue_project` on activate |
| `BackgroundScan` | Path capture for deferred vector callback |
| `Folder.discover_repository` | `force` on project root bypasses stale `is_repo=0` |
| `Folder.read_dir_update` | `copy_from` except `is-ignored`, `is-repo`; sync flags after save |
| `Copyable` | `GLib.error` if `except` names use underscores (must be GObject dash names) |
| `Folder.read_dir` | `scan_idle()` when `scanning` map empties |

---
